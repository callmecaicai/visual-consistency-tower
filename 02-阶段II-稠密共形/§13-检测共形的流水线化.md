## 1 · R-CNN → Fast R-CNN → Faster R-CNN (Girshick 等, 2014–2015)

**共形贡献**：R-CNN 先用 Selective Search 生成区域，再对每个区域做 CNN 分类——第一次把深度学习引入检测。Fast R-CNN 共享 backbone 特征、RoI Pooling 使训练端到端。Faster R-CNN 把区域提议也塞进网络（RPN）——「backbone + RPN + RoI head」成为稳定模板。

🔗 [https://arxiv.org/abs/1506.01497](https://arxiv.org/abs/1506.01497)

## 2 · YOLO v1–v3 (Redmon 等, 2016–2018) / SSD (Liu 等, ECCV 2016)

**共形贡献**：一阶段检测——把检测看成一次稠密回归，每个位置直接出框和类别。放弃两阶段的精度换速度，为实时检测奠基；SSD 引入多尺度特征图上的 anchor 分配。

🔗 [https://arxiv.org/abs/1804.02767](https://arxiv.org/abs/1804.02767) (YOLOv3)

🔗 [https://arxiv.org/abs/1512.02325](https://arxiv.org/abs/1512.02325) (SSD)

# 这几个疑点我来逐个打穿

## 一、为什么需要"正负样本划分"——不就是和 GT 算 IoU 吗？

你直觉正确的一半：**IoU 确实是直接和 GT 算的**，没有神秘的地方。但你漏掉了一个关键事实：

**网络一次 forward 会吐出几万个预测框**（Faster R-CNN RPN 大约 ~20000 个 anchor，SSD300 有 8732 个，YOLOv3 有 10000+），**而一张图的 GT 通常只有几到几十个**。

这就引出了两个无法回避的问题：

### 问题 1：哪些 anchor 该学"这是一个对象"，哪些该学"这是背景"？

这是一个**分配问题**（assignment problem）。网络需要一个规则告诉每个 anchor：

- "你负责预测某个具体的 GT"（→ 正样本，算分类 loss + 回归 loss）
- "你什么对象都不对应，就学做背景"（→ 负样本，只算分类 loss，不算回归 loss——因为没有 GT 框可回归）
- "你情况模糊，不参与训练"（→ ignore，不贡献梯度）

**IoU 阈值就是这个分配规则**：

- IoU ≥ 0.7 → 正样本（你和某个 GT 重合度高，学预测它）
- IoU ≤ 0.3 → 负样本（你和所有 GT 都离得远，学做背景）
- 0.3 < IoU < 0.7 → 忽略（你卡在中间，梯度给了也是噪声，干脆不学）

### 问题 2：回到你的核心疑问——同一个 GT 会被多个 anchor 同时监督吗？

**会**。这是设计意图，不是 bug。

一个典型场景：图里有一只狗，GT 框是 (100, 100, 200, 300)。backbone feature map 上那一带的位置上有 9 个 anchor。可能有 **3 个 anchor 同时达到 IoU ≥ 0.7**——比如 128² 的 1:1、128² 的 1:2、256² 的 1:2 三个。

这 3 个 anchor 会**同时作为正样本**，每个都去学:

- "你应该分类为'dog'"
- "从你这个 anchor 为起点，应该往 (dx, dy, dw, dh) 精修"

三个 anchor 因为形状不同，回归的 (dx, dy, dw, dh) 是不同的目标值（因为从不同起点出发到同一终点的向量不同）。

**这样做的本质作用是什么？**

- **冗余监督** = **正样本数量扩充**。一张图几十个 GT，每个 GT 可以激活几个正样本 anchor，一张图就有上百个正样本参与训练。否则如果"一个 GT 只分配一个 anchor"，一张图只有几十个正样本，训练信号极其稀疏。
- **鲁棒性**。推理时哪怕某个 anchor 没检测出来，其他 anchor 还能补上，多个重叠框再通过 NMS 留下最好的那个。
- **为 NMS 铺路**。一阶段检测本来就预期会有大量重叠预测，NMS 去重是明牌操作。

**但这也是个问题**。"一个 GT 激活多个 anchor" 本质是在训练时鼓励网络输出重复框，然后靠后处理 NMS 去重——这是一个**人工设计的训练-推理不一致**。你训练时让它产生重复，推理时又让它别重复，这是不优雅的。

**DETR 要干掉的就是这个**——它用 Hungarian matching 强制**一个 GT 只对应一个预测**（one-to-one matching），训练时就不鼓励重复，推理时直接不需要 NMS。这是后话，但现在你可以看到 IoU 阈值匹配这个设计在哲学上为什么是临时方案。

### 一阶段的分配规则略有不同

- **YOLOv1**：没有经典 anchor boxes。每个 GT 分配给“中心所在网格”，再由该网格中 `B` 个 box predictor 里与 GT IoU 最大的那个负责——**一个 GT 对一个 box predictor，硬分配**。
- **YOLOv2/v3**：每个 GT 分配给"中心所在网格"的"形状最匹配（IoU 最大）的那个 anchor"——**仍然是一对一**。
- **SSD**：用 IoU 阈值（>0.5 全正），和 Faster R-CNN 风格一致——**一对多**。

有趣的事：YOLO 早期主线一直偏向"一对一"责任分配，这使它天生正样本少；SSD/Faster R-CNN 是"一对多"，正样本多但有重复。两种哲学从一开始就分岔了。

## 二、"Coarse-to-fine" 在深度学习领域的一致性

你问到点子上了。**Coarse-to-fine 是深度学习里反复撞见的一个组织原语**，不是检测独有的。

### 它的数学本质

任何"高精度、高自由度"的任务，如果直接端到端出最终答案：

- 解空间太大（所有可能的精细答案）
- 训练信号太稀疏（对的答案只有一个，错的答案有无穷多）
- 梯度方向不稳定（初期预测离目标远，loss landscape 平坦）

**Coarse-to-fine 的策略是**：

> 先在一个**低自由度的粗空间**里找到大致答案（容易学），然后在这个粗答案周围的**局部高自由度空间**里精修（只需要学小修正）。
> 

这是一个**分治 + 尺度递降**的思路。每一层只解决一个"自己尺度内的子问题"。

### 在深度学习各领域里的同构实践

| 领域 | 粗阶段 | 精阶段 |
| --- | --- | --- |
| **检测 (Faster R-CNN)** | RPN 从 anchor 粗修到 proposal | RoI head 从 proposal 精修到最终框 |
| **检测 (Cascade R-CNN)** | IoU≥0.5 阶段出框 | IoU≥0.6 阶段再修 → IoU≥0.7 阶段再修（三级级联） |
| **姿态估计** | 先估 body 根关节 | 再逐级估子关节 |
| **光流估计 (PWC-Net)** | 最低尺度估粗光流 | 上采样 + warp + 残差精修，逐尺度提升 |
| **深度估计 (MVSNet 族)** | 低分辨率 cost volume 出粗深度 | 高分辨率局部精修 |
| **语言生成 (LLM chain-of-thought)** | 先生成大纲 / 粗推理 | 再生成具体答案 |
| **扩散模型** | T 步去噪里前几步出大结构 | 后几步修细节（这是 stochastic 的 coarse-to-fine） |
| **GAN (ProGAN/StyleGAN)** | 从 4×4 开始训练 | 逐步加层到 1024×1024 |
| **语音识别 (CTC + rescoring)** | CTC 粗出字符序列 | LM 精修 |
| **人体 mesh 重建** | 先估 SMPL β,θ 参数 | 再逐顶点精修 |
| **NeRF 渲染** | coarse network 粗采样 | fine network 在重要区域重采样 |

### 本质的一致性（不是比喻，是同一个数学结构）

把任何任务写成"从 x 预测 y"，其中 y 的空间是高维复杂的。Coarse-to-fine 本质是把这个预测写成：

$$
y = f_{\text{coarse}}(x) + \sum_{k=1}^{K} f_{\text{fine},k}(x, y_{k-1})
$$

每一级：

- **输入**：前一级的粗答案 + 原始 x
- **输出**：一个小的修正项
- **loss**：每一级都可以独立监督

这和**残差网络的 ResNet 思想**在结构上同构——ResNet 的 F(x) + x 也是"从一个粗起点（x）加修正（F(x)）到精答案"。

**更深一层的一致性**：Coarse-to-fine 本质是**在解空间上做分治 + 归纳偏置的注入**。"从 anchor 起点做偏移"这个归纳偏置本身就是 coarse-to-fine 的具象——anchor 是粗答案，偏移是精修。去掉 anchor 等于放弃这个归纳偏置，改为让网络从零学到一个粗-精分解（DETR 的 object query 就是在做这件事——query 是可学习的粗答案）。

**最深一层的一致性**：这是**玻尔兹曼式的采样原理**。高维空间里直接跳到目标态概率极低，但**先跳到一个能量低的中间态，再从中间态做小涨落**，总概率就高得多。Coarse-to-fine 就是把"一步大跳"拆成"一步大跳 + 很多小跳"的 MCMC 采样链结构。所以扩散模型是 coarse-to-fine 的极化版本——把 K 推到几十几百步。

## 三、为什么 YOLO v1 感觉像 CNN 架构的思路

**因为它就是**。你这个直觉非常对。

YOLOv1 本质上是一个**分类 CNN 的直接变体**：

```
分类 CNN:  image → CNN → GAP → fc → softmax(C)
YOLOv1:   image → CNN → reshape → fc → tensor(7×7×30)
```

结构上的差异只有两点：

1. 最后输出不是一个 C 维向量，而是一个 7×7×30 的张量。
2. 损失函数不是 cross-entropy，而是"带 objectness mask 的 MSE"。

**YOLOv1 的创新不在架构上**，它的架构就是 GoogLeNet 改的。它的创新在**把检测任务 reduce 成回归任务**的任务定义上——"这个输出张量的每个格子代表什么"的约定上。

所以你说它"像 CNN 的思路"——对，它**就是用分类 CNN 的工具链硬打一个检测任务**。它的美在于承认"检测不一定需要复杂流水线，一个分类 CNN 换个 head 就能干"。

这种"把任务格式改造成分类 CNN 能吃的形状"的哲学，后来在很多地方复现：

- **Segmentation 的 FCN**：把 fc 换成 conv，分类 CNN 变分割
- **Pose estimation 的 heatmap**：把 pose 表示成每个关节的 heatmap，变成"每个关节一个分割图"的任务，分割 CNN 直接干
- **OCR detection 的 CTPN/EAST**：把文本框表示成 dense heatmap，分类 CNN 直接干

这条线的精神都是 **"把新任务硬翻译成分类 CNN 能干的形式"** ——不改架构，改任务表示。这是稠密视觉任务早期（2014-2017）非常主流的打法。

**但这种思路有天花板**——它依赖"任务能被表示成固定大小张量"。检测任务的 k 是可变的，强行塞进固定 7×7×30 必然有信息损失（一个格子只能预测 B=2 个对象，密集场景直接爆）。这个天花板是 YOLO 家族后来一直在打补丁的事情（v2 的 anchor、v3 的多尺度），也是 DETR 彻底换路的原因。

## 四、YOLOv2 的 k-means anchor，为什么 work

### 先看 Anchor 是在解决什么

YOLOv1 每个网格直接回归 2 个框的 (x, y, w, h)——**从零预测绝对坐标**。这有两个问题：

1. **回归目标范围太大**。w 可以从 10 像素到 500 像素，网络要学一个非常宽的回归范围，梯度不稳。
2. **每个框没有"身份"**。B=2 的两个框功能上对称，训练时用"哪个框和 GT IoU 更大就分配给它"的规则，这让两个框都在学差不多的东西，缺乏分化。

### Anchor 做了什么

给每个 anchor 一个**预设的形状（w_a, h_a）**。网络不再回归 w, h，而是回归**相对 anchor 的偏移**：

$$
w_{\text{pred}} = w_a \cdot \exp(t_w), \quad h_{\text{pred}} = h_a \cdot \exp(t_h)
$$

$t_w, t_h$ 是网络输出，大部分时间接近 0。这把"宽泛的尺寸回归"变成"在 anchor 附近的小偏移"，回归范围收紧，训练稳定得多。

更重要的是**分化**：k=5 个 anchor 有 5 种不同形状，第 1 个 anchor 擅长高瘦物体（人），第 2 个擅长矮胖（车），第 3 个擅长大正方形（桌子）……——**每个 anchor 变成一个专家**，负责一个形状模态。网络不用"一个 head 学所有形状"，而是"5 个 head 各学一种形状"。

### k-means 的妙处

Faster R-CNN 的 9 个 anchor 是手工定的（3×3），来自经验。但**手工定的 anchor 不一定匹配数据集**：

- COCO 里大多数物体是什么形状？没人先验地知道。
- 如果 anchor 形状和真实数据分布对不上，大部分 anchor 会离 GT 远，永远是负样本——**anchor 被浪费**。

YOLOv2 的观察：

> 既然 anchor 的意义是"预设 k 个代表性形状"，那最好的 k 个代表性形状应该是**数据里真实形状的 k 个聚类中心**。
> 

**做法**：

1. 把训练集所有 GT 框的 (w, h) 拿出来，不管位置
2. 做 k-means 聚类，但距离函数不是欧氏距离，而是 **1 - IoU(box_a, box_b)**——因为 IoU 才是和检测 loss 对齐的度量
3. 收敛后的 k 个聚类中心，就是 k 个 anchor 尺寸

**为什么 work**（本质解释）：

- **Anchor 的作用是给回归提供好起点**。起点离目标近 → 回归偏移小 → 训练稳 → 精度高
- 手工 anchor 在 COCO 上的平均 IoU（anchor vs closest GT）大约 0.55
- k-means 学出来的 anchor 平均 IoU 可以到 0.65-0.70
- 这 10-15 个百分点的"起点距离"收紧直接让回归目标的方差降下来，梯度质量变好
- 同时 recall 提升——因为 anchor 更贴合真实形状分布，几乎每个 GT 都能找到高 IoU 的 anchor 去匹配，不会出现"没有 anchor 能覆盖这个 GT"的情况

**更深的视角**：k-means anchor 是把**数据的先验统计**注入到架构的归纳偏置里。这是 anchor-free 时代之前的最优解——既不彻底放弃 anchor，又尽可能减少其中的人工成分。从这个角度说，它是"人工先验"向"可学习先验"过渡的中间形态。

**到了 DETR**：这个先验直接变成可学习的 query embedding——k-means 那步都省了，让网络自己学出 anchor 形状模态。这是同一逻辑的终点。

## 五、YOLOv3 的 logistic vs softmax / direct sigmoid 到底在说什么

这里混了两件不同的事，分开说。

### 5.1 分类：从 softmax 换成 independent logistic

**Softmax**：

$$
p_i = \frac{e^{z_i}}{\sum_j e^{z_j}}, \quad \sum_i p_i = 1
$$

**数学含义**：**互斥分类**。C 个类别相互竞争一个概率，总和必须等于 1。一个对象是 "cat" 就不能是 "dog"。

**Independent logistic (sigmoid per class)**：

$$
p_i = \sigma(z_i) = \frac{1}{1 + e^{-z_i}}, \quad p_i \in [0, 1] \text{ 独立}
$$

**数学含义**：**多标签分类**。C 个类别各自独立判断"是 / 不是"，一个对象可以同时是 "Person" 和 "Woman" 和 "Adult"——这三个都可以是 1。

### 为什么 YOLOv3 做这个切换

因为**数据集的标签体系不是互斥的**：

- COCO 里没问题（cat vs dog 是互斥的）
- 但 **Open Images** 这种数据集里，同一个对象会被标上多个重叠类别：
    - "Person" ∧ "Woman" ∧ "Human face"
    - "Vehicle" ∧ "Car" ∧ "Sedan"

**softmax 强行让它们互斥是错的**——如果 "Woman" 的概率上去，softmax 会自动压低 "Person"。但 "Person" 明明也应该高！

**independent logistic 允许多个类别概率同时高**，正好符合多标签的语义。

**训练上的区别**：

- softmax + cross-entropy loss
- sigmoid + binary cross-entropy per class

### 5.2 Objectness：用 logistic (sigmoid)

Objectness 是"这个预测框里是不是有对象"的**二分类问题**——和多类分类无关，天然就是 sigmoid：

$$
p_{\text{obj}} = \sigma(z_{\text{obj}}) \in [0, 1]
$$

这和 YOLOv1/v2 一直在用的 sigmoid 一样，没啥新鲜的。论文提一嘴只是为了对齐表述。

### 5.3 Direct location prediction with sigmoid

这个和 anchor 偏移回归有关。YOLOv2/v3 对**中心坐标 (x, y)** 的预测用 sigmoid 约束：

$$
b_x = \sigma(t_x) + c_x \\
b_y = \sigma(t_y) + c_y
$$

其中 $(c_x, c_y)$ 是当前网格的左上角整数坐标，$sigma(t_x) in (0, 1)$ 是网格内的相对位置。

**为什么？**

回想 Faster R-CNN 对中心的回归是：

$$
b_x = x_a + w_a \cdot t_x
$$

$t_x$ 无约束，理论上可以是任何值——这意味着一个网格的 anchor **有可能预测到离它很远的位置**的对象。这在训练早期极不稳定：

- 网络初始化随机 → $t_x$ 可能输出一个大值 → 预测框飞到图像另一角 → loss 梯度巨大混乱

**YOLOv2 的修正**：

把 $t_x$ 先过 sigmoid，强制 $sigma(t_x) in (0, 1)$，中心只能在**当前网格内部**。这等于说"**每个网格只负责预测中心在自己范围内的对象**"——这个硬约束消除了初期的不稳定性。

**为什么叫 "direct"**：因为相比 Faster R-CNN 的"预测一个无约束的 offset 然后加到 anchor 上再看落到哪"，YOLOv2 是"预测一个 0-1 之间的数直接告诉我中心在这个格子的哪里"——更直接的位置编码。

**小结表**：

| 量 | YOLOv3 怎么处理 | 为什么 |
| --- | --- | --- |
| 分类 | sigmoid per class + BCE | 允许多标签，适配 Open Images 这种数据集 |
| objectness | sigmoid + BCE | 二分类本来就该 sigmoid |
| 中心 (x,y) | $\sigma(t) + c_{\text{grid}}$ | 约束在网格内，训练稳定 |
| 尺寸 (w,h) | $w_a \cdot \exp(t_w)$ | 保证 w,h > 0，exp 允许大范围尺度 |

## 把这五点串起来看

你问的这些不是散点——它们映射到了一组**更深的设计哲学**：

1. **正负样本分配**：如何把稀疏的 GT 扩展成密集的训练信号（冗余监督）——**训练信号密度问题**。
2. **Coarse-to-fine**：如何把高自由度任务分解成多步低自由度任务——**任务分解问题**。
3. **YOLOv1 的 CNN 直打**：如何把新任务翻译成旧架构能吃的张量形状——**表示约简问题**。
4. **k-means anchor**：如何把人工先验换成数据先验——**归纳偏置注入问题**。
5. **Sigmoid vs Softmax**：如何让输出的概率结构匹配标签的真实结构——**概率模型匹配问题**。

五个问题每一个都不只是检测的。每一个在 NLP、语音、生成模型里都有同构的实例。**理解这五个你就理解了深度学习里 "任务适配" 这层工程美学的一大半**。

下一组（FCOS 的 anchor-free 和 DETR 的 query）会把 1 和 4 这两条线**彻底重构**。准备好之后随时问我。

## 3 · FPN —— *Feature Pyramid Networks for Object Detection* (Lin, Dollár, Girshick, He, Hariharan, Belongie, CVPR 2017)

**共形贡献**：top-down + lateral connection 让 backbone 自身成为多尺度金字塔，每个尺度特征都具备语义 + 分辨率双属性。不只是检测用的一个 neck——它把「多尺度特征」**架构化**成稠密任务的默认基础设施，被后续所有稠密架构无差别继承。

🔗 [https://arxiv.org/abs/1612.03144](https://arxiv.org/abs/1612.03144)

## 4 · RetinaNet / Focal Loss —— *Focal Loss for Dense Object Detection* (Lin, Goyal, Girshick, He, Dollár, ICCV 2017)

**共形贡献**：一阶段检测长期输给两阶段，根因是**极端类不平衡**（负样本 >> 正样本）。Focal Loss 用 $(1-p_t)^\gamma$ 压抑易分负样本，让一阶段追上两阶段精度。损失函数显式识别「共形失衡」——度量本身的一次进化。

🔗 [https://arxiv.org/abs/1708.02002](https://arxiv.org/abs/1708.02002)

## 5 · Mask R-CNN —— *Mask R-CNN* (He, Gkioxari, Dollár, Girshick, ICCV 2017)

**共形贡献**：在 Faster R-CNN 的 RoI head 上加一个并行的 mask 分支，用 RoIAlign 替代 RoIPool 解决量化误差。一个架构同时输出框 + 类别 + 掩码（+ 关键点 / 全景）——「多任务稠密头」的标准答案。

🔗 [https://arxiv.org/abs/1703.06870](https://arxiv.org/abs/1703.06870)

## 6 · Panoptic FPN —— *Panoptic Feature Pyramid Networks* (Kirillov, Girshick, He, Dollár, CVPR 2019)

**共形贡献**：把语义分割（stuff）与实例分割（thing）统一在同一套 FPN 特征之上——提出「全景分割」这个任务定义本身。稠密任务开始主动**合并**自己的前奏。

🔗 [https://arxiv.org/abs/1901.02446](https://arxiv.org/abs/1901.02446)

## 本组合力的残差

到 2018–2019，CNN 时代的检测 + 分割已经被 FPN + RoI 这套流水线做到成熟。但架构内部**塞满了手工先验**：

- **Anchor 设计**：尺度、长宽比、步长都是人工调的超参数。
- **NMS**：后处理里隐藏一个不可微的、启发式的去重步骤。
- **RoI Pooling / Align**：把连续的 box 离散化到 grid，损失信息。
- **匹配**：哪个 anchor 对应哪个 GT，靠 IoU 阈值规则决定，非可微。

这些手工先验每一个都是**共形事件之外**的人类决定——它们让系统在训练分布附近工作得很好，但一旦脱离这个分布就开始失效。端到端共形还没有达成。

→ 下一组必须把这些手工先验全部拆掉：**让匹配变成可学习的 Hungarian；让 anchor 消失；让后处理退回到网络内部**。DETR 式的查询范式要出场。
