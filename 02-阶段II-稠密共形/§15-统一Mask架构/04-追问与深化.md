
# §5 · N 个可学习 query 的工程细节

## 5.1 预设方式

```python
# 典型实现 (PyTorch)
self.query_embed = nn.Embedding(N, d_model)  # N=100, d_model=256
```

就这么简单。N × d_model 大小的可学习矩阵，随机初始化（通常 Xavier 或 normal），训练中和其他所有参数一起被优化。

**大小选择**：

- DETR / MaskFormer: N=100
- Mask2Former (base): N=100
- Mask2Former (large + 多任务): N=200

**选择逻辑**：单图最多实例数的安全上界。COCO 单图最多 ~70 实例，100 够用。剩下的 query 学到输出 "no-object"（空预测）。

**初始化不加先验**——这一点特别值得注意。你可能会觉得"既然要检测 100 个对象，是不是应该初始化成 100 个不同位置的 anchor"。DETR 原论文做过对比：**加位置先验没什么帮助，甚至有害**——反而 Hungarian matching 在训练中自发诱导每个 query 学会自己的"偏好槽位"（某个大致位置 / 某种对象尺寸），这是**涌现式的特化**。

这和 CNN filter 的特化是同一种现象——没人告诉 filter "你要学边缘"，但训练自发诱导。

## 5.2 用共形理论理解 cross-attention

回忆你的骨架：

- 自注意力 = 同场内部的相关性测量
- 残差连接 = 把原始信号叠回（防止信号丢失）

cross-attention 的精确位置：

**同一个 decoder layer 内部**的结构（伪代码）：

```
query_stream  = Q_in
Q1 = query_stream + SelfAttn(query_stream)         ← 同场内的 query-query 共形测量
Q2 = Q1          + CrossAttn(Q1, pixel_stream)     ← 跨场的 query-pixel 共形对齐
Q_out = Q2       + FFN(Q2)                         ← 非线性扩容
```

**三个共形操作，三次残差回流**：

- SelfAttn：**在 query 场内部**测量共形（N 个 query 之间的关系——哪些 query 应该"分工不同"）
- CrossAttn：**在 query 场和 pixel 场之间**测量共形（每个 query 在 pixel 场里找最像自己的区域）
- FFN：**单 token 内部**的非线性变换（升维压缩后扩大表征能力）

每一次都是 `stream + 该步提取的增量`——**共形测量的结果作为残差回注到 query stream**。这就是为什么 transformer 的叙事永远是"x = x + f(x)"——**每个 sub-layer 都是残差里的一个支路**。

## 5.3 自注意力是自残差 → 交叉注意力是跳跃残差？

**这个类比部分对，但不够准确。让我给你更精确的版本**。

物理 skip connection 的本意（ResNet）：**把浅层信号直接跨过几层接到深层**——跨越的是**深度维度**（layer 1 → layer 10）。

cross-attention 跨越的不是深度维度，而是**表征场维度**：

- query 场（100 个 N 维向量，抽象对象槽位）
- pixel 场（HW 个 D 维向量，稠密视觉信号）

这两个场是**平行的两条数据流**，不是一条流的不同深度。所以 cross-attention 不是"把早期信号跨层接回来"——它是"**在两条独立数据流之间建立对齐通道**"。

**更精确的共形理论重述**：

| 操作 | 共形对象 | 残差目的 |
| --- | --- | --- |
| 物理 skip connection | 同一流的不同深度 | 防止深度引起的信号衰减 |
| Self-attention (在 skip 内部) | 同场 token 之间 | 在同场里聚合上下文 |
| Cross-attention (在 skip 内部) | 异场 token 之间 | 把另一个场的相关信息注入到本场 |

**cross-attention 的正确类比不是"跳跃残差"，而是"异构场之间的共形桥"**。query 场和 pixel 场像两个独立的语言系统，cross-attention 是它们之间的翻译器——它不复制任何东西，它**建立对应关系并把关系转化成数据流动**。

这个理解在你后面的工作（SAM、Grounding DINO、CLIP）里会极其有用——它们都是**在不同的异构场之间用点积建立共形桥**。

# §6 · Mask embedding 编码的是形状还是位置？

**都不是——它编码的是"在 pixel embedding 空间里的一个判别方向"**。

## 6.1 数学上看

每个 mask query 经过 transformer decoder 得到 mask embedding $e_k in mathbb{R}^D$。

生成 mask 的公式：

$$

m_k(i) = sigma(e_k cdot E_{pix}[i]) quad text{for each pixel } i

$$

$e_k$ **在 pixel embedding 空间里定义了一个方向向量**：

- 和 $e_k$ 方向对齐的像素 → 点积大 → sigmoid 接近 1 → 属于 mask k
- 和 $e_k$ 方向正交的像素 → 点积 ≈ 0 → sigmoid ≈ 0.5 → 不属于
- 和 $e_k$ 方向相反的像素 → 点积小 → sigmoid ≈ 0 → 明确不属于

所以 $e_k$ 本质是**在 pixel 表征空间里切一刀——这一侧的像素属于我**。

## 6.2 形状信息在哪里？

**形状是浮现的，不是编码的**。

机制：

- 同一个对象的所有像素，经过 backbone + pixel decoder 后，它们的 pixel embedding 会**自然聚集在 pixel 表征空间的一个簇里**（训练诱导的——因为它们要被同一个 mask embedding 点积激活）
- mask embedding 找到这个簇的"中心方向"，画一条判别线
- 这条线在**图像平面上投影**出来的形状，就是 mask 的形状

所以形状是**pixel embedding 空间的聚类 + 判别方向**联合决定的——不是 mask embedding 自己"知道" 这个对象是方的还是圆的。

## 6.3 位置信息在哪里？

**位置靠 positional encoding 间接进入**：

- pixel embedding 里已经注入了 2D positional encoding
- 所以同一个物体的像素不仅在"语义上聚集"，还在"位置上聚集"（PE 让相近像素的 embedding 也相近）
- mask embedding 的判别方向 implicit 包含了"这个方向偏好什么位置"

这就是为什么 mask embedding 不需要显式位置——**pixel embedding 自带位置信息，判别方向自然就带了位置偏好**。

## 6.4 和 SAM 的 prompt encoder 对比

| 维度 | MaskFormer mask embedding | SAM prompt encoder |
| --- | --- | --- |
| 位置信息来源 | 隐式（PE 通过 pixel embedding 渗透） | 显式（点/框通过 PE 直接编码坐标） |
| 形状信息来源 | 浮现（pixel embedding 聚类+判别方向） | 浮现（同样机制） |
| Query 来源 | 内生（可学习参数） | 外生（用户提供） |
| 表达能力 | 只能学一类"query 原型" | 可以表达任意点/框/mask prompt |

**共同点极其重要**：两者都是"**在 pixel embedding 空间里定义一个判别方向**"的变体。SAM 的 prompt encoder 把用户的点/框编码成一个 D 维向量，这个向量到了 transformer decoder 里和 pixel embedding 做 cross-attention 的方式——**和 MaskFormer 完全一致**。SAM 的创新不在"mask 怎么生成"（这是 MaskFormer 的范式），在于"**query 从内生变成外生**"。

所以你之前的直觉"query 是指向某个位置的点"——**对 SAM 是对的，对 MaskFormer 不完全对**。但在**架构机制上**两者等价，只是 query 的内容来源不同。

这也是为什么 SAM 可以直接在 MaskFormer-style 架构上建起来——**底层的"query × pixel 点积生成 mask"这个原语不变**。

# §7 · 点积 —— 深度学习里的底层原语

你问到了一个根子上的问题。展开。

## 7.1 先澄清一个表述偏差

你说："注意力是一种残差操作，找差异的"。

**前半对，后半需要调整**。点积**找的是"对齐度 / 相似性"**，不是"差异"。

但你的直觉可以救回来——把表述改成：

> 注意力通过**找相似**来决定**哪些邻居是我需要补足的**，聚合这些邻居得到残差信号，回注到当前流。
> 

精确化：

- 点积 = 相似度测量（基础操作）
- softmax(点积) = 注意力权重（基于相似度的选择）
- 加权平均 V = 按相似度聚合邻居（retrieval）
- stream + attention(stream) = 把聚合来的邻居信号**作为残差**加回流（skip connection 的作用）

所以**点积找相似，残差把相似邻居的信号补回来**。两件事，不是一件。

## 7.2 点积为什么在深度学习里无处不在

第一性原理层面，点积有四个无可替代的性质：

### (1) 点积是"方向对齐度"的最简表达

$$

a cdot b = |a| |b| costheta

$$

去掉长度，点积就是**夹角的余弦**——纯方向对齐度。

在高维空间里，方向的表达能力极强：

- D=256 时可以容纳 $\sim 2^{256}$ 个近乎正交的方向
- "学一个概念" = "学一个方向"
- "测量两个概念的相似性" = "两个方向的夹角" = "点积"

### (2) 点积是 GPU 的天然母语

所有深度学习计算最终归约到 **GEMM（general matrix multiply）**——$C = A cdot B^T$ 的批量化。

- GPU 硬件专门为 GEMM 优化（Tensor Core、cuBLAS）
- 你写什么 novel 操作，只要能表达成矩阵乘，都会是快的
- 不能表达成矩阵乘的操作（稀疏、条件控制）天然慢

**工程约束反向塑造了架构设计**——因为点积快，大家就用点积来表达一切"相关性"。

### (3) 点积在高维里统计上"稀疏激活"

高维随机向量的点积近似 0（用高斯分布算，$mathbb{E}[acdot b]=0$，方差 $propto 1/D$）。

- 随机两个 D 维单位向量，点积方差 $\approx 1/D$
- D=256 时，随机点积在 ±0.06 范围内波动
- 所以当两个向量的点积远超这个阈值（比如 0.5），这是一个**极强的信号**——它们真的学到了对齐

这就是为什么 CLIP / 对比学习 / retrieval 能 work——**高维里点积的信噪比天然很高**。

### (4) 点积连续可微，梯度良好

$nabla_a (a cdot b) = b$，$nabla_b (a cdot b) = a$——**最简单的梯度结构**。

对比其他相似度度量：

- 欧氏距离 $|a - b|_2$：可微但梯度涉及平方根
- 余弦相似度 $a cdot b / (|a||b|)$：可微但分母增加计算复杂度
- KL 散度：不对称、需要概率分布输入

**点积是"信息量/计算量"比最高的相似度度量**。

## 7.3 点积的家族谱——深度学习里所有相似度机制都是它的变体

| 操作 | 数学形式 | 应用 |
| --- | --- | --- |
| **卷积** | $\sum_{ij} W_{ij} x_{ij}$ | 局部 patch 和 filter 的点积 |
| **全连接** | $W \cdot x$ | 输入和权重向量的点积 |
| **分类头** | $E_{pix} \cdot W_{cls}$ | feature 和类原型的点积（§4 讲过） |
| **Self-attention** | $Q K^T$ | token 和 token 的点积 |
| **Cross-attention** | $Q K^T$ | query 和 key 的点积 |
| **CLIP** | $E_{img} \cdot E_{text}$ | 图像和文本的点积 |
| **对比学习** | $E_a \cdot E_p$ vs $E_a \cdot E_n$ | anchor 和正/负样本的点积 |
| **Mask 生成** | $e_k \cdot E_{pix}$ | mask embedding 和 pixel embedding 的点积 |
| **Prompt tuning** | $p \cdot h$ | prompt embedding 和 hidden state 的点积 |

**全部都是点积**。只是点积的两个操作数来源不同：

- 卷积：一个是局部 patch，一个是 filter 权重
- Attention：两个都是动态的 token embedding
- Classification：一个是动态 feature，一个是静态类原型
- CLIP：两个都是动态的跨模态 embedding

**深度学习的大部分可学习模块，本质上就是在优化"什么样的点积结构是好的"**。

## 7.4 和共形理论的终极挂接

**共形 = 两个结构之间的同构对应**。

在连续可微的向量空间里，**"共形性的最经济度量"就是点积**——因为：

1. 它测量方向对齐度（结构同构最基本的信号）
2. 它连续可微（可以被梯度下降优化）
3. 它线性、可批量化（硬件友好）
4. 它在高维里信噪比高（学到的共形关系清晰）

所以深度学习的"找共形"这件事，几乎必然落到点积上——**不是因为点积特别神秘，而是因为所有其他选项都有某种不可接受的代价**。

**一句话**：**点积是"在连续可微、GPU-friendly、高维友好的前提下，测量共形性的唯一经济解"**。它的无处不在不是偶然，是数学 + 工程 + 统计三重约束下的必然。

## 7.5 回到你的共形理论主线

你原来的表述：

> 深度学习的大厦中，相关性及其关键，注意力是一种残差操作。
> 

现在可以精确化成：

> **深度学习的大厦里，一切可学习的"相关性结构"最终都归约到点积。点积是共形测量的基本原子；残差连接是把这些共形测量的结果聚合回 data stream 的结构；注意力是其中一种特殊形式——用归一化点积作为权重，用加权平均作为聚合。**
> 

这样你就有三层分工：

- **点积** = 共形测量的原子操作
- **注意力** = 基于共形测量做选择性信息聚合的机制
- **残差连接** = 把聚合结果回注到数据流的架构模式

三者结合起来，就是 Transformer 的基础骨架。所有创新（MaskFormer 的 mask prediction、SAM 的 prompt encoding、CLIP 的跨模态对齐）都是在这三个原语的组合空间里做变形。
