## 阶段 II 的零度设定：可见性作为投影表面

阶段 I 把图像裁决为标签。阶段 II 则让被标签遮蔽的图像内部返回。

但阶段 II 处理的空间不是世界本身的三维空间，而是可见性被传感器捕获后的投影表面：

```text
x: Ω -> A
```

其中 `Ω` 是图像定义域，`A` 是像素、特征或局部观测值。

阶段 II 的暴力设定是：

> **这张投影表面上的每个位置、区域、对象和提示，都必须能向系统索要回答。**

这意味着阶段 II 不是简单扩展输出维度，而是把可见性第一次交给空间法律统治：标签不再能替整张图作最终裁决，图像内部必须自己出场。

这个设定同时给出阶段 II 的边界。`Ω` 上的每个点可以被度量，但 `Ω` 之外的背面、历史、意图、因果和三维持续状态仍然缺席。阶段 II 的全部成功，都仍然停在“投影可度量”层；它不能单独给出公共语义和世界状态。

## 一、从阶段 I 继承的三条残差（精确接力）

阶段 I 终章已经把判别共形的极限写清楚：

1. **空间内部坍缩** —— GAP 把 H×W 压成单点，图像内部的几何关系被强行折叠。
2. **度量对位置、形状、边界盲视** —— 交叉熵不看「在哪」，只看「是什么」。
3. **输出空间退化** —— 单分类远不能承载真实世界的结构化输出（框、掩码、关键点、深度图、流场、轨迹）。

阶段 II 的展开就是对这三条残差的**结构化回应**。它不是「检测 / 分割 / 深度」三个子领域的并置——那是论文检索视角。本体论视角下，这三个子领域是同一个共形命题（度量穿透）在不同输出空间上的同时实例化。

## 二、阶段 II 的本体论命题

<aside>
🔷

**让度量成为一个在图像定义域内处处定义的场。**

</aside>

阶段 I 的最小公式是：

```text
Image -> Label
图像 -> 标签
```

阶段 II 的最小公式则变成：

```text
Image-as-domain -> Field / Object / Interface
图像作为空间定义域 -> 场 / 对象 / 接口
```

这意味着视觉世界不再被压缩成一个标签，而是被展开为一个可度量的空间定义域。在这个定义域中，每个位置、每个区域、每个对象、每个外部提示，都可以要求模型给出局部回答。

这句话有三重含义：

- **密度**：度量不再只在输出端存在，而是在每一个空间位置都有一份损失。
- **结构化**：度量不再是标量（是/否、对/错），而是带内部结构的对象——坐标、区域、掩码、深度值、向量场。
- **可解耦**：语义、位置、边界、实例这些维度开始**独立承受共形压力**，而不是挤在同一个分类头里。

这是共形事件中**度量维度的第一次真正深化**。阶段 I 的度量是判别式的（离散类别）；阶段 II 的度量是空间化的（稠密场）；阶段 III/IV 之后才走向结构化关系与生成分布。

进一步说，阶段 II 内部不是一个单一闭合，而是三种闭合形态的连续展开：

| 闭合形态 | 最小公式 | 主要对象 | 出口残差 |
|---|---|---|---|
| 场闭合 | `x -> y(Ω)` | 像素、边界、深度、流、法线等空间场 | 每个位置被判定，但像素不是对象 |
| 对象闭合 | `x -> {(region_i, class_i, score_i)}` | box、mask、实例、stuff/thing | 对象依赖 anchor、NMS、RoI、assignment、query 等绑定制度 |
| 接口闭合 | `(x, q) -> output_q` | query、prompt、text、example、task token | 接口打开，但语义仍由外部语言/示例/概念表赋予 |

这三层的细化见 [§10-阶段II导论-场闭合对象闭合接口闭合](§10-阶段II导论-场闭合对象闭合接口闭合.md)。

## 三、六维度上的具体位移

| 维度 | 阶段 I 的状态 | 阶段 II 的位移 |
| --- | --- | --- |
| 度量 | 交叉熵 + Top-1 | Dice / IoU / mAP / Focal / Hungarian 匹配 / PQ / RMSE —— 每一种度量都重新定义「错」 |
| 载体 | 单图单模态 | 仍为单图，但空间维度被充分调用；出口处语言作为「语义载体」首次进入 |
| 主体性 | 无主体 | 仍为无主体；但「提示化」（SAM 式）让系统对外部指令做局部响应——准主体性的雏形 |

阶段 II 的核心移动发生在 **度量** 和 **材料**；载体与主体性在本阶段只是轻微位移，真正的跃迁留给阶段 III 及以后。

## 四、阶段 II 的内部接力线（残差自生下一组）

阶段 II 不是按年代或子任务平铺的，是按**残差传递**串起来的：

1. **全卷积转向**（FCN / DeconvNet / SegNet / U-Net / U-Net++）—— 去掉 FC，让每一个空间位置都承受一份损失。
    - 残差：像素级共形达成，但感受野碎片化、尺度不一致。
2. **空洞卷积与多尺度**（DeepLab v1–v3+ / PSPNet / HRNet / SegFormer）—— 在保持分辨率的同时扩大感受野，让「全局上下文」与「像素精度」共存。
    - 残差：语义稠密已解决，但「实例」与「语义」割裂——两个像素属于同一个对象这件事没有被建模。
3. **检测共形的流水线化**（R-CNN 家族 / YOLO / SSD / FPN / RetinaNet / Mask R-CNN / Panoptic FPN）—— 用区域提议 + 特征金字塔把「实例」作为一级原语引入。
    - 残差：架构塞满手工先验（anchor / NMS / RoI），端到端共形未达成。
4. **Anchor-free 与端到端查询**（CornerNet / CenterNet / FCOS / DETR / Deformable DETR / DINO-DETR / Co-DETR）—— Hungarian 匹配把 NMS 拆除，查询成为稠密新原语。
    - 残差：查询范式统一，但每种稠密任务仍有自己的头，语义 / 实例 / 全景 / 深度各走各的。
5. **统一 Mask 架构**（MaskFormer / Mask2Former / OneFormer / SegFormer / SegGPT / EoMT）—— 区域、对象、实例、stuff/thing 等可离散稠密任务强烈收束到「一组 mask 查询 + 每个查询一个标签」；连续场任务仍保留 field-to-field 的闭合形态。
    - 残差：架构统一了，但**类别边界仍锁在训练集之内**——80 类就是 80 类。
6. **提示化与开放词汇**（SAM / SAM 2 / Grounding DINO 1.5 / DINO-X / T-Rex2 / YOLO-World / Florence-2 / APE / ODISE / CAT-Seg / FC-CLIP / Depth Anything）—— 类别从闭集拉成任意语言 / 点 / 框提示。
    - 残差：**开放词汇是借来的**——语义挂载 CLIP / 语言模型的外部表示，视觉侧自己不产语义。

第六步的残差直接定义阶段 III 的入口：稠密空间已经能被稳定调用，但公共语义载体仍在系统外部。下一阶段要解决的不是再画更准的 mask，而是让视觉区域进入公共语义空间，并能被语言、指令和概念真正调用。

## 五、阶段 II 的截止线

**做完了**：

- 度量穿透到像素、实例、位置、深度值、向量场——稠密度量成为视觉系统的主导评价形式，但边界模糊、标注噪声、开放类别、实例身份、时序一致性和任务后果仍会以残差返回。
- 对于可离散为区域、对象、实例、stuff/thing 的稠密任务，阶段 II 末端出现「backbone + query/prompt + matching/readout」的强统一形态；但深度、光流、法线、低层恢复、医学配准等连续回归任务不必强行纳入 mask-query 范式。
- 类别闭集被提示化撬开——80 类锁死的时代结束。

**没做（留给阶段 III / V 与反身维度）**：

- **语义的自生**。阶段 II 的开放词汇都在借 CLIP 的语义。第 81 类能被识别，是因为 CLIP 的对比预训练见过这个词——不是视觉侧「结构地」推出的。
- **结构化关系**。两个实例之间的关系（「在上面」、「握着」、「导致」）仍不在稠密输出里。
- **反身度量**。IoU / mAP / PQ / RMSE 仍然外部给定；网络不能质疑或修订自己的度量。

前两条指向阶段 III 与 Pre-LLM 的终结；第三条指向总纲的反身维度——那是整个大厦的终极残差。

阶段 II 的自我否定可以压成三层：

1. 场闭合让每个像素都有答案，但像素不是对象，因此场闭合逼出对象闭合。
2. 对象闭合让实例成为输出单位，但对象依赖绑定制度，因此对象闭合逼出 query/set/prompt 闭合。
3. 接口闭合让外部提示可以调用区域和对象，但语义仍来自外部语言和概念数据，因此接口闭合逼出阶段 III 的公共语义闭合。

最短的结论是：**空间越可度量，越暴露出意义不在空间里。**

## 六、阶段 II 的存在、无与成为

| 环节 | 存在 / 设定 | 无 / 反面 | 成为 / 技术制度 | 成功 | 内在残差 |
|---|---|---|---|---|---|
| 投影表面 | 图像内部必须被度量 | 标签只看最后一点 | FCN、U-Net、SegNet | 场闭合 | 像素不是对象 |
| 上下文化 | 每点必须获得远处上下文 | 局部精度与感受野冲突 | DeepLab、PSPNet、HRNet、SegFormer | 多尺度场闭合 | 上下文不是对象关系 |
| 对象化 | 连续场中必须出现实体 | 场没有可数对象 | R-CNN、YOLO、SSD、Mask R-CNN | 对象闭合 | 对象依赖绑定制度 |
| 集合化 | 对象必须端到端绑定 | anchor、NMS、RoI 外置规则 | DETR、DINO-DETR、Co-DETR | set / query 闭合 | query 仍是黑箱 |
| Mask 统一 | 区域任务必须共享原语 | 任务头分裂 | MaskFormer、Mask2Former、OneFormer、EoMT | 区域 / 实例架构统一 | 类别闭集 |
| 提示化 | 外部提示必须调用区域 | 任务规格固定在训练集 | SAM、SegGPT、Grounding DINO | 接口闭合 | 语义主权外部化 |
| 开放词汇 | 任意概念要能进入稠密输出 | 视觉自身没有概念 | Grounding DINO、CAT-Seg、Florence-2、SAM 3 | 开放接口 | 语言遮蔽视觉 |

这张表说明，阶段 II 的“空间胜利”不是线性进步，而是投影表面在不同闭合制度中连续被切开。每一次切开都扩大了可见性，也制造了新的不可见者。
