# 阶段 II 导论补页：场闭合、对象闭合、接口闭合

类型: 理论/分析
阶段: II · 稠密共形的扩张
共形维度: 度量, 接口, 载体
关键贡献: 把阶段 II 从“分割/检测/开放词汇任务集合”改写为三种主导闭合形态：场闭合、对象闭合、接口闭合。
序号: 10
残差 / 它催生了什么: 空间被处处度量后，像素不是对象；对象被分离后，对象生成依赖外部制度；接口被打开后，语义仍不属于视觉自身。这三层自我否定共同推出阶段 III 的公共语义闭合。

## 一、阶段 II 的纯存在态

阶段 I 的纯存在态是：

```text
Image -> Label
图像 -> 标签
```

阶段 II 的纯存在态不再是把图像压成一个点，而是把图像展开成一个定义域：

```text
Image-as-domain -> Field / Object / Interface
```

视觉世界不再作为整体被判别，而是作为一个空间定义域被处处要求作答。在这个定义域中，每个位置、每个区域、每个对象、每个外部提示，都有权向模型索要局部回答。

因此，阶段 II 的展开不是“输出维度变多”，而是三种闭合形态的连续展开。

## 二、三种闭合形态

### 1. 场闭合 Field Closure

```text
x -> y(Ω)
```

图像被看作定义在空间域 Ω 上的输入场，模型必须为每个位置输出类别、深度、法线、流、边界或其他局部量。

代表路线：

- FCN、DeconvNet、SegNet、U-Net
- DeepLab、PSPNet、HRNet、SegFormer
- DPT、Depth Anything 等连续场读出

场闭合的度量包括 pixel CE、Dice、IoU、mIoU、boundary score、RMSE、AbsRel 等。它的成功是：度量第一次真正进入图像定义域。

它的残差是：每个位置都能被判定，但像素不是对象。两个同类像素是否属于同一个实体、一个区域是否构成对象、一个边界是否承载实例身份，这些问题不能由逐像素场本身完全回答。

### 2. 对象闭合 Object Closure

```text
x -> {(region_i, class_i, score_i)}
```

图像不只是空间场，而是包含可数实体。检测、实例分割和全景分割让“对象”成为输出单位。

对象闭合的最小对象可以写成：

```text
object_i = {
  support / box,
  category,
  confidence,
  assignment identity,
  relation to other objects
}
```

代表路线：

- R-CNN、Fast/Faster R-CNN、YOLO、SSD
- FPN、RetinaNet、Mask R-CNN、Panoptic FPN
- FCOS、DETR、Deformable DETR、DAB/DN/DINO-DETR、Co-DETR

对象闭合的度量包括 mAP、AR、PQ、Hungarian matching cost、box/mask IoU 等。它的成功是：连续图像场中出现了可数实体。

它的残差是：对象越稳定，越暴露出对象生成依赖 anchor、NMS、RoI、IoU threshold、assignment、query 等外部制度。对象并不是自然从像素场里“自己长出”，而是被一套绑定制度组织出来。

### 3. 接口闭合 Interface Closure

```text
(x, q) -> output_q
```

图像的空间和对象不再只由模型内部固定任务头读取，而是被 query、prompt、text、example、instruction 等外部接口调用。

代表路线：

- MaskFormer、Mask2Former、OneFormer、EoMT
- SAM、SAM 2、SAM 3
- Grounding DINO、YOLO-World、APE、CAT-Seg、FC-CLIP、Florence-2、SegGPT

接口闭合的度量包括 prompted mask quality、open-vocabulary AP、grounding score、PCS、VOS/VPS 指标、跨接口一致性等。它的成功是：输出空间不再完全由训练时类别表固定，用户、文本和示例开始规定模型该在空间中调用什么。

它的残差是：prompt 越强，越暴露出语义并不属于视觉自身。视觉区域可以被语言、示例、用户意图临时调用，但公共语义载体仍来自外部文本、图文对齐、人工命名和概念数据引擎。

## 三、阶段 II 的三层自我否定

| 层级 | 成功标准 | 如何成功 | 自我否定 | 推出下一层 |
|---|---|---|---|---|
| 场闭合 | 每个像素/位置都有答案 | 度量进入 Ω，形成稠密场 | 像素不是对象，位置正确不等于实体成立 | 对象闭合 |
| 对象闭合 | 每个实例可被分离 | box、mask、identity、assignment 成为输出单位 | 对象依赖 anchor、NMS、RoI、matching、query 等制度 | 接口闭合 |
| 接口闭合 | 外部提示可调用区域/对象 | prompt、text、example、task token 进入稠密读出 | 语义仍由外部语言/示例/概念表赋予 | 阶段 III 公共语义闭合 |

阶段 II 的自我否定不是“mask 还不够准”，而是：

> 空间越可度量，越暴露出意义不在空间里。

这句话是阶段 II 通向阶段 III 的真正入口。阶段 III 不是外部给视觉接上语言，而是阶段 II 自己在成功中发现：空间处处能回答，仍然不足以说明这些回答如何进入公共意义。

## 四、使用纪律

后续阅读阶段 II 时，所有方法先判定它主要扩张哪一种闭合：

1. 它是在增强场闭合、对象闭合，还是接口闭合？
2. 它的度量落在像素、区域、对象、prompt，还是跨接口一致性上？
3. 它留下的残差来自表征场、对象绑定制度、接口调用方式，还是语义来源？
4. 它是否只是提高某个指标，还是改变了“谁有权向图像索要答案”？

这样阶段 II 才不会退化成分割、检测、深度、开放词汇的技术展厅，而能保持为稠密视觉意识的内在展开史。
