# §10-B · 空间标注制度：从标签数据到图像定义域的法律

阶段 I 的材料制度是标签数据：一张图属于什么类。

阶段 II 的材料制度是空间标注：图像内部哪里算对象，哪里算边界，哪里算实例，哪里算 stuff，哪里算 ignore region。

这不是中性监督。它是可见性第一次被空间法律切开。

## 一、从标签到空间法律

阶段 I 的标签回答：

```text
image -> class
```

阶段 II 的空间标注回答：

```text
Ω -> label / box / mask / keypoint / depth / flow
```

PASCAL VOC、COCO、Cityscapes、ADE20K、LVIS、Objects365、SA-1B、FLD-5B 等数据制度，不只是给模型更多样本，而是在规定：

- 哪些像素属于同一对象；
- 哪些区域应该被视为同一个实例；
- 哪些类别是 thing，哪些是 stuff；
- 哪些边界需要标，哪些可以忽略；
- 哪些深度、关键点、框或掩码可作为监督代理。

## 二、标注不是事实本身

box 不是对象本身，而是对象在投影表面上的粗支撑。

mask 不是对象本身，而是可见区域或标注制度承认的区域。

keypoint 不是身体本身，而是任务指定的稀疏结构点。

depth map 不是完整三维世界，而是从单视角证据推定的几何代理。

因此，阶段 II 的材料轴必须接受一条纪律：

> **空间标注定义的是任务可度量对象，不等于世界对象。**

## 三、空间标注的残差

空间标注越密，越暴露出它的制度性：

- 标注边界常有歧义；
- 不同数据集对 stuff / thing 的切分不同；
- occlusion 和 amodal object 常被简化；
- 长尾类别难以穷尽；
- promptable mask 可以扩展空间支撑，但语义仍需外部概念制度。

所以阶段 II 的材料胜利不是“图像内部终于真实透明”，而是“图像内部第一次被可训练制度切开”。这套制度产生了稠密视觉的工程奇迹，也留下了阶段 III 和阶段 V 的根残差。
