## 1 · CornerNet / CenterNet (Law & Deng, ECCV 2018 / Zhou 等, 2019)

**共形贡献**：CornerNet 把框预测改成「左上 + 右下两个关键点」；CenterNet (Objects as Points) 改成「中心点 + 宽高回归」。anchor 从定义里消失——检测变成「关键点定位 + 属性回归」。

🔗 [https://arxiv.org/abs/1808.01244](https://arxiv.org/abs/1808.01244) · [https://arxiv.org/abs/1904.07850](https://arxiv.org/abs/1904.07850)

## 2 · FCOS —— *Fully Convolutional One-Stage Object Detection* (Tian, Shen, Chen, He, ICCV 2019)

**共形贡献**：最彻底的 anchor-free——每个特征位置直接回归「到框四条边的距离」+ 类别 + centerness。检测被拉回全卷积范式，和分割在结构上完全对称。

🔗 [https://arxiv.org/abs/1904.01355](https://arxiv.org/abs/1904.01355)

## 3 · DETR —— *End-to-End Object Detection with Transformers* (Carion, Massa, Synnaeve, Usunier, Kirillov, Zagoruyko, ECCV 2020)

**共形贡献**：一组可学习的 object query 对 backbone 特征做 cross-attention，Hungarian 匹配在训练时**一次性**把查询与 GT 对齐——NMS 从架构里彻底消失，训练是严格端到端。稠密共形走向纯神经的标志事件。

🔗 [https://arxiv.org/abs/2005.12872](https://arxiv.org/abs/2005.12872)

## 4 · Deformable DETR (Zhu, Su, Lu, Li, Wang, Dai, ICLR 2021)

**共形贡献**：DETR 收敛慢（500 epoch）、小物体差——根因是 cross-attention 在高分辨率特征图上看什么都「模糊一片」。Deformable Attention 让每个 query 只对少量可学习参考点做采样，收敛压到 50 epoch，小物体精度显著提升。

🔗 [https://arxiv.org/abs/2010.04159](https://arxiv.org/abs/2010.04159)

## 5 · DAB-DETR / DN-DETR / DINO-DETR (IDEA, 2022)

**共形贡献**：DAB 把 query 显式写成 4-d anchor box 并在层间迭代；DN 引入「带噪 GT」作为去噪训练信号稳定 bipartite 匹配；DINO-DETR 把两者 + 对比式去噪合成 SOTA 方案。这一系列让 DETR 真正达到 Faster R-CNN 级的训练效率。

🔗 [https://arxiv.org/abs/2203.03605](https://arxiv.org/abs/2203.03605)

## 6 · Co-DETR —— *DETRs with Collaborative Hybrid Assignments Training* (Zong, Song, Liu, ICCV 2023)

**共形贡献**：在 DETR 训练里并联多个辅助 head（ATSS / Faster R-CNN 式密集匹配），让同一 encoder 同时承受「一对一匹配」和「一对多匹配」两种监督。多年来 COCO 榜上常驻 SOTA 之一。

🔗 [https://arxiv.org/abs/2211.12860](https://arxiv.org/abs/2211.12860)

## 本组合力的残差

到 DINO-DETR / Co-DETR，检测这一任务的架构问题基本被解决：端到端、anchor-free、匹配可微、精度领先、训练效率合理。架构统一的脚步并没有停在检测——

- 分割（MaskFormer / Mask2Former）和检测开始**互相模仿**：同一组 query + 同一种 Hungarian 匹配，把 mask 和 box 都当作「查询的属性」。
- 全景 / 实例 / 语义三者开始坍缩为同一个架构。
- 但每种稠密任务仍然在**各自的数据集**上训、用**各自的 head**——任务级的统一还差最后一步。

→ 下一组的使命：**用一种架构接管所有稠密任务**。MaskFormer / Mask2Former / OneFormer / EoMT 出场。

下面几页将顺次把 DETR 主线、YOLO 的工业吸收线，以及 object query 的本体论外溢打穿。它们共同承担这一组的深度展开，并把本页从“检测去先验”自然推进到下一组“统一 Mask 架构”。
