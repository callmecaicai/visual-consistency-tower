# §10-A · 阶段 II 的投影表面：从图像定义域到世界缺席

阶段 II 不是直接处理世界空间，而是处理可见性被传感器捕获后的投影表面。

```text
x: Ω -> A
```

`Ω` 是图像定义域，可以是像素网格、patch 网格、feature field 或 token field。`A` 是在该位置可观测、可编码、可训练的局部值。

## 一、投影表面不是世界本身

阶段 II 的所有模型首先面对的是 `H x W x C` 的观测平面。这个平面来自某个传感器、某个时间点、某个视角和某个采样制度。

因此，`Ω` 能承载：

- 像素类别；
- 边界；
- box；
- mask；
- 深度估计；
- 光流；
- prompt 所指区域。

但 `Ω` 不能直接承载：

- 图像背面；
- 被遮挡对象的完整形状；
- 拍摄前后的历史；
- 行动意图；
- 因果关系；
- 三维持续实体。

阶段 II 的第一条纪律是：**空间闭合首先是投影表面闭合，不是世界状态闭合。**

## 二、投影表面的可度量化

阶段 I 的度量落在整图标签上。阶段 II 的度量落入 `Ω`：

```text
L = Σ_{p in Ω} ℓ(π_p(z), y_p)
```

这不是“多算几个 loss”，而是视觉制度的改写：图像内部第一次获得独立发言权。

FCN 让每个像素承担损失；DeepLab / PSPNet / HRNet 让每个位置获得上下文；R-CNN / YOLO / DETR 让投影表面中出现可数对象；SAM / Grounding DINO 让外部提示可以调用投影区域。

这些工作都在同一个平面上完成：让 `Ω` 从被标签压扁的背景，变成可度量、可对象化、可调用的场。

## 三、投影表面的内在残差

投影表面越可度量，它的边界越明显。

每个像素可以有答案，但像素不是对象。

每个对象可以有 box / mask，但 box / mask 只是投影支撑，不是三维实体。

每个 prompt 可以调用区域，但 prompt 的意义来自外部任务、语言、示例或概念制度。

因此，阶段 II 的出口残差不是“mask 还不够准”，而是：

> **投影表面不能单独给出公共语义，也不能单独给出世界状态。**

这条残差一分为二：公共语义进入阶段 III，世界状态进入阶段 V。
