# 阶段 II · 空洞与多尺度这一组：六个答案如何围着同一道题转圈

## 零、先把这组的题干写清楚

前一组（FCN/DeconvNet/SegNet/U-Net）留下的残差写成一道题：

> 稠密预测要求**每个像素**都有一个覆盖「足够远的上下文」的感受野，同时保留「像素级的空间精度」。但 CNN 的算子天然把这两者对立——要远，就得 pool，就丢分辨率；要清，就不能 pool，就看不远。
> 

这一组六篇论文，**不是六个不同任务的解答，而是针对同一道题的六种不同哲学**。要看清它们各自的本质，就要把它们并排摆到这道题前。

---

## 一、DeepLab v1 / v2 —— 空洞卷积 + CRF：「扩张算子 + 外挂结构先验」

### 做了什么优化

**1. 把 pool 换成 atrous**

标准 ResNet-101 的 5 个 stage 每次 stride=2 下采样，输出 stride = 32。DeepLab 的做法：**保留前 3 个 stride=2，但把最后 2 个 stage 里的 stride=2 改成 stride=1**，结果分辨率只下采样到 1/8。

问题来了：原本那些 stride=2 的 conv 给后续 conv 带来的感受野翻倍没了。怎么补？用 **atrous convolution**。

atrous convolution 的机制：在 3×3 卷积核的采样点之间**插入洞**。dilation rate = r 的 3×3 卷积，采样 9 个点，但这 9 个点在原图上的间距是 r——**参数量、计算量、采样点数不变，但覆盖半径扩大 r 倍**。

所以 DeepLab 把后续 conv 的 dilation 设成 2、4，**恰好补偿掉被去掉的两次下采样带来的感受野损失**。这是一个干净的守恒变换：**感受野不变，分辨率 ×4**。

**2. DenseCRF 做结构化精修**

网络输出是 H×W×C 的 softmax 概率图，CRF 把它当作一个图模型的 unary potential，然后加上一个 **fully-connected pairwise potential**：

$\psi_p(x_i, x_j) = \mu(x_i, x_j) \left[ w_1 \exp\left(-\frac{|p_i-p_j|^2}{2\theta_\alpha^2} - \frac{|I_i-I_j|^2}{2\theta_\beta^2}\right) + w_2 \exp\left(-\frac{|p_i-p_j|^2}{2\theta_\gamma^2}\right) \right]$

意思是：位置接近且颜色相近的像素（第一项）、位置接近不管颜色的像素（第二项），倾向于同一标签。用 mean-field 迭代 ~10 次做近似推断。

### 本质

- **atrous**：用「扩张算子」在 CNN 内部绕开"下采样换感受野"的锁。注意它不是真的解锁——它还是用局部卷积核，只是让采样点稀疏化。代价是**网格伪影**（gridding artifacts）：连续的 dilation 卷积采样点落在固定格子上，有些位置永远被跳过。
- **CRF**：承认 CNN 的稠密输出是"软的、碎的"，然后**外挂一个手工结构先验**来修边界。CRF 是 pre-deep-learning 时代图像分割的主力，DeepLab 做的是"CNN 拿 unary，CRF 拿 pairwise"这种新旧结合。

### 效果

- v1 (VOC 2012)：71.6 mIoU，加 CRF 后 ~72.7。
- v2 (VOC 2012)：加入 ASPP 和多尺度测试后 79.7 mIoU。
- CRF 约贡献 **+1~2 mIoU**，但推理慢了一大截（mean-field 迭代 CPU 约 0.5 秒/张）。

### ASPP（v2 加入）

v1 的单一 dilation 只覆盖一个尺度。v2 的 ASPP 把 4 个并行分支（dilation 6/12/18/24）的输出拼起来——**同一层 feature 同时被不同"视野半径"观察**。这是把空洞卷积从"单尺度工具"变成"多尺度模块"的关键一步。

## 二、DeepLab v3 —— 「让网络自己吞下 CRF」

### 做了什么优化

**1. 扔掉 CRF**

v3 的大胆主张：结构化输出应该由网络内部完成，不需要外挂。证据就是 v3 本身的分数——**不用 CRF 也能刷到 85.7 mIoU**。

**2. ASPP 加 global image pooling**

纯 ASPP 有个问题：dilation 过大时，3×3 核的 9 个采样点可能大部分落在 feature map 外面（padding），变成退化的 1×1 卷积。v3 的修正：在 ASPP 里加一条**全局平均池化支路**——直接把整张 feature 压成 1×1，再广播到原尺寸。这一条专门负责"图像级上下文"，补上 atrous 在大 dilation 时的失效区。

**3. ASPP 里加 BN**

原来的 ASPP 没 BN，训练难收敛。v3 给每个 ASPP 分支加 BN，让 ASPP 可以**从头训练**而不是只能作为 fine-tune 模块。这是把 ASPP 从"后处理套件"变成"主力架构组件"的工程关键。

**4. 级联 atrous 模块 + Multi-grid**

把 ResNet 的最后几个 block 复制成多个 stage，每个 stage 用不同 dilation。Multi-grid：同一 block 内部三个 3×3 conv 分别用 dilation (1, 2, 4)，避免所有 conv 都落在同一格子上——**缓解网格伪影**。

### 本质

**神经网络部分吸收了图模型在语义分割 benchmark 中承担的角色**。CRF 原本是 CNN 稠密输出的外部边界平滑和上下文一致性补丁，v3 证明在特定语义分割设置里，这个补丁可以被更好的 feature + 更好的多尺度模块替代。这不是“所有图模型式结构推理被消灭”，而是 CRF 在这一类 benchmark 中的后处理地位被网络内部机制部分接管。

### 效果

- VOC 2012 test: **85.7 mIoU**（不用 CRF，超过 v2+CRF）。
- Cityscapes val: ~79.3 mIoU。
- 成为接下来两年所有"语义分割 baseline 表格"的标配第一行。

## 三、DeepLab v3+ —— 「空洞路线 ∪ encoder-decoder 路线」

### 做了什么优化

**1. 给 v3 的 ASPP 顶部加一个轻量 decoder**

v3 的输出是 stride 16 的 feature，直接 16× 双线性插值到原图——边界糊。v3+ 的改法：

```
ASPP 输出 (stride 16)
  → 4× 上采样 (stride 4)
  → 和 backbone 的 stride-4 浅层 feature concat
  → 3×3 conv
  → 4× 上采样到原图
```

这个 decoder 很轻（两次上采样 + 一次 concat + 两次 conv），但**把边界锐度从 U-Net 级别拉回来了**。

**2. Atrous separable convolution**

Depthwise separable conv（MobileNet 的做法：把 3×3×C 拆成 depthwise 3×3 + pointwise 1×1）和 atrous 结合。在 decoder 和 ASPP 里广泛使用——**同样精度下计算量减半**。

**3. Xception backbone 变体**

用改造版 Xception（更深的 depthwise separable）替代 ResNet 做 backbone。

### 本质

**「空洞全局上下文」和「encoder-decoder 局部边界」是正交的两件事，不是互斥的两条路**。v3+ 把它们拼在一起，各自做各自擅长的事——ASPP 负责远，decoder 负责细。这是一个工程意义上的整合，但也是一次认知整合：**多尺度感受野 vs 高分辨率恢复，是两个独立维度的问题，要分别解**。

### 效果

- VOC 2012 test: **89.0 mIoU**（JFT 预训练 + Cityscapes 预训练）。
- Cityscapes test: **82.1 mIoU**。
- **CNN 时代语义分割的事实巅峰**——之后的 CNN-only 工作基本没能显著超过它。后续的进步都要么借助 Transformer，要么借助更强的预训练。

## 四、PSPNet —— 「显式全局池化」的另一种多尺度哲学

### 做了什么优化

**Pyramid Pooling Module (PPM)**：在 backbone 末端（ResNet 输出 stride 8 的 feature），做 4 个并行的 **average pooling**：

- 1×1 pool（= 全局平均池化，得到一个向量）
- 2×2 pool（把 feature 分成 2×2 网格，每格取均值）
- 3×3 pool
- 6×6 pool

每个分支：pool → 1×1 conv 降通道 → 双线性上采样到原 feature 尺寸 → 和原 feature concat → 3×3 conv → 输出。

### ASPP vs PPM 的哲学差异（这是这组最有趣的对照）

这两个都是"多尺度上下文注入"，但**在机制层是两种不同的东西**：

|  | **ASPP（DeepLab）** | **PPM（PSPNet）** |
| --- | --- | --- |
| 算子 | 空洞卷积 | 平均池化 |
| 尺度的意义 | 采样的**半径**（看多远） | 空间被分成几份做**统计摘要** |
| 保留什么 | 保留稀疏空间信息 | 保留每个子区域的**平均响应** |
| 丢弃什么 | 丢弃大 dilation 下的连续性 | 丢弃子区域内的**所有空间细节** |
| 融合方式 | 并行分支 concat | 并行分支上采后 concat |
| 失败模式 | 网格伪影（大 dilation） | 平均池化会抹掉细节类别（小目标） |

**更深层的哲学区别**：

- **ASPP 是"稀疏扫视"**：我用不同步长的稀疏探头同时扫周围，每个探头的输出还是空间分布的。
- **PPM 是"分区统计"**：我把图切成不同粒度的块，每块归纳出一个"这块大致是什么"的摘要，再把摘要广播回去。

类比到 LLM：**ASPP 像多种步长的 sliding window attention**，PPM **更像 pool-based summary token + broadcast**（有点像 Perceiver 或者早期 reduction 的做法）。它们都在解决"每个位置要看到远处"这个问题，但 ASPP 让远处**依然是结构化的**，PPM 把远处**折成摘要**。

### 本质

PSPNet 敏锐地发现：**很多语义分割错误不是"看不清"，而是"看错场景"**——比如把水面上的船预测成车。这类错误的来源是"局部 feature 没告诉你当前是在"水 / 路 / 室内""这种场景先验。PPM 里那个 1×1 全局池化分支是**专门注入场景级先验**的。

### 效果

- Cityscapes val: 78.4 mIoU（CNN 时代非 DeepLab 线最强）。
- ADE20K val: 44.9 mIoU（2016 ImageNet Scene Parsing 冠军）。
- 比 DeepLab v2 略强，但在 v3+ 面前不占优势。PPM 的真实遗产是**"显式全局池化 + 广播"这个动作被后来的很多架构重用**——比如 segmentation transformer 里的 class tokens、object queries，都有这种"全局摘要然后广播"的影子。

## 五、HRNet —— 「别下采样再上采样，一直并行保着」

### 做了什么优化

前面所有方法都有一个共同架构：**先下采样到瓶颈，再想办法恢复**（不管是 skip、dilation、还是 decoder）。HRNet 直接质疑这个前提：**为什么非得先下采样？**

具体做法：

- Stage 1：只有一个 high-res 分支（原图 1/4）。
- Stage 2：保留 high-res 分支，**新增一个 1/8 分支并行跑**。
- Stage 3：保留 high-res 和 1/8，再新增 1/16 分支。
- Stage 4：四个分支 1/4、1/8、1/16、1/32 **全部并行持续到最后**。
- 每个 stage 末尾做**跨分辨率交换**：高 res 下采样后加到低 res，低 res 上采样后加到高 res，所有分支互相更新。

### 本质

这是对整个稠密任务架构范式的一个**哲学挑战**：

- FCN/DeepLab/PSPNet 的隐含假设：**高分辨率特征必须从低分辨率特征恢复**（先下采样出语义，再上采样出细节）。
- HRNet 的主张：**高分辨率特征从头到尾都保持**，语义不是来自"下采样的末端"，而是来自"多分辨率分支的并行交流"。

这是一次"分辨率是稠密任务的一级公民"的架构级表态。对比 encoder-decoder：encoder-decoder 承认了"高分辨率会死在瓶颈里，然后必须复活"，而 HRNet 说"不要让它死就好了"。

### 效果

- **COCO 关键点**：HRNet 直接把 AP 推到 77.0（远超 Simple Baseline），这个领域直到今天依然被 HRNet 家族主导（配合 transformer head 演化）。
- **Cityscapes seg**：81.8 mIoU（与 DeepLab v3+ 相当）。
- **Detection / Mask R-CNN 做 backbone**：+1-2 AP 稳定。

### 为什么它没有一统江湖

- 参数量和计算量都比 ResNet 高。
- 在纯语义分割上只是"和 DeepLab v3+ 打平"，没有决定性优势。
- **但它的哲学胜利了**：后来几乎所有注意力到稠密任务的架构都要处理"多分辨率表示如何共存"这个问题。Swin Transformer、SegFormer 的分层结构都是在回应"HRNet 提出的问题"，只是用了不同的算子。

## 六、SegFormer —— 「让多尺度归纳偏置被 backbone 自己吸收」

### 做了什么优化

**Mix Transformer (MiT) encoder**：一个分层 ViT。

- **Overlapping patch embedding**：和 ViT 的 non-overlap patch 不同，MiT 用**带重叠的 patch**（stride < kernel），保留局部连续性。
- **分 4 个 stage，每 stage 下采样**：stride 4/8/16/32，产出四个尺度的 feature——和 ResNet 的输出尺度完全一致，后续的多尺度消费可以无缝对接。
- **Efficient Self-Attention (SRA)**：标准 self-attention 是 O(N²)，MiT 在 K 和 V 上做 stride 降维（pool）后再做 attention，把复杂度降到 O(N²/R)。这是为了让 ViT 在高分辨率输入下活下来。
- **Mix-FFN**：在 FFN 里塞一个 3×3 depthwise conv，**隐式引入位置信息**，所以 SegFormer 不用显式 positional embedding——输入分辨率可变时直接适应。

**MLP-only decoder**：

- 每个尺度的 feature → 1×1 conv 统一通道 → 上采样到 1/4 → 四个尺度 concat → 1×1 conv → 分类头。
- **没有 ASPP、没有 PPM、没有 dilation、没有 CRF**。

### 本质

这是阶段 II 的终结姿势：**多尺度不再是 decoder 的责任，而是 backbone 的内生属性**。

回看前面所有论文：DeepLab、PSPNet、HRNet 都在 backbone 之后堆专门的多尺度模块（ASPP/PPM/HRNet 的分支交换也算）。SegFormer 说：**如果 backbone 天生就是分层的、天生就能捕捉长程依赖（self-attention）、天生就能跨尺度交互（每层都是全局 attention），那 decoder 只需要把 feature 拼起来就够了**。

更深一层：**"为什么需要 ASPP / PPM" 这个问题的根源是 CNN 的局部性先验**——CNN 的卷积核只能看局部，所以必须在 decoder 处设计专门的模块来"伪造全局视野"。一旦 backbone 换成 self-attention，看远处的代价不再主要由层数和下采样承担，而转移到 token 数、attention 稀疏化和层级设计上。全局视野不是免费的，只是代价形态变了。

### 效果

- ADE20K: MiT-B5 达到 **51.8 mIoU**（同等精度下参数量 / FLOPs 显著低于 Swin + UperNet 组合）。
- Cityscapes: 84.0 mIoU。
- **鲁棒性**：对噪声、模糊、天气扰动的退化，在若干测试设置中比 ResNet + DeepLab 更小。这是 ViT 作为 backbone 的一个重要红利，但不能离开数据、训练 recipe 和评测协议单独归因。
- 工程意义：在强预训练、高质量 dense feature、任务边界不极端复杂的条件下，**"薄 decoder + 好 backbone"成为 2022-2024 年稠密任务的重要趋势**，直接通向 Mask2Former 这类"通用 mask head + ViT backbone"架构。

## 七、六者并列比较：同一道题的六种姿势

| 方法 | 解锁感受野的手段 | 多尺度的处理位置 | 保真边界的手段 | 对"CNN 局部性锁"的态度 |
| --- | --- | --- | --- | --- |
| **DeepLab v1/v2** | atrous + CRF 外挂 | ASPP 并行 dilation | CRF 外挂 | 用扩张算子绕 |
| **DeepLab v3** | atrous + global pool | ASPP + image pool | 网络内部吃掉 CRF | 用扩张算子绕，但更稳 |
| **DeepLab v3+** | atrous | ASPP | 轻量 decoder | 用扩张算子绕 + 对称结构补 |
| **PSPNet** | global pool | PPM 多粒度分区池化 | 无专门处理 | 用"分区摘要"绕 |
| **HRNet** | 多分辨率并行 + 交换 | 分支交换即多尺度 | 高 res 从未丢失 | **重构架构来回避** |
| **SegFormer** | self-attention | 分层 Transformer 天生多尺度 | 极简 MLP decoder | **换算子来解锁** |

这张表里有一条暗线：前四个都在 CNN 的局部性、分辨率和感受野绑定中重分配代价（dilation 是稀疏采样，PPM 是摘要回注，HRNet 是并行保真）。SegFormer 则换成分层 Transformer，把多尺度更多前移到 backbone 内部；从此“专门多尺度 decoder 必须很重”这件事开始松动。

换句话说，这组里可以标出两次更深的结构转向：

- 一次是 HRNet 在**架构范式**层面质疑"必须先下采样"；
- 一次是 SegFormer 在**算子和预训练接口**层面把全局依赖与多尺度组织前移到 backbone。

其他的（DeepLab 三代、PSPNet）都是同一个锁里的精巧绕行——它们工程价值极高（v3+ 至今仍是 CNN 时代最强 baseline），但在你"第一性原理谱系"的尺子下是**同一层级的兄弟**，不是**不同层级的阶梯**。

---

## 八、这一组留下的内部小残差 + 指向阶段 III 的出口

从这张表能清楚看到**阶段 II 的自我终结点**：

- DeepLab 系的 atrous 路线最终把自己做到了**必须和 encoder-decoder 合并**才能再进步（v3+）。
- 之后几年纯 CNN 的语义分割几乎没有新节点，直到 HRNet 提出"重构而非修补"，但 HRNet 也没能拿下决定性优势。
- SegFormer 用 Transformer **直接让多尺度成为 backbone 的内生属性**，专门的多尺度 decoder 模块从此退场。

这是一个非常干净的范式闭环：**问题被提出（FCN/U-Net 留下的感受野残差）→ 在 CNN 主导范式内多次重分配代价（DeepLab 三代 + PSPNet + HRNet）→ 换算子与预训练制度后，decoder 压力显著下降（SegFormer）**。SegFormer 站在这一组最后一位，已经把稠密视觉送到更强 backbone 与更开放接口的门口。

阶段 II 真正没解决、必须交给下一组的残差只有一条：**"语义 ≠ 实例"**。这条 DeepLab/PSPNet/HRNet 都绕不开——因为它们的任务定义就是"每个像素一个类别"，同一类里的两个对象本质上无法区分。要进一步共形，必须引入"实例"作为一级原语——这是下一组（检测流水线化）要接的棒。

---
