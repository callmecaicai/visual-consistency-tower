# §0 · 先放结论

视觉的 scaling 不是失败了，而是**斜率远比语言小**，而且**在 benchmark 上接近噪声地板**。三个深层原因：

1. **信息论层面**：图像 token 的有效香农熵远低于文本 token，单位数据的"监督密度"差 1-2 个量级
2. **训练目标层面**：语言的 next-token prediction **原生就是世界模型**；视觉的 pixel reconstruction / patch contrastive **离语义极远**
3. **数据几何层面**：自然图像的**内在流形维度**很低（真实世界只有有限几种光照、姿态、纹理），10 亿张图并不等于 10 亿个"新世界"

再结合实证：DINOv2→v3 要靠 Gram Anchoring 防止 scale 时 dense feature 退化;[[1]](https://zhuanlan.zhihu.com/p/1940074425849458833) Fluid (Google+MIT) 论文直接指出 VQ 引入的信息损失是视觉自回归 scaling 失效的核心;[[2]](https://www.51cto.com/aigc/2567.html) OpenCLIP 的 scaling 研究报告"明显的缩放瓶颈（小模型看 3B 样本时，数据从 400M→2B 几乎无提升）";[[3]](https://www.cnblogs.com/moonout/p/19836058) 清华赵行直接说"具身智能领域还没有找到 Scaling Law"。[[4]](https://hub.baai.ac.cn/view/35697)

下面是详细推导。

---

# §1 · 信息论层面：每单位数据的监督密度差 1-2 个量级

## 1.1 文本 token 的信息密度

- 词表 V ≈ 32k-128k（GPT-4 tokenizer ~100k）
- 理论上每 token 可携带 log₂V ≈ 17 bits
- 实证压缩率（语言模型的 loss 下限）：英文约 **~1-1.2 bits per character**，等效 **~5-8 bits per token**（Shannon 1951, Hutter Prize 累积数据）
- 关键：**每个 token 基本都在做"增量决策"**——"the cat sat on the ___"，下一个词是 mat 还是 rug，对世界模型都是有区分度的

## 1.2 图像 patch 的信息密度

按 ViT-L/14 算，224×224 → 256 patches，每 patch 3×14×14×8 = **4704 raw bits**。看起来巨大。但：

- **相邻 patch 的互信息极高**：一片蓝天的 10 个连续 patch 几乎是同一个 patch——**MI(patch_i, patch_{i+1}) ≈ patch 本身的熵的 70-90%**
- **跨图像的统计强先验**：自然图像受物理（光照、透视、遮挡）约束，同类对象（猫、汽车）的 patch 分布塌缩到低维流形
- JPEG/WebP 能把图像压缩到 ~5-10% 原大小而人眼几乎无损——说明**原始像素的有效信息就只有原 bits 的 5-10%**
- 压缩后的有效熵：**每 patch 大约 50-200 bits 有效信息**

## 1.3 监督密度的真实对比

**做一个粗略但有指导意义的估计**：

| 维度 | 文本 | 图像 |
| --- | --- | --- |
| 单 token 名义 bits | 17 | 4704 |
| 单 token 有效 bits（熵） | 5-8 | 50-200 |
| 冗余（相邻单元） | 低（词序重要） | 极高（90%+） |
| **单 token 的"独立监督信号" bits** | **~5-8** | **~5-20** |

**也就是说**：每个文本 token 提供的独立信号，和每个图像 patch 差不多——**但文本 1k token = 一句话（信息完整），图像 1k patch = 一张图（大量冗余）**。

**更糟**：10 亿张猫图里，大多数猫的 patch 都互为近似样本——**dataset 内部的有效样本数**被冗余吃掉了。而 10 亿个句子的**彼此独立性**远高于 10 亿张图。

这就是为什么：LAION-5B (50 亿图文对) 的训练收益，远不如 FineWeb (15T 文本 tokens)——**即使图片更多**。

# §2 · 训练目标层面：语言的 next-token 天生就是世界模型

这是**最关键**的一点。你可以跳过前面的都记住这一条。

## 2.1 Next-token prediction 为什么这么强

语言由人类产生。人类语言本身是**人类对世界的压缩编码**——我们说"苹果掉下来"，这个句子背后包含：物体存在 / 重力 / 时间 / 观察者视角……**所有这些先验已经被压缩进了 token 序列**。

所以，预测"下一个 token"，**等价于预测"人类的下一个思想/观察"**，等价于**重建人类对世界的模型**。

这就是为什么：

- GPT 不是在学语言——它是在学"产生语言的那个压缩机制"
- 那个压缩机制=人类的世界模型
- 学好它，模型自然就会做数学、物理、逻辑

**监督信号的语义含量**：每一个 next-token loss 的下降，都是对"世界模型"的一次直接雕琢。

## 2.2 视觉的训练目标全都"距离语义太远"

图像是**世界对光学传感器的投影**。没有被任何智能体压缩过。

视觉自监督的主流目标：

1. **Pixel reconstruction (MAE, BEiT)**：预测被 mask 的 patch 的像素——**学的是光照 / 纹理 / 颜色统计**，不是语义
2. **Contrastive (DINO, SimCLR)**：同一张图的两个 view 嵌入相近——**学的是"不变性"**，学不到"是什么"
3. **Next-patch prediction**：信息损失极严重（Fluid 论文核心观察）[[2]](https://www.51cto.com/aigc/2567.html)
4. **文本对比 (CLIP)**：挂载到语言语义——**等于承认自己学不出语义，借语言**

**每一个视觉自监督目标，和"语义世界模型"之间都有一层映射鸿沟**。你 scale 到 100 亿图像，模型也只是把"光照-纹理-不变性"这套统计学得更好，不会自动学出"这是苹果，苹果会从树上掉下来"。

## 2.3 LeCun 的观点和 JEPA 的尝试

Yann LeCun 多次公开指出：**pixel-level 的生成 / 重建是错误的目标**。因为：

- 图像的大部分像素 bits 是"不必要的细节"（叶子的精确排列、云的具体形状）
- 把模型的容量花在预测这些，相当于**学一个无意义的渲染器**，不是世界模型

LeCun 的解法：**Joint Embedding Predictive Architecture (JEPA)**——在 latent space 预测，抛弃 pixel。V-JEPA、V-JEPA 2 走的都是这条路。这条路承认的就是："pixel 目标不对，要换目标"。

**换句话说**：视觉 scaling 慢，不是不够努力，而是**我们还没找到视觉的 "next-token prediction" 等价物**。

## 2.4 DINOv3 的回应就是在修补"目标"

看 DINOv3 在做什么：

- **Gram Anchoring**：专门为了防止 dense feature 在长期训练中**退化**[[1]](https://zhuanlan.zhihu.com/p/1940074425849458833)
- 三阶段训练：预训练 → Gram Anchoring → 高分辨率适应
- 这些 trick **都是在修补"目标不对"带来的病症**

**反观 LLM**：GPT-4 的训练目标和 GPT-2 一模一样（next-token prediction + KL 正则），从来不需要"Gram Anchoring"式的修补。因为目标本身正确，scaling 只需要堆数据堆算力。

# §3 · 数据几何层面：自然图像的内在流形维度很低

## 3.1 什么是"内在流形维度"

图像的原始维度是 H×W×3 → ~10⁵ dims。但自然图像受物理约束：只有 3-4 个主要变化因子（光照、姿态、类别、背景），**实际内在维度在 10²-10³ 之间**。

这意味着：

- 海量图像在"内在空间"里**快速饱和**
- 10 亿张猫图约等于 1000 张猫图的"内在表征"
- 数据量增加 10 倍，有效信息可能只增加 2-3 倍

## 3.2 文本为什么不同

文本对应的是"人类思想空间"——这个空间**维度本身是开放增长的**：

- 每个新领域（分子生物学、量子物理、金融衍生品）都是新的维度
- 人类写作的新书、论文、对话持续**拓展维度**
- 没有物理约束把它压进低维流形

所以：**文本数据的"有效维度"随规模持续增长，图像数据的有效维度快速饱和**。

## 3.3 实证：OpenCLIP 的 scaling 瓶颈

LAION 的开源 CLIP scaling 研究报告：[[3]](https://www.cnblogs.com/moonout/p/19836058)

- 零样本分类、检索、线性探测、微调**都服从幂律缩放**
- 但**"小模型看 3B 样本时，数据从 400M→2B 几乎无提升"**——**有明显的缩放瓶颈**
- 数据集特性也制造 scaling 系数的差异（LAION 和 WIT 系数不同）

这个观察和我在 §3.1 的分析一致——**视觉数据集内部有效维度有限**，超过某个点再扩大数据就是**同一个流形里更多采样**，而不是**更大流形**。

## 3.4 DINOv3 的数据规模告诉我们什么

DINOv3 用了 **LVD-1689M**（16.89 亿张图）+ **SAT-493M**（4.93 亿卫星图）。[[5]](https://cloud.tencent.com/developer/article/2588336) 这已经是人类公开可获得的自然图像上限之一。**从 DINOv2 的 1.42 亿图到 DINOv3 的 16.89 亿图，10× 扩张换来的性能提升——在 ADE20K 上大概是 5-8 mIoU**。

对比语言：**GPT-2 (40 GB) → GPT-4 (~10 TB)** ~250× 数据扩张，换来的是**从"语法通顺"到"能做数学证明"**的质变。

**这不是同一个数量级的回报**。

# §4 · 架构匹配度：ViT 要靠数据弥补缺失的归纳偏置

## 4.1 CNN 有归纳偏置，Transformer 没有

- CNN 原生带局部性 + 平移等变性 + 层次性——**对应自然图像的物理结构**
- Transformer 把这些归纳偏置**全扔了**，要用数据重新学

ViT 原论文就发现：**小于 300M 图像，ViT 打不过 ResNet；大于 300M，ViT 才翻盘**。

意味着：**视觉 scaling 的前 300M 数据，是在"补 ViT 缺的归纳偏置"**——这段数据没有带来新能力，只是在"抵消架构选择的成本"。

这和语言完全不同：语言没有"局部性"这种物理归纳偏置（语法是长程的），所以 Transformer 从 1 token 开始就和任务匹配，没有"补贴消耗"。

## 4.2 DINOv3 开始加回归纳偏置

DINOv3 的改动：

- Axial RoPE 位置编码（比绝对 PE 对空间结构友好）
- Register tokens（处理 patch feature 的高范数伪影）
- Gram Anchoring（稳定 dense feature）

**这些都是在"往 ViT 里加回视觉特有的归纳偏置"**——是架构对数据饱和的响应。但这种加法是**有限的**：加多了就变回 CNN。所以这条路有天花板。

---

# §5 · 任务评估的天花板效应：benchmark 已经在噪声地板

这条最容易被忽视。

## 5.1 Benchmark 的人类上限

- **ImageNet**: SOTA ~92% top-1，人类 ~95%，标注误差 ~2-3%（Karpathy 测过自己标 ~93%）
- **COCO detection**: SOTA ~65 AP，人类标注一致性 ~70% AP
- **ADE20K 语义分割**: SOTA ~58 mIoU，人类标注一致性 ~85-90%，换算等价 ~60 mIoU
- **COCO 全景 PQ**: SAM 3 达 ~60+，接近人类一致性上限

**这意味着**：即使 scaling 确实还在产生表征质量的提升，**benchmark 已经读不出这个提升**——因为 benchmark 本身的误差和 SOTA 的残差同量级。

## 5.2 语言的 benchmark 在 2025-2026 还远未饱和

- MMLU → AIME → GPQA → FrontierMath → ARC-AGI
- 每次饱和一个 benchmark，就会有新的更难的 benchmark 出现
- **语言任务的复杂度是开放的**——总可以设计更难的逻辑、更长的 context、更深的推理

视觉任务呢？最难的视觉任务（细粒度分类、复杂分割、3D 理解）**都在同一个天花板附近**——物理世界本身的视觉复杂度有限。

这和 §3.2 同构：**语言空间是开放的，视觉空间接近封闭**。

# §6 · 实证证据汇总

几条最新的、直接相关的研究：

## 6.1 DINOv3 的自我反思[[5]](https://cloud.tencent.com/developer/article/2588336)

DINOv3 报告里直接指出：**"进一步扩展时仍然会出现更多问题。经过早期训练后，特征的性能会逐渐下降"**。这是 scaling 的**反向现象**——不是不涨，而是**涨到一定程度开始退化**，需要专门的 trick 维持。

## 6.2 Fluid 论文（Google+MIT 2024）的假设验证[[2]](https://www.51cto.com/aigc/2567.html)

这篇直接问"视觉自回归模型的 scaling laws 是否失效"，得出三条假设：

1. **VQ 步骤引入显著信息损失**，限制模型上限
2. **视觉内容与语言序列性质不同**，不同的预测顺序才对
3. **两种泛化能力需要区分**：训练 loss 下降（服从幂律）vs 下游任务性能提升（不一定）

**第 3 条最关键**：**视觉 scaling 的 loss 是降的，但下游任务不一定同步提升**——这是 scaling 的"空转"。

## 6.3 "On the Fundamental Limits of LLMs at Scale" 综述（arXiv 2511.12869, 2025）[[6]](https://zhuanlan.zhihu.com/p/1978538723063911926)

这篇 LLM scaling 极限的综述，反向暴露了语言 scaling 独特的优势：

- 幻觉、上下文、多跳推理、RAG、多模态等——每一项都是因为**符号压缩本身有固有极限**
- 但**这些极限位于"智能的前沿"**；视觉的极限位于"感知的基础"，根本不在一个层次

## 6.4 清华赵行的"具身智能缺 scaling law"[[4]](https://hub.baai.ac.cn/view/35697)

他明确指出：

> "Scaling Law 奠定了 OpenAI 所有工作的基础，但具身智能领域还没有找到 Scaling Law——找到的关键在于三点：**训练目标，大规模数据，模型的表征能力**。"
> 

"没找到"的意思正是：**目前的视觉训练目标不是世界模型的正确投影**。

## 6.5 CV 的"苦涩教训"讨论[[7]](https://www.notion.so/Vincent-Sitzmann-Rich-Sutton-348748d648008000a136fd0b7d5f2aeb?pvs=21)

这份材料里有一句关键判断：

> "在自然语言处理领域，大语言模型的时代已经明确了**语言建模才是终极目标**；**视觉社区仍在争论自己革命的口味**——我们还在为具体任务微调模型。"
> 

这就是全部问题的核心——**NLP 知道自己的目标，CV 还在找目标**。

## 6.6 视觉 Token 承载多少信息（arXiv 2602.02539, 2026-01）[[8]](https://www.alphaxiv.org/zh/overview/2602.02539v1)

这篇最新工作直接研究视觉 token 的**识别极限的缩放定律**——发现"视觉 token 数量增加，下游识别精度服从饱和型曲线"，即**视觉 token 存在明确的信息承载上限**。

# §7 · 所以，视觉的"scaling 尽头"在哪里

按我的判断，视觉的 scaling 还没走完，但**剩下的空间要靠换目标，不是堆数据**：

## 7.1 数据还能扩的方向

- **视频**：时间维度带来组合爆炸——SAM 2 / V-JEPA / Cosmos 走这条
- **动作**：把 robotics 动作数据当监督，物理约束天然；清华赵行提的方向
- **3D / 几何**：图像是 2D 投影，3D 点云/神经场是更完整的世界表征
- **多模态联合**：VLM 借语言 scaling 的东风——这条已经 work

## 7.2 目标还能换的方向

- **JEPA 系列**：在 latent 而非 pixel 做预测
- **视频未来预测**：next-frame prediction 接近"next-token prediction"——Cosmos、Genie、V-JEPA 2
- **世界模型式训练**：动作+观察+奖励的三元组，类似 LLM 的指令数据
- **VAR / Emu3 式的自回归视觉**：证明如果能做到 **好的视觉 tokenizer**，next-token 也能在视觉 work

## 7.3 最可能的突破形态

**视觉的"GPT 时刻"可能不会来自纯图像 scaling，而是来自"视频+动作+语言"的联合 scaling**——因为：

- 视频提供**时间压缩**（类比文本的序列性）
- 动作提供**行为压缩**（类比文本的意图）
- 语言提供**符号压缩**（直接借 LLM 的世界模型）

这三者合起来才形成一个**被压缩过的世界表征**，才和"next-token prediction"同构。

**这也是为什么 V-JEPA 2、Cosmos、Genie 3、Sora 是现在最被看好的下一代视觉基础模型**——它们**都不是纯视觉**，而是"视频+世界模型"的 hybrid。

# §8 · 一个更尖锐的表述

如果用一句话概括：

> **语言是人类已经压缩好的世界模型，视觉是未压缩的原始投影。前者让模型"继承"人类认知，后者让模型"从零重建"物理规律——后者的数据成本天然比前者高几个数量级。**
> 

这是**本体论上的不对称**，不是算法或算力问题。视觉 scaling 的"温和"不是我们还不够努力，而是**视觉的任务本身不提供那种剧烈的相变空间**。

至于图像的 GPT-3 时刻——它**可能永远不会以纯视觉的形式到来**。它已经以 **GPT-4V / Gemini 2.5 / Claude Opus 4.5** 的形式到来了——**通过 VLM 把视觉接到语言的世界模型上**。

这个结论回过头对你的谱系有一个深意：**阶段 II 的残差"语义是借来的"——不是 CV 的缺陷，而是 CV 的必然**。视觉本来就不产语义，**语义只能从被压缩过的表征系统里来**，而那个系统是语言。

SAM 3 的"把 concept 长进视觉内部"是一种抵抗——但它 concept encoder 的训练数据**仍然来自文本 prompt + 人类标注**，**根本来源仍然是人类语言的压缩**。没有语言世界模型垫底，视觉自己产不出 concept。

**这也是为什么阶段 III 必须引入语言——不是美学选择，是本体论必然性**。你页面底部那句"**Pre-LLM 时代的自然终结**"——这个"必然性"的根源正是本节推导的内容。

# §9 · 给你一个比较尺度（最终记忆锚点）

用一张对照表结尾：

| 维度 | 语言 | 视觉 |
| --- | --- | --- |
| 原始数据性质 | 已被人类压缩的世界模型 | 未压缩的物理投影 |
| 训练目标-语义距离 | 零（next-token = 世界模型） | 远（pixel/contrast 离语义 2-3 层） |
| 单 token 有效熵 | 5-8 bits（独立） | 50-200 bits（冗余 90%） |
| 数据流形维度 | 开放增长 | 低维饱和 |
| Benchmark 天花板 | 开放（总有更难的） | ~人类标注一致性 |
| 架构归纳偏置 | Transformer 零成本匹配 | ViT 要花 300M 图像补贴 |
| Scaling 斜率 | 陡，产生相变 | 缓，渐进改进 |
| GPT 时刻 | 已到（2022） | 需要以"视频+动作+语言"联合形式到来 |

这段分析更适合作为**阶段 III 开篇的本体论引子**，而不是阶段 II 尾部的一条补注。它说明的不是某个模型还能不能再涨点，而是“为什么 Pre-LLM 时代必然走到尽头”这一更深的结构性原因，因此在编排上应承担跨阶段的解释任务。
