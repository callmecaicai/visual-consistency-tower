# 两个底层思路的第一性原理拆解 + SegFormer 的深层讲解

## 一、PPM 的深层遗产：「聚合 → 广播」是一个原语

### 把 PPM 抽象成一个最小算子

抛开 "pyramid" "pooling" 这些名词，PPM 做的其实是两步：

$\text{aggregate}: \mathbb{R}^{H\times W\times C} \longrightarrow \mathbb{R}^{k\times C}$

$\text{broadcast}: \mathbb{R}^{k\times C} \longrightarrow \mathbb{R}^{H\times W\times C'}$

- **aggregate**：把 H×W 的空间**折叠**成 k 个摘要向量（k=1 是全局，k=4 是 2×2 网格，k=9 是 3×3 网格⋯⋯）。这是一次**信息瓶颈**。
- **broadcast**：把 k 个摘要**再分发**回每个空间位置（原始 PPM 用双线性插值分发）。这是一次**反向分布**。

这个 "折叠-分发" 的二元循环，**不是池化的专利，是一个几乎可以独立命名的架构原语**。一旦把它看成原语，就会发现它在后来的很多关键算子里以不同皮肤反复出现。

### 它的家族谱

| 算子 | aggregate 是什么 | broadcast 是什么 | k 的含义 |
| --- | --- | --- | --- |
| **PPM (2017)** | 静态平均池化（固定网格） | 双线性插值 | 固定（1, 4, 9, 36） |
| **SENet (2018)** | 全局平均池化 → MLP | 逐通道乘法 | k=1（通道注意力） |
| **Non-local (2018)** | 每个位置 attention-weighted 聚合 | attention-weighted 分发 | k=N，动态 |
| **ViT class token (2020)** | class token attend 到所有 patch | 其他 patch attend 回 class token | k=1，可学习 |
| **DETR object queries (2020)** | N 个 query 各自 cross-attend 图像 | query 再解码出 box/mask | k=N_obj，可学习 |
| **Perceiver (2021)** | fixed latents cross-attend 输入 | latents 再 cross-attend 回输出 | k=固定，可学习 |
| **Mask2Former mask queries (2022)** | mask query 聚合 pixel features | query · pixel features → mask | k=N_mask，可学习 |

### 第一性原理：为什么这个原语如此顽强

它在做的事本质上是**解决一个信息几何问题**：

> 稠密预测的输出是 N 个位置的独立决策，但这些决策的正确性依赖一个 **M ≪ N** 维的场景状态（"这是室内""这是一群车""焦点在左上"）。**算子需要一种方式让 M 维状态免费广播到 N 个决策**。
> 

有两种办法解决：

- **暴力法**：让每个位置单独去"看"全图（self-attention 的 O(N²)）。
- **瓶颈法**：先压成 k 维（k ≈ M），再广播（O(N·k)）。

**PPM 是瓶颈法的静态版**（k 固定、aggregate 非参数、broadcast 非参数）。

**cross-attention 是瓶颈法的动态版**（k 可学习、aggregate 参数化、broadcast 参数化）。

从 PPM 到 object query，**本质上就是把这个原语从"静态、硬编码、无参数"升级到"动态、软性、全可学习"**。object query 不是从天而降的新概念，它就是"可学习的金字塔池化柱"——只是柱的位置、形状、内容都让网络自己决定。

这个视角的实际用处：当你看 SAM 的 prompt encoder、Mask2Former 的 mask query、Perceiver IO 的 output query、甚至 LLM 里的 soft prompt，你看到的其实都是**同一个原语在不同载体上的复刻**。PPM 的那颗 1×1 全局池化种子，在 2017 年就埋下了这个范式。

### 一个更锋利的说法

> **凡是需要"全局结构介入每个局部决策"的场景，最终都会演化到某种「低维 token 作为广播源」的架构**。PPM 是这个洞察的第一次原始形态，object query 是它的成熟形态。
>

## 二、HRNet 的哲学胜利：「多分辨率共存」是一个问题，不是一个答案

### HRNet 提出的问题（而不是它的解）

HRNet 的真正贡献**不是它的具体架构**——事实上它的具体架构（并行四支 + 跨支交换）在今天几乎没人直接用。它真正贡献的是把一个隐藏假设拖到明面：

> **CNN 范式把「深度 / 语义 / 低分辨率」这三件事耦合到了一条流水线上——加深就得下采样，下采样就意味着低分辨率。这三件事有必要耦合吗？**
> 

把这个问题挑明之后，后来所有稠密架构都必须回答它。不同架构的区别就变成**"你对这三者耦合关系的立场是什么"**。

### 三件事的解耦谱系

把"深度 / 语义 / 分辨率"这三个轴画出来：

| 架构 | 深度轴 | 语义轴 | 分辨率轴 | 耦合关系 |
| --- | --- | --- | --- | --- |
| **VGG / ResNet** | 单条流水 | 语义随深度增长 | 分辨率随深度降低 | 三者**强耦合** |
| **FCN / U-Net / DeepLab** | 单条流水 + 恢复头 | 同上 | 末端恢复 | 耦合同上，加个"补救头" |
| **HRNet** | 多条并行流水 | 语义随深度增长，但**在每个分辨率上都有**一条 | 各分辨率**独立保持** | **分辨率与深度解耦**，靠跨支交换共享语义 |
| **Swin / SegFormer** | 单条分 stage 流水 | 语义随深度增长 | stage 间下采样 | **看似耦合，但每个 stage 内部有全局 attention，所以下采样不再是"语义代价"** |

### HRNet 和 Swin/SegFormer 的真正关系

表面看：Swin/SegFormer 回到了"先下后上"的分层结构——这看上去像**反 HRNet**。实际上它们是**绕过 HRNet 的问题而不是正面回答**。关键差别在这里：

**HRNet 被迫保持高分辨率的原因**，是 CNN 的局部算子让它拿不到全局上下文——高分辨率一旦丢了就永远丢了。**所以它必须让高分辨率分支从第一层一直活到最后**。

**Swin/SegFormer 可以放心下采样**，是因为 attention 算子在**任何一个 stage 内部**都能跨越整个 feature map 建立依赖。高分辨率不需要一路保持——只要在**早期 stage 中有过**，后面的全局信息依然能在语义判定中引用。

换一个更锋利的说法：

- **HRNet 的多分辨率并行，是 CNN 的"障碍性解决方案"**——不保高分辨率就做不好稠密任务。
- **Swin/SegFormer 的分层下采样，是 Transformer 的"解禁方案"**——换了算子之后，高分辨率丢了也没事，因为全局依赖不靠分辨率保持。

所以 HRNet 的哲学胜利是**把这个隐藏耦合暴露出来**；但它自己给出的解（物理上保住分辨率）其实是被算子约束逼出来的次优解。真正的答案是：**换算子后，耦合自然松解，根本不需要物理保住分辨率**。

### 第一性总结

HRNet 的问题提纲比它的解决方案更长寿。它的问题是：

> **稠密任务要求每个像素的输出既包含"远"（场景上下文）又包含"近"（边界细节）。这两件事必须在同一个表示里共存。你的架构如何让它们共存？**
> 

三种合法回答：

1. **物理并行保持**（HRNet）——多分支并行跑不同分辨率，互相交换。
2. **分层 Transformer**（Swin/SegFormer）——每个 stage 内部靠 attention 搞定"远"，stage 间下采样搞定"近→远"的抽象。
3. **全保高分辨率的 ViT**（原始 ViT + 大 patch）——每个 patch 都和所有 patch 交互，不下采样。

选 1 是 CNN 时代的最高答卷；选 2 是 2021 年后的主流；选 3 是计算力足够时的奢侈方案（SAM 就走这条）。HRNet 的遗产是**让这个选择题被正式提出**。

## 三、SegFormer：「让 backbone 吃掉专用模块」的证明

### 先把"为什么它能这么简单"讲透

SegFormer 的论文价值观可以浓缩成一句话：

> **"好 backbone + 弱 decoder" > "平庸 backbone + 强 decoder"。如果前者成立，阶段 II 后半段（ASPP / PPM / HRNet 的分支交换）都不再必要。**
> 

论文本身由两部分组成：**MiT (Mix Transformer) backbone** 和 **All-MLP decoder**。全部功夫在 backbone 上，decoder 刻意保持平庸。这个设计不是为了炫技简洁，是**用刻意弱的 decoder 反证 backbone 的强度**——decoder 越弱，越能证明 backbone 已经把所有活干了。

### 3.1 MiT (Mix Transformer) 的四个关键工程修正

原版 ViT 不能直接用在稠密任务上，有三个致命缺陷：

- **缺陷 A**：单一 patch size，没有多尺度 feature（ResNet 给 4 个尺度，ViT 只给 1 个）。
- **缺陷 B**：self-attention 是 O(N²)，高分辨率下爆显存（1024×1024 输入、patch=16 → N=4096，attention 矩阵 16M 项 × B 个 batch × h 个头 × L 层）。
- **缺陷 C**：依赖 positional embedding，换分辨率就崩。

MiT 对每个缺陷都做了一个精准的局部修正：

### (a) Overlapping Patch Embedding（补 CNN 的局部连续性）

原版 ViT 把图切成不重叠的 16×16 patch，然后每个 patch 独立线性投影。问题：**相邻 patch 之间没有任何共享信息**，一个物体如果恰好落在 patch 边界就被分割成两个 token 分别处理。

MiT 的做法：用 **stride < kernel 的 conv** 实现 patch embedding。第一层用 kernel=7、stride=4、padding=3，所以每个 patch 和相邻 patch 在空间上**重叠 3 个像素**。

这是一次**对 CNN inductive bias 的主动引入**——承认"相邻像素应该共享信息"是一个有用的先验，不要因为赶时髦用纯 ViT 就丢掉。

### (b) Hierarchical 4-stage 结构（补 ResNet 的多尺度输出）

MiT 有 4 个 stage，每个 stage 开头一次 overlapping patch merge（stride=2 的 conv），分辨率砍半。输出四个尺度：**1/4, 1/8, 1/16, 1/32**——**和 ResNet 完全一致**。

这个对齐不是偶然：它让 MiT 可以直接替换任何用 ResNet 做 backbone 的 dense prediction 方法（Mask R-CNN、DeepLab、UperNet 都能无缝对接）。**这是一次 API 级别的兼容设计**，让社区迁移无痛。

### (c) Efficient Self-Attention (SRA)（把 O(N²) 压下来）

这是最关键的数学修正。标准 self-attention：

$\text{Attn}(Q, K, V) = \text{softmax}(QK^\top / \sqrt{d}) V, \quad Q,K,V \in \mathbb{R}^{N\times d}$

复杂度 O(N²·d)。MiT 借鉴 PVT 的做法——**不对称地压缩 K 和 V**：

$\hat K = \text{Reshape}(N/R^2, R^2\cdot d)(K), \quad \hat K = \text{Linear}(R^2 d, d)(\hat K)$

同样处理 V。Q 保持不变。attention 变成：

$\text{Attn}(Q, \hat K, \hat V), \quad Q\in\mathbb{R}^{N\times d},\ \hat K, \hat V\in\mathbb{R}^{(N/R^2)\times d}$

复杂度 O(N²·d / R²)。在早期 stage N 大（N = H/4 × W/4），用大 R（=8）压得狠；深层 stage N 小，R 降到 1。

**机制注解**：这个操作看起来像工程 hack，但它实际上**复活了前面讨论的"aggregate-broadcast 原语"**：

- Q 保持 N 个位置（每个位置独立提问）。
- K 和 V 被空间下采样（在局部窗口内池化）——**这是 aggregate**。
- attention 把压缩后的 K/V 的信息分发回 N 个 Q——**这是 broadcast**。

所以 SRA 本质上是**把标准 self-attention 改造成"aggregate + broadcast"**。这和 PPM 的精神完全一致，只是从静态（平均池化 + 插值）升级到了动态（query-dependent attention）。这是 PPM 那个种子在 Transformer 时代的直系后裔。

### (d) Mix-FFN 代替 Positional Embedding（用 conv 偷偷编码位置）

原版 ViT 的 FFN：`Linear → GELU → Linear`。MiT 改为：

```
Mix-FFN: Linear → DWConv 3×3 → GELU → Linear
```

那个 3×3 depthwise conv 在 FFN 中间偷偷做了一件关键的事：**它让每个 token 的 FFN 输出依赖于它的 8 个空间邻居**。

为什么这能替代 PE？因为 attention 是置换等变的（交换两个 token 的位置不会改变结果），所以必须有某种方式告诉网络"位置"。PE 是显式编码。Mix-FFN 是**通过卷积核的局部性隐式编码**——每个 token 知道"谁是我的邻居"就知道自己的位置。

**好处**：**任意分辨率输入都能无痛使用**。不需要 PE 插值、不需要为不同分辨率训练不同 PE。这对稠密任务至关重要——实际部署时输入分辨率千变万化（从 512×512 到 2048×2048 都有）。

这也是第二次对 CNN inductive bias 的主动引入：**用 conv 补 attention 的无序性**。

### 3.2 All-MLP Decoder：故意让它平庸

Decoder 的全部操作：

```
For i in {1,2,3,4}:  # 四个 stage 的 feature
    F_i = MLP(C_i → C)(F_i)        # 1×1 线性统一通道
    F_i = Upsample 1/4(F_i)        # 双线性上采到 1/4

F = Concat([F_1, F_2, F_3, F_4])   # 拼起来：4C 通道
F = MLP(4C → C)(F)                 # 降回 C
F = MLP(C → N_cls)(F)              # 分类头
F = Upsample 原图(F)
```

**全部操作**：4 个 1×1 linear + 一次 concat + 2 个 linear + 几次双线性上采样。**没有 conv 3×3、没有 dilation、没有 ASPP、没有 PPM、没有 attention**。

这个 decoder 弱到什么程度？它**连相邻像素的空间关系都不建模**——所有操作都是逐像素的 linear。空间关系全部托付给 backbone 中的 attention 做过。

这是一个反证法：**如果这么弱的 decoder 能打败 DeepLab v3+ 那种精心设计的 decoder，那只能说明 backbone 已经把一切都做了**。

### 3.3 效果（要诚实地看）

- **ADE20K val**：
    - SegFormer-B0（3.8M 参数）：37.4 mIoU
    - SegFormer-B5（84.7M 参数）：51.8 mIoU
    - 对比 DeepLabV3+ + ResNeXt-101 (62M)：46.4 mIoU——**同等参数下大幅领先**
    - 对比 Swin-L + UperNet (234M)：53.5 mIoU——**SegFormer-B5 用 1/3 参数打到 97%**
- **Cityscapes test**：83.1 mIoU（当年 SOTA 水平）
- **鲁棒性（Cityscapes-C）**：对 15 种 corruption 的平均 mIoU 保持能力比 DeepLabV3+ 高约 **10 mIoU**。这是 ViT-based backbone 的一个意外红利——self-attention 的全局视野让它对局部 corruption 比 CNN 稳定得多。

### 3.4 SegFormer 在谱系里的真正位置

它的重要性不在**具体分数**，而在**范式宣言**：

> **当 backbone 的算子内生支持"长程依赖 + 局部连续 + 多尺度分层"时，阶段 II 积累的所有专用稠密模块（ASPP/PPM/HRNet fusion/CRF）都变成历史遗物。**
> 

这个宣言一旦成立，稠密任务的研究重心从"如何设计更好的 decoder 模块"彻底转向"如何设计更好的 backbone"。后续两年的发展完全验证了这个转向：

- **Mask2Former (2022)**：把 SegFormer 的"弱 decoder"进一步抽象成 "mask query decoder"——decoder 模块化、backbone 可替换。语义 / 实例 / 全景统一。
- **SAM (2023)**：ViT-H backbone + 一个极简 mask decoder + prompt encoder。稠密任务从此"被 prompt 化"。
- **EoMT (2024)**：发现连 mask decoder 都可以被 encoder 内部的 attention 吸收——**连 "decoder 这个概念" 本身都开始消失**。

所以 SegFormer 是一个**范式分水岭**。在它之前，稠密任务的进步靠"更好的多尺度模块"；在它之后，进步靠"更好的 backbone + 更通用的输出协议"。阶段 II 终止于它手中，阶段 III 从它身后升起。

### 3.5 诚实的局限

- **SRA 是近似的全局 attention**，不是真正的 full self-attention。对真正长程依赖（跨图两端）建模不如原版 ViT。
- **小模型（B0/B1）下优势有限**，因为 decoder 过于弱，backbone 不够强时撑不起来。
- **没有实例概念**，还是纯语义分割。实例、全景、开放词汇这些都是下一组的事。
- **Mix-FFN 的 3×3 conv 是一种妥协**——承认了 pure ViT 在稠密任务上不够好。这是 "纯 Transformer 胜利"的一个工程性削弱。

## 尾声：这一组的真正终点

回头看，SegFormer 的存在让你能清晰看到阶段 II 这组的**真正闭环逻辑**：

1. FCN/U-Net 留下"感受野-分辨率"二难。
2. DeepLab 用 atrous 绕。
3. PSPNet 用 "aggregate-broadcast 原语" 绕——这个原语会在未来成为 object query 的祖先。
4. HRNet 用"多分辨率并行"绕——它把耦合问题挑到明面。
5. SegFormer 换了算子（attention），于是以上所有绕法都不需要了。

但这组遗留给下一组的东西不是工程技巧，是**两个概念原语**：

- **「aggregate-broadcast」**：PPM 的灵魂 → object query 的灵魂。
- **「多分辨率表示必须共存」**：HRNet 的提问 → 所有现代分层 backbone 都必须回答。

这两个原语会在阶段 III / 阶段 IV 的查询式稠密架构（DETR / Mask2Former / SAM）里被整合成一个更高级的东西——**"可学习的 query 从多尺度 feature 中聚合并广播"**。那时我们会看到 PSPNet 和 HRNet 以一种新形态重新相遇。
