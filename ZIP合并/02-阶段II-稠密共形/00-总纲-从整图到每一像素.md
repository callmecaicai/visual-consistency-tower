## 一、从阶段 I 继承的三条残差（精确接力）

阶段 I 终章已经把判别共形的极限写清楚：

1. **空间内部坍缩** —— GAP 把 H×W 压成单点，图像内部的几何关系被强行折叠。
2. **度量对位置、形状、边界盲视** —— 交叉熵不看「在哪」，只看「是什么」。
3. **输出空间退化** —— 单分类远不能承载真实世界的结构化输出（框、掩码、关键点、深度图、流场、轨迹）。

阶段 II 的展开就是对这三条残差的**结构化回应**。它不是「检测 / 分割 / 深度」三个子领域的并置——那是论文检索视角。本体论视角下，这三个子领域是同一个共形命题（度量穿透）在不同输出空间上的同时实例化。

---

## 二、阶段 II 的本体论命题

<aside>
🔷

**让度量成为一个在图像定义域内处处定义的场。**

</aside>

这句话有三重含义：

- **密度**：度量不再只在输出端存在，而是在每一个空间位置都有一份损失。
- **结构化**：度量不再是标量（是/否、对/错），而是带内部结构的对象——坐标、区域、掩码、深度值、向量场。
- **可解耦**：语义、位置、边界、实例这些维度开始**独立承受共形压力**，而不是挤在同一个分类头里。

这是共形事件中**度量维度的第一次真正深化**。阶段 I 的度量是判别式的（离散类别）；阶段 II 的度量是空间化的（稠密场）；阶段 III/IV 之后才走向结构化关系与生成分布。

## 三、六维度上的具体位移

| 维度 | 阶段 I 的状态 | 阶段 II 的位移 |
| --- | --- | --- |
| 度量 | 交叉熵 + Top-1 | Dice / IoU / mAP / Focal / Hungarian 匹配 / PQ / RMSE —— 每一种度量都重新定义「错」 |
| 载体 | 单图单模态 | 仍为单图，但空间维度被充分调用；出口处语言作为「语义载体」首次进入 |
| 主体性 | 无主体 | 仍为无主体；但「提示化」（SAM 式）让系统对外部指令做局部响应——准主体性的雏形 |

阶段 II 的核心移动发生在 **度量** 和 **材料**；载体与主体性在本阶段只是轻微位移，真正的跃迁留给阶段 III 及以后。

## 四、阶段 II 的内部接力线（残差自生下一组）

阶段 II 不是按年代或子任务平铺的，是按**残差传递**串起来的：

1. **全卷积转向**（FCN / DeconvNet / SegNet / U-Net / U-Net++）—— 去掉 FC，让每一个空间位置都承受一份损失。
    - 残差：像素级共形达成，但感受野碎片化、尺度不一致。
2. **空洞卷积与多尺度**（DeepLab v1–v3+ / PSPNet / HRNet / SegFormer）—— 在保持分辨率的同时扩大感受野，让「全局上下文」与「像素精度」共存。
    - 残差：语义稠密已解决，但「实例」与「语义」割裂——两个像素属于同一个对象这件事没有被建模。
3. **检测共形的流水线化**（R-CNN 家族 / YOLO / SSD / FPN / RetinaNet / Mask R-CNN / Panoptic FPN）—— 用区域提议 + 特征金字塔把「实例」作为一级原语引入。
    - 残差：架构塞满手工先验（anchor / NMS / RoI），端到端共形未达成。
4. **Anchor-free 与端到端查询**（CornerNet / CenterNet / FCOS / DETR / Deformable DETR / DINO-DETR / Co-DETR）—— Hungarian 匹配把 NMS 拆除，查询成为稠密新原语。
    - 残差：查询范式统一，但每种稠密任务仍有自己的头，语义 / 实例 / 全景 / 深度各走各的。
5. **统一 Mask 架构**（MaskFormer / Mask2Former / OneFormer / SegFormer / SegGPT / EoMT）—— 所有稠密任务坍缩为「一组 mask 查询 + 每个查询一个标签」。
    - 残差：架构统一了，但**类别边界仍锁在训练集之内**——80 类就是 80 类。
6. **提示化与开放词汇**（SAM / SAM 2 / Grounding DINO 1.5 / DINO-X / T-Rex2 / YOLO-World / Florence-2 / APE / ODISE / CAT-Seg / FC-CLIP / Depth Anything）—— 类别从闭集拉成任意语言 / 点 / 框提示。
    - 残差：**开放词汇是借来的**——语义挂载 CLIP / 语言模型的外部表示，视觉侧自己不产语义。

第六步的残差直接定义阶段 III 的入口：**语义共形不能永远依赖外部挂载，系统必须自己生成语义**。

## 五、阶段 II 的截止线

**做完了**：

- 度量穿透到像素、实例、位置、深度值、向量场——稠密度量不再是一个开放问题。
- 架构在稠密任务上坍缩成「一个 backbone + 一组 mask 查询 + 一种匹配损失」——范式统一完成。
- 类别闭集被提示化撬开——80 类锁死的时代结束。

**没做（留给阶段 III / V 与反身维度）**：

- **语义的自生**。阶段 II 的开放词汇都在借 CLIP 的语义。第 81 类能被识别，是因为 CLIP 的对比预训练见过这个词——不是视觉侧「结构地」推出的。
- **结构化关系**。两个实例之间的关系（「在上面」、「握着」、「导致」）仍不在稠密输出里。
- **反身度量**。IoU / mAP / PQ / RMSE 仍然外部给定；网络不能质疑或修订自己的度量。

前两条指向阶段 III 与 Pre-LLM 的终结；第三条指向总纲的反身维度——那是整个大厦的终极残差。
