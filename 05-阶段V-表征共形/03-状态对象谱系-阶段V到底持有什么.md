# 状态对象谱系：阶段 V 到底持有什么

类型: 核心语法补页 / 状态对象登记
阶段: V · 表征共形
核心命题: 阶段 V 不能再笼统说“表征更好”，必须说明视觉系统到底持有了哪一种状态对象。
残差 / 它催生了什么: 状态对象越多，越需要证据等级、持有等级和反 probe 协议来防止概念膨胀。

## 一、为什么需要状态对象谱系

阶段 V 的危险在于把所有好 feature 都叫 world state。为避免膨胀，本页把“状态”拆成可登记对象：

```text
sample identity
visual feature
dense structure
object support
geometry
multi-view geometry
persistent 3D/4D state
predictive state
physical / causal state
metric state
```

它们不是同一级对象。阶段 V 的理论硬度来自区分这些对象，而不是把它们合并成一个大词。

## 二、状态对象总表

| 状态对象 | 最小定义 | 代表前沿 | 典型证据 | 对应章节 |
|---|---|---|---|---|

| 对象支撑 | 对象边界、前景、部件、区域归属 | LOST、TokenCut、STEGO、SAM-like teachers | object discovery、mask transfer | §44 |
| 几何深度 | depth、normal、metric scale、camera-aware geometry | Depth Pro、Depth Anything V2、Metric3D、UniDepth | depth、normal、boundary、scale | §46 |

## 三、状态对象审计卡

每个阶段 V 前沿方法都必须能填写这张卡。

| 字段 | 需要回答的问题 |
|---|---|
| 状态对象 | 它声称或实际持有什么状态？ |
| 证据等级 E | 证据停在 E0/E1，还是进入 E4/E5？ |
| 持有等级 H | 它是 H1 不变性，H3 预测式状态，还是 H4 部分世界状态？ |
| 接口 | 这个状态通过什么接口被调用？ |
| 变换族 | 它在哪些变换下保持稳定或等变？ |
| 失败模式 | 它在哪些反例中破裂？ |
| 阶段残差 | 它遮蔽了什么，如何推向下一层反身？ |

## 四、H 分级与状态对象的关系

| H 等级 | 对应状态对象 | 典型误判 |
|---|---|---|
| H0 | 任意有用 feature | probe 高就说理解 |
| H1 | 样本同一性、部分实例稳定 | 把图像实例当物理对象 |
| H2 | dense 结构、对象支撑 | 把 objectness signal 当对象持有 |
| H3 | 预测状态、部分 dense predictive state | 把 latent prediction 当完整 world model |
| H4 | 几何、多视图、遮挡、对应、布局的组合状态 | 把几何状态当物理/因果状态 |
| H5 | 度量状态与反身审计 | 把研究者反身误写成模型反身 |

## 五、阶段 V 的判断纪律

1. DINO / DINOv2 / DINOv3 是基础设施与 dense feature 强证据，不自动等于 H4。
2. JEPA / V-JEPA / V-JEPA 2.1 是预测式状态强候选，不自动等于完整世界模型。
3. Depth / pose / pointmap 是几何状态，不自动等于物理、因果与行动状态。
4. RADIO / PE / SigLIP2 是聚合表征与接口能力，不自动等于统一状态闭合。
5. IntPhys2 / CausalVQA 等负证据必须进入阶段 V，而不是被当作外部 benchmark 附录。

## 六、一句话

> 阶段 V 的对象不是“强表征”，而是“可登记、可审计、可反驳的状态对象”。
