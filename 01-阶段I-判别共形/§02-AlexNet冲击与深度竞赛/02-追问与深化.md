## §3 · 跨域回声：1×1 对高光谱遥感意味着什么？

先直接回答：**不仅仅是通道压缩。1×1 对高光谱遥感做的，是光谱端元的可学习线性混合**。

### 高光谱遥感张量的几何身份

一张高光谱图像的 shape 是 $(H, W, B)$，其中：

- $H \times W$ = 空间维度（比如 $512 times 512$）
- $B$ = 光谱波段数，典型 100–400（AVIRIS 224 bands, PRISMA 239 bands, EnMAP 224 bands）

**每个像素** $(i, j)$ **是一个** $B$ **维光谱向量** $\mathbf{s}_{ij} \in \mathbb{R}^B$，表示该地面点在 $B$ 个波长上的反射率。

### 1×1 卷积作用在这个张量上是什么

1×1 卷积 $W \in \mathbb{R}^{B \times C_{out}}$ 在张量上的操作是：

$\mathbf{y}*{ij} = W^\top \mathbf{s}*{ij}, \quad \text{对每个 } (i, j) \text{ 独立}$

也就是说——**空间上完全并行，每个像素位置独立做一次光谱维的线性变换**。

### 这与经典遥感的哪些操作同构？

这里的对应关系极其清晰，几乎 1-to-1：

| **1×1 卷积的设置** | **经典遥感对应操作** | **物理意义** | $C_{out} ll B$，无监督 | PCA / MNF 降维 | 提取主要光谱变化方向 |
| --- | --- | --- | --- | --- | --- |
| $C_{out} ll B$，有监督 | LDA / Fisher 判别投影 | 找最具判别力的光谱基 | $C_{out} = $ 已知端元数 | 线性光谱解混 (LSU) | 学习纯净物质的丰度 |
| $C_{out} > B$ | 过完备字典 | 稀疏表示 | $C_{out} = B$，近似正交 | 光谱基重组 | 学一组新的光谱坐标系 |

**所以 1×1 不只是"压缩通道"——它是"用梯度下降学一组最优的光谱基/端元"**。经典遥感需要人工决定 PCA 还是 MNF 还是某个物理模型，1×1 让数据+任务**自己告诉你**最该用的那组基。

### 实战意义

- 高光谱分类头部第一步通常是 $B = 224 \to C_{out} = 32$ 的 1×1，物理上对应"学 32 个最适合下游任务的光谱端元"
- 后续 3×3 空间卷积在这 32 维低冗余通道上做空间结构提取——**比直接在 224 维上做快 49 倍**
- 如果把 1×1 的权重可视化，会看到它学出类似植被红边（680–750 nm）、水吸收带（940 nm）、矿物特征（SWIR）这些**物理上有意义的光谱基**——这就是 1×1 对你做高光谱研究的直接价值

<aside>
💎

**§3 要义**：1×1 在高光谱上不是「通道压缩」，而是**用梯度下降学一组最优的光谱端元基**。它把 PCA/MNF/LDA/LSU 等传统遥感方法作为它的特例——数据+任务决定最优基，而不是人工决定。

</aside>

## §4 · "度量不变" 指的是哪种度量？(关键澄清)

你抓到了一个真正的矛盾点。让我把它讲清楚——上一轮我没区分"**空间度量**"和"**通道度量**"，造成了歧义。

### 一个张量上同时存在两套度量

高维张量 $(H, W, C)$ 上实际上存在**两个独立的度量空间**：

1. **空间度量**：$H times W$ 网格上的距离。两个像素 $(i_1, j_1)$ 与 $(i_2, j_2)$ 之间的空间距离 = $sqrt{(i_1-i_2)^2 + (j_1-j_2)^2}$。
2. **通道度量**：$C$ 维特征向量之间的距离。两个像素处的特征向量 $\mathbf{f}*{ij}$ 与 $\mathbf{f}*{i'j'}$ 之间的 L2 距离 / 余弦距离。

1×1 卷积的准确行为是：

| 维度 | 1×1 对它做了什么 |
| --- | --- |
| **空间度量** | **严格保持**（不改变分辨率、不改变空间邻域关系，平移严格等变） |
| **通道度量** | **彻底重塑**（256 维 → 64 维，度量空间被整体换掉） |

所以我之前说"度量不变"，精确的表述应该是："**空间度量保持，通道度量被重塑**"。

### 那 256 → 64 是什么样的通道度量重塑？

这是一次**线性投影 + 学到的降维**：

$\mathbf{y} = W^\top \mathbf{x}, \quad W \in \mathbb{R}^{256 \times 64}$

- 这个投影**不是**保持 L2 距离的等距投影（除非 $W$ 是正交矩阵，一般不是）
- 它是"**学到的最有信息量的 64 维子空间**"——通过梯度下降，$W$ 的列向量张成的子空间在对下游任务而言是"最重要的 64 个方向"
- 本质上是**主成分式的判别性降维**：保留任务最需要的信息，丢弃冗余

### 为什么这种"通道度量的重塑"不破坏共形流？

因为它**是线性的**，而且每个空间位置用的是**同一个** $W$。这两条保证了：

1. 线性 → Lyapunov 链中 $\sigma(J_l)$ 是 $W$ 的奇异值（可解析控制，通过 He/Xavier 初始化锁在 1 附近）
2. 空间位置共享 $W$ → 1×1 仍严格保持平移等变性 → 空间对称群守恒

**所以"通道度量被重塑但共形流不破坏"**，因为纵向去噪关心的是雅可比奇异值守恒，不是度量空间的同一性。**你完全可以把度量空间换掉，只要换的动作本身是线性且可控的**。

<aside>
💎

**§4 要义**：张量 $(H,W,C)$ 上同时存在两套正交度量——**空间度量**（严格被 1×1 保持）与**通道度量**（被 1×1 线性重塑）。纵向去噪定理只要求雅可比奇异值守恒，不要求度量空间同一性——**「保共形 + 变度量」**是现代架构的根本语法。

</aside>

## §5 · 1×1-3×3-1×1 三明治：不只是节流，而是 information-bottleneck-like 结构

这是你问的第三个点，而且是最重要的一个。我之前说"二阶节流"只讲了计算账，没讲几何本质。

### 先把计算账讲完

在 ResNet-50 的 bottleneck block 里：

- 进：1×1 把 256 通道降到 64
- 中：3×3 在 64 通道上做空间卷积
- 出：1×1 把 64 升回 256

计算量对比（不含 bottleneck 的"直 3×3"方案）：

- 直做：$3 times 3 times 256 times 256 = 589,824$ 次乘加
- Bottleneck：$1 cdot 1 cdot 256 cdot 64 + 3 cdot 3 cdot 64 cdot 64 + 1 cdot 1 cdot 64 cdot 256 = 69,632$ 次

**约 8.5 倍节省**。计算账是清楚的。但这只是表面理由——真正的理由是信息论的。

### 几何本质：information-bottleneck-like 的工程拓扑

这个三明治在做的是：**强迫信号通过一个窄通道，学到"对下游任务最关键的低维表征"**。

这与 **Tishby 的 Information Bottleneck 原理**形成启发性同构：一个好的表征应该同时满足：

$\min_{T} I(X; T) - \beta \cdot I(T; Y)$

(最小化表征与输入的互信息 = 压缩冗余；最大化表征与目标的互信息 = 保留关键)

Bottleneck 架构并不严格优化 Tishby IB 目标，但它把“任务相关压缩”的工程形态写进了拓扑：

- 第一个 1×1：$256 to 64$ = 显式压缩 $I(X; T)$
- 中间 3×3：在被压缩后的 64 维表征上做空间计算 = 在最少冗余的空间里学结构
- 第二个 1×1：$64 to 256$ = 投回宽空间，让后续层继续用宽表征
- 外围残差连接：保证即使瓶颈学糟了，恒等分量也能穿过 → **给瓶颈学习提供安全网**

### 用通信工程的类比

**这就是"信号 → 编码压缩 → 传输处理 → 解码展开"的架构化**：

| 通信阶段 | Bottleneck 中的对应 |
| --- | --- |
| 编码器压缩 | 1×1 降维 256→64 |
| 窄信道传输与处理 | 3×3 在 64 通道上做空间卷积 |
| 解码器展开 | 1×1 升维 64→256 |
| 物理冗余 | 残差连接保留恒等分量 |

**一个合理的谱系归纳**：凡是“宽 → 窄 → 处理 → 宽”这种结构，都可以被看成 information-bottleneck-like 的拓扑原型：它强迫信息经过低维通道再展开，从而在计算预算和任务相关压缩之间建立折中。从 ResNet bottleneck，到 Transformer 的 $d_{model} \to d_k$ 压缩再到 $d_{model}$，到 VAE 的 latent 码，到 MAE 的编码-解码，都能在这一意义上被比较；但这种比较是 G/P 层读法，不是严格等价。

<aside>
💎

**§5 要义**：1×1-3×3-1×1 三明治不只是计算节流技巧，而是一种 **information-bottleneck-like 的结构**——「宽→窄→处理→宽」强迫信号穿过低维瓶颈，学到「对任务最关键的低维表征」。这个几何原型可用于比较 ResNet / Transformer / VAE / MAE 等架构，但它不是严格 IB 目标的直接实现。

</aside>

## §6 · 为什么这四种架构都"必须"用 1×1

一个一个讲清楚每一个的必要性。

### §6.1 Inception bottleneck：不加 1×1 会爆炸

Inception 的核心思想是"同一层并行用多尺度卷积，然后 concat 所有分支"：

$\text{Output} = \text{concat}(\text{conv}*{1\times 1}, \text{conv}*{3\times 3}, \text{conv}_{5\times 5}, \text{maxpool})$

**问题**：每个分支产生 $C$ 个通道，四个分支 concat 就是 $4C$——**通道数每层翻 4 倍**，两层后就 16 倍，参数与显存立即爆炸。

**1×1 的解法**：

- 在 3×3 和 5×5 之前各插一个 1×1 先把通道降到 $C/2$ 或 $C/4$
- 在 maxpool 之后插一个 1×1 压通道
- 保证四个分支 concat 后通道数与输入同量级

**所以 Inception 里 1×1 是并行拓扑的"通道预算管理器"**。没有它，Inception 多分支的想法根本不可行。

### §6.2 ResNet 1×1-3×3-1×1：深度换计算的关键方式

ResNet-18/34 用的是简单的 3×3-3×3 block。但 ResNet-50/101/152 必须换成 1×1-3×3-1×1 bottleneck——**否则训 152 层是算力上不可承受的**。

**为什么必须**：

- 深度从 34 提到 152 是约 4.5 倍
- 每层保持 256 通道的 3×3 卷积，总计算量按深度线性增长
- 用 bottleneck 把每块计算量降 8.5 倍 → 152 层的总计算量与 34 层 basic block 持平

**1×1 让"加深"从"算力灾难"变成"可承受的代价"**——这是 ResNet 能做到 1000+ 层的架构根基。

### §6.3 MobileNet / Xception 的 Depthwise Separable：把"空间"和"通道"彻底解耦

这一条是"维度解耦"思想的**极限版本**。标准 3×3 卷积一次做两件事：

1. 空间邻域混合（看 3×3 邻域）
2. 通道间线性混合（每个输出通道是所有输入通道的加权和）

Depthwise separable 把它**拆成两步**：

| **步骤** | **操作** | **参数量** | **做的事** |
| --- | --- | --- | --- |
| 2. Pointwise | 1×1 卷积 | $C \cdot C'$ | 只混通道，不混空间 |

标准 3×3 卷积的参数 = $9 C C'$；depthwise separable = $9 C + C C' approx C C'$（当 $C'$ 大时）。**参数降到 1/9，准确率几乎不降**。

**为什么 work**：这其实是一个深刻的几何事实——**图像里的"空间相关"和"通道相关"是两种近似独立的结构**。标准卷积把它们耦合处理是冗余的，depthwise separable 把它们正交化，用两个小矩阵的张量积近似一个大矩阵（类似于 CP / Tucker 张量分解）。

**1×1 在这里的必要性**：它是"通道混合"那一半。没有 1×1，depthwise conv 就只是各通道自说自话，永远学不到跨通道组合——depthwise separable 本身就是 1×1 的一个特定拼法。

### §6.4 Transformer 的 FFN：1×1 在序列空间的复用

这是整个框架中最重要的一条，因为它把"1×1 是根本算子"这一命题推广到了所有现代架构。

Transformer 的每个 block 有两部分：

$\text{Attention}: \text{token 维做全局混合，特征维不混}$

$\text{FFN}: \text{token 维独立，特征维做 MLP (expand} \to \text{activate} \to \text{contract)}$

关键观察：**FFN 对每个 token 独立做一个 2 层 MLP**。

如果你把 token 想象成"一维序列里的像素"，那：

| **CNN** | **Transformer** |
| --- | --- |
| 1×1 conv（通道混合） | FFN（特征维混合） |

**FFN 就是 1D 情形下、token-wise 的 1×1 卷积**——序列空间严格保形（每个 token 独立处理），特征空间做大幅升维-非线性-降维（典型 $d to 4d to d$，其实就是一个 bottleneck 的"反向"版本）。

更有趣的是，**Attention + FFN 的 block 结构 = 空间混合算子 + 通道混合算子的交替**——这和 CNN 的 3×3 + 1×1 交替**结构上同构**。

**所以 1×1 不只是 CNN 的一个技巧**——它是"在一个维度上恒等、在另一维度上做 MLP"这一根本算子的原型，**被 Transformer 以 FFN 的名字继承到了序列空间**。

<aside>
💎

**§6 要义**：Inception / ResNet-50+ / MobileNet / Transformer FFN 都**必须**用 1×1——在前三者中它是通道预算管理器与维度解耦器，在 Transformer 中它以 FFN 的名字承担「特征维 MLP」的角色。**1×1 是贯穿 CNN 与 Transformer 的根本算子**。

</aside>

## §7 · "保共形 + 变度量"这句话的精确含义

现在可以把这句话拆开讲清楚。

### 每个张量算子都在"选择维度"

一个多维张量 $(H, W, C)$ 上的算子，总是要决定**对每一个维度做什么**——是保持、是局部混合、是全局混合、还是独立变换。我列出四个基本选择：

| 算子对一个维度的动作 | 几何含义 |
| --- | --- |
| **恒等 (保)** | 这个维度完全不变（如 1×1 对空间维） |
| **局部混合 (小变)** | 邻域加权（如 3×3 卷积对空间维） |
| **全局混合 (大变)** | 所有位置互相看（如 self-attention 对 token 维，或 GAP 对空间维） |
| **任意线性重塑 (MLP)** | 整个维度坐标系被换掉（如 FC / 1×1 对通道维） |

### 现代架构 = 在每个维度上选择不同动作的算子堆叠

把几个核心算子对"空间维 + 通道维"两个维度的选择列出来：

| **算子** | **空间维动作** | **通道维动作** |
| --- | --- | --- |
| 1×1 卷积 | **恒等** | 任意线性重塑 |
| Self-attention | **全局混合** | 恒等（特征维不混） |
| GAP | **全局平均**（极限等变） | 恒等 |

### 所以"保共形 + 变度量"的意思是

**一个好的架构算子，总是在某些维度上严格保形（等变或恒等，维持 Lyapunov 守恒），同时在另一些维度上做大幅重塑（学习表征）**。如果一个算子同时在所有维度上都强烈改变，它就会破坏共形流；如果它在所有维度上都不变，它就不能学东西。

**"保"和"变"的组合设计，就是现代架构工程的全部**。看懂了这个，你看任何新架构（Swin、ConvNeXt、Mamba、MoE）都能一眼识别它在每个维度上的动作——以及它为什么必须这么设计。

<aside>
💎

**§7 要义**：每个张量算子都在多个维度上选择一组**〈恒等 / 局部混合 / 全局混合 / 任意线性重塑〉**的动作。现代架构工程的全部，就是**在哪些维度上保共形、在哪些维度上变度量**的组合设计。这是解读阶段 II-V 所有新架构（Swin / ConvNeXt / Mamba / MoE / SAM）的通用钥匙。

</aside>

## §8 · GAP 的具体计算流程

你说没搞懂 GAP 具体怎么做。我把它和传统 FC 并排写出来，一步步对比。

### 输入

设 CNN 最后一个卷积层输出张量 $(H, W, C)$，比如 $(7, 7, 2048)$（ResNet-50 的最后阶段）。

### AlexNet/VGG 式（传统 FC 头）

```
Conv output (7, 7, 2048)
  → Flatten  (100352,)           ← 空间结构被破坏
  → FC  (100352 → 4096)          ← 参数 411M  (!!!)
  → ReLU + Dropout
  → FC  (4096 → 4096)            ← 参数 16.7M
  → ReLU + Dropout
  → FC  (4096 → 1000)            ← 参数 4M
  → Softmax
```

总参数 ≈ 432M，其中 95% 在第一个 FC。这就是为什么 AlexNet 的参数量里 FC 头占 90%。

### NiN / GoogLeNet / ResNet 式（GAP 头）

```
Conv output (7, 7, 2048)
  → 对每个通道 c ∈ [1, 2048]：
      v_c = mean over all 49 spatial locations of feature map c
      = (1/49) · Σ_{i,j} x[i, j, c]
  → 得到向量 (2048,)              ← 参数 0
  → FC  (2048 → 1000)             ← 参数 2M
  → Softmax
```

总参数 ≈ 2M，比 FC 头省 200 倍。

### GAP 的几何意义（为什么它还能 work）

这里有个关键观察：**GAP 只有在 CNN 学到了"每通道 = 某个语义类别检测器"的情况下才 work**。

- 训练前：通道 $c$ 可能混杂多种语义
- 训练中：由于 GAP 后直接接 1000 类分类头（每类对应一个通道的线性组合），梯度会**反向逼迫 CNN 的最后一层学出"通道** $c$ **在输入图片含有类别** $c$ **时强响应"**这种性质
- 训练后：通道 $c$ 的 7×7 空间响应图 = "这张图里哪些位置看起来像类别 $c$"的热图
- 对 7×7 空间取均值 = "这张图整体有多像类别 $c$"

这就是为什么 GAP 不仅省参数，还提供了**天然的可解释性**——每个通道的空间响应图就是类别激活图 (CAM)，**Zhou et al. 2016 的 CAM 论文发现的就是 GAP 的这个几何副产品**。

### 你可以这样理解

FC 头：把 2048 个通道的 7×7=49 个空间位置当成 100352 个完全无关的特征，用一个巨大的线性变换去组合——**空间结构被破坏**，参数全用来学"哪些像素组合表示哪类"。

GAP 头：先把每个通道的空间图压成一个数（"该类特征的总强度"），再用一个小线性变换决定类别——**先用 CNN 学到位置无关的语义检测器，再用 GAP 聚合**。

**后者是更符合"平移不变的图像分类"任务的结构化做法**。

<aside>
💎

**§8 要义**：GAP 在参数上省 200 倍的同时带来**天然可解释性**——每个通道的空间响应图就是类别激活图 (CAM)。Zhou 2016 发现的 CAM 实际上是 GAP 的几何副产品。GAP 把「空间维度的极限等变投影」与「类别检测器的训练引导」**一体化**。

</aside>

## §9 · 反卷积可视化的具体流程

用户没搞懂这个。我一步一步写。

### 目标

给定一个训练好的 CNN，想知道**"第 5 层第 127 号滤波器到底被什么图像 pattern 激活？"**。

### 方法（Zeiler & Fergus 2014）

### Step 1: 前向，找到该滤波器的最强响应

1. 拿一批训练图片 ${x_1, x_2, ldots}$，每张前向经过 CNN
2. 在第 5 层记录第 127 号滤波器的激活图（空间尺寸假设是 14×14）
3. 找出该滤波器在所有图片中最大响应的位置 $(i^*, j^*)$ 和图片 $x^*$

### Step 2: 构造"只保留这个激活"的张量

把第 5 层的所有其他通道、以及第 127 号通道的其他位置全部置 0，**只保留** $(i^*, j^*, 127)$ 这一个激活值。

$a_5^{\text{vis}}[i, j, c] = \begin{cases} a_5[i^*, j^*, 127] & \text{if } (i, j, c) = (i^*, j^*, 127) \\ 0 & \text{otherwise} \end{cases}$

### Step 3: 反向投影到像素空间

现在要把这个稀疏的第 5 层激活**反投影**回 224×224 的像素空间。对每一层，反向时用一个"反操作"：

| **前向操作** | **反向操作 (unconv)** | Max-Pooling | Un-pooling：用前向记录的 "argmax 位置"，把值放回那个位置，其余位置填 0 |
| --- | --- | --- | --- |
| ReLU | ReLU（保持负值被截断；只关心正激活路径） | Conv (用 W) | Transposed conv (用 W 的翻转转置) |

一层一层反向算下去，最终得到一张 224×224×3 的"反投影图"。

### Step 4: 观察

这张 224×224 的图显示：**正是输入图像的什么 patch / pattern 激活了这个滤波器**。

### 经验结果（ZFNet 的核心发现）

- 第 1 层的滤波器：Gabor-like 边缘检测器（与 V1 简单细胞一致）
- 第 2 层：角点、弯线、同心圆
- 第 3 层：网格纹理、车轮、人脸部件
- 第 4 层：狗头、鸟嘴、完整物体部件
- 第 5 层：完整物体类别（蜂鸟、汽车、花朵）

**这个观察第一次把"CNN 是层次化共形特征学习器"从假设变成可视证据**。

### 更重要的一点

Zeiler & Fergus 用反卷积查看 AlexNet 时发现第一层有**混叠坏滤波器**（过于高频、不对齐），这直接启发他们把 AlexNet 第一层的 11×11 stride-4 改成 7×7 stride-2——**ZFNet 比 AlexNet 在 ImageNet 上准确率提升约 2%，完全靠这个通过反卷积诊断出的修改**。

**所以反卷积不是纯观察工具，它是第一次让架构调参基于几何证据而非试错的工具**。

<aside>
💎

**§9 要义**：反卷积可视化 = 共形流的**可观测工具**。它把「CNN 是层次化共形特征学习器」从假设变成可视证据——第 1 层 Gabor 边缘 → 第 2 层角点/弯线 → 第 5 层完整物体。ZFNet 改进 AlexNet 第一层就是基于这个观察——**第一次让架构调参从试错升级为几何证据**。

</aside>

## §10 · GPU 并行如何与深度学习/优化共形

这是你最后一问，也是最深的一问。我展开讲。

### 核心命题：深度学习的算子代数天然适合大规模并行

几乎所有传统 ML 算法（SVM 对偶优化、GBDT 的贪婪树构造、EM 算法）都有**序列依赖**——当前步的计算依赖上一步的完整结果。这让它们难以并行。

**深度学习的所有算子只有两种**：

1. **线性 (Conv, FC, Attention matmul)**：线性性意味着**在任意维度上都可分解** ($W(x_1 + x_2) = Wx_1 + Wx_2$)
2. **逐点非线性 (ReLU, Sigmoid, LayerNorm)**：逐点意味着**每个张量元素独立**

**两条性质合起来 → 深度学习从算子层面就是"embarrassingly parallel"**。这不是偶然——这是深度学习能承接 Moore's Law 与 GPU 架构的根本原因。

### 四种并行范式与它们的共形位置

### §10.1 数据并行 (Data Parallelism)

**做法**：每个 GPU 复制一份完整模型，分发不同 minibatch 样本；反向后 all-reduce 梯度。

**共形基础**：Loss 关于 batch 是**线性可加**的：

$\mathcal{L} = \frac{1}{B} \sum_{i=1}^{B} \ell(f(x_i), y_i)$

$\nabla_\theta \mathcal{L} = \frac{1}{B} \sum_{i=1}^{B} \nabla_\theta \ell(f(x_i), y_i)$

**这个“梯度对 batch 的线性性”是数据并行能直接成立的关键共形条件**。如果 loss 不是样本可加的（比如对比损失、triplet loss），数据并行要做额外的通信设计（跨 GPU 维护负样本库）。

**与深度学习的共形**：SGD 本身就是 full-batch gradient 的蒙特卡罗近似——**把 batch 继续切到多 GPU，只是在统计意义上再多一层采样，与深度学习的随机梯度性质完全共形**。

### §10.2 模型并行 (Model Parallelism, AlexNet 用的)

**做法**：把模型的不同部分放在不同 GPU。AlexNet 把 conv 通道一分为二，两块 GTX 580 每块放一半。

**共形基础**：卷积对通道维的**可分块性**——可以把通道分成两组，每组独立 conv，少数跨 GPU 层做信息交换。

**代价**：通信成本。AlexNet 的 cross-GPU 通信只在特定几层发生（精确设计），其余层通道独立——这本质上是**通道维分块 + 稀疏跨块通信**的架构。

### §10.3 张量并行 (Tensor Parallelism, Megatron-LM)

**做法**：把**单个权重矩阵**切到多 GPU。例如把 $W \in \mathbb{R}^{d \times d}$ 按列切成 $W = [W_1, W_2]$ 放两块 GPU：

$y = Wx = [W_1 x, W_2 x] = \text{concat}(y_1, y_2)$

每块 GPU 各算半个结果，最后 all-gather 合并。

**共形基础**：矩阵乘的**块结构性**——矩阵可以按行或按列切，且运算保持正确。

**深度学习共形**：Transformer 的 FFN 是 $d \to 4d \to d$ 的两层 MLP，中间的 $4d$ 维隐层天然可以切片到多 GPU；attention 的多头 ($h$ 头) 也天然可以按头切到多 GPU——**Transformer 架构的 head 和 FFN 扩张结构本身就为张量并行量身定制**。

### §10.4 流水并行 (Pipeline Parallelism)

**做法**：把层分段放到不同 GPU（GPU1 放第 1-12 层，GPU2 放 13-24 层...），像工厂流水线一样，minibatch 的不同 micro-batch 同时在不同 GPU 上处理不同阶段。

**共形基础**：深度网络的**层间依赖是线性链**（下一层输入 = 上一层输出）——这让流水作业成立。如果层间是随机图、有反馈回路，就没法流水。

### 更深的命题：GPU 的硬件结构与深度学习的算子代数是相互成就的

这是很少被明说但极其重要的一点：

**GPU 本来是为图形渲染设计的——每个像素独立着色、大量并行算 shader**。它的硬件假设是：

1. 海量的轻量线程（thousands of cores）
2. 线程之间几乎无依赖
3. SIMD（单指令多数据）
4. 显存带宽大于延迟

**而深度学习的算子（conv/matmul/elementwise）恰好全部符合这个硬件假设**。

| GPU 硬件假设 | 深度学习算子的匹配 |
| --- | --- |
| 海量轻量并行 | matmul 的 $O(n^3)$ 乘加天然并行 |
| 线程无依赖 | 逐点非线性对每个元素独立 |
| SIMD | conv 对每个空间位置做相同运算 |
| 高带宽 > 低延迟 | 张量运算是批量吞吐型，对延迟不敏感 |

**这个匹配不是巧合**——它是 90 年代 GPU 为图形 shader 设计的并行模式，恰好在 2010s 被 CNN 发现**可以复用于视觉表征学习**（因为二者都建立在 2D 网格上的独立像素计算）。

### GPU 并行与共形演化的"时间尺度维度"

把上面所有合起来：

> GPU 并行是共形坐标第六维"**时间尺度**"的物理承载。它把"训练一个模型需要多久"从物理常量变成了**可扩展变量**——并行度决定了 wall-clock time，从而决定了给定 wall-clock budget 下可训模型的**规模**。
> 

**Scaling law 的几何完整性**需要三个可扩维度：**数据 × 参数 × 算力**。前两个由数据集和架构决定，第三个由 GPU 并行决定。**只有三者同时可扩，Chinchilla 公式** $N \propto P$ **才有意义**。

**AlexNet 在 2012 年用 GPU 并行赢下 ImageNet，历史意义不在于 GPU 省了训练时间，而在于它示范了"算力 × 并行度"作为可扩独立维度**——这个示范直接催生了 NVIDIA 从游戏显卡公司转型成 AI 算力公司（CUDA、cuDNN、V100、A100、H100 的整条产品线），也催生了 Google TPU、Meta MTIA、华为昇腾——整个 2015–2025 的 AI 算力基础设施，都是顺着 AlexNet 这条线长出来的。

### 一句话收束

**深度学习与 GPU 并行的"共形"不是一个隐喻——它在三个层面同时成立**：

1. **算子代数层面**：线性+逐点非线性恰好符合 SIMD 假设
2. **拓扑层面**：层间线性链、通道可分块、batch 可加恰好符合四种并行范式
3. **scaling 层面**：算力作为可扩维度让 scaling law 三角形闭合

**AlexNet 把这三层同时打通了**——这是它 2012 年那场胜利的真正历史体量。

<aside>
💎

**§10 要义**：GPU 并行与深度学习**在算子代数、拓扑、scaling 三层同时共形**。AlexNet 2012 的真正遗产不是 GPU 省训练时间，而是示范了「算力 × 并行度」作为共形坐标第六维（时间尺度）的物理承载——**让 scaling law 的三角形（数据 × 参数 × 算力）闭合**。整条 NVIDIA → TPU → 昇腾的算力基础设施都是沿着这条线长出来的。

</aside>

## §11 · 本轮总结 + 下一步建议

你这一轮问的所有点，合起来说明一件事：**1×1 卷积 / GAP / 反卷积 / GPU 并行不是 AlexNet 时代的"实现细节"，它们是深度学习架构语法的四个基石算子/思想**：

1. **1×1** = "保一维 + 变一维" 的维度解耦根本算子
2. **GAP** = 空间维度上的极限等变投影，连接 CNN 与可解释性
3. **反卷积** = 共形流的可观测工具，让架构调参基于证据
4. **GPU 并行** = 算力作为共形坐标第六维的物理承载

这四个思想一直贯穿到 Transformer、Diffusion、VLM 时代——没有一个被淘汰。

### 下一步建议

我建议你下一步可以考虑把以下三条核心提炼**立卡**写进 🏛️ 父页或独立成页（不动手，先和你确认方向）：

1. **「维度动作矩阵」**：把每个算子在每个维度上的动作（恒等/局部/全局/MLP）做成一张大表，作为整个谱系的通用解读工具——阶段 II-V 每一篇都可以被快速归位
2. **「information-bottleneck-like 架构谱系」**：从 ResNet bottleneck → Transformer $d \to 4d \to d$ → VAE → MAE，这条线可以单独成页
3. **「深度学习-GPU 共形命题」**：从算子代数/拓扑/scaling 三层证明它们天然共形，作为第六维的锚点
