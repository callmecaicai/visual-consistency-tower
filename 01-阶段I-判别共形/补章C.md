## **A-4 · 损失函数谱系 · 从"误差度量"到"几何公理的注入口"**

先给出本章中心断言：

**损失函数不是“衡量误差的标尺”那么简单——它是塑造训练后表征空间几何形状的最显式、最可声明的公理注入口。** 但它不是唯一注入口：架构、数据、优化、增强、归一化、batch 采样与训练时长也都在隐式注入几何假设。40 年损失函数史可以读成一条从“点预测 → 分布预测 → 距离预测 → 等价关系预测”的本体论轨迹，每一代 loss 都在以更结构化的方式注入“语义是什么”这条公理。

一旦用这个视角看，A-4 列表里的所有损失（CE / Triplet / ArcFace / Focal / IoU / Perceptual / InfoNCE）就不再是"不同任务的不同工具"，而是**同一个几何构造工程里不同精度的刻度**。

**一 · 损失函数的真实身份——几何公理的注入口**

经典机器学习里损失函数的定义是"预测与真值的距离"。这个定义对线性模型、决策树、SVM 都够用——因为这些模型的"假设空间"由算法本身决定，loss 只决定"哪个解最优"。

**深度网络打破了这个分工**。

现代深度网络通常高度过参数化，表达容量远超训练集规模。

在足够容量与优化成功的条件下，它们可以拟合非常复杂的标签分配。

所以 loss 的作用**不是"挑一个最好的假设"，而是"告诉模型：哪种形状的假设算 '好'"**

换言之：**Loss 定义了"最优解的几何形状"，而不只是"最优解的质量标准"**。

这是为什么近 10 年损失函数工程变成了一个独立的深度学习子领域——在架构和优化之外，**loss 是第三条塑造模型的轴**。

本补章讨论的是阶段 I 的度量轴如何向后续阶段外溢；它不是说所有任务都能由 CE 谱系推出，也不是说 loss 是唯一塑形来源。

**二 · Cross-Entropy 的条件唯一性——不是随意选择，而是约束交集**

**§2.1 四重必要性论证**

CE 看起来是“自然选择”，但它的地位不是历史偶然。更准确地说，在经典单标签分类、概率输出、局部评分、最大似然这些约束同时成立时，softmax + CE 是最自然、最稳定、最可扩展的交集解：

**论证 1 — 最大似然**：softmax + CE 精确等价于对类别分布做最大似然估计。任何从指数族（exponential family）假设出发的概率模型，CE 是自然参数化下的 log-likelihood。

**论证 2 — Strict Proper Scoring Rules**（Gneiting & Raftery 2007）：在所有**严格 proper 的评分规则**（预测真实分布时得分唯一最优）里，**log-score 是唯一"局部的"**——"局部"的意思是"得分只依赖于**观察到的那个类别**的预测概率"。其他 proper scoring rules（Brier score、quadratic loss）需要看**所有类别**的预测。局部性对 SGD 友好：梯度直接指向观察到的类别，不需要其他类别的参与。

**论证 3 — 信息几何**：softmax + CE 对应的梯度是 **Fisher-Rao 自然梯度在参数简化下的形式**——也就是说，CE 的 SGD 隐式执行的是"信息几何上的最速下降"，而不是欧氏空间的最速下降。这让 CE 的优化动力学在类别流形上是**几何正确**的。

**论证 4 — 神经崩溃（Neural Collapse, Papyan 2020）**：CE + weight decay 训好的网络，在典型监督分类训练后期可能出现类内压缩、类间等角分离、分类器与类均值对齐等结构。这说明 CE + 深度架构 + 标准训练协议会强烈诱导特定类别几何，但不是无条件普遍定理。

**把这四个论证合起来**：CE 不是单纯“好用的 loss”，而是在「最大似然、局部得分、信息几何、类几何诱导」这些条件交叠处最自然的解。你换成别的 loss（如 Hinge、MSE-on-softmax），通常会丢掉其中某些条件。

**§2.2 为什么 Hinge / SVM 在深度分类主线中退居边缘**

早期 R-CNN 等方法还用 Hinge（"max-margin"），但 2015 之后在深度分类主线中退居边缘。原因不是“CE 精度永远更高”（实际上 Hinge 在一些设置下也不差），而是：

**Hinge 不是概率解释**：无法自然给出 class posterior，与下游贝叶斯推断不兼容

**Hinge 不是局部评分**：梯度来自违反 margin 的那几个类别，而不是观察到的类别本身

**Hinge 的梯度在正确分类区域变为零**：一旦样本被正确分类且 margin 足够，梯度消失——这**切断**了"知识继续注入"的通道

**Hinge 不诱导 Neural Collapse**：因为它不推类均值到单纯形 ETF

**本体论断言**：CE 不是“一个普通 loss”，它是经典单标签分类任务在一组常用概率约束下的公理化默认选择。在这个意义上，B-18 讨论“Softmax+CE 的条件唯一性”是回到了大量深度学习论文的真正出发点。

**三 · 度量学习族——从"预测类别"到"构造距离"的拓扑转向**

**§3.1 一个决定性的哲学转向**

2006 Hadsell-Chopra-LeCun 的 Contrastive Loss 做了一件当时几乎没人意识到其深度的事：

**不预测类别标签——直接规定"表征空间里哪些点应该靠近、哪些点应该远离"。**

$$
\mathcal{L}_{\text{contrastive}}
=
(1-Y)\cdot d(x_i,x_j)^2
+
Y\cdot \max(0, m-d(x_i,x_j))^2
$$

**这是分类问题到拓扑问题的转译**：

经典分类：输入 → 类别标签（集合上的函数）

度量学习：输入 → 表征空间（距离结构的构造）

**为什么这个转向深刻？** 因为它**把"语义"从"离散类别"重新定义为"几何距离"**。两张猫的图相似——不是因为它们都被贴上"cat"标签，是因为它们在某个表征空间里距离小。**标签从目标变成了副产品**。

**§3.2 Triplet Loss（FaceNet 2015）——度量学习的工业化**

$$
\mathcal{L}_{\text{triplet}}
=
\max(0, d(a,p)-d(a,n)+\alpha)
$$

Triplet 的三元组结构（anchor / positive / negative）把 "同类" 和 "异类" 放在**同一个比较单元**里。这一步微妙但关键：

**Contrastive Loss**：分别处理正对和负对——训练时它们是两种不同的 loss 项

**Triplet Loss**：**一个样本同时参与正对和负对**——loss 衡量"正对比负对近多少"

这是**从绝对距离到相对距离的转变**。绝对距离没有语义（"0.3 是近还是远？"），**相对距离才是语义**（"A 比 B 更像 anchor"）。

**FaceNet 的成功与人脸任务的几何特殊性**：人脸有**极强的 intra-class 变化（表情、角度、光照）+ 极强的 inter-class 相似性（都是人脸）**。这恰好是度量学习最有效的场景——因为"类别"本身是一个弱概念，"相似度"才是真正的语义。

**§3.3 Center Loss（2016）——CE 和度量学习的协同**

$$
\mathcal{L}_{\text{center}}
=
\mathcal{L}_{\text{CE}}
+
\lambda \cdot \sum_i \|\phi(x_i)-c_{y_i}\|^2
$$

Center Loss 把 CE（分类 loss）和一个"向类均值靠近"的 loss 加在一起。这看似平庸，但**它预告了 2020 年的 SupCon（Supervised Contrastive）**：**同时用标签 + 度量**。

**关键观察**：Center Loss 显式把类均值 $c_{y_i}$ 作为可学习参数。这等价于**神经崩溃（Neural Collapse）在 2020 年前的工程化先兆**——Papyan 2020 证明标准 CE 训练**自然地**诱导了类均值的单纯形 ETF 结构，Center Loss 只是**提前把这件事做进了 loss**。

**四 · Angular Margin 家族——超球面几何的终极形态**

L-Softmax → A-Softmax → CosFace → ArcFace 是 2016-2019 人脸识别领域的**四连跳**。每一代只改一个超参，但几何命题逐步锐化。

**§4.1 基础几何**

把 CE 的 logit 重写成：

$$
\text{logit}_y = \|W_y\|\|\phi(x)\|\cos\theta_y
$$

其中 $theta_y$ 是权重向量 $W_y$ 和特征 $phi(x)$ 的夹角。**如果把 $W_y$ 和 $phi(x)$ 都归一化到单位球面**，logit 就**只剩夹角**：

$$
\text{logit}_y = s \cdot \cos\theta_y
$$

（$s$ 是尺度因子）。**这一步把所有语义转入"角度"**——不再关心向量长度，只关心方向。

**§4.2 四代 margin 的几何锐化**

**SphereFace (A-Softmax, 2017)**：$\cos\theta_y \to \cos(m\theta_y)$，在角度上乘一个因子。但 $\cos(m\theta)$ 的单调性在 $\theta > \pi/m$ 破坏，需要 trick。

**CosFace (2018)**：$\cos\theta_y \to \cos\theta_y - m$，直接在 cosine 值上减。简单、单调、稳定。

**ArcFace (2019)**：$\cos\theta_y \to \cos(\theta_y + m)$，在角度上加 margin。**角度 margin 是几何上最自然的——它直接对应超球面上的测地距离**。

**这四代的真正收敛点**：**超球面上的类原型分布**。ArcFace 的 margin 强制每个类的 anchor 角度和其他类至少相差 $m$——这是**超球面上的 Tammes 问题**（将 $K$ 个点均匀放在球面上最大化最小距离）的近似求解。

**§4.3 为什么这条线只对人脸有效**

Angular Margin 家族在人脸识别中最自然、最强，因为身份类别具有相对稳定的类内紧致和类间排斥结构；在通用分类、检测等任务中，它的收益通常不如在人脸上稳定，需要视类别结构和任务目标而定。这是一个值得认真对待的边界条件：

**原因**：Angular margin 假设"**类别在超球面上分布是重要的**"——但这个假设**对大部分任务不成立**：

通用 ImageNet 分类：类别之间有层级关系（dog → animal），球面等距分布破坏这种层级

检测：前景-背景严重不平衡，球面几何不是主要矛盾

分割：稠密预测，"类原型"概念弱

**深层本体论**：Angular Margin 的成功**揭示了人脸任务的特殊几何**——人脸是一个"**所有类都在一个紧凑流形上**"的任务，超球面是这个流形的良好近似。换到流形不是超球面的任务（如 ImageNet 的类层级），这条假设失效。

**这是 loss 工程的一个通用教训**：**loss 注入的几何公理必须匹配任务的几何**。错配 = 失败。

**五 · Focal Loss 和 IoU Loss——任务-loss 不一致问题的两种修复**

**§5.1 Focal Loss（RetinaNet 2017）的三重身份**

$$
\text{FL} = -(1-p_t)^\gamma \log p_t
$$

**身份 1（表面）**：解决检测中 easy negative 淹没 hard positive——对 easy example 下权重，hard example 上权重。

**身份 2（中层）**：把 **"在线 hard example mining"** 写进 loss 的闭式。之前 OHEM (Shrivastava 2016) 需要显式采样 hard examples，Focal Loss 把同样的效果做成了可微 loss。

**身份 3（本体论）**：Focal Loss 是**"class imbalance 的隐式先验"**。

$gamma = 0$：退化为 CE（假设 class balanced）

$\gamma \to \infty$：只关心置信度极低的样本（假设极端 imbalance）

中间的 $gamma$（2.0 常用）：一个连续调节的"imbalance prior"

**深层含义**：**Focal Loss 不是"修改 CE"，是"在 CE 里偷偷插入一条 class imbalance 的贝叶斯先验"**。检测任务里前景：背景 ≈ 1:1000，这个先验是结构性的，不加就崩。

**§5.2 IoU / GIoU / DIoU / CIoU 的本体论命题**

IoU Loss=1−∣A∩B∣∣A∪B∣\text{IoU Loss} = 1 - \frac{|A \cap B|}{|A \cup B|}IoU Loss=1−∣A∪B∣∣A∩B∣

**背景问题**：检测任务的评估指标是 IoU，但训练用的是 L1/L2 bounding box loss。**评估度量和训练 loss 不一致** —— 训练优化 L1 不一定降 IoU。

**IoU Loss 的本体论**：**把评估度量直接做成可微 loss**。GIoU → DIoU → CIoU 是 IoU 的**可微性修正**：IoU 在不相交时梯度为零，需要修正让梯度在任何位置都有。

**通用命题**：**"loss 工程的通用公式" = 把任务的**真实评估度量**做成可微代理**。这是从 2017 之后 loss 工程的主流方法。其他例子：

分割：Dice Loss / Lovász Loss 是 F-score / Jaccard 的可微代理

排序：NDCG Loss 是 NDCG 的可微代理（信息检索）

生成：LPIPS 是 "人类感知相似度" 的可微代理

**深层洞察**：**"evaluation metric → differentiable proxy"** 是一条通用工程原则——它说明 **loss 和 metric 的分离是临时的工程残差，最终会被消灭**。

**六 · Perceptual Loss（2016）——距离本身被学出来**

$$
\mathcal{L}_{\text{perceptual}}
=
\sum_l \|\phi_l(x)-\phi_l(\hat{x})\|_2^2
$$

其中 $phi_l$ 是预训练 VGG 的第 $l$ 层特征。

**这是一个深刻的本体论转折**：

**之前的 loss**：在"输入空间"或"输出空间"度量距离（像素 L2、类概率 CE）。

**Perceptual Loss**：在**另一个神经网络的特征空间**度量距离——**距离本身是被"另一个模型"定义的**。

**含义**：

**"像素距离"不是感知距离**——两张语义相同、像素不同的图（旋转 1 像素）像素 L2 大，人类感觉一样

**预训练网络的特征空间承载了"感知相关的距离结构"**

**Perceptual Loss = 用另一个模型的表征当作"真实距离的代理"**

**这预告了后面一整条路线**：

**LPIPS (Zhang 2018)**：学习"人类感知相似度"的可学习距离

**CLIP guidance**：用 CLIP 的特征空间当作"文本-图像对齐度量"

**Knowledge Distillation**：teacher 的 feature/output 空间当作 student 的 loss target

**GAN discriminator loss**：discriminator 学的是"真实 vs 生成"的距离度量

**共同结构**：**loss 从"固定度量"升级为"学习度量"**。这是 loss 工程的**元级别升级**：不再预设距离，让距离本身成为学习对象。

**本体论断言**：**Perceptual Loss 引入了"距离的递归性" —— loss 可以用另一个 loss 训出来的模型定义，这条递归在 2020 后被 Self-Supervised Learning / GAN / Diffusion 全面利用**。

**七 · InfoNCE——损失函数的"阶段性终点"**

$$
\mathcal{L}_{\text{InfoNCE}}
=
-\log
\frac{\exp(\text{sim}(q,k^+)/\tau)}
{\sum_i \exp(\text{sim}(q,k_i)/\tau)}
$$

InfoNCE (van den Oord 2018, SimCLR 2020) 统一了之前几乎所有 loss 的本质：

**形式上是 CE**：softmax + log → CE 的结构

**内容上是度量学习**：正对 $(q, k^+)$ 要比负对近

**语义上是对比**：学"什么算一对"，不是"属于什么类"

**InfoNCE 的本体论身份**：**它融合了 "CE 的形式 + 度量学习的几何 + 对比学习的等价类定义" 三者的全部优势**。从 2020 之后，所有的自监督学习、多模态学习（CLIP）、偏好学习（DPO 的原型）几乎都建立在 InfoNCE 上。

**§7.1 InfoNCE → 统一框架**

把之前的 loss 全部重写为 InfoNCE 的特例：

| **Loss** | **InfoNCE 的角度** |
| --- | --- |
| **CE** | "正类 = 真标签类 prototype，负类 = 其他类 prototypes"，温度 $tau=1$ |
| **Triplet** | 单正单负的退化 InfoNCE |
| **Contrastive (2006)** | 单对的退化 InfoNCE |
| **ArcFace** | 在超球面 + angular margin 的 InfoNCE |
| **SimCLR** | "正类 = 同图增强，负类 = batch 其他图" |
| **CLIP** | "正类 = 对应 caption，负类 = 其他 caption" |

**这个统一不是修辞——它是 2022-2025 年 LLM/多模态训练的实际代码形态**。

**InfoNCE 的温度 $tau$** 控制"正负对区分的硬度"——小 $tau$ 对比强烈，大 $tau$ 对比柔和。**这等价于 CE 里 softmax 的温度**，把温度推到 0 等价于 Hinge，推到 ∞ 等价于均匀分布。**所以所有上面提到的 loss 本质上是 InfoNCE 家族里的不同温度 + 不同正负对定义**。

**§7.2 InfoNCE 的几何**

InfoNCE 优化的几何等价于：

**Alignment**：正对 (q, k⁺) 距离尽量小

**Uniformity**：所有样本在超球面上均匀分布

Wang & Isola 2020《Understanding Contrastive Representation Learning Through Alignment and Uniformity on the Hypersphere》给出了严格分解。**这是对比学习的几何本质**：**同一对象的不同视图聚到一点 + 不同对象在超球面上均匀铺开**。

**这个几何和 Neural Collapse 形成启发性同构**——Neural Collapse 里类均值趋向单纯形 ETF（球面上等距），InfoNCE 里每个实例的不同增强视图聚成点，点之间倾向于在球面上分散。两者可以在 $N_{\text{class}} \to N_{\text{sample}}$ 的极限想象中被比较；但这仍是 G/P 层谱系映射，不是严格等价。

**八 · 损失函数的演化轨迹——从点到分布到关系**

把 40 年损失函数按**注入对象的结构化程度**排列：

| **时代** | **Loss** | **注入对象的结构** | **本体论内容** |
| --- | --- | --- | --- |
| **1986** | MSE | 点（标量） | "预测应等于真值" |
| **1986-** | CE | 点（类别分布） | "预测应匹配类别分布" |
| **2006** | Contrastive | 对（距离） | "同类近、异类远" |
| **2015** | Triplet | 三元组（相对距离） | "正对比负对近" |
| **2017** | Focal | 类别 + 难度 | "hard example 更重要" |
| **2018** | InfoNCE | 集合（等价类） | "同一等价类内聚、跨类分散" |
| **2020+** | CLIP / DPO / RLHF | 多模态对齐 / 偏好 | "模态间一致"、"人类偏好结构" |

**轨迹方向**：注入对象的结构化程度**单调上升**：

点 → 对 → 三元组 → 集合 → 关系网络

**本体论意义**：**Loss 从"判断单个输出"演化为"约束整个表征空间的关系结构"**。这是 "点预测 → 分布预测 → 关系预测" 的完整谱系。

**这恰好对应父页的核心公理**：

**相关性是共形本体**——现代 loss 承载的不是"单点预测的准确性"，而是"**样本间的相关性结构**"。从 CE 到 InfoNCE 再到 DPO，每一代 loss 都在更精细地编码"样本之间应该有什么关系"。

**九 · 共形六轴 · A-4 归位**

| **方法** | **主要作用轴** | **本体操作** |
| --- | --- | --- |
| CE / Softmax | **度量**（类单纯形） | 类原型几何 |
| Triplet / Contrastive | **度量**（表征空间） | 拓扑关系 |
| ArcFace | **度量**（超球面） | 球面 Tammes 分布 |
| Focal | **材料**（重采样） | 隐式 class imbalance 先验 |
| IoU / Dice | **度量**（任务指标） | metric = loss 消融 |
| Perceptual | **主体性**（另一模型的空间） | 距离被另一模型学出 |
| InfoNCE | **主体性 + 度量** | 等价类定义 + 超球面均匀性 |

**核心命题**：**Loss 是"共形结构"在"目标轴"上的显式注入器**。40 年 loss 演化 = 共形结构从"点"到"关系网"的精细化过程。

**十 · A-4 立卡命题**

**「CE 的条件唯一性四重论证」** —— CE 在“最大似然 / local proper scoring / 信息几何 / Neural Collapse 倾向”这些约束交集上成为经典分类默认解。不是偶然选择，但也不是所有任务的无条件必要选择。

**「度量学习 = 分类的拓扑化」** —— 从 Contrastive Loss 开始，损失函数从"预测标签"转向"构造距离"。**语义 = 几何**，不是"语义 → 几何"。

**「Angular Margin 的任务特异性」** —— 超球面 Tammes 几何只匹配"所有类都在紧凑流形上"的任务（人脸）。这是 loss 工程的通用教训：loss 注入的几何必须匹配任务的几何。

**「Loss = metric 的闭式化」** —— IoU/Focal/Dice 都是"把任务评估指标做成可微代理"的实例。评估和训练的分离是临时工程残差，终将被消灭。

**「Perceptual Loss = 距离的可学习化」** —— Loss 从"固定度量"升级为"学习度量"。这条升级在 2020 后被 GAN/KD/CLIP/Self-Supervised 全面利用。

**「InfoNCE = 损失函数的阶段性终点」** —— CE + 度量学习 + 对比学习的三重融合。2020 后所有前沿训练（SimCLR/CLIP/DPO）的底层 loss 实际上是 InfoNCE 的变体。

**「Loss 轨迹 = 相关性本体的精细化」** —— 点预测 → 对预测 → 三元组 → 等价类 → 关系网。Loss 承载的不是"单点正确"，是"样本间相关性结构"。这直接等价于父页"相关性是本体"公理在 loss 轴上的展开。
