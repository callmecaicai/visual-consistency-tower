# A-3 · 学习率调度与 Batch Size 的几何

<aside>
🧭

**章首协议（接回第一原理层）**：本章按“稳定差异 → 闭合半径 → 出口残差”三问读取。

- **稳定差异**：学习率、batch size、噪声尺度与训练时长之间的可换算关系。
- **闭合半径**：把阶段 I 的“能训练”从静态初始化推进到动态训练制度，使优化轨迹在时间上保持稳定。
- **出口残差**：训练 recipe 可以显著改变最终几何，但它仍服务于外部标签和固定 loss，不能解决标签制度本身的遮蔽。

</aside>

## 一 · 本页定位

学习率和 batch size 不是“调参边角料”。在阶段 I 里，它们承担的是时间尺度上的共形任务：

```text
梯度方向  +  步长尺度  +  batch 噪声  +  训练时长
      ->  一条可控的参数轨迹
```

如果初始化回答“信号能否穿过层”，优化器回答“梯度如何被记忆和缩放”，那么学习率调度与 batch size 回答的是：

> **同一套损失曲面上，参数应该以什么时间尺度移动，移动时保留多少随机噪声。**

---

## 二 · 学习率：不是速度旋钮，而是轨迹形状

学习率过大，优化会越过低损区域；学习率过小，训练会困在缓慢局部。阶段 I 的关键发现是：学习率不是常数，而是一条时间函数。

| 调度 | 机制 | 共形含义 |
| --- | --- | --- |
| Step decay | 到固定 epoch 降低学习率 | 人工切换训练相位。 |
| Exponential decay | 连续收缩步长 | 把大步探索逐渐压成小步收敛。 |
| Cosine decay | 平滑退火到低学习率 | 让轨迹从探索自然过渡到收束。 |
| Warmup | 起步阶段逐渐升高学习率 | 防止早期梯度尺度尚未稳定时撕裂表征。 |
| Cyclical / restart | 周期性升降学习率 | 让优化轨迹反复跳出局部盆地。 |

从共形角度看，学习率调度是在时间轴上控制“更新尺度是否仍与当前表征几何相容”。它不是单纯让训练更快，而是决定模型以什么节奏从粗闭合走向细闭合。

---

## 三 · Batch Size：噪声不是污染，而是训练动力

小 batch 带来更强随机噪声，大 batch 提供更稳定梯度估计。二者不是简单优劣，而是两种训练动力：

| Batch 形态 | 优势 | 风险 |
| --- | --- | --- |
| 小 batch | 噪声强，容易探索，隐式正则更强 | 梯度估计抖动大，硬件效率低 |
| 大 batch | 梯度稳定，吞吐高，可规模化 | 噪声不足，可能收敛到泛化较差区域 |
| 极大 batch | 分布式训练效率高 | 需要 warmup、LARS/LAMB 等额外稳定机制 |

SGD 可以被近似读成 SDE：梯度均值决定漂移项，batch 噪声决定扩散项。这个读法的关键不是把 SGD 变成物理学，而是说明：

> **训练中的随机性本身参与塑造解的几何。**

因此，batch size 改变的不只是速度，也改变优化轨迹偏好的解区域。

---

## 四 · Linear Scaling 与 Warmup

大 batch 训练的经典经验法则是 linear scaling：

```text
batch size 放大 k 倍，learning rate 近似放大 k 倍
```

但这条规则只有在梯度统计足够稳定时才成立。训练早期，模型表征和 BN 统计都尚未稳定，直接使用大 lr 容易让参数轨迹撕裂。因此 warmup 不是工程妥协，而是大 batch 条件下的时间共形修复：

```text
先让表征统计进入可换算区域
再让大步长发挥吞吐优势
```

---

## 五 · Weight Decay 与 AdamW：正则和优化器不能混在一起

Weight decay 的经典解释是高斯先验或 L2 正则，但在深网中它还承担轨迹导航功能：持续压低权重范数，使优化更偏向平坦区域。

AdamW 的关键修正是把 weight decay 从 loss 梯度中解耦出来。原因很直接：在 Adam 里，L2 惩罚会被自适应二阶矩重新缩放，导致“正则强度”不再是我们以为的强度。

所以 AdamW 的本质是：

> **把参数空间的几何偏好，从自适应梯度尺度中重新分离出来。**

这说明阶段 I 的训练工艺不是独立小技巧，而是一组互相耦合的度量协议。

---

## 六 · 四层正则的替代性

阶段 I 的正则化可以按作用位置分成四层：

| 层级 | 代表机制 | 作用 |
| --- | --- | --- |
| 输入层 | 数据增强、Mixup、CutMix | 改写训练分布。 |
| 激活层 | Dropout、DropBlock | 随机化前向路径。 |
| 权重层 | Weight decay、spectral norm | 改写参数几何偏好。 |
| 输出层 | Label smoothing、KD | 软化目标分布。 |

这些正则并不完全独立。强数据增强可以部分替代 Dropout；强 teacher 可以部分替代 label smoothing；大模型规模和大数据也会改变正则的有效位置。

这条规律可以写成阶段 I 的工作命题：

> **正则化不是降低容量，而是在过大容量中塑造可被选择的解形状。**

---

## 七 · Label Smoothing、KD 与 Information Bottleneck 的关系

Label smoothing 可以被看成知识蒸馏的退化形式：

```text
teacher = uniform distribution
```

KD 则使用一个更有结构的 teacher 分布，保留类间混淆关系。二者共同降低目标端的尖锐度，迫使模型不要把所有表征容量都压向 one-hot 的无穷置信。

这与 Information Bottleneck 形成启发性同构：表征不应保留与标签无关的输入细节，也不应对标签噪声过度拟合。但这里仍需边界：

> Label smoothing / KD / IB 可以被并置比较，但它们不是严格同一个目标。IB 是信息论目标，LS/KD 是输出分布工程。

---

## 八 · 本页残差

学习率、batch size、weight decay、warmup 与正则化共同让阶段 I 的训练 recipe 变得可迁移。但它们都没有改变阶段 I 的根制度：

```text
所有训练工艺仍服务于 image -> label
```

因此，A3 的出口残差不是“还需要更好调参”，而是：

> **再好的训练制度，也只能让标签闭合更稳，不能让标签闭合自动变成空间闭合。**
