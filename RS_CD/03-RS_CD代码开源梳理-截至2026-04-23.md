# RS_CD 代码开源梳理

更新时间：`2026-04-23`

这份文档只服务一个目的：判断 `RS_CD` 当前主线论文里，哪些**已经有可访问的开源代码**，哪些只是**挂了官方仓库但代码还没真正放出**，哪些则是**目前还没查到官方代码**。

默认判定标准尽量保守：

- `已明确开源`：论文页或官方仓库明确给出 GitHub，并且仓库里已经有代码文件、训练/推理说明，或至少已有可运行结构。
- `仓库已建但未完全放出`：官方仓库存在，但 README 明说代码仍在整理、即将发布，或者当前只有极少文件，尚不足以直接复现。
- `暂未查到官方代码`：截至本次核查，没有查到论文作者或论文页指向的官方代码仓库。

但对当前项目的**实用判断**还要补一条：

- 如果官方仓库已经公开，并且至少放出了**推理代码**或**训练 / 推理中的一部分主干代码**，那它就已经进入“可跟踪、可学习、可接入”的开源链。

按这个更实用的口径，`UniChange` 可以视为**已经进入开源链**。

## 二、最值得直接复现的开源主线

如果目标是**快速搭 baseline 或对齐主线演进**，下面这一组最实用：

1. `FC-EF / FC-Siam`  
   这是现代深度 CD 的起点，代码最直接，结构最清楚。  
   仓库：[rcdaudt/fully_convolutional_change_detection](https://github.com/rcdaudt/fully_convolutional_change_detection)

2. `STANet` + `SNUNet-CD`  
   这两篇基本代表 CNN 范式成熟段。  
   仓库：[justchenhao/STANet](https://github.com/justchenhao/STANet)  
   仓库：[likyoo/Siam-NestedUNet](https://github.com/likyoo/Siam-NestedUNet)

3. `BIT` + `ChangeFormer`  
   这是 Transformer 进入 CD 的最关键两篇。  
   仓库：[justchenhao/BIT_CD](https://github.com/justchenhao/BIT_CD)  
   仓库：[wgcban/ChangeFormer](https://github.com/wgcban/ChangeFormer)

4. `TTP / ChangeCLIP / BAN / MTP`  
   这是基础模型迁移阶段最值得读代码的一组。  
   仓库：[KyanChen/TTP](https://github.com/KyanChen/TTP)  
   仓库：[dyzy41/ChangeCLIP](https://github.com/dyzy41/ChangeCLIP)  
   仓库：[likyoo/BAN](https://github.com/likyoo/BAN)  
   仓库：[ViTAE-Transformer/MTP](https://github.com/ViTAE-Transformer/MTP)

5. `AdaptOVCD / Seg2Change / UniChange / Change-Agent / TEOChat`  
   这是当前前沿里最有实操价值、且代码已经能抓到的一组。  
   仓库：[Dmygithub/AdaptOVCD](https://github.com/Dmygithub/AdaptOVCD)  
   仓库：[yogurts-sy/Seg2Change](https://github.com/yogurts-sy/Seg2Change)  
   仓库：[NKU-HLT/UniChange](https://github.com/NKU-HLT/UniChange)  
   仓库：[Chen-Yang-Liu/Change-Agent](https://github.com/Chen-Yang-Liu/Change-Agent)  
   仓库：[ermongroup/TEOChat](https://github.com/ermongroup/TEOChat)

## 三、几个需要特别区分的情况

### 1. `OpenDPR` 不是“没仓库”，而是“仓库已建但代码还没完全放出”

当前官方仓库是：

- [guoqi2002/OpenDPR](https://github.com/guoqi2002/OpenDPR)

但 README 里明确写了：

- `We are currently organizing the code and will release it soon.`

所以它不该被记成“已可复现”，更准确的状态是：

- **官方仓库已存在**
- **当前仍偏占位**
- **可以继续跟踪，但暂时不宜当作已开源成熟代码**

### 2. `RSCC` 更像“数据与评测基座已开”

`RSCC` 不是传统意义上的单模型 repo。它开出来的是：

- 数据集
- 推理代码
- 评测代码
- 训练脚本

这对变化理解阶段其实非常关键，因为它给的是**统一实验基座**。  
仓库：[Bili-Sakura/RSCC](https://github.com/Bili-Sakura/RSCC)

### 3. `AnyChange` 的代码入口在统一库里

`AnyChange` 论文里给出的代码入口是：

- [Z-Zheng/pytorch-change-models](https://github.com/Z-Zheng/pytorch-change-models)

这类情况后面会越来越多：不是每篇论文单开一个小 repo，而是被并入一个统一 change library。对复现来说这反而更实用。

### 4. `UniChange` 目前更准确的状态是“实用开源，但未完整放全”

当前官方仓库是：

- [NKU-HLT/UniChange](https://github.com/NKU-HLT/UniChange)

但截至本次核查：

- README 写明 `2026/04/04` 已发布 training and inference code；
- 同时又把 `dataset` 和 `checkpoints` 继续列在待发布项里。

所以它不适合被记成“完全没开源”，也不适合被记成“已经完整开源可复现”。  
更准确的状态是：

- **仓库公开**
- **部分训练 / 推理代码已放出**
- **数据与权重仍未完整放出**
- **但按当前项目的实用标准，已经可以算进入开源链**

## 四、额外值得长期关注的开源基础设施

这些不一定对应单篇论文，但对后续复现实验很重要：

- [likyoo/open-cd](https://github.com/likyoo/open-cd)  
  现在已经是 RS-CD 里最重要的通用训练/复现底座之一。

- [xwmaxwma/rschange](https://github.com/xwmaxwma/rschange)  
  更偏工具箱，持续收进新方法。

- [Z-Zheng/pytorch-change-models](https://github.com/Z-Zheng/pytorch-change-models)  
  现在也在向 `torchange` 统一。

- [wenhwu/awesome-remote-sensing-change-detection](https://github.com/wenhwu/awesome-remote-sensing-change-detection)  
  更适合查方法、数据集、代码入口。

## 五、当前结论

如果只问一句：

**RS_CD 这条线里，代码开放情况其实已经不差。**

尤其是：

- 前两阶段（CNN / Transformer）主干基本都能找到官方代码。
- 第三阶段（基础模型迁移）已经有一批质量不错的官方 repo。
- 第四、第五阶段里，真正麻烦的不是“完全没有开源”，而是**前沿论文的放码速度不一致**。

截至当前，最需要继续盯的几个空位是：

- `Semantic-CD`
- `CoRegOVCD`
- `Delta-LLaVA`
- `SAM-CD`
- `UniChange` 的数据与 checkpoints 完整开放情况

其中 `OpenDPR` 需要继续观察，因为它已经有官方仓库，但还不能算完全放出。
