# RS_CD 前沿补遗（2025-01 至 2026-04）：从开放词汇 CD 到变化理解系统

时间范围：`2025-01-01` 至 `2026-04-23`  
适用范围：只补充 `RS_CD` 这条线的最新前沿，不重写更早历史。  
在新的 `5 阶段` 总纲里，这篇主要展开的是 **第 3-5 阶段**。  
说明：下面只使用论文原始页面、会议 open access、OpenReview 页面做初步梳理。这里的目标不是定论，而是把“**哪些是真正的新阶段**”和“**哪些只是旧阶段里的更强模型**”分开。

## 一句话总判断

截至 `2026-04-23`，RS-CD 最近一轮真正的推进，不再只是“把二值变化图做得更准”，而是同时出现了四个收束方向：

1. **开放词汇变化检测（OVCD）开始从概念走向成熟框架。**
2. **变化检测开始从 mask 输出走向 change understanding：referring、caption、QA、MLLM 一体化。**
3. **监督不再是默认前提，weak / training-free / time-series 自监督开始成为正面主线。**
4. **配准、跨时相语义可比性、用户查询条件这三条旧残差重新回到舞台中央。**

如果沿用前一版总纲的阶段语言，那么我会把这批工作压成一个新的补充阶段：

## 最新残差阶段：`⑦ 开放词汇变化理解阶段`

它不再把任务定义成：

- 给我两张图，输出一张二值 change map。

而开始定义成：

- 给我两张或多张图；
- 你要能回答**指定类型**的变化；
- 最好不用或少用专门 CD 标注；
- 最终不仅要指出哪里变了，还要说清**变了什么、为什么算变化、能否继续追问**。

这就是为什么最近 15 个月最重要的论文，不是同一类 backbone 的微小优化，而是围绕下面几条残差展开：

- **概念在两时相之间如何可比？**
- **开放类别变化如何被 query 化？**
- **语言接口如何不牺牲空间 grounding？**
- **数据稀缺时，监督还能不能继续退场？**
- **变化能否成为一个可描述、可问答、可调度的内部对象？**

## 先分清两类工作

### A. 真正在改写阶段边界的

- `Semantic-CD`
- `RSCC`
- `Referring Change Detection`
- `AdaptOVCD`
- `OpenDPR`
- `CoRegOVCD`
- `Seg2Change`
- `UniChange`
- `Decoding the Delta / Delta-LLaVA`
- `Remote Sensing Change Detection via Weak Temporal Supervision`

### B. 重要，但更像旧阶段内的强基线增强

- `M-CD`：Mamba 把监督二值 CD 做强
- `DDPM-CD`：把 diffusion feature 当更强预训练表征
- `RemoteVAR`：把生成式 autoregressive 引入 change map 预测
- `DiffRegCD`：把 registration 与 CD 联合起来
- `SAT-Cap`：把 captioning 做得更轻更强
- `SkySense V2`：把 RS 基础模型底座做得更统一

这类工作很重要，但它们大多没有改写“变化是什么”的定义，只是在现有阶段里把能力做得更强。

## 1. 开放词汇 CD 正式成形

这一支是最近最清楚的新主线。

### 1.1 `Semantic-CD`：开放词汇语义 CD 被正式提出

- 论文：[Semantic-CD: Remote Sensing Image Semantic Change Detection towards Open-vocabulary Setting](https://arxiv.org/abs/2501.06808)
- 时间：`2025-01-12`
- 关键信号：
  - 明确把 `semantic change detection` 推向 `open-vocabulary`；
  - 把 `CLIP` 语义先验正式拉进双时相 CD；
  - 用解耦的二值 CD + 语义 CD 结构，避免“哪里变了”和“变成什么”纠缠在一个头里。

我的判断：
这篇的意义不只是“加了 CLIP”，而是第一次明确承认：**变化类别不应再被训练集标签空间锁死。**

它解决的残差：
- 封闭类别 semantic CD 无法泛化到真实场景里的新类别。

它留下的残差：
- 语义先验仍是借来的；
- 两时相之间的开放概念响应还不够稳定；
- 还不是 training-free。

### 1.2 `AdaptOVCD`：training-free OVCD 开始系统化

- 论文：[AdaptOVCD: Training-Free Open-Vocabulary Remote Sensing Change Detection via Adaptive Information Fusion](https://arxiv.org/abs/2602.06529)
- 时间：`2026-02-06`
- 关键信号：
  - 不再单押一个 foundation model；
  - 显式把 `SAM-HQ + DINOv3 + DGTRS-CLIP` 放进多层自适应融合；
  - 目标不是“开放词汇语义理解”本身，而是 **zero-shot arbitrary-category change detection**。

我的判断：
到这一步，OVCD 已经不是“用 CLIP 试试看”的概念验证，而是变成了一个完整工程范式：  
**数据层对齐 -> 特征层差异 -> 决策层语义筛选。**

它解决的残差：
- training-free 路线里多模型串联误差太大；
- radiometric shift 和阈值不稳影响很重。

它留下的残差：
- pipeline 仍然很重；
- 本质仍是多基座拼接，不是统一变化空间。

### 1.3 `OpenDPR`：OVCD 的瓶颈被明确改写为“类别识别”

- 论文：[OpenDPR: Open-Vocabulary Change Detection via Vision-Centric Diffusion-Guided Prototype Retrieval for Remote Sensing Imagery](https://arxiv.org/abs/2603.27645)
- 时间：`2026-03-29`
- 状态：论文页标注 `Accepted by CVPR 2026`
- 关键信号：
  - 直接指出：OVCD 的主瓶颈不是 change proposal，而是 **category identification**；
  - 认为普通 VLM 的图文匹配不足以承载细粒度 land-cover 类别；
  - 用 diffusion 生成离线 prototype，在视觉空间做 prototype retrieval；
  - 同时承认 change localization 仍弱，于是又补了一个弱监督 `S2C` 模块。

我的判断：
这篇非常关键，因为它把一个模糊问题说透了：

**OVCD 不是“先找到变化，再贴个词”这么简单。真正难的是：变化区域一旦被找到，类别到底靠什么认。**

它解决的残差：
- 纯 CLIP / image-text matching 对细粒度地物类别不够用。

它留下的残差：
- 视觉 prototype 是否足够覆盖长尾类别仍未彻底解决；
- 仍是组合系统，而非统一时序语义模型。

### 1.4 `CoRegOVCD`：两时相概念响应的“可比性”被正式问题化

- 论文：[CoRegOVCD: Consistency-Regularized Open-Vocabulary Change Detection](https://arxiv.org/abs/2604.02160)
- 时间：`2026-04-02`
- 关键信号：
  - 把 training-free OVCD 的核心困难明确写成：**dense concept responses are difficult to compare directly across dates**；
  - 不再满足于“比较两张图的语义图”；
  - 转向 `posterior discrepancy`、competition-aware calibration、geometry-aware consistency。

我的判断：
如果说前面的 OVCD 论文还在“把开放词汇拉进 CD”，那么这篇开始处理更深的 residual：

**同一个概念在 t1 与 t2 上，是否真的可比。**

这其实是 OVCD 目前最本质的问题之一。  
因为跨时相变化、光照、季节、尺度、纹理扰动，会让开放词汇响应本身不稳定。

它解决的残差：
- 直接比较 dense semantic response 容易碎、噪、假变化。

它留下的残差：
- 仍然没有真正的多时相世界状态；
- 更多是在做后验校准，而不是直接学习变化本体。

### 1.5 `Seg2Change`：OVS 到 OVCD 的通用适配器出现

- 论文：[Seg2Change: Adapting Open-Vocabulary Semantic Segmentation Model for Remote Sensing Change Detection](https://arxiv.org/abs/2604.11231)
- 时间：`2026-04-13`
- 关键信号：
  - 直接回答“开放词汇分割模型怎么迁移到 change detection”；
  - 建了 `CA-CDD` 这类 category-agnostic CD 数据；
  - 用 `category-agnostic change head` 承接任意类别过渡。

我的判断：
这篇的重要性在于，它把 OVCD 从“多模型拼装 pipeline”推进成了更像标准范式的 **adapter 化问题**。

它解决的残差：
- OVS 模型强在单时相语义，弱在直接处理时相转移。

它留下的残差：
- 依然没有从根上解决“变化状态空间”的问题；
- 更像 OVS -> CD 的适配，而不是原生多时相理解。

## 2. 从开放词汇走向“可指代变化”

### `Referring Change Detection in Remote Sensing Imagery`

- 论文：[Referring Change Detection in Remote Sensing Imagery](https://openaccess.thecvf.com/content/WACV2026/html/Korkmaz_Referring_Change_Detection_in_Remote_Sensing_Imagery_WACV_2026_paper.html)
- arXiv 版本时间：`2025-12-12`
- 会议：`WACV 2026`，`2026-03`
- 关键信号：
  - 不再默认“检测所有变化”；
  - 允许用户用自然语言指定想看的变化类型；
  - 同时引入 `RCDGen`，用 diffusion 生成指定类别变化的数据。

我的判断：
这篇意味着 CD 的问题定义又前进了一步：

**变化检测不再是统一出口，而是 query-conditioned。**

也就是说，未来系统可能不是给一张总 change map，而是根据不同用户问题输出：

- 建筑新增在哪里？
- 被水淹没的区域在哪里？
- 只看道路相关变化。

它解决的残差：
- “所有变化一锅端”不符合真实应用的任务接口。

它留下的残差：
- 数据仍稀缺，类别不均衡严重；
- referring 还是局部 query，不是完整 change understanding。

## 3. 从 change captioning 走向 change understanding

这一支和你目录里已有的 `⑥-A / ⑥-B / ⑥-C` 是直接连续的，但 2025-2026 的推进比之前更清楚。

### 3.1 `SAT-Cap`：captioning 的效率线继续推进

- 论文：[Change Captioning in Remote Sensing: Evolution to SAT-Cap -- A Single-Stage Transformer Approach](https://arxiv.org/abs/2501.08114)
- 时间：`2025-01-14`
- 关键信号：
  - 把多阶段融合改成 single-stage；
  - 强调 change captioning 的效率和 object semantics 提取。

我的判断：
它不是新阶段，但它说明了旧的 captioning 线在 2025 年还在继续往“更轻、更可用”推进。

### 3.2 `RSCC`：change captioning 数据规模突然上来了

- 论文页：[RSCC: A Large-Scale Remote Sensing Change Caption Dataset for Disaster Events](https://openreview.net/forum?id=yn2fJYBKEB)
- arXiv 版本：[arXiv:2509.01907](https://arxiv.org/abs/2509.01907)
- arXiv 时间：`2025-09-02`
- OpenReview 发布时间：`2025-09-18`
- 关键信号：
  - `62k+` pre/post-disaster image pairs；
  - 把 change captioning 拉到真正大规模 benchmark 级别；
  - 场景落在灾害变化分析，明显更接近真实应用。

我的判断：
这篇不是方法突破，而是任务基础设施突破。  
它意味着“变化描述”这条线终于有了像样的数据规模，不再只是小数据集上的精致实验。

### 3.3 `UniChange`：BCD / SCD 开始被 MLLM 统一

- 论文：[UniChange: Unifying Change Detection with Multimodal Large Language Model](https://arxiv.org/abs/2511.02607)
- 时间：`2025-11-04`
- 关键信号：
  - 直接把 `BCD` 和 `SCD` 统一到一个 `MLLM` 框架里；
  - 引入 `[T1]`、`[T2]`、`[CHANGE]` 三个 special tokens 作为统一任务接口；
  - 用 text prompts 代替固定分类头，允许从多源数据集联合吸收知识，即使类定义冲突。

我的判断：
`UniChange` 很关键，但它和 `Semantic-CD`、`OpenDPR` 不是同一跳。

`Semantic-CD` 更像是在把开放词汇语义带进双时相 CD；  
`UniChange` 则是在回答另一个问题：

**能不能不再把 BCD 和 SCD 拆成两套模型，而是让一个 MLLM 统一承接两种任务。**

所以它的地位更接近：

- 从“变化语义外显”走向“变化任务统一化”；
- 从专用网络走向 `query-guided unified CD model`。

它解决的残差：
- BCD / SCD 长期分裂；
- 多源数据集标签定义冲突，知识难以共享；
- 固定分类头限制变化类别的外延。

它留下的残差：
- 统一的是任务接口，不等于已经统一了时序世界状态；
- 仍然主要围绕双时相变化任务组织；
- 空间 grounding 与高阶推理的一体化仍未完全解决。

### 3.4 `Decoding the Delta / Delta-LLaVA`：变化理解一体化真正出现

- 论文：[Decoding the Delta: Unifying Remote Sensing Change Detection and Understanding with Multimodal Large Language Models](https://arxiv.org/abs/2604.14044)
- 时间：`2026-04-15`
- 关键信号：
  - 直接指出普通 MLLM 在 RS change understanding 上有 **temporal blindness**；
  - 提出 `Delta-QA`，把 pixel-level segmentation 和 VQA 放到一个 benchmark 里；
  - 引入 `Change-Enhanced Attention`、`Change-SEG`、`Local Causal Attention`，显式做时相差分与防泄漏。

我的判断：
这篇很可能是最新一轮里最像“新阶段信号”的工作之一。

因为它不再满足于：

- change mask；
- 单句 caption；
- 单轮 QA。

而是想把 **分割 + 问答 + 多时相推理** 做成统一 change understanding。

它解决的残差：
- 通用 MLLM 对多时相差异“看不见”；
- 语言推理与精细空间定位分裂。

它留下的残差：
- 仍是双/三时相任务导向；
- 还没有真正变成长期、多状态、可反事实的世界模型。

## 4. 监督正在退场，但方式变了

这一支不是单一模型线，而是最新阶段的“数据与训练范式重写”。

### 4.1 `The Change You Want To Detect`

- 论文：[The Change You Want To Detect: Semantic Change Detection In Earth Observation With Hybrid Data Generation](https://openaccess.thecvf.com/content/CVPR2025/html/Benidir_The_Change_You_Want_To_Detect_Semantic_Change_Detection_In_CVPR_2025_paper.html)
- 会议：`CVPR 2025`，`2025-06`
- 关键信号：
  - 提出 `HySCDG` 生成式 pipeline；
  - 用真实 + inpainted 的 hybrid 数据生成 `FSC-180k`；
  - 明确把语义 change detection 的数据问题抬到主位。

我的判断：
这篇说明一个非常现实的趋势：  
**不是所有前沿都在改模型，很多前沿是在改数据制造方式。**

### 4.2 `OPTIMUS`

- 论文：[OPTIMUS: Observing Persistent Transformations in Multi-temporal Unlabeled Satellite-data](https://arxiv.org/abs/2506.13902)
- WACV 页面：[OPTIMUS @ WACV 2025](https://openaccess.thecvf.com/content/WACV2025/html/Yu_OPTIMUS_Observing_Persistent_Transformations_in_Multi-Temporal_Unlabeled_Satellite-Data_WACV_2025_paper.html)
- arXiv 时间：`2025-06-16`
- 会议：`WACV 2025`，`2025-02`
- 关键信号：
  - 不依赖 change labels；
  - 直接从时间序列顺序恢复与 change point detection 出发；
  - 目标不是二值双时相 benchmark，而是持续变化监测。

我的判断：
这篇的重要性在于，它把问题从“配对双时相”挪向了“长期时间序列中的 persistent transformation”。

这和你后面想走的“状态持有”是相通的。

### 4.3 `Remote Sensing Change Detection via Weak Temporal Supervision`

- 论文：[Remote Sensing Change Detection via Weak Temporal Supervision](https://arxiv.org/abs/2601.02126)
- 时间：`2026-01-05`
- 关键信号：
  - 不新标注像素级 change；
  - 通过给单时相数据补不同时间观测，假设真实双时相大多不变、跨地点配对大多变化；
  - 用 object-aware map 和 iterative refinement 处理弱标签噪声。

我的判断：
这是最近非常值得注意的一条线。  
它说明监督并不是简单“有 / 无”二分，而正在被改写成：

- 强监督；
- 弱时间监督；
- 合成监督；
- training-free 推断。

## 5. 旧阶段里的强增强：生成先验、状态空间与配准

这些工作未必构成新阶段，但对你后续读法很重要，因为它们往往揭示“旧阶段哪里还不稳”。

### `M-CD`

- 论文：[A Mamba-Based Siamese Network for Remote Sensing Change Detection](https://openaccess.thecvf.com/content/WACV2025/html/Paranjape_A_Mamba-Based_Siamese_Network_for_Remote_Sensing_Change_Detection_WACV_2025_paper.html)
- 会议：`WACV 2025`，`2025-02`

判断：
它说明监督二值 CD 仍在吃“更大感受野 + 线性复杂度状态空间模型”红利。  
但它没有改写阶段边界，更多是 `③/④` 的强增强。

### `DDPM-CD`

- 论文：[DDPM-CD: Denoising Diffusion Probabilistic Models as Feature Extractors for Remote Sensing Change Detection](https://openaccess.thecvf.com/content/WACV2025/html/Bandara_DDPM-CD_Denoising_Diffusion_Probabilistic_Models_as_Feature_Extractors_for_Remote_WACV_2025_paper.html)
- 会议：`WACV 2025`，`2025-02`

判断：
这篇说明 generative prior 开始被正式当作 CD 的预训练表征，而不是只服务于图像生成。

### `RemoteVAR`

- 论文：[RemoteVAR: Autoregressive Visual Modeling for Remote Sensing Change Detection](https://arxiv.org/abs/2601.11898)
- 时间：`2026-01-17`

判断：
它把 autoregressive generation 拉进了 dense CD，说明“生成先验进入 CD”已经不只是一条 diffusion 支线。

### `DiffRegCD`

- 论文：[DiffRegCD: Integrated Registration and Change Detection with Diffusion Features](https://arxiv.org/abs/2511.07935)
- arXiv 时间：`2025-11-11`
- 状态：论文页标注 `Accepted to WACV 2026`

判断：
这篇很重要，因为它提醒了一条被很多 benchmark 低估的老残差：

**真实变化检测并不总是建立在完美配准之上。**

所以“registration residual”在 2025-2026 明显重新抬头了。

## 6. 基座与系统层：不是 CD 方法，但会反过来改写 CD

### `SkySense V2`

- 论文：[SkySense V2: A Unified Foundation Model for Multi-modal Remote Sensing](https://openaccess.thecvf.com/content/ICCV2025/html/Zhang_SkySense_V2_A_Unified_Foundation_Model_for_Multi-modal_Remote_Sensing_ICCV_2025_paper.html)
- 会议：`ICCV 2025`，`2025-10`

判断：
它不是 CD 论文，但它会直接影响下一轮 CD。  
因为 OVCD、change understanding、training-free pipeline 的上限，很大程度上取决于你手里有没有一个更像“RS-native”的基础模型。

### `CHOICE`

- 论文页：[CHOICE: Benchmarking the Remote Sensing Capabilities of Large Vision-Language Models](https://openreview.net/forum?id=GZ5LI6Fetp)
- OpenReview 发布时间：`2025-09-18`

判断：
它的重要性在于：开始系统测 VLM 在 RS 里的真实能力边界。  
这对 change understanding 尤其关键，因为很多“看起来会说”的模型，在 RS 时序理解上并不稳。

### `REMSA`

- 论文页：[REMSA: An LLM Agent for Foundation Model Selection in Remote Sensing](https://openreview.net/forum?id=F1ZD5wi02T)
- OpenReview 时间：`2025-09-19`，`2026-02-11` 更新

判断：
它不是 change detection 方法，但它很像未来系统层的一部分：  
**不是训练一个万能 CD 模型，而是让 agent 从多个基础模型中选最合适的组合。**

这和你现在这套知识库 + agent 的路线其实是同构的。

## 7. 这一轮前沿之后，新的总残差是什么

如果把最近 15 个月所有这些工作压成一条总残差，我会写成：

**变化已经开始被开放词汇化、可指代化、可描述化，但它仍然没有成为一个系统自己持有的稳定时序状态空间。**

更具体地说，当前前沿最深的未决问题有五个：

1. **跨时相语义可比性**
   同一个概念在两个时相上的响应，到底如何校准，才不把季节、光照、分辨率差当成语义变化。

2. **从 query-conditioned 到 state-conditioned**
   现在很多方法还能说是“给一个词，查一种变化”；还没真正做到“系统内部先持有变化状态，再按需回答问题”。

3. **从双时相到多时相**
   `OPTIMUS`、`Delta-LLaVA` 已经露出这条线，但主流还是 pairwise。

4. **从 borrowed semantics 到 owned semantics**
   大量高层能力来自 `SAM / CLIP / DINO / diffusion / LLM` 的借用，而不是 RS-CD 自己形成的变化本体。

5. **从 pipeline 到 unified model**
   现在最强方法往往是组合系统；但组合越强，系统越碎。统一模型尚未真正出现。

## 8. 因而，最新残差阶段应该怎样命名

我建议把 `RS_CD` 总纲里的最后一段，进一步细分成：

- `⑦-A 开放词汇变化检测`
- `⑦-B 可指代变化检测`
- `⑦-C 变化理解系统`

如果只取一个总名，我会叫它：

**⑦ 开放词汇变化理解阶段**

因为它的共同点不是具体模型结构，而是：

- 变化开始接受语言条件；
- 变化开始摆脱固定标签表；
- 变化开始走向弱监督 / 免训练；
- 变化开始从“检测结果”变成“可被理解、可被追问的对象”。

## 10. 最短结论

如果把最近这批论文再压成一句最短的话，我会写成：

**RS-CD 最新的前沿，不再只是“找出变化”，而是在逼系统回答：面对任意用户指定的语义变化，能否在少监督甚至免训练条件下，稳定地定位、命名、描述，并继续对话。**

这意味着 `RS_CD` 这条线已经开始从：

**变化检测**

走向：

**变化理解系统。**
