# 反身脊柱

贯穿五阶段的纵向追问：每一阶段的度量从哪里来，命中了什么，遮蔽了什么，又如何被下一阶段改写？

审计文件指出：「如果反身是深渊，大厦的每一层都应该在自己的位置上报告它对反身问题的贡献或回避。」本文件是这条纵向脊柱的实现。

---

## 追问模板

每阶段结束时回答以下五个问题：

1. **本阶段用什么度量判定闭合？** （损失函数、评价指标、匹配策略）
2. **这个度量命中的对象是什么？**
3. **这个度量遮蔽了什么对象？**
4. **被遮蔽对象如何以残差形式返回？**
5. **下一阶段是否改写了度量本身？**

反身的硬定义：

> 反身 = 度量本身从外部裁判，变成系统或理论必须解释、审计、修订的对象。

反身还必须区分主体，避免把“理论在反思指标”偷换成“模型自己已经反身”：

| 类型 | 名称 | 谁在反身 | 判定边界 |
|---|---|---|---|
| `Rfl-T` | 理论反身 | 研究者 / 理论框架 | 审计度量合法性，说明命中什么、遮蔽什么 |
| `Rfl-C` | 共同体反身 | benchmark、审稿、leaderboard、评估协议 | 共同体修订评价对象与评价流程 |
| `Rfl-M` | 模型反身 | 模型内部 | 模型能发现、选择、修订或生成度量 |

当前大厦主要已经启动 `Rfl-T`，部分进入 `Rfl-C`；`Rfl-M` 仍是终极残差，不能提前宣称完成。

## 阶段 I · 判别共形

| 问题 | 回答 |
|---|---|
| 闭合度量 | 交叉熵 + Top-1/Top-5 accuracy |
| 由谁固定 | 人工设计——ImageNet 竞赛规则直接规定了度量 |
| 命中对象 | 整图级类别可分性；同物异形仍归一类 |
| 遮蔽对象 | 空间内部、对象边界、部件关系、位置与形状 |
| 残差返回 | 标签闭合吃不掉空间内部，逼出检测、分割、关键点、深度等稠密接口 |
| 下一阶段是否改写度量 | 是。阶段 II 把单标签 accuracy 改写为 IoU / Dice / mAP / PQ 等空间化度量 |
| 被质疑过吗 | 阶段内基本未被质疑。Label smoothing（§04）软化了目标分布，但没有质疑“分类正确率”本身作为度量的合法性。 |
| 反身贡献 | **零**。阶段I的网络完全不知道自己在被什么度量评判。 |

---

## 阶段 II · 稠密共形

| 问题 | 回答 |
|---|---|
| 闭合度量 | Dice / IoU / mAP / Focal Loss / Hungarian 匹配 / PQ / RMSE——每种稠密任务有自己的度量 |
| 由谁固定 | 人工设计——COCO 评估协议、PASCAL VOC 协议、Cityscapes 协议各自规定 |
| 命中对象 | 空间局部、框、mask、实例、全景区域、深度值等结构化输出 |
| 遮蔽对象 | 开放词汇语义、对象关系、语言调用、用户意图、跨模态公共意义 |
| 残差返回 | 稠密场闭合已经压进空间内部，但类别表、关系结构和语义调用仍外部给定 |
| 下一阶段是否改写度量 | 是。阶段 III 把闭合从空间指标推向图文对齐、caption、VQA、grounding 等公共语义接口 |
| 被质疑过吗 | 部分质疑：Hungarian 匹配（DETR）用可学习的匹配代替了 NMS 的硬规则；但匹配的代价函数本身仍是人工设计的。Mask2Former 统一了三种分割的度量接口，但没有让网络自己选择度量。 |
| 反身贡献 | **微弱**。匹配策略从硬编码走向可学习，但度量函数本身仍然外部给定。 |

## 阶段 III · 语言共形

| 问题 | 回答 |
|---|---|
| 闭合度量 | 对比损失（CLIP）、caption 损失、VQA accuracy、grounding IoU、指令遵循度 |
| 由谁固定 | 混合——对比损失由架构设计固定；VQA/caption 的评价依赖人类标注和语言模型打分（GPT-4 judge） |
| 命中对象 | 图像与语言之间的共享语义、命名、检索、描述、问答和指令调用 |
| 遮蔽对象 | 语言未编码的视觉细节、空间关系、细粒度属性、感知是否真实参与推理 |
| 残差返回 | MMVP / MMStar / CLIP-blindness / “思考越长越不看图”等问题暴露公共语义闭合并不等于视觉自持语义 |
| 下一阶段是否改写度量 | 部分是。阶段 IV 把“能说出/对齐”改写为“能反向生成/编辑”；阶段 V 则把“借语言”改写为“能否自持状态” |
| 被质疑过吗 | 开始被质疑：MMVP / MMStar 暴露了现有 benchmark 的盲区；「62% 故障源自感知」说明度量在错误的层级上评判。但没有工作让 VLM 自己修订自己的评价标准。 |
| 反身贡献 | **初现**。外部审计（benchmark 批判）开始质疑度量的合法性，但系统内部仍无自我质疑能力。 |

## 阶段 IV · 生成共形

| 问题 | 回答 |
|---|---|
| 闭合度量 | FID / IS / CLIP Score / 文本跟随 / 编辑一致性 / 人类偏好或 Arena 排名——不同工艺线对应不同指标 |
| 由谁固定 | 混合——FID/IS 等统计度量由研究共同体固定；文本跟随与编辑一致性由产品任务固定；人类偏好把一部分度量移交给外部评审者 |
| 命中对象 | 样本分布、视觉质量、文本条件满足、编辑前后不变量与变化量 |
| 遮蔽对象 | 生成过程中内部表征是否持有对象、几何、对应和可预测状态 |
| 残差返回 | 生成主体能为观者产出样本，但未必为自己持有世界；FID/CLIP Score/偏好指标之间也没有单一终局 |
| 下一阶段是否改写度量 | 是。阶段 V 把“产出样本是否好”改写为“内部表征是否持有并预测世界状态” |
| 被质疑过吗 | 明显被质疑：FID 与真实可用性常常脱节，CLIP Score 不等于编辑质量，人类偏好又带入审美偏见与平台目标；阶段 IV 因此出现“多指标并存但无终局指标”的局面 |
| 反身贡献 | **弱到中等**。阶段 IV 首次显化“度量并不唯一”，但度量修订仍主要由人类和平台完成，系统本身并未真正决定自己该被怎样评估。 |

## 阶段 V · 表征共形

| 问题 | 回答 |
|---|---|
| 闭合度量 | 对比损失 / 自蒸馏损失 / JEPA 预测损失只是训练侧度量；评估侧还必须同时看 linear probe、dense transfer、对应、深度与几何一致性、结构显化能力 |
| 由谁固定 | 仍主要由人类研究共同体固定，但已经不再有单一合法指标；不同任务实际上在争夺“什么才算世界状态”的解释权 |
| 命中对象 | 对象、部件、边界、几何、遮挡、对应、状态预测与迁移能力 |
| 遮蔽对象 | 什么才算合法世界状态、什么才算合法表征度量、度量能否被系统内部修订 |
| 残差返回 | 单一 probe 无法代表世界状态持有；不同评估对象之间的冲突把度量本身推为对象 |
| 下一阶段是否改写度量 | 尚未闭合。阶段 V 把度量问题推出台面，但系统自主修订度量仍是终极残差 |
| 被质疑过吗 | 这是阶段 V 的核心内容：单一 probe 无法合法代表世界状态持有，loss 本身也隐含强先验；阶段 V 首次把“度量命中了什么对象”作为显式问题推到台面 |
| 反身贡献 | **强，但未闭合**。阶段 V 让度量本身成为对象，从而真正开启反身问题；但系统仍未能自主修订自己的度量，所以反身闭合尚未完成。 |

---

## 纵向观察

反身能力的演进轨迹：

```
I:  度量完全外部给定，系统无感知
II: 匹配策略开始可学习，但度量函数仍外部
III: 外部审计开始质疑度量合法性，系统内部仍无自我质疑
IV: 多指标并存，度量冲突被显化，但修订权仍在人类与平台
V:  度量本身成为对象；系统是否能自主选择「关注什么」=「用什么度量」成为终极问题
```

每次新增阶段内容时，回来更新对应行。
