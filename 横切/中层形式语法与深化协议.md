# 中层形式语法与深化协议

类型: 协议/深化路线
阶段: 横切
关键贡献: 在公理层与阶段叙事之间补入一层形式语法，使闭合、半径、残差、共形、主体性与反身获得可继承的判定边界。
残差 / 它催生了什么: 解决“公理层 -> 历史叙事 -> 具体论文/任务”之间缺少中层形式约束的问题；后续应回灌到公理层、阶段章首协议和应用域实例化模板。
定位标签: 中层形式语法, 闭合, 残差, 半径, 共形, 主体性, 反身, 阶段偏序, 领域实例化

本文不是新公理，而是公理层向正文和应用域落地时必须经过的中间语法层。

当前大厦的根语言已经成立：[真正的开端](../00-公理层/真正的开端.md) 把“可见性 -> 差异 -> 稳定差异 -> 相容性”设为发生链，[第一性原理](../00-公理层/第一性原理.md) 再用公理零与三条工作公理把它压成可执行的理论结构。相容性仍是第一计算原则，但不再是最初开端。

真正需要补的是中间一层：

```text
公理层
  ↓
中层形式语法
  ↓
阶段解释
  ↓
具体任务实例
```

现在的问题不是“太抽象”，而是：


中间少了“形式语法层”。没有这一层，“闭合”“残差”“主体性”“反身”会在哲学解释和论文评论之间来回跳。加上这一层，它们仍然抽象，但会有理论硬度。

## 一、问题重述：缺的是形式约束，不是实验指标

第一大问题应重新表述为：

> 概念缺少“形式约束”，而不是缺少“实验指标”。

不应先问“闭合半径怎么量化”，而应先问：

1. 什么样的失败，才算闭合失败。
2. 什么样的扩张，才算闭合半径扩张。
3. 什么样的残差，才有资格推动阶段展开。

也就是说，要补的是判定规则，而不是某个具体 benchmark。

[一致性半径与一致性闭包](../00-公理层/03-一致性半径与一致性闭包.md) 已经给出正确方向：模型演进不是不断加功能，而是让同一套内部表征在更大的差异、模态、时间和干预下仍保持自洽；一致性不是输出相同，而是内部世界不裂；一致性闭包则是把更多彼此冲突的约束压进同一内部状态而不崩。

但这还缺一个形式层。已新增公理层文件：

```text
00-公理层/04-闭合、残差与半径的形式语法.md
```

该文件不应写成 RS_CD，不应写成实验指标，而应写成所有视觉任务都能继承的抽象协议。

## 二、最小形式核

可以定义一个视觉系统为：

```text
S = (X, Z, Π, Γ, M, D)
```

其中：

| 符号 | 含义 |
|---|---|
| `X` | 观测域，例如图像、视频、多模态输入、遥感时相、医学影像 |
| `Z` | 内部表征域 |
| `Π` | 接口族，例如分类、检测、分割、描述、问答、生成、行动 |
| `Γ` | 变换族，例如视角、尺度、噪声、遮挡、模态切换、时间推进、编辑、行动介入 |
| `M` | 度量族，即什么算稳定、什么算正确、什么算闭合 |
| `D` | 动力机制，即残差如何转化为模型、表征、任务或度量的更新 |

对应地：

```text
表征: f: X -> Z
接口: πᵢ: Z -> Yᵢ
变换: γ ∈ Γ
度量: m ∈ M
```

闭合可以抽象成：

> 当输入经过某类变换 `γ` 后，系统在内部表征 `Z` 与外部接口 `Yᵢ` 上仍能保持相容，则称系统在 `γ` 下闭合。

更硬的表述是：

> Closure holds when different paths through representation, transformation, and interface approximately commute.

中文即：

> 闭合不是“结果一样”，而是“从不同路径进入同一个对象、语义、状态或功能时，不发生结构性矛盾”。

这接住现有大厦的核心判断：一致性不是表面输出不变，而是同一内部表征在变化、扰动、模态切换、局部改写与行动介入后，仍能维持同一个可调用世界。

## 三、闭合半径：变换族上的可承受范围

闭合半径不应写成一个总分数。那会把理论压扁。

更合适的定义是：

> `R(S)` 是系统 `S` 在给定表征、接口和度量下，能够承受的变换族 `Γ` 的范围。

也就是说，闭合半径不是一个数，而是一个结构：

```text
R(S) = {
  R_view,
  R_scale,
  R_noise,
  R_context,
  R_modality,
  R_semantic,
  R_temporal,
  R_edit,
  R_generation,
  R_action,
  R_metric
}
```

这些分量不直接等价。它们共同描述系统能把多大范围的差异压进同一个内部世界。

这有两个好处：

1. 保持抽象，不把理论压成某个任务上的 accuracy 曲线。
2. 允许实例化，到了 RS_CD、医学影像、自动驾驶、视频理解、机器人视觉，每个领域都可以把这些半径具体化。

后续应在 [一致性半径与一致性闭包](../00-公理层/03-一致性半径与一致性闭包.md) 中补入这条判断：

> 一致性半径不是单一 leaderboard 分数，而是系统在一组变换族、接口族和度量族上维持闭合的结构化能力。

## 四、残差：闭合失败的结构性余项

残差不应继续只作为阶段叙事词使用。

更硬的判定是：

> 残差不是任何错误。残差是某个闭合结构在系统性条件下反复失败，并暴露出当前表征-接口-度量链条无法吸收的不相容。

建议把残差分成四层：

| 层级 | 名称 | 含义 |
|---|---|---|
| `R1` | 表面误差 | 单次预测错误、偶发失败 |
| `R2` | 分布残差 | 换数据、换域、换扰动后系统性退化 |
| `R3` | 接口残差 | 分类、分割、语言、生成、行动等接口之间互相矛盾 |
| `R4` | 度量残差 | 指标本身无法表达真正要闭合的对象 |

只有 `R2` 以上才值得进入理论叙事。只有 `R3`、`R4` 才有资格推动阶段性展开。

这样，“残差”不会变成万能解释词。

后续每章可增加小型残差卡：

```text
本章残差属于哪一类？
它暴露的是表征不足、接口不足、度量不足，还是动力不足？
它能否在同一阶段内部修补？
如果不能，它为什么逼出下一种闭合形式？
```

## 五、共形使用纪律

[术语分层表](../00-公理层/术语分层表.md) 已经要求术语回退到五问：纯存在态是什么，保留了什么稳定差异，形成了什么闭合制度，成功遮蔽了什么，留下了什么内在残差；也区分了共形的技术层级，例如公理层共形、对称群共形、稠密场共形、接口共形等。

这里需要继续强化，因为“共形”很容易从哲学层滑到技术层。

强制规则：

> 凡使用“共形”，必须说明是哪一级共形。

最低应回答五问：

1. 共形发生在哪两个结构之间？
2. 由什么度量判定？
3. 保留了什么稳定差异？
4. 闭合在哪个接口上成立？
5. 残差从哪里返回？

示例：

| 类型 | 两个结构 | 度量 | 闭合接口 | 残差 |
|---|---|---|---|---|
| 判别共形 | 图像差异结构 ↔ 标签空间结构 | 分类损失 / accuracy | 整图标签 | 空间内部被压扁 |
| 稠密共形 | 图像定义域 ↔ 像素/区域输出场 | IoU / Dice / mAP / PQ | 空间局部 | 开放语义、关系结构、跨模态调用 |
| 语言共形 | 视觉表征 ↔ 公共语义空间 | 对比损失、caption、VQA、grounding | 语言调用 | 借语言获得语义，但未必自持语义 |

## 六、主体性：残差回路内化程度

主体性不应过早工程化，也不应理解成“像人”“会对话”或“能调用工具”。

更适合大厦的定义是：

> 主体性 = 系统把外部残差转化为内部展开压力的能力。

主体性越高，系统越不是被动拟合器，而越能把不相容度组织成内部更新。

建议将主体性写成五个抽象层：

| 层级 | 抽象定义 | 关键问题 |
|---|---|---|
| `A0` 被动拟合 | 残差完全来自外部损失 | 系统只被度量 |
| `A1` 内部表征调适 | 残差能改变表征 | 系统能学习稳定差异 |
| `A2` 接口调适 | 残差能改变任务接口 | 系统能适配新调用方式 |
| `A3` 度量调适 | 残差能暴露并修订评价准则 | 系统开始质疑“什么算好” |
| `A4` 展开调适 | 系统能主动组织新数据、新任务、新度量来吸收残差 | 系统成为展开的共同发起者 |

原有主体性刻度可以保留：

```text
无主体性 -> 准主体性 -> 共享主体性 -> 生成主体性 -> 持有主体性 -> 反身主体性
```

但需要补一句：

> 这些不是人格等级，而是残差回路内化程度。

## 七、反身：度量对象化

[反身脊柱](反身脊柱.md) 当前追问三件事：本阶段度量是什么，度量由谁固定，度量是否被质疑。这是正确起点，但还差一步。

更硬的定义是：

> 反身 = 度量本身从外部裁判，变成系统或理论必须解释、审计、修订的对象。

反身不是“我知道我错了”，而是系统或理论开始追问：

1. 当前度量为什么合法？
2. 它命中了什么对象？
3. 它遮蔽了什么残差？
4. 它是否应该被替换、组合或内生化？

建议把反身脊柱的三问扩成五问：

1. 本阶段用什么度量判定闭合？
2. 这个度量命中的对象是什么？
3. 这个度量遮蔽了什么对象？
4. 被遮蔽对象如何以残差形式返回？
5. 下一阶段是否改写了度量本身？

这与阶段 V 的定位高度一致：阶段 V 不再有单一合法指标，不同任务在争夺“什么才算世界状态”的解释权。阶段 V 首次把“度量命中了什么对象”显式推到台面。

这条线是整座大厦从“视觉史解释”走向“理论框架”的关键。

## 八、五阶段：闭合型态的偏序结构

第二大问题应重写为：

> 五阶段不是时间线，而是“闭合型态的偏序结构”。

解决历史线性化风险的方法，不是反复声明“不是线性”，而是给五阶段一个更抽象的结构解释。

[五阶段总览](../00-公理层/五阶段总览.md) 已经升级为五问读取法：每层问“纯存在态是什么、保留了什么稳定差异、形成了什么闭合制度、成功遮蔽了什么、内在残差如何逼出下一阶段”。五阶段也被解释为“闭合制度在成功中生产遮蔽，并由内在残差逼迫重写”的五次主要折返。

需要进一步改成：

> 五阶段不是历史年表，而是五种主导闭合型态。它们之间不是严格线性顺序，而是残差驱动下的偏序关系。

五种主导闭合型态：

| 阶段 | 主导闭合型态 |
|---|---|
| I | 标签接口闭合 |
| II | 空间场闭合 |
| III | 公共语义闭合 |
| IV | 反向生成闭合 |
| V | 世界状态与度量反身闭合 |

一个方法可以跨阶段，但它的主阶段取决于它主要扩张了哪种闭合。

示例：

| 方法/路线 | 结构判断 |
|---|---|
| CLIP dense adaptation | 语言闭合 + 稠密闭合 |
| diffusion feature | 生成闭合反向输血给表征闭合 |
| MAE / DINO / iBOT | 重建、自监督、稠密结构、表征持有之间的桥接节点 |
| SAM | 稠密场闭合 + prompt 接口闭合，但不等于语言语义完全闭合 |
| VLM | 语言闭合强，但视觉空间闭合可能弱 |

对应横切页已新增：

```text
横切/阶段偏序与桥接节点.md
```

该页应定义：

| 概念 | 定义 |
|---|---|
| 主阶段 | 该方法主要解决的闭合残差 |
| 副阶段 | 该方法借用或回流的其他闭合能力 |
| 桥接节点 | 不能归入单层，但改变了层间关系的方法 |
| 反向输血 | 高阶段机制回流增强低阶段任务 |

## 九、领域实例化协议

需要新增的不是“RS_CD 闭合协议”，而是通用的领域实例化协议。

对应横切页已新增：

```text
横切/领域实例化协议.md
```

任何应用域都按同一套抽象模板进入大厦：

1. 本领域的观测域 `X` 是什么？
2. 本领域保留的稳定差异是什么？
3. 本领域的主要变换族 `Γ` 是什么？
4. 本领域的接口族 `Π` 是什么？
5. 本领域的闭合半径主要沿哪些方向扩张？
6. 本领域的残差属于表征、接口、度量、时间，还是主体性？
7. 本领域能否反向修正公理层或阶段结构？

RS_CD 只是这个模板的第一个实例：

| 形式项 | RS_CD 实例 |
|---|---|
| `X` | 多时相遥感观测 |
| 稳定差异 | 在配准、季节、尺度、传感器和语义扰动下仍值得保留的变化关系 |
| `Γ` | 时间推进、成像扰动、地理空间变换、语义查询、尺度变化 |
| `Π` | 二值变化、语义变化、开放词汇变化、描述、问答、报告、决策 |
| 当前残差 | 变化是否只是被定位，还是被内部状态化 |

RS_CD 的角色不是替代公理，而是验证大厦的抽象协议是否能穿过一个复杂真实领域。

[RS_CD 总纲](../RS_CD/00-RS_CD总纲-从代数差分到开放词汇变化检测.md) 已经把它定位为“变化对象被不断抬升到更高表征层”：从像素与光谱差值，到监督特征差异、全局关系差异、foundation prior 下的开放变化，最后到可描述、可提问、可统一调度的变化理解。这非常适合作为领域实例化，而不是公理替代品。

## 十一、最短执行口径

以后深化任何正文，先问四组问题：

1. **形式核**：这篇内容里的 `X, Z, Π, Γ, M, D` 分别是什么？
2. **闭合**：它扩张的是哪一种闭合半径，而不是提升了哪个单一分数？
3. **残差**：它暴露的是 `R1/R2/R3/R4` 哪一级残差？
4. **位置**：它是主阶段推进、桥接节点，还是高阶段对低阶段的反向输血？

只要这四组问题稳定，公理层、阶段解释和应用域实例就不会再断开。
