阶段 III 现在已经很有分量了。它不再只是“CLIP/VLM/多模态模型史”，而是已经被你压成了一个非常清楚的纯存在态：

> **Visual Difference → Public Linguistic Contract**
> 视觉差异必须进入公共语言契约。

这个设定是对的，而且非常适合作为你当前研究主方向。阶段 III 的独特性就在于：它不是单纯让视觉“更强”，而是让视觉第一次接受一个外部公共制度的裁决。阶段 I 是标签制度，阶段 II 是空间制度，阶段 III 则是**公共语义制度**。你现在已经抓住了这一点：语言提供公共概念、指代表达、任务指令和评价话语，视觉提供图像证据、空间定位和部分感知支撑；阶段 III 的残差也来自这张契约不是视觉自身的主权。这个判断非常重要。([CallMeCaicai][1])

但因为阶段 III 是你最主要的研究方向，它还需要比其他阶段更硬、更丰富、更一致。我的总体判断是：

> **阶段 III 的灵魂已经立住，但它还没有完全成为“公共语义契约的逻辑学”。**
> 它现在强在总纲、核心语法和 §27 清算；弱在章首协议未统一、若干表格未渲染、部分章节仍像模型谱系、以及“语言主权 / 视觉主体性 / 公共语义制度”三者的边界还可以更锋利。

# 1. 阶段 III 的最高层定位应再提升一步

你现在说阶段 III 是“语义从外挂到骨架”，这个标题有力量，但“骨架”这个词容易让人误读成：语言已经成为视觉自身的内部本体。更准确的说法应该是：

> **阶段 III 不是语言成为视觉的本体，而是语言成为视觉进入公共语义世界的契约制度。**

也就是说，阶段 III 的核心不是“语言进入视觉”，而是：

```text
可见性经由阶段 II 成为投影表面；
投影表面上的区域、对象、属性、关系，
必须进入人类公共语义制度，
才能被命名、指代、调用、评价。
```

你在核心语法页已经写得很好：阶段 III 的语言不是单一对象，而是名称语言、描述语言、对齐语言、指代语言、指令语言、推理语言、评价语言七种角色；并且这些语言角色逐层进入视觉系统不同位置。([CallMeCaicai][2])

建议把阶段 III 的总标题从：

```text
语义从外挂到骨架
```

升级为类似：

```text
语言共形：视觉进入公共语义契约
```

或者：

```text
语言共形：可见性进入公共命名、指代与调用制度
```

“骨架”可以保留在内部小标题里，但总标题最好强调“契约”。因为阶段 III 最深的地方不是语言变成骨架，而是视觉把自己的公共语义主权交给语言制度。结语里你已经说得非常准：视觉与语言签了一张临时协议，语言提供语义主权，视觉提供空间定位；协议有效期内，VLM 是最强视觉系统，但到期日是视觉必须自己能生成、能持有。([CallMeCaicai][3])

# 2. 总纲仍有旧“三问协议”，需要升级为新五问协议

这是当前阶段 III 最大的一致性问题。

你的阶段 III 核心语法页已经使用“五问版”：纯存在态、稳定差异、闭合制度、成功标准、遮蔽对象、内在残差、下一步重写。它还给出了 `C_III=(Z_image-text, Π_caption/VQA/grounding/instruction, Γ_modality/language/prompt, M_contrastive/IT/judge, D_alignment/IT/projector)` 这个闭合制度，非常好。([CallMeCaicai][2])

但阶段 III 总纲、§21、§22、§23、§24、§25、§26、§27、结语的章首协议仍然大量写成旧的“三问读取法”：稳定差异、闭合半径、出口残差。比如总纲第 319 行仍写“三问读取”，§22、§23、§24、§25、§26、§27 的定位卡也仍然用三问协议。([CallMeCaicai][1])

这会削弱你新公理层的执行力。阶段 III 是你的主方向，所以它应该成为全站第一批“完全五问化”的阶段。

建议每个主章统一改成：

```text
本章零度设定：
本章的无 / 反面：
本章的成为：
本章闭合制度 C=(Z,Π,Γ,M,D)：
本章成功标准：
本章遮蔽对象：
本章内在残差：
本章如何逼出下一章：
命题等级：F/M/G/P
```

例如 §22 可以这样改：

```text
零度设定：
图像与文本必须进入同一个可比较空间。

无 / 反面：
全图图文对齐无法保留空间、计数、方向、状态等未被 caption 稳定命名的视觉差异。

成为：
CLIP / ALIGN / SigLIP / MetaCLIP 等将 web-scale 图文配对变成公共语义监督制度。

闭合制度：
C_22=(Z_global_image-text, Π_zero-shot/retrieval/prompt, Γ_image-text/noisy-caption, M_contrastive/sigmoid, D_web-scale_alignment)

成功标准：
图像和文本能在共享 embedding 空间中互相检索、zero-shot 分类、支撑开放词汇接口。

遮蔽对象：
局部结构、空间关系、计数、方向、状态、OCR、遮挡。

内在残差：
caption 可描述的视觉子空间不等于完整视觉空间。
```

这会让 §22 真正成为阶段 III 的“对齐制度出生页”。

# 5. “语言主权”概念要继续加硬

阶段 III 现在最核心的词是：

```text
语义主权
```

你多处说视觉的语义主权在语言手里：总纲说阶段 III 是共享而非自主，结语说视觉的语义空间不是自生的，而是通过与语言对齐获得的，本质未变是语义主权始终在语言手里。([CallMeCaicai][1])

这个概念非常好，但还没有被正式定义。建议新增一个小节：

```text
语义主权：谁决定什么算意义？
```

定义可以这样写：

```text
语义主权不是模型能不能输出语义词，而是谁规定语义空间的坐标、粒度、边界、评价和纠错方式。

阶段 III 中，视觉获得公共语义调用权，但没有获得语义主权。因为：

1. 概念坐标来自语言分布；
2. 监督材料来自 caption / alt-text / label / instruction；
3. 评价标准来自 VQA、caption、human preference、LLM judge；
4. 推理后端常由 LLM 承担；
5. 未被语言命名的视觉差异容易被压扁。
```

然后给一个表：

| 权能    | 阶段 III 是否拥有 | 说明                            |
| ----- | ----------- | ----------------------------- |
| 命名权   | 部分拥有        | 能输出名称，但名称来自语言制度               |
| 指代权   | 部分拥有        | 能指到区域，但指代词来自语言                |
| 调用权   | 较强          | prompt/instruction 可以调用任务     |
| 评价权   | 很弱          | 多由 benchmark、LLM judge、人类偏好控制 |
| 生成权   | 未完成         | 推向阶段 IV                       |
| 状态持有权 | 未完成         | 推向阶段 V                        |

这样“语义主权”就不再是漂亮词，而是可分析对象。

# 6. 阶段 III 的“语言”还应再分成三层：语词、语义制度、社会评价

你现在已经写了七种语言角色，非常好。([CallMeCaicai][2])
但对研究方向而言，还可以再抽象成三层：

```text
Linguistic Form：文本 token、caption、prompt、phrase、instruction。
Semantic Institution：人类命名习惯、概念边界、类别体系、长尾语义。
Evaluative Regime：benchmark、judge、preference、rubric、safety policy。
```

阶段 III 的关键不是“文本进来了”，而是这三层一起进来了。尤其你结语里已经说：阶段 III 的语言不是纯文本 token，而是社会语义制度，包括 caption、alt-text、人类标签、object tag、instruction、preference、benchmark question、LLM prior 和 judge rubric。([CallMeCaicai][3])

建议在核心语法页“公共语义的材料层”后面加一个三层表：

| 层级    | 内容                                | 技术表现                                              | 残差         |
| ----- | --------------------------------- | ------------------------------------------------- | ---------- |
| 语言形式层 | token、caption、prompt、phrase       | CLIP text encoder、VLM prompt、referring expression | 语言歧义       |
| 语义制度层 | 类别、概念、命名习惯、文化偏见                   | web-scale alt-text、标签空间、LLM prior                 | 未命名视觉被压扁   |
| 评价制度层 | benchmark、rubric、preference、judge | VQA、human eval、LLM-as-judge、DPO/RLHF              | 能讨好评测不等于看见 |

这会让阶段 III 的“公共语义契约”更加丰富，也更适合作为研究方向。

# 7. §21 很清楚，但应该更明确是“阶段 II 对象制度被语言消费后的极限”

§21 已经成熟。它把外挂范式写成：detector / region proposal → region feature + object tag → BERT-style fusion → MLM / ITM / region-word alignment / VQA；并明确指出 region feature 是二手视觉，已经被 detector 类别表、box 制度和 Visual Genome 标注体系解释过一次。这个判断非常重要。([CallMeCaicai][5])

但它的章首协议仍是旧三问，而且它还可以和阶段 II 接得更深：

```text
§21 的本质不是“早期 VLM 不够好”，而是：
阶段 II 的对象闭合制度被语言系统消费后，发现对象制度本身太贫乏。
```

也就是说，§21 是阶段 II 的“对象闭合”被语言拿来使用后的失败。建议在 §21 开头新增一句：

```text
§21 不是阶段 III 的真正出生，而是阶段 II 对象制度在语言系统中的第一次试用。它证明：如果语言只能消费 detector 产生的二手对象摘要，那么公共语义的上限就被阶段 II 的 box/class 制度锁死。
```

这样 §21 会更像“过渡失败页”，而不是普通早期 VLM 综述。

# 8. §22 是阶段 III 的真正出生页，应把“数据配方”升到材料制度层

§22 现在非常强。你把 CLIP 的本体写成“监督经济学置换”——用 web-scale 弱对齐替代精标强监督；机制展开也说明 CLIP 不是多一个文本塔，而是把互联网上的人类描述当作视觉监督材料，并让图像和文本共享公共相似度空间。这个判断很好。([CallMeCaicai][6])

但因为阶段 III 是你的主研究方向，§22 可以再深化一层：**CLIP 的材料制度是什么？**

你已经写了 WIT 400M、query-based 构造、500k 高频英文词/bigram、每个 query 最多 20k 对，以及 MetaCLIP 复现数据配方说明关键秘密在 query balancing，不是模型结构。([CallMeCaicai][6])

建议把这些单独升格成：

```text
§22-B 图文材料制度：web-scale 弱语义如何制造公共视觉空间
```

内容可包括：

```text
1. caption 不是描述真相，而是人类选择性说出的图像侧面；
2. query balancing 不是数据技巧，而是公共语义分布的治理；
3. web-scale 噪声不是简单缺陷，而是语言社会制度进入视觉的材料形态；
4. MetaCLIP 说明 CLIP 的秘密很大一部分在材料制度，而非架构。
```

这样 §22 会成为阶段 III 的“材料地基”，对应阶段 I 的 ImageNet、阶段 II 的 COCO/SA-1B 空间标注制度。

同时，你在 §22 对 CLIP-blindness 的根因写成“人类在 caption 里不写的视觉属性，监督信号里就不存在；CLIP 学到的是 caption-完备的视觉子空间，不是完整视觉空间”。这非常好，但建议用更严格的措辞：

```text
caption-stabilized visual subspace
```

也就是“由 caption 稳定支持的视觉子空间”。它不是“caption 完备”，而是“caption 能稳定命名的那部分”。([CallMeCaicai][6])

# 9. §23 很重要，但应拆清“语言锚定稠密”与“几何边界节点”

§23 的机制展开很清楚：语言进入稠密任务有三种注入位置：text-as-classifier、text-as-query、text-as-region-prompt；也明确说 CLIP dense 化的真实难点是全图对齐没有 pixel-level 监督，直接用于 dense prediction 容易语义塌陷，所有位置都倾向图像主体。([CallMeCaicai][7])

这页有两个方向值得继续强化。

第一，§23 应明确自己处理的是：

```text
公共语义如何落回投影表面。
```

阶段 II 已经把图像变成投影表面，§22 把图像全局拉进语言相似度空间，§23 的问题是：语言能不能重新落回局部空间。这就是“指代机制”的核心。

建议加：

```text
§23 的本体不是 open-vocab detection 本身，而是公共语义从全图向量返回投影表面的过程。
```

第二，Depth Anything 作为边界节点非常好。你已经明确说它不应被简单归入语言条件几何，而更像阶段 III 内部的反证节点：深度、几何和连续场任务可能更依赖强视觉 SSL、伪标签和视觉数据规模，而不是语言对齐。([CallMeCaicai][7])

建议把这个边界再做成一张表：

| 任务类型                  | 是否天然需要语言     | 更接近阶段   |
| --------------------- | ------------ | ------- |
| 开放词汇检测                | 需要           | III     |
| referring / grounding | 需要           | III     |
| 开放词汇分割                | 需要，但受空间盲限制   | III/II  |
| 深度 / 法线 / 对应          | 不天然需要        | V       |
| 变化状态 / 多时相            | 不只需要语言，更需要状态 | V / 应用域 |

这样 §23 不会把所有 dense task 都吸进语言共形。

# 10. §24 很稳，但“光荣退位”稍微过强

§24 现在比以前稳定得多。你已经不再说 Seq2Seq 胜出，而是把统一多模态骨干分成三类：多目标 VLP 骨干、Seq2Seq 任务统一、冻结 LLM 桥接；并指出真正通向 §25 的不是一般 Seq2Seq，而是视觉 token 被送入已完成语言预训练的 LLM 后端。([CallMeCaicai][8])

但 §24 的一句话收束是：

> 统一骨干是 VLM 的过渡骨架——它的历史任务是铺好 LLM 能够插入视觉的标准接口，然后光荣退位。([CallMeCaicai][8])

“光荣退位”有点过强。统一骨干并没有完全退位，很多多模态基础模型、OCR/grounding/caption/data engine、工业多任务系统仍会需要 unified backbone 或 unified interface。建议改成：

```text
统一骨干不是被淘汰，而是被重新定位：
它从“最终语义主体”退到“标准视觉接口与多任务材料组织者”。
```

这样更准确。

§24 很适合补一个“统一的三种层级”表：

| 统一类型 | 统一了什么                               | 没统一什么   |
| ---- | ----------------------------------- | ------- |
| 目标统一 | ITC / ITM / LM 等 loss               | 世界状态    |
| 任务统一 | caption / OD / OCR / grounding 输出格式 | 语义主权    |
| 后端统一 | 视觉 token 接入 LLM                     | 视觉证据真实性 |

这会让 §24 更硬。

# 11. §25 是阶段 III 顶点章，但“VLM 是 LLM 的眼睛”需要边界化

§25 是全阶段最有研究价值的部分之一。它已经非常成熟：三代 VLM 谱系、视觉-LLM 接口、训练配方、LLM 后端与视觉瓶颈的条件分工、视觉 token 工程战争、隐性失败模式都写出来了。尤其这条公式很好：

```text
VLM 能力 = 语言后端能力
         + 视觉证据带宽
         + connector 保真度
         + 数据/指令配方
         - 语言先验接管风险
```

这条公式应该保留并上升为阶段 III 的核心机制公式。([CallMeCaicai][9])

但 §25 有一句：

> VLM 是 LLM 的眼睛，不是视觉的嘴。([CallMeCaicai][10])

这句话很锋利，但建议加边界。它适用于“LLM 后端主导的通用指令型 VLM”，不适用于所有多模态架构，尤其不适用于原生多模态共同训练、视觉专家路径、video/world-model 系统。

建议改成：

```text
在主流 LLM 后端式 VLM 中，VLM 更像 LLM 长出眼睛，而不是视觉长出嘴；但这不是所有多模态系统的本质定义，而是阶段 III 最典型的语义主权结构。
```

同时，§25 的核心研究方向应该从“VLM 强弱”升级为：

```text
视觉证据在语言后端中的生存问题。
```

也就是：

```text
视觉 token 如何进入 LLM？
进入后如何不被文本 prior 淹没？
推理过程中视觉证据如何持续被调用？
多图、视频、高分辨率、遥感变化这种视觉证据密集任务如何不被语言模板化？
```

建议新增一节：

```text
视觉证据生存论：进入 LLM 后，图像还剩下什么？
```

结构：

| 问题   | 技术现象                                  | 残差                |
| ---- | ------------------------------------- | ----------------- |
| 证据压缩 | projector / Q-Former / token pooling  | 细节丢失              |
| 证据稀释 | CoT 越长越不看图                            | 视觉衰减              |
| 证据污染 | 多图 token 混淆                           | 差异比较失败            |
| 证据替代 | LLM prior takeover                    | no-image baseline |
| 证据回看 | visual verification / self-correction | 推向 V / 反身         |

这会直接服务你的研究方向。

# 12. §26 是阶段 III 的输出闭环页，但应更强调“单向闭合”

§26 现在已经很准确地说，它建立的是 `language -> spatial output` 的单向闭合，不是行动闭环，也不是生成闭环；并指出缺两条回路：`spatial output -> visual verification -> corrected output` 和 `language -> new observation`。第一条推向阶段 V 的反身度量与状态持有，第二条推向阶段 IV 生成共形。([CallMeCaicai][11])

这非常好。建议把这条作为 §26 的标题核心：

```text
§26 语言条件可提示视觉：单向空间输出闭合
```

因为“语言条件可提示视觉”听起来像能力命名，而“单向空间输出闭合”更能凸显它的残差。

另外，§26 中“没有真正的视觉动作空间——能做的只是选择现有像素”这一点很关键。([CallMeCaicai][12])
建议加深：

```text
§26 的空间输出不是行动，也不是生成；它是选择、标注、指示、切分现有可见性。  
它仍然停留在投影表面内部。
```

这能更好地把它和阶段 IV / V 分开。

# 13. §27 很强，但“账单证据”需要加证据等级

§27 已经是阶段 III 最成熟的终章。它不再把账单压成 caption bias 单根因，而是多根因结构：caption distribution bias、global alignment bottleneck、connector/token bottleneck、LLM prior takeover、benchmark language shortcuts、reasoning-time visual decay。它还正确强调换视觉塔、换 connector、换 LLM、换数据、换评测都是有用的范式内缓解，不是无用补丁。([CallMeCaicai][13])

但因为 §27 会成为你研究方向的关键证据库，建议给四张账单增加“证据等级”：

| 账单                         | 证据等级                       | 当前风险                  |
| -------------------------- | -------------------------- | --------------------- |
| MMVP / CLIP-blindness      | 较强，具体 benchmark 和类别盲区      | 需要区分 CLIP 与 VLM 的继承关系 |
| MMStar / no-image baseline | 强，适合揭露 benchmark shortcuts | 不能推出所有 VLM 都不看图       |
| 62% 感知故障                   | 中-强，取决于标注协议                | 需要说明来源与分类标准           |
| Look Light, Think Heavy    | 中，机制启发强                    | attention 衰减不等同于因果证明  |

然后每条账单加：

```text
对应根因：
对应阶段残差：
可被什么改进缓解：
什么结果会反驳这个账单：
```

这会让 §27 从“账单清算”升级成“研究计划”。

# 14. 阶段 III 需要新增一页：视觉证据带宽

我认为这是阶段 III 最值得你深入的研究方向之一。

你已经在 §25 提到视觉 token 数量是 VLM 核心瓶颈，高分辨率、多图、视频会压爆 LLM 上下文；主要路线包括 pooling/perceiver、dynamic resolution、AnyRes/tiling、token merging/pruning、visual memory。([CallMeCaicai][10])

但这应该成为独立页：

```text
§25-A 视觉证据带宽：语言后端能吞下多少可见性？
```

核心命题：

```text
阶段 III 的真正 bottleneck 不是“有没有视觉 token”，而是视觉证据通过何种带宽进入语言制度。
```

可写成公式：

```text
Visual Evidence Surviving in LLM
=
visual encoder fidelity
× token budget
× connector preservation
× attention persistence
× task relevance
- language prior takeover
```

这比一般 VLM 工程综述更有理论价值。它能连接：

* CLIP 全图压缩；
* Q-Former / Perceiver 压缩；
* LLaVA projector 简化；
* Qwen2-VL dynamic resolution；
* InternVL 强视觉塔；
* token pruning / merging；
* Look Light Think Heavy；
* 多图/视频/遥感变化任务失败。

这页会非常适合你作为研究方向的核心专题。

# 16. 阶段 III 当前最大内容缺口：缺少“数据制度”的系统页

阶段 I 有 ImageNet，阶段 II 有空间标注，阶段 III 的材料不是单纯“图文对”。它应该包括：

```text
caption / alt-text / web image-text pairs
object tags / region phrases
instruction data
synthetic GPT-4V captions
human preference data
benchmark questions
LLM judge rubrics
safety / refusal / policy data
```

你在核心语法和结语已经提到公共语义材料层，但还没有单独展开。([CallMeCaicai][2])

建议新增：

```text
§20-B 公共语义材料制度：从 caption 到 judge rubric
```

结构：

| 材料                         | 代表                 | 进入系统的位置      | 主要偏差              |
| -------------------------- | ------------------ | ------------ | ----------------- |
| alt-text / caption         | CLIP/ALIGN         | 对齐目标         | 省略、社会偏见、标题化       |
| object tag / region phrase | OSCAR/GLIP         | 指代/grounding | 类别表与标注限制          |
| instruction data           | LLaVA/InstructBLIP | 调用机制         | 语言模板化             |
| synthetic caption          | ShareGPT4V 等       | SFT 数据       | 生成模型偏差回流          |
| preference data            | RLHF/DPO           | 输出规范         | 讨好偏好而非看清          |
| benchmark questions        | VQA/MMBench/MMStar | 评价制度         | language shortcut |
| judge rubric               | LLM-as-judge       | 度量制度         | 评价外包              |

这页会成为阶段 III 的“材料轴”，非常关键。

# 17. RS 遥感专题应该留，但要重新标识为领域回灌

§27 下有遥感 VLM、多时相 VLM 和 RS-VLM 十五篇关键工作专题。它们对你的研究方向可能很有价值，但挂在 §27 主线下容易让阶段 III 主线被应用专题拉偏。导航中这些专题直接在 §27 下出现。([CallMeCaicai][13])

建议改成：

```text
§27-A 领域回灌：遥感 VLM 放大了阶段 III 的空间残差
§27-B 领域回灌：多时相 VLM 放大了阶段 III 的时间残差
```

并在专题开头写：

```text
本专题不是阶段 III 主线，而是领域实例化回灌。
遥感之所以重要，是因为它放大了语言共形的两个弱点：
1. 空间图斑定位必须精确；
2. 多时相状态不能只靠 caption/QA。
```

这样 RS 专题会服务主理论，而不是抢主线。

# 最核心的一句话

阶段 III 现在应该被写成：

> **投影表面上的视觉差异，第一次被迫进入人类公共语义制度；语言给它命名、指代、调用、评价，也因此取得了语义主权。阶段 III 的胜利，是视觉终于能被公共语言使用；阶段 III 的失败，是语言太强，以至于它能替视觉说话。**

阶段 III 的胜利：

```text
视觉进入公共语义契约。
```

阶段 III 的失明：

```text
公共语义不是视觉自己的世界。
```

阶段 III 的研究核心：

```text
视觉证据如何在语言制度中生存？
语言主权如何塑造、放大、遮蔽视觉？
哪些视觉能力必须逃离语言，转向生成或状态持有？
```

你现在的阶段 III 已经很接近完成，但如果它是你的主研究方向，就应该再向前一步：
不要只写“语言如何进入视觉”，而要写成：

> **公共语义制度如何接管视觉，以及视觉如何在这张契约中获得能力、丧失主权、欠下账单，并最终逼出生成与自持两条道路。**
