# 领域实例化协议

类型: 协议/应用域
阶段: 横切
关键贡献: 给应用域提供统一入口模板，使 RS_CD、医学影像、自动驾驶、视频理解、机器人视觉等领域都能继承同一套公理层形式语法。
残差 / 它催生了什么: 防止某个应用域吞掉公理层；要求所有领域实例先说明观测域、稳定差异、变换族、接口族、闭合半径与残差类型。
定位标签: 应用域, 领域实例化, RS_CD, 观测域, 变换族, 接口族, 闭合半径

应用域不是公理替代品。

应用域的任务是检验：

> 公理层和中层形式语法能否穿过一个复杂真实领域，并在不改写根语言的情况下解释其演进。

---

## 一、通用模板

任何应用域进入大厦前，先填写七项：

| 项 | 问题 |
|---|---|
| `X` 观测域 | 本领域观察到的原始材料是什么？ |
| 稳定差异 | 本领域真正要保留的差异是什么？ |
| `Γ` 变换族 | 本领域必须承受哪些扰动、迁移、时间或干预？ |
| `Π` 接口族 | 本领域要求系统通过哪些接口输出或被调用？ |
| 闭合半径 | 本领域的闭合主要沿哪些方向扩张？ |
| 残差类型 | 残差属于表征、接口、度量、时间，还是主体性？ |
| 反向修正 | 本领域是否迫使公理层、阶段边界或度量协议修订？ |

## 三、RS_CD 作为第一个实例

RS_CD 的角色不是替代公理，而是验证抽象协议是否能穿过复杂领域。

| 形式项 | RS_CD 实例 |
|---|---|
| `X` | 多时相遥感观测，包括光学、SAR、多尺度、多传感器图像 |
| 稳定差异 | 在配准误差、季节变化、尺度变化、传感器差异和语义扰动后，仍值得保留为“变化”的关系 |
| `Γ` | 时间推进、成像扰动、地理空间变换、尺度变化、传感器切换、语义查询 |
| `Π` | 二值变化、语义变化、开放词汇变化、变化描述、问答、报告、决策 |
| 闭合半径 | 从像素差分扩到全局关系、foundation prior、开放词汇、变化理解 |
| 当前残差 | 变化是否只是被定位，还是被系统内部状态化 |
| 主线锚点 | II 稠密场 + III 语言语义 + V 世界状态；后续可能形成应用域反向压力 |

RS_CD 当前最深残差：

> 变化语义已经能被开放词汇化、可指代化、可描述化，但多数系统仍在借用外部 foundation model / VLM 的语义空间，而没有形成自己稳定的变化状态空间。

## 四、领域不能越权

应用域只能做三件事：

1. 提供真实复杂材料。
2. 检验公理层和阶段结构是否能稳定落地。
3. 在出现无法吸收的残差时，反向提示公理层或横切协议修订。

应用域不能直接替代：

- 公理层根语言
- 五阶段主线
- 横切形式语法
- 术语分层表

如果某个领域材料看起来要求“新阶段”，先判断它是：

```text
现有阶段的应用实例？
跨阶段桥接节点？
高阶段机制对低阶段的反向输血？
还是确实暴露了现有闭合型态无法吸收的 R4 残差？
```

只有最后一种才有资格推动阶段结构修订。

## 五、写作入口

新增应用域专题时，文件开头建议包含：

```md
类型: 应用域/领域实例
主阶段:
副阶段:
桥接/反向输血:
观测域 X:
稳定差异:
变换族 Γ:
接口族 Π:
闭合半径:
残差类型:
主线反哺:
定位标签:
```

这样，应用域就不会退化为论文卡堆积，而会成为公理层的压力测试场。
