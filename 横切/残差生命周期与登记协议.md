# 残差生命周期与登记协议

类型: 横切/协议  
阶段: 横切  
共形维度: 残差, 动力, 反身  
关键贡献: 把“残差驱动”从阶段叙事词升级为登记制度，规定什么失败有资格进入理论链条，什么失败只停留在任务误差。  
残差 / 它催生了什么: 解决“任何失败都能被叫残差”的概念泛化风险；催生章节残差登记卡与阶段自我否定写法。  
定位标签: 残差生命周期, 登记残差, 结构残差, 展开残差

## 一、残差不是任何错误

残差的硬定义见 [闭合、残差与半径的形式语法](../00-公理层/04-闭合、残差与半径的形式语法.md)：

> 残差是某个闭合结构在系统性条件下反复失败，并暴露出当前表征-接口-度量链条无法吸收的不相容。

因此：

- 单次预测错，不自动构成残差。
- benchmark 分数低，不自动构成残差。
- 只有当失败可重复、可定位、且指向当前闭合结构无法吸收的不相容时，才进入残差登记。

## 二、残差生命周期

| 阶段 | 名称 | 判定问题 | 处理方式 |
|---|---|---|---|
| `L0` | 表面误差 | 是否只是单次错误或噪声？ | 不进入理论叙事，只作为案例保留 |
| `L1` | 候选残差 | 是否在同类条件下反复失败？ | 暂列候选，继续收集反例 |
| `L2` | 登记残差 | 是否能定位到 `Z / Π / Γ / M / D` 某个环节？ | 登记到章节残差卡 |
| `L3` | 结构残差 | 是否证明同阶段小修小补无法吸收？ | 进入阶段推进论证 |
| `L4` | 展开残差 | 是否迫使系统引入新的表征、接口、度量或动力？ | 作为下一闭合型态的入口 |
| `L5` | 吸收残差 | 新机制是否部分消化它？ | 写入新阶段的成功标准 |
| `L6` | 剩余残差 | 新机制吸收后留下了什么更深不相容？ | 写入下一轮自我否定 |

## 三、登记卡

每个章节若声称某个失败是残差，至少填写：

```text
残差名称:
生命周期阶段: L0 / L1 / L2 / L3 / L4 / L5 / L6
对应形式核:
- X:
- Z:
- Π:
- Γ:
- M:
- D:

失败事实:
- 在什么条件下反复出现？
- 为什么不是偶发错误？

定位:
- 暴露的是表征不足、接口不足、变换半径不足、度量不足，还是动力不足？

阶段地位:
- 能否在当前阶段内部修补？
- 若不能，它逼出哪一种新的闭合型态？

吸收结果:
- 后续机制吸收了什么？
- 又留下了什么剩余残差？
```

## 四、进入阶段叙事的门槛

| 层级 | 能否进入正文 | 能否推动阶段展开 |
|---|---|---|
| `L0` 表面误差 | 否 | 否 |
| `L1` 候选残差 | 可作为证据脚注 | 否 |
| `L2` 登记残差 | 可进入章节主线 | 通常否 |
| `L3` 结构残差 | 应进入阶段结语 | 可能 |
| `L4` 展开残差 | 必须进入阶段推进 | 是 |
| `L5/L6` 吸收与剩余 | 必须进入下一阶段自我否定 | 是 |

最重要的纪律：

> 只有结构残差以上，才有资格把“下一阶段为何出现”写成内在推进。
