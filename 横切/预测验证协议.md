# 预测验证协议

类型: 横切/验证协议
阶段: 横切
共形维度: 度量, 反身, 证据
关键贡献: 把 [预测性判据登记簿](预测性判据登记簿.md) 从“可证伪判断清单”推进为可执行的验证卡片制度。
残差 / 它催生了什么: 如果预测只有口号而没有验证任务、对照模型、支持条件和失败条件，理论仍然不能承担真正风险。本文件给每条预测提供最小审计格式。
定位标签: prediction audit, falsification, validation protocol, evidence level

本文件不新增预测。

它只规定：每一条已经登记的生成性预测，怎样进入验证。

---

## 一、为什么需要验证协议

[预测性判据登记簿](预测性判据登记簿.md) 已经完成第一步：把理论命题分成描述性统一、生成性预测和启发性映射。

但“可证伪方式”还不等于真正验证。真正验证至少要说清：

1. 用什么任务或 stress test 验证；
2. 用哪些模型做对照；
3. 什么结果算支持；
4. 什么结果算失败；
5. 证据强度属于哪一级；
6. 验证结果写回哪里。

否则预测登记簿会退化为未来判断清单，而不是理论冒险台。

---

## 二、预测验证卡模板

每条生成性预测进入验证时，都必须补成同一张卡。

| 字段 | 填写要求 |
|---|---|
| 预测编号 | 对应登记簿编号，例如 `P3`、`P-V4`。 |
| 对应阶段 | I / II / III / IV / V / 横切 / 应用域。 |
| 对应内在残差 | 说明它检验的是哪种闭合制度的遮蔽返回。 |
| 最小验证任务 | 只写最小可执行任务，不先追求全覆盖。 |
| 候选数据集 / benchmark | 优先一手 benchmark、官方数据集或可复现 stress test。 |
| 对照模型 | 至少包含旧闭合制度代表模型和新闭合制度候选模型。 |
| 支持条件 | 什么结果会支持预测。 |
| 失败条件 | 什么结果会推翻或迫使修正预测。 |
| 证据等级 | 使用 `E0-E6`：分数、迁移、显现、密集接口、结构一致性、预测/反事实、反身证据。 |
| 当前状态 | 待设计 / 待运行 / 已运行 / 支持 / 失败 / 需修正。 |
| 更新记录 | 记录日期、结果摘要和下一步。 |

---

## 三、证据等级

| 等级 | 名称 | 含义 |
|---|---|---|
| `E0` | 分数证据 | linear probe、classification、单 benchmark 分数。 |
| `E1` | 迁移证据 | 下游迁移有效，但未直接证明结构状态。 |
| `E2` | 显现证据 | attention map、object discovery、聚类或可视化中出现结构。 |
| `E3` | 密集接口证据 | depth、segmentation、correspondence 等 dense task 有效。 |
| `E4` | 结构一致性证据 | 遮挡、跨图对应、视角变化、局部编辑下保持结构。 |
| `E5` | 预测 / 反事实证据 | 缺失状态预测、反事实一致、多假设不确定性表达。 |
| `E6` | 反身证据 | 系统或评估协议能发现某个度量不适合当前状态对象。 |

阶段 V 的预测原则上不能只停在 `E0-E1`；如果一条预测声称检验世界状态持有，至少应争取 `E3` 以上证据。

---

## 四、最小执行顺序

第一轮不追求把所有预测一次验证完，只做三类代表：

| 优先级 | 预测 | 理由 |
|---|---|---|
| P0 | `P3` 生成质量高不等于世界状态持有 | 直接检验阶段 IV -> V 的边界。 |
| P0 | `P-V5` 单一 probe 会错误排序阶段 V 方法 | 直接检验反 Probe 协议。 |
| P1 | `P2` 语言闭合强的模型会暴露视觉残差 | 直接检验阶段 III 的借语言代价。 |

每次验证只要完成一张卡即可，不要求一次写成论文级实验设计。

---

## 五、写回规则

验证完成后，同步更新三个位置：

1. [预测性判据登记簿](预测性判据登记簿.md)：把结果列从“待验证”改为“支持 / 失败 / 需修正”。
2. 对应阶段正文：如果预测失败，必须回写到该阶段的自我否定或残差页。
3. 本文件：在“验证卡实例”中保留最小记录。

## 六、验证卡实例占位

后续每次验证都在这里追加。

### `P3` · 生成质量高不等于世界状态持有

| 字段 | 内容 |
|---|---|
| 预测编号 | `P3` |
| 对应阶段 | IV -> V |
| 对应内在残差 | 外显样本越强，越暴露最终样本不是理解本身。 |
| 最小验证任务 | 受控编辑中身份、部件、几何、遮挡连续性审计。 |
| 候选数据集 / benchmark | 待定。 |
| 对照模型 | 待定。 |
| 支持条件 | 生成偏好或图文分数强，但结构一致性、身份保持或遮挡连续性弱。 |
| 失败条件 | 生成质量提升稳定同步带来状态一致性提升，且跨模型排序不反转。 |
| 证据等级 | 目标 `E4`。 |
| 当前状态 | 待设计。 |
| 更新记录 | 2026-04-25 建卡。 |
