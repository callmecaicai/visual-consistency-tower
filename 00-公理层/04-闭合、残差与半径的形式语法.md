## 一、最小形式核

定义一个视觉系统：

```text
S = (X, Z, Π, Γ, M, D)
```

| 符号 | 名称 | 含义 |
|---|---|---|
| `X` | 观测域 | 图像、视频、多模态输入、遥感时相、医学影像、传感器观测 |
| `Z` | 内部表征域 | 系统保留稳定差异的位置 |
| `Π` | 接口族 | 分类、检测、分割、描述、问答、生成、编辑、行动等输出或调用接口 |
| `Γ` | 变换族 | 视角、尺度、噪声、遮挡、域迁移、模态切换、时间推进、编辑、行动介入 |
| `M` | 度量族 | 判定什么算稳定、正确、相容、闭合的准则 |
| `D` | 动力机制 | 残差如何转化为参数、结构、接口、任务或度量更新 |

相应地：

```text
表征: f: X -> Z
接口: πᵢ: Z -> Yᵢ
变换: γ ∈ Γ
度量: m ∈ M
动力: d ∈ D
```

这不是为了把大厦改写成数学论文，而是为了避免同一术语在公理层、阶段叙事和论文评论之间滑动。

## 二、闭合

闭合的最低定义：

> 当输入经过某类变换 `γ` 后，系统在内部表征 `Z` 与外部接口 `Yᵢ` 上仍能保持相容，则称系统在 `γ` 下闭合。

更硬的形式表述：

> Closure holds when different paths through representation, transformation, and interface approximately commute.

中文即：

> 闭合不是“结果一样”，而是“从不同路径进入同一个对象、语义、状态或功能时，不发生结构性矛盾”。

因此，闭合至少涉及三条路径：

```text
路径 A: x -> f(x) -> πᵢ(f(x))
路径 B: x -> γ(x) -> f(γ(x)) -> πᵢ(f(γ(x)))
路径 C: x -> f(x) -> 内部变换或接口迁移 -> πⱼ(...)
```

如果这些路径在相关度量 `m` 下仍指向同一个可调用对象、语义、状态或功能，闭合成立。

### 非塌缩条件

闭合成立不仅要求路径相容，还要求非塌缩。

一个系统可以非常一致，但一致得毫无信息：所有图像都映射成同一个向量、所有问题都回答“这是一张图片”、所有生成都回到模糊均值图，这些都可能在某些变换下稳定，却不是理解。

因此，非塌缩闭合至少满足四个条件：

1. **路径相容**：不同路径进入同一对象、语义、状态或功能时不发生结构性矛盾。
2. **差异保真**：闭合必须保留某种非平凡稳定差异，不能靠抹平所有差异获得稳定。
3. **接口可调用**：被保留的差异必须能在至少一个接口 `Π` 上产生有效调用。
4. **外部抵抗**：内部闭合必须经受观测、任务、交互或后果的外部约束。

所以，自洽不是理解；能在外部抵抗下保持非塌缩自洽，才接近理解。

## 三、闭合失败

闭合失败不是任何错误，而是路径之间出现结构性矛盾。

最低判定：

1. 失败在同一类条件下可重复出现。
2. 失败不是单一噪声或偶发样本造成。
3. 失败暴露出 `Z`、`Π`、`Γ`、`M` 或 `D` 至少一处无法吸收当前不相容。

因此：

- 一张图片分类错，不一定是闭合失败。
- 换域后系统性退化，是闭合失败候选。
- 分类、分割、语言描述、生成结果互相矛盾，是更强的闭合失败。
- 指标本身无法表达要闭合的对象，是最深的闭合失败。

## 四、闭合半径

闭合半径不应定义为单一分数。

> 闭合半径 `R(S)` 是系统 `S` 在给定表征、接口和度量下，能够承受的变换族 `Γ` 的范围。

它是结构，而不是标量：

```text
R(S) = {
  R_view,
  R_scale,
  R_noise,
  R_context,
  R_modality,
  R_semantic,
  R_temporal,
  R_edit,
  R_generation,
  R_action,
  R_metric
}
```

| 分量 | 含义 |
|---|---|
| `R_view` | 视角、姿态、投影变化下仍能闭合 |
| `R_scale` | 尺度、分辨率、粒度变化下仍能闭合 |
| `R_noise` | 噪声、压缩、遮挡、缺失下仍能闭合 |
| `R_context` | 背景、上下文、组合变化下仍能闭合 |
| `R_modality` | 图像、文本、音频、视频、传感器之间仍能闭合 |
| `R_semantic` | 开放概念、关系、属性、意图变化下仍能闭合 |
| `R_temporal` | 时序推进、长期状态、多轮交互下仍能闭合 |
| `R_edit` | 局部改写时同时守住不变量与变化量 |
| `R_generation` | 从内部反向产出观测时仍保持整体自洽 |
| `R_action` | 感知、语言、计划、动作属于同一个世界 |
| `R_metric` | 度量被审计或改写后仍能维持理论一致 |

一致性半径不是 leaderboard 分数，而是系统在一组变换族、接口族和度量族上维持闭合的结构化能力。

## 五、残差层级

残差是闭合失败的结构性余项。

> 残差不是任何错误。残差是某个闭合结构在系统性条件下反复失败，并暴露出当前表征-接口-度量链条无法吸收的不相容。

建议四级：

| 层级 | 名称 | 含义 | 理论地位 |
|---|---|---|---|
| `R1` | 表面误差 | 单次预测错误、偶发失败 | 不进入阶段叙事 |
| `R2` | 分布残差 | 换数据、换域、换扰动后系统性退化 | 可进入章节问题 |
| `R3` | 接口残差 | 分类、分割、语言、生成、行动等接口互相矛盾 | 可推动章节或阶段展开 |
| `R4` | 度量残差 | 指标本身无法表达真正要闭合的对象 | 推向反身与新闭合型态 |

只有 `R2` 以上才值得进入理论叙事。只有 `R3`、`R4` 才有资格推动阶段性展开。

## 六、共形五问

凡使用“共形”，必须回答五问：

1. 共形发生在哪两个结构之间？
2. 由什么度量判定？
3. 保留了什么稳定差异？
4. 闭合在哪个接口上成立？
5. 残差从哪里返回？

示例：

| 类型 | 两个结构 | 度量 | 闭合接口 | 残差 |
|---|---|---|---|---|
| 判别共形 | 图像差异结构 ↔ 标签空间结构 | 分类损失 / accuracy | 整图标签 | 空间内部被压扁 |
| 稠密共形 | 图像定义域 ↔ 像素/区域输出场 | IoU / Dice / mAP / PQ | 空间局部 | 开放语义、结构关系、跨模态调用 |
| 语言共形 | 视觉表征 ↔ 公共语义空间 | 对比损失、caption、VQA、grounding | 语言调用 | 借语言获得语义，但未必自持语义 |
| 生成共形 | 内部潜空间 ↔ 可见观测分布 | 似然、去噪损失、偏好、编辑一致性 | 样本生产 | 为观者产出，不为自己持有世界 |
| 表征共形 | 图像观测 ↔ 世界状态表征 | probe、迁移、对应、几何一致性、预测损失 | 状态持有 | 度量自身的合法性返回 |

## 七、主体性层级

主体性不是人格化，不是会对话，也不是工具调用能力。

> 主体性 = 系统把外部残差转化为内部展开压力的能力。

| 层级 | 名称 | 抽象定义 | 关键问题 |
|---|---|---|---|
| `A0` | 被动拟合 | 残差完全来自外部损失 | 系统只被度量 |
| `A1` | 内部表征调适 | 残差能改变表征 | 系统能学习稳定差异 |
| `A2` | 接口调适 | 残差能改变任务接口 | 系统能适配新调用方式 |
| `A3` | 度量调适 | 残差能暴露并修订评价准则 | 系统开始质疑“什么算好” |
| `A4` | 展开调适 | 系统能主动组织新数据、新任务、新度量来吸收残差 | 系统成为展开的共同发起者 |

原有刻度仍可使用：

```text
无主体性 -> 准主体性 -> 共享主体性 -> 生成主体性 -> 持有主体性 -> 反身主体性
```

但这些不是人格等级，而是残差回路内化程度。

---

## 八、反身定义

反身不是“我知道我错了”。

> 反身 = 度量本身从外部裁判，变成系统或理论必须解释、审计、修订的对象。

反身最低五问：

1. 本阶段用什么度量判定闭合？
2. 这个度量命中的对象是什么？
3. 这个度量遮蔽了什么对象？
4. 被遮蔽对象如何以残差形式返回？
5. 下一阶段是否改写了度量本身？

当系统只在外部度量下优化，它没有反身。  
当系统或理论开始追问“这个度量为什么合法”，反身启动。  
当度量能被内部修订、组合或主动生成，反身才接近闭合。

## 九、章节残差卡

后续每个主章至少应能填出这张卡：

```text
本章形式核:
- X:
- Z:
- Π:
- Γ:
- M:
- D:

闭合判断:
- 本章要求哪条路径近似交换？
- 闭合在哪个接口上成立？
- 闭合半径扩张到哪一组变换？

残差判断:
- 残差层级: R1 / R2 / R3 / R4
- 暴露的是表征、接口、度量、时间，还是主体性不足？
- 能否在同一阶段内部修补？
- 若不能，为什么逼出下一种闭合型态？

反身判断:
- 本章度量命中了什么？
- 本章度量遮蔽了什么？
- 被遮蔽对象如何返回？
```

这张卡不是强制每篇正文都完整展示，但它应成为写作和审计时的后台协议。
