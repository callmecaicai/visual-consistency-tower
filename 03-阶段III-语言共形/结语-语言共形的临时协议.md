## 一、阶段 III 做了什么（技术总结）

§21→§27 的残差链回顾：

```
§21 外挂范式锁死（detector 上限）
  ↓ 残差：视觉塔必须端到端训练 + 数据必须 web-scale
§22 CLIP 范式置换（对比对齐规模化）
  ↓ 残差：全图级对齐不够，稠密任务需要语言渗透到像素
§23 语言锚定稠密任务（开集检测/分割/深度）
  ↓ 残差：每个任务仍有独立骨干，语言只是外挂分类头
§24 统一多模态骨干（对比+caption+VQA+grounding 归一化）
  ↓ 残差：骨干统一了，但语义后端仍是浅层的
§25 VLM 诞生（LLM 作为语义后端）
  ↓ 残差：理解闭环了，但不能反向操作像素
§26 语言条件可提示视觉（语言→像素级输出）
  ↓ 残差：单向闭环，缺反向循环；且语言是借来的
§27 借语言的代价（四张账单 + 两条分叉）
```

### 阶段 III 的公理性陈述

> **阶段 III 的公理**：视觉的语义空间不是自生的，是通过与语言的对齐而获得的。这个对齐从外挂（§21）走到骨架（§25-§26），深度递增，但本质未变——视觉的语义主权始终在语言手里。

### 三条结构性主线

这一层不能按论文堆，而应按四条机制线收束：

| 机制线 | 核心问题 | 代表位移 | 阶段 III 的贡献 | 留下的残差 |
|---|---|---|---|---|
| 命名机制 | 视觉差异如何进入公共概念 | region tag / label → caption → text embedding | 让视觉第一次稳定进入可共享语义空间 | 命名来自语言分布，视觉未必自持概念 |
| 指代机制 | 语言如何指到图像中的对象、区域、属性、关系 | image-text matching → grounding → dense grounding | 让“说什么”开始落到“图中哪里” | 指代正确不等于空间状态被持有 |
| 调用机制 | prompt / instruction 如何把视觉接口从固定任务变成可调用任务 | fixed head → prompt → instruction → `<seg>` token | 把视觉模型改写成可被用户调用的服务 | 接口扩张不等于状态闭合 |
| 借语言的代价 | 语言先验如何遮蔽视觉细节 | CLIP-blindness / no-image baseline / hallucination | 正式命名语言闭合对视觉闭合的遮蔽 | 模型能说对，但未必看对 |

因此，阶段 III 的自我否定不是“语言还不够强”，而是：

> 语言让视觉获得公共语义，同时也让视觉可能以为自己看见了。

### 阶段 III 的展开骨架

| 展开环节 | 技术对象 | 理念功能 | 自我否定 |
|---|---|---|---|
| 外挂 | ViLBERT、LXMERT、UNITER、VinVL | 语言接到二手 region feature 上 | detector 上限锁死语义 |
| 对齐 | CLIP、ALIGN、LiT、SigLIP | 图像和文本进入同一相似度空间 | 全图语义压扁空间细节 |
| 稠密锚定 | GLIP、Grounding DINO、CAT-Seg | 语言开始指向框、mask、region | 指代不等于状态持有 |
| 统一骨干 | BLIP、CoCa、OFA、Florence-2 | 多任务图文理解共用骨干 | 接口统一不等于世界统一 |
| LLM 后端 | Flamingo、BLIP-2、LLaVA、Qwen-VL | 视觉进入语言推理空间 | 视觉主体性借自语言 |
| 像素调用 | LISA、GLaMM、PixelLM | 指令可产生像素级输出 | 语言到像素仍是单向闭合 |
| 账单清算 | MMVP、MMStar、Look Light Think Heavy | 命名语言遮蔽未命名视觉 | IV 需要生成，V 需要自持 |

## 二、阶段 III 没做什么（四个残差）

### 残差 ①：语义主权仍在语言手里

视觉模型没有自己的概念空间。所有"理解"都是通过语言的坐标系实现的。CLIP-blindness 是这个结构性缺陷的症状，不是 bug。

### 残差 ②：不能反向合成

视觉可被语言描述、锚定、操作，但不能被语言反向合成——"画出""演出""延续"不在阶段 III 的能力范围内。→ 阶段 IV

### 残差 ③：依赖配对语料

语义对齐绑定 caption / 配对语料。若要摆脱对文本描述的依赖，必须走纯视觉自监督。→ 阶段 V

### 残差 ④：感知粒度的结构性缺失

MMVP / 62% 感知故障 / Look Light Think Heavy 共同指向：VLM 的视觉塔在细粒度感知上存在结构性缺陷。换视觉塔、换连接器、换数据和扩大训练规模可以显著缓解这些问题，但只要语义主权仍由语言后端接管，阶段 III 的借来性就不会消失。

### 出口残差的三重压缩

| 残差 | 含义 | 推向哪里 |
|---|---|---|
| 借来的语义 | 视觉进入公共语义，但语义稳定性来自语言空间，而非视觉自身 | 阶段 V 的自持状态 |
| 被语言遮蔽的视觉 | 模型能说对，但未必看对；能回答，但未必 grounding | 阶段 III 的账单与反例库 |
| 接口扩张不等于状态持有 | prompt 越强，任务越灵活，但系统未必持有更稳定的视觉世界 | 阶段 IV 的生成证明与阶段 V 的状态持有 |

## 三、哲学思考

### 公共语义不是自然语言本身

阶段 III 的“语言”不是纯文本 token，而是社会语义制度：caption、alt-text、人类标签、object tag、instruction、preference、benchmark question、LLM prior 和 judge rubric 共同构成公共语义空间。

因此，语言共形的本体代价不是“文本噪声”，而是视觉模型把人类公共语义制度当作自己的语义骨架。它获得了可共享、可指代、可对话的能力，也继承了省略、偏见、礼貌、文化背景和评价外包。

### 三层自我否定

| 层级 | 成功 | 自我否定 |
|---|---|---|
| 命名的自我否定 | 视觉差异被语言命名 | 语言没有命名的视觉差异被压扁 |
| 指代的自我否定 | 语言能指到框、mask、region | 指到某处不等于持有对象关系、几何状态和时间延续 |
| 调用的自我否定 | prompt / instruction 让视觉系统灵活可用 | 接口成功不等于视觉主体性，回答成功不等于视觉自持 |

阶段 III 的失败不是语言不够强。阶段 III 的失败恰恰来自语言太强：它能替视觉完成许多公共语义工作，因此遮蔽了视觉自身仍未闭合的部分。

阶段 II 结语的哲学思考预言了七个维度的缺失。阶段 III 结束时的回顾：

| 维度 | 阶段 II 结束时 | 阶段 III 结束时 | 进展 |
|---|---|---|---|
| 投影（柏拉图） | 只有 2D 投影 | 仍然只有 2D 投影 | 无 |
| 先验范畴（康德） | 无先验，只有统计共现 | 语言提供了"借来的范畴" | 有，但是借的 |
| 语言游戏（维特根斯坦） | 没有自己的语言游戏 | 进入了人类的语言游戏，但作为外围 | 有，但不是自己的 |
| 身体（梅洛-庞蒂） | 无身体 | 仍然无身体 | 无 |
| 上手/在手（海德格尔） | 只有在手模式 | 仍然只有在手模式 | 无 |
| 生成性（谢林） | 被认识的自然 | 仍然被认识的自然 | 无 → 阶段 IV |
| 无名之见（老子） | 只有"有名"的视觉 | "有名"更强了，但"无名"仍缺 | 恶化 |

> 阶段 III 在康德和维特根斯坦维度上有进展，但代价是在老子维度上恶化——语言越强，视觉自身的"无名"感知越被压扁。这就是"借语言的代价"的哲学本质。

---

## 四、阶段 III 的一句话

> 视觉与语言签了一张临时协议：语言提供语义主权，视觉提供空间定位。协议有效期内，VLM 是最强的视觉系统。但协议到期日已经写在账单上——视觉必须自己能生成（阶段 IV）、自己能持有（阶段 V），才能从合同人变成主权者。
