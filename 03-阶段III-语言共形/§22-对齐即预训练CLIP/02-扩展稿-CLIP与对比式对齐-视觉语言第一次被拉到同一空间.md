**主题**：阶段 II 终章把「开放词汇」外包给了 CLIP——第 81 类能被识别，是因为 CLIP 见过这个词。这一节就是要把 CLIP 本身拆开看：为什么「两个独立编码器 + 对比损失」这套机制能让语言在工程上第一次成为视觉可用的标签字典？它又为什么终究停在「挂件」阶段？五篇一起，是对这同一个问题的连续追问。

## 1 · CLIP —— *Learning Transferable Visual Models From Natural Language Supervision* (Radford 等, OpenAI, ICML 2021)

**共形贡献**：400M 网络图文对 + InfoNCE 对比损失 + 双塔结构（ViT / ResNet 图像塔 + Transformer 文本塔）。对比损失把图文对齐写成「在 batch 里找配对」的判别问题；模型学到的是图像向量与文本向量在共享球面上的**相对几何**。zero-shot 迁移用 `a photo of a {class}` 做 prompt，把任意类别名变成查询向量——**开放词汇在工程上第一次达成**。

🔗 [https://arxiv.org/abs/2103.00020](https://arxiv.org/abs/2103.00020)

## 2 · ALIGN —— *Scaling Up Visual and Vision-Language Representation Learning With Noisy Text Supervision* (Jia 等, Google, ICML 2021)

**共形贡献**：和 CLIP 几乎同期、独立提出；把数据量推到 1.8B **未经清洗**的 alt-text 图文对——比 CLIP 多一个量级而且更脏。核心论点：**规模可以吃下噪声**，数据清洗可以被尺度定律替代。工程上把「大规模 + 对比损失」这条路从 CLIP 的一次性实验变成**可复现的范式**。

🔗 [https://arxiv.org/abs/2102.05918](https://arxiv.org/abs/2102.05918)

## 3 · LiT —— *LiT: Zero-Shot Transfer with Locked-image Text Tuning* (Zhai 等, Google, CVPR 2022)

**共形贡献**：把图像塔**冻住**（用已有的强视觉编码器，例如 JFT 上的 ViT），只训练文本塔与投影。论点：图像的表征已经在监督 / 自监督预训练里学得足够好，对比预训练的工作其实是**让文本去适应图像**，而非对称地让两边都动。这是对「CLIP 是对称双塔」这一默认的第一次松动——也是「语言是挂件」从直觉变成工程事实的证据。

🔗 [https://arxiv.org/abs/2111.07991](https://arxiv.org/abs/2111.07991)

## 4 · SigLIP —— *Sigmoid Loss for Language Image Pre-Training* (Zhai 等, Google, ICCV 2023)

**共形贡献**：把 softmax-based InfoNCE 换成**逐对 sigmoid 二分类**——每一对 $(I_i, T_j)$ 独立判「是否配对」，不再要求 batch 内归一化。两个后果：(1) 不需要 all-gather 做 softmax 分母，**小 batch 也能训练**；(2) 对超大 batch（百万级）更容易扩展。把 CLIP 训练的硬件门槛显著降低，让对比损失从「竞赛式」（整批相对排序）变成「独立式」（每对独立判）。

🔗 [https://arxiv.org/abs/2303.15343](https://arxiv.org/abs/2303.15343)

## 5 · EVA-CLIP —— *EVA-CLIP: Improved Training Techniques for CLIP at Scale* (Sun 等, BAAI, 2023)

**共形贡献**：把视觉塔推到 1B+ 参数、图文对推到 2B 级；把 MIM 预训练好的 EVA 作为 CLIP 的图像初始化。工程论点：CLIP 的性能上限不是数据决定的，是**视觉塔初始化的质量**决定的——从 MIM 重建任务预训练出的特征，再做对比对齐，远胜从头开始。这条路把「对比式对齐」与「自监督视觉」第一次显式串起来——预示阶段 V（表征共形）的伏笔已经从阶段 III 内部伸出头。

🔗 [https://arxiv.org/abs/2303.15389](https://arxiv.org/abs/2303.15389)

# 关于 §22 五篇的机制追问

## 一、InfoNCE 到底在做什么：不是「对齐」，是「对齐相对几何」

### 损失式

设 batch 里有 N 对 $(I_i, T_i)$，分别编码到 L2 归一化向量 $v_i, t_i$，损失是双向对称的 InfoNCE：

$L = -\frac{1}{2N}\sum_i \left[ \log\frac{\exp(v_i \cdot t_i / \tau)}{\sum_j \exp(v_i \cdot t_j / \tau)} + \log\frac{\exp(v_i \cdot t_i / \tau)}{\sum_j \exp(v_j \cdot t_i / \tau)} \right]$

第一项是「以图找文」，第二项是「以文找图」。$tau$ 是温度。

### 关键点 1：它不学「相等」，学「最相似」

这个式子很容易被误读为「让配对的 $v$ 和 $t$ 彼此相等」——不是。L2 归一化之后，cosine 就是点积；InfoNCE 的目标是让**配对项的 cosine 严格大于所有非配对项的 cosine**。这是一个**排序目标**：只要排序对，绝对数值不重要。

### 关键点 2：它对齐的是「空间的相对几何」，不是「绝对位置」

训练完成之后：

- 单看一个 $v$ 或一个 $t$，它们的绝对坐标没有语义——两次训练可以得到两套完全不同的坐标系。
- 但两个空间里的**相似度排序是对齐的**：图里两张照片 A、B，A 和文本「猫」更近、B 和文本「狗」更近——这种**相对比较结构**是可迁移的。

这才是 zero-shot 能工作的真正基础：**不是**「图像向量 = 类别向量」，**是**「在相似度排序上两套空间协同变形」。这与阶段 I §01 里「共形的第一要义不是保值，是保相关性」精神一致——CLIP 是这条原理在视觉-语言两套空间之间的第一次大规模工程化。

### 关键点 3：配对的定义决定了能学到什么

CLIP 的正样本是「网页上图像旁边那段 alt-text」。这意味着：

- 能学到：物体名、粗粒度属性（颜色、风格）、场景标签、名人脸、地标。
- 学不到：空间关系（「A 在 B 左边」）、计数（「三只」）、方向、否定（「不是猫」）、因果。

因为 alt-text 里这些结构性信息**极少出现**——图说就是几个词。对比损失本身是无辜的，**监督信号的语义密度直接规定了能学到什么的上限**。这条就是 §27（MMVP / CLIP-blindness）账单的数学根。

## 二、为什么必须是两塔：对比式对齐的工程逼迫

CLIP 为什么不用一个统一的「图文融合」编码器，像 BERT 那样把图文拼成一个序列过一遍？

### 答：对比损失的**用法**决定了塔必须分开

对比损失要求在每个 batch 里做 N×N 的所有两两比较。如果用融合式（单流）编码器：

- 每一对 $(I_i, T_j)$ 都要**单独过一次**编码器才能拿到联合特征——batch 内计算量变成 $O(N^2 cdot text{forward})$。
- 对 batch = 32K 来说这是灾难性的。

两塔结构则：

- $v_i = f(I_i)$：只需 N 次图像前向。
- $t_j = g(T_j)$：只需 N 次文本前向。
- 所有 N×N 两两比较只是 **N×N 的点积**——几乎零成本。

所以 CLIP 的两塔**不是一种风格选择，是「对比损失 + 大 batch」这对组合的工程必然**。

### 后果 A：zero-shot 推理的结构红利

这个结构还有一个更重要的副产品：**文本侧可以完全离线**。

- 训练一次 → 固定 1000 个类别的文本描述 → 各自算一次文本向量 → 缓存。
- 推理一张图 → 一次图像前向 → 和缓存的文本向量做 cosine → argmax。

单张图像的 zero-shot 分类成本就是「一次图像前向 + 一次矩阵点积」——几乎不比普通分类贵。**这是 CLIP 能在工业界泛滥的工程根本原因**。阶段 II 末端的开放词汇检测（Grounding DINO / YOLO-World / ODISE）全部继承这套结构——先把 CLIP 文本向量缓存成「类别原型」，视觉侧再来查。

### 后果 B：「语言是挂件」被写死在架构里

在 CLIP 里，图像塔和文本塔**在前向过程中从未相遇**——它们只在损失函数那一刻相遇。这意味着：

- 图像塔的推理不需要语言。
- 文本塔的推理不需要图像。
- 两塔的交互是**一次性、非交互式**的——就是一次点积。

这就是为什么阶段 III 总纲把 §22 的残差写成「对齐是拉到同一空间，不是放到同一骨干」。**两塔结构把「语言只在损失处相遇」这件事钉在了架构层面**——要打开它，就得继续走向 §23 的稠密锚定、§24 的统一骨干、§25 的 LLM 后端，直到 §26 的像素级输出。

## 三、zero-shot 迁移的真实机制：不是「模型会推理」，是「prompt 查了字典」

CLIP 的 zero-shot 常被宣传为「模型无需微调就能分类新类别」，给人一种「模型理解了语言」的印象。拆开看是这样的：

### 步骤

1. 给出类别列表 `["cat", "dog", "car", ...]`。
2. 用 prompt 模板（通常是 `"a photo of a {class}"`）把每个类名变成完整句子。
3. 文本塔把每个句子编码成向量，缓存起来。
4. 图像来了之后，图像塔编码成向量。
5. 和所有类别文本向量做 cosine，取最大。

### 这是「查字典」，不是「理解」

- 模型从来不知道「猫」是什么。它只知道：「这张图的视觉向量，和由字符串 `a photo of a cat` 产生的文本向量，离得近」。
- 类名换成乱码（`a photo of a xqyzblorp`），模型就崩——它没有语义实体观念。
- prompt 写法（`a photo of a` vs `a picture of a` vs `an image of a`）能改变精度 1–3 个点——因为文本塔对句式敏感，而**句式不是语义**。
- 这就是 prompt engineering 兴起的根：CLIP 让 prompt 变成了一个**需要调的超参数**。

### 为什么这仍然是突破

虽然机制是「查字典」，但它做到了一件阶段 II 所有开放词汇工作做不到的事：**词典的 key 可以是任意自然语言短语**。阶段 II 的开集检测要么依赖预定义词表、要么只能泛化到训练集词汇的近邻；CLIP 的字典是 400M 图文对训练出来的，近似覆盖了网络图说所能表达的概念空间。

**「工程可用的开放词汇 = 一个足够大的文本-向量字典 + 一个能查这个字典的图像编码器」**——这就是 CLIP 的本质工程贡献。它是**判别式**的极限：系统能对任意 x 做出 y 的命名，但**不能反向从 y 合成 x**。这条判别-生成的分界线就是 §27 残差 A 要交给 IV 的东西。

## 四、温度 $\tau$ 与 batch size：softmax 分母上的两个旋钮

### 温度 $\tau$

InfoNCE 里的 $\tau$ 控制 softmax 的锐度：

- $\tau$ 小 → softmax 接近 argmax → 对比压力极大 → 模型被逼着把配对和非配对**严格**分开 → 训练不稳定。
- $\tau$ 大 → softmax 平 → 对比压力弱 → 学得慢但稳。

CLIP 把 $\tau$ 做成**可学习参数**（初始 0.07，通常稳定到 0.01 附近）。这不是小细节——**可学习** $\tau$ **允许模型早期用大** $\tau$ **平稳启动，后期自动缩小** $\tau$ **去精炼**。是一种内置的课程。

### Batch size 的决定性作用

InfoNCE 的 softmax 分母包含 batch 内所有非配对项——**这就是「负样本」**。负样本越多，判别边界越精细：

- batch = 256 时，每个锚点只有 255 个负样本——边界粗糙。
- batch = 32K 时，每个锚点有 32767 个负样本——精细到足以区分「金毛」和「拉布拉多」。

CLIP 用 batch = 32768（256 GPUs，每卡 128）。这也是 CLIP 训练门槛高的根：**不是参数量大，是 batch size 大**。batch = 32K 要求显存、通信、all-gather 全部工程化。这条门槛直接催生了 SigLIP。

### SigLIP 改造的本质

把 InfoNCE 里的 softmax-over-batch 换成 per-pair sigmoid：对每一对 $(i,j)$，

$L_{ij} = -\log \sigma\!\left( z_{ij} \cdot (t \cdot v_i \cdot t_j + b) \right)$

其中 $z_{ij}=+1$ 若 $i=j$，否则 $-1$；$t, b$ 是可学习的温度和偏置。

关键差别：**每对** $(i,j)$ **的损失独立**，不再依赖 batch 内其他对——

- 不需要 all-gather 做分母。
- 小 batch 不再降质——只是少了些负对，不影响每个现存对的学习信号。
- 超大 batch 更容易扩展——只要增加每 GPU 处理的对数。

从**共形视角**看：InfoNCE 把对齐做成了一场「竞赛」（每个锚点必须打败 batch 内所有其他候选）；SigLIP 把它做成了一场「独立判分」（每对独立决定是否配对）。两者的理论极限相同（都能对齐），但**工程代价分布完全不同**。这条改造让对比式对齐**民主化**，不再是 big lab 专利——也让 §23 之后的 LLM 集成式 VLM 训练时可以把对比目标作为一个廉价的辅助损失挂上去。

## 五、LiT 与 EVA-CLIP：对「对称假设」的两次削弱

CLIP 的默认是「图像塔和文本塔对称地从头训练」。LiT 和 EVA-CLIP 都在这个假设上打洞——一个往「视觉已经够强，只需训练语言」的方向；一个往「视觉初始化决定上限」的方向。

### LiT：冻住图像塔

- 用预训练好的强视觉编码器（监督 JFT 或自监督 DINO），**冻住权重**。
- 只训练文本塔 + 两塔之间的投影。
- 效果：zero-shot 精度**高于**对称训练的 CLIP。

机制解读：图像塔在监督 / 自监督预训练里已经学到了「看」——只需要让文本塔学会「指」。对称训练反而让图像塔的好表征被对比损失**稀释**（对比损失相对于分类监督 / MIM 重建是更弱、更粗粒度的信号）。

本体论解读：**「语言是挂件」在这里被赤裸地写出来**——你可以冻住视觉、只训练语言，依然能得到好的对齐；反过来不行（冻住语言、只训练视觉，精度大幅下降）。**两塔本质不对称**——视觉是主，语言是副。LiT 的存在是这条不对称性在工程上的自白。

### EVA-CLIP：用 MIM 初始化视觉塔

- 先用 MAE / BEiT 式的 mask image modeling 预训练视觉塔。
- 再用对比损失做 CLIP 训练。
- 效果：同等数据量下精度更高；同等精度下数据量更少。

机制解读：MIM 学到的是「像素层面的完形填空」——给视觉塔一个**稠密的、空间结构完整的**初始表征；CLIP 的对比损失本身太粗（只在全局向量上压），容易把稠密结构稀释。MIM 初始化相当于**先给视觉塔灌上稠密先验，再让对比损失只微调「向量级语义」**。

本体论伏笔：EVA-CLIP 这条路实际上已经在**预演阶段 V**——「不依赖配对语料的视觉自组织」这条线在 MIM / DINO / JEPA 里继续发展，最后在 §27 送给 V 的那条残差 B 就是从这里展开的。**EVA-CLIP 是阶段 III 内部已经探出头的 V 的尾巴**。这也是为什么「对比预训练的上限由视觉初始化决定」这句话这么重——它暗示着：如果视觉能自己组织起足够强的语义，语言那一侧的作用就会持续缩水。

## 小结：§22 的谱系内逻辑

- **CLIP**：把「网页图文对 + 对比损失 + 双塔」这一套钉下；zero-shot 用 prompt 查字典，**开放词汇在工程上达成**。
- **ALIGN**：推到 1.8B 噪声数据，证明**规模能吃下清洗**——CLIP 从个案变成范式。
- **LiT**：冻住视觉塔——**对称假设第一次被削弱**；「语言是挂件」从直觉变成工程事实。
- **SigLIP**：softmax 换 sigmoid——**批内归一化被拆**；训练工程门槛塌一半。
- **EVA-CLIP**：MIM 初始化视觉塔——**自监督视觉与对比式对齐第一次显式串联**，为阶段 V 埋伏笔。

五篇合起来完成一件事：**把「视觉-语言对齐」从一个模糊愿望，工程化为一个可复现、可扩展、可民主化的预训练范式**。但也一起暴露了同一个结构极限：**两塔永远不在前向中相遇**。

## 本组合力的残差

CLIP 与四篇改造把对齐做实了，但同时把三条结构性残差暴露给后续：

1. **语言仍只在损失和相似度空间中发挥作用**——两塔结构在前向中从未相遇，语言只是在损失处提供排序监督。视觉骨干不「读」语言，它只产生一个「可被任意文本向量查询」的特征。这条残差推向 §23 的稠密锚定与 §24 的统一骨干：从全图 lookup 走向 region、task prompt 与多任务共享接口。
2. **语义被词袋化**——监督信号是 alt-text（平均几个词、noisy、非结构化），模型学到的是「概念名」而非「结构化描述」。空间关系、计数、方向、否定、因果——这些在 alt-text 里极少出现，也就没被学到。这条残差**最终流向 §27 账单**（MMVP / CLIP-blindness），并继续分化成：生成侧的反向合成残差（送 IV）与表征侧的自组织残差（送 V）。
3. **对齐依赖配对语料**——CLIP 的整套工作建立在「网络上本来就有 400M+ 图-文对」这一经验前提上。没有配对语料就做不起来；非英语、低资源领域（医疗、工业）、需要结构化描述的场景全部退化。这条残差**被 §27 送给 V**——「不依赖配对语料的语义自组织」（JEPA / DINO 那条线）要回答的就是它。

第 1 条定义 §23 / §24 的入口；第 2 条最终汇入 §27 账单；第 3 条被 §27 转交给 V。**§22 是阶段 III 的出生点，也是阶段 III 两条出口残差的源头之一**——「对齐 → 骨干」这条线从这里走向 §23-§26，「配对 → 自组织」这条线从这里走向 V。
