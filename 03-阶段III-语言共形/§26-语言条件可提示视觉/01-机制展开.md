# §26 · 机制展开：从理解到像素级输出的闭环

§26 的问题不是“VLM 会不会说出答案”，而是语言指令能否触发空间化产出：box、region、mask、grounded response。这里建立的是语言到像素级输出的单向闭合，不是行动闭环，也不是生成闭环。

---

## 一、三种范式

| 范式 | 代表 | 机制 | 优点 | 残差 |
|---|---|---|---|---|
| 外部组装 | Grounded-SAM | GroundingDINO / 检测器定位，再交给 SAM 分割 | 灵活、可替换、部署快 | 多模块误差级联，非端到端 |
| 坐标作为 token | Shikra、Ferret、KOSMOS-2 | bbox / region 被编码进文本序列 | 对话接口自然 | 坐标文本化粗糙，mask 表达弱 |
| embedding-as-mask | LISA、GLaMM、PixelLM | LLM 输出 `<SEG>` 等 token，再驱动 mask decoder | 端到端语言推理到 mask | 分辨率、速度、多 mask 和数据成本仍有限 |

这三类不是简单淘汰关系。Grounded-SAM 式 pipeline 在工程灵活性上很强；embedding-as-mask 在端到端闭环上更完整。

---

## 二、LISA 的位置

LISA 的关键是：

```text
instruction / question
  -> LLM reasoning
  -> <SEG> token
  -> mask decoder
  -> pixel mask
```

`<SEG>` token 把语言推理的结果转成可喂给 mask decoder 的隐变量。它不是把 mask 写成字符串，而是在语言推理后打开一个空间输出通道。

因此，LISA 是 reasoning segmentation 中最清楚地打通“语言推理 -> mask 输出”的代表范式。它优于纯 pipeline 的地方是闭环完整；它弱于 pipeline 的地方是工程可替换性、部署速度和开放组合弹性。

---

## 三、任务差异

| 任务 | 输入 | 输出 | 主要考什么 |
|---|---|---|---|
| referring expression | “左边那只红色的狗” | bbox / mask | 空间、属性、关系定位 |
| reasoning segmentation | “可以坐的东西” | mask | 常识与视觉证据绑定 |
| grounded conversation | 多轮对话 + 指代 | mask + 文本 | 指代延续、上下文维护 |

三类任务越往后，语言能力越重要；但语言越重要，越容易由语言先验接管视觉证据。

---

## 四、为什么仍未真正闭环

§26 建立的是：

```text
language -> spatial output
```

它还缺两条回路：

1. `spatial output -> visual verification -> corrected output`：输出后能否看回去、审计、修正。
2. `language -> new observation`：能否生成、编辑、延续一个新的视觉状态。

第一条把问题推向阶段 V 的反身度量与状态持有；第二条把问题推向阶段 IV 的生成共形。
