# 阶段 III · 多时相视觉语言模型专题：时间维度上的图斑产出（TEOChat / ChangeChat / DeltaVLM / TAMMs / ViLaCD-R1）

作者 / 机构: Irvin 等 (Stanford, ICLR 2025) / Deng 等 (ICASSP 2025) / Deng+Zhou+Wu (北京外国语大学, MDPI RS 2025 / arxiv 2507.22346) / Guo 等 (2025/06 arxiv 2506.18862) / Ma+Feng 等 (2025/12 arxiv 2512.23244)
共形维度: 结构 + 度量 + 时序
关键贡献: 把 §27 的「单时相 RS-VLM 图斑产出」这条线沿时间轴再推一步。TEOChat 把 LLaVA 架构的视觉编码器做成「时间共享」从而支持任意长度的 EO 序列；ChangeChat 用指令微调把 bi-temporal change captioning / quantification / localization 合并为 VLM 对话任务；DeltaVLM 把 bi-temporal 处理升级为指令引导的差异感知，用 CSRM 机制和 Q-former 直接从差异特征提取查询相关信号；TAMMs 让 frozen MLLM 同时做变化理解与未来帧预测，首次把 ControlNet 嫁接到 RS 时序；ViLaCD-R1 用 SFT + RL 训练一个多图推理器 MIR 输出块级粗 mask，再由 Mask-Guided Decoder 精化，第一次端到端打通「VLM → 像素变化 mask」。

🎯
**主题**：多时相 RS-VLM 是 §27 三条残差中汇入 V 反身度量那一条（多时相 / 变化推理仍无原生架构）的专题展开，也正对应你当前研究方向的核心痛点。五篇论文画的是「从把多张图塞进 VLM、到让 VLM 理解图间差异、到让 VLM 直接输出变化图斑」这条完整曲线。曲线的终点（端到端 VLM 输出像素级变化 mask）在 2025/12 才刚出现（ViLaCD-R1），并且是两阶段 MIR + MGD 而非真正的单一模型；曲线更远的点（矢量多边形 + 时间、>2 时相 SITS、多传感器时序）还没人踏进去。这就是你的研究空间。

## 1 · TEOChat —— *arxiv 2410.06234 / ICLR 2025* (Irvin 等, Stanford)

**共形贡献**：首个面向时间序列地球观测数据的视觉语言助手。此前所有 RS-VLM（GeoChat / LHRS-Bot / Qwen-VL 衍生）都只支持单图输入，TEOChat 把这个根本约束打破。

**三件关键事**：

**（1）TEOChatlas 指令数据集**：554,071 条指令样本，涵盖单图任务与时序任务。时序任务包括建筑变化检测（bbox + 描述）、建筑损毁评估（xBD 语料）、语义变化检测（SECOND / HRSCD 语料）、时序场景分类（fMoW-temporal）。**单个数据集就是量级超过 §27 所有 RS-VLM 指令数据的总和**——这说明如果想做时序 RS-VLM，数据首先要按 10 倍扩展。

**（2）时间共享视觉编码器**：使用 LLaVA 风格架构，ViT 在所有时间帧间权重共享，输出的时间帧 token 拼接进入 LLM（LLaMA 2）。这是最简单的时序处理方式——**并没有真正的时间 attention**，仅靠 LLM 的自回归来处理时间关系。这是首篇工作的典型设计选择：用最小的架构改动换最大的功能扩展。

**（3）训练效率**：LoRA 微调 + 8-bit 量化，让序列长度达到 8 帧也能在一张 A4000（16GB）上训练。这个工程方案让中小实验室也能复现时序 VLM——一定程度上降低了时序 VLM 研究的门槛。

**关键结果与局限**：在 xBD 建筑损毁评估、HRSCD 语义变化检测、AID 时序分类等任务上超过单图 VLM 基线和专用模型。**但 TEOChat 的输出仍以文本和 bbox 为主，不产出像素 mask；时间推理能力受限于 LLM 的自回归而非架构层的 temporal attention**。这两点局限后续被 DeltaVLM 和 TAMMs 分别从不同方向弥补。

🔗 [https://arxiv.org/abs/2410.06234](https://arxiv.org/abs/2410.06234)

## 2 · ChangeChat —— *arxiv 2409.08582 / ICASSP 2025* (Deng 等)

**共形贡献**：首个专门为 bi-temporal RS 变化分析设计的 VLM。与 TEOChat 的「通用时序」不同，ChangeChat 把焦点收窄到双时相——**这恰好覆盖了 90% 的实际 RS 变化检测业务场景**（灾前灾后、建设前后、季节前后、年前年后）。

**三件关键事**：

**（1）ChangeChat-87k 指令数据集**：用规则方法 + GPT 辅助流水线从 LEVIR-CD / LEVIR-CC 等数据集自动构造 87k 条指令-响应对。任务类型包括：change captioning（文本描述变化）、category-specific quantification（定量化某一类别的变化量）、change localization（给出变化区域的 bbox 或描述方位）。

**（2）基于 Vicuna 的 LLM + 双路 ViT**：两张时相图像分别过共享 ViT，特征拼接送入 Vicuna；与 TEOChat 同样没有显式 temporal attention。**但 instruction prompting 里嵌入了「第一张图」/「第二张图」/「变化前后对比」的任务 token**——靠 prompt 工程驱动 LLM 在语言空间处理时序关系。

**（3）Chain-of-thought 变化分析策略**：论文里提出了「先识别 → 再比较 → 再描述」的 CoT prompting 模板，这一点是对 §26 推理时规模律在 RS 双时相上的早期试探。

**关键结果**：在 LEVIR-CC（10,077 对双时相图 + 50,385 条变化描述句）上 BLEU-4 / METEOR / ROUGE-L / CIDEr 各项均超过非 VLM 基线（RSICCFormer、PSNet 等）。category-specific quantification 这一任务当时是首次提出，没有对比基线。**但 localization 只能到粗粒度方位描述或 bbox，不产出 mask**——这是 ChangeChat 留给 DeltaVLM 和 ViLaCD-R1 的空白。

🔗 [https://arxiv.org/abs/2409.08582](https://arxiv.org/abs/2409.08582)

## 3 · DeltaVLM —— *arxiv 2507.22346 / MDPI Remote Sensing 2025* (Deng+Zhou+Wu, 北京外国语大学信息学院)

**共形贡献**：把 bi-temporal VLM 的能力维度从「看懂变化 + 描述变化」扩展到「围绕变化进行多轮对话」。换句话说，TEOChat 和 ChangeChat 是「问一句答一句」的 VQA 范式；DeltaVLM 是「针对变化区域展开 6 类任务的 multi-turn 对话」的 agent 范式。

**三件关键事**：

**（1）RSICA 任务定义**：Remote Sensing Image Change Analysis——统一 change captioning + change VQA + change classification + change counting + change localization + multi-turn dialog 为同一任务体系。这是 §27 的「任务 token」思想在时序上的直接延拓。

**（2）ChangeChat-105k 指令数据集**：在 ChangeChat-87k 基础上扩到 105k，新增多轮对话（最多 5 轮，平均 3.2 轮）、开放式 QA、计数任务。这是目前**最大的 bi-temporal RS 指令数据集**，也是 ViLaCD-R1 等后续工作的训练基座。

**（3）三件架构创新**：

- **Fine-tuned bi-temporal 视觉编码器**：在 RS 双时相对上微调 CLIP ViT，让两张图的特征在 embed 空间中「差异被放大」。具体做法是引入 temporal contrastive loss（同时相的不同区域应相似 vs 跨时相的同区域应有差）。
- **视觉差异感知模块 + CSRM（Cross-Semantic Relation Measuring）**：不直接拼接两张图的特征，而是先算差异图再提炼。CSRM 计算两张图的 patch 间的跨语义相似度矩阵，用来识别「语义层面的变化」而非「像素层面的变化」（后者会被光照 / 季节 / 图像配准误差污染）。
- **Instruction-guided Q-former**：受 BLIP-2 启发，用 Q-former 从差异特征里提炼与当前指令相关的压缩表示再送入 frozen LLM。Q-former 的 queries 由指令嵌入初始化，这让模型能「听懂」用户想关心哪一类变化。

**冻结 LLM + 只调视觉**：与 §23 BLIP-2 / LLaVA 的思路一致——LLM 完全冻结，只训练视觉编码器 + CSRM + Q-former。训练成本极低。

**关键结果**：在 ChangeChat-105k 单轮 / 多轮对话、LEVIR-CC、LEVIR-MCI 等 benchmark 上全面超过 TEOChat / ChangeChat / GeoChat。**但仍然不产出 mask**——这一点要到 ViLaCD-R1 才被解决。

🔗 [https://arxiv.org/abs/2507.22346](https://arxiv.org/abs/2507.22346)

## 4 · TAMMs —— *arxiv 2506.18862* (Guo 等, 2025/06)

**共形贡献**：第一篇把「变化理解」与「未来帧预测」耦合在同一 MLLM 框架里的工作。用户给若干历史时相，模型既要描述已发生的变化，也要生成 / 预测下一个时相的图像。这正面回应了 RS 领域里一条长期被搁置的子方向——**时序 forecasting**——而 VLM 的介入让 forecasting 第一次带上了语义约束。

**四件关键事**：

**（1）Temporal Adaptation Modules（TAM）**：在 frozen MLLM（LLaVA / Qwen-VL 类）的视觉 → 语言桥接处插入轻量的 temporal adapter。TAM 的设计参考了 Video-LLaVA 但针对 RS 的长时间跨度（年级 / 月级）而非视频的帧级做了改造——用结构化的序列编码（位置嵌入 + 相对时间戳）而非等间隔帧嵌入。这是目前 RS-VLM 里最接近「原生 temporal attention」的设计。

**（2）Semantic-Fused Control Injection（SFCI）**：为了让模型能生成未来帧图像，引入增强版 ControlNet。MLLM 输出的高层语义推理（自然语言描述）与结构先验（前一时相图 + 前一时相 mask / 分割图）双路注入 ControlNet，指导未来帧生成。**这把 §25 原生多模态（图文共生成）的思想迁到了时序**。

**（3）Temporal Consistency Score（TCS）度量**：RS 时序 forecasting 长期缺乏好的度量——PSNR / SSIM 只看像素级相似度，不看是否符合「历史演化趋势」。TCS 是第一个专门度量「预测变化 vs 历史变化趋势」一致性的指标，基于光流 + 语义变化率的加权和。**这条度量本身就是 §28 残差 3 的一个早期回应**，但只覆盖 forecasting 一侧，不覆盖语言对变化的忠实度。

**（4）IceInPot/TAMMs 数据集**：贡献了一个专门为 RS SITS 理解 + forecasting 设计的多任务 benchmark。

**关键结果**：在变化描述和未来帧生成两个任务上都显著超过 baseline（单独的 CLIP-based caption 模型、单独的 ControlNet）。TAMMs 用 TCS 指标证明了「多任务耦合」的增益——这呼应 §24 的「指令混合反哺单任务」现象。

**给你的实战意义**：TAMMs 的 TAM 设计直接可以作为你方向的架构模板——在 GeoPixel / GeoGround 的基座上加一个 TAM-like 模块处理多时相序列，再把 SFCI 改造为「变化 mask 生成」而非「未来帧生成」——这就是你实验 3 的一个具体开路方案。

🔗 [https://arxiv.org/abs/2506.18862](https://arxiv.org/abs/2506.18862)

## 5 · ViLaCD-R1 —— *arxiv 2512.23244* (Ma+Feng 等, 2025/12)

**共形贡献**：第一个让 VLM 直接产出像素级变化 mask 的端到端框架（严格说是两阶段）。这是 §27 末 + §28 最前沿的真正交点——前面四篇要么不产出 mask（TEOChat / ChangeChat / DeltaVLM），要么产出的是未来帧图像而非变化 mask（TAMMs）——ViLaCD-R1 把 VLM + 像素级变化 mask 这对组合第一次做通。

**两阶段设计**：

**（1）Multi-Image Reasoner（MIR）**：基于 VLM（InternVL / Qwen2-VL 类），用 **SFT + RL** 在块级双时相推理任务上训练。SFT 阶段用标注数据学「给定 patch 对判断是否有变化」；RL 阶段用 GRPO 或 PPO 在「判断正确性 + 描述合理性」的复合奖励下继续训。**RL + VLM + CD 的组合是 §26 规模律与推理时刻思想的 RS 时序实例**——DeepSeek-R1 的范式被直接搬到了 RS 变化检测上。

**（2）Mask-Guided Decoder（MGD）**：MIR 输出的是块级粗 mask（分辨率与 patch grid 对齐）；MGD 以这个粗 mask 作为引导，与原图特征融合，上采样到原图分辨率的高质量 mask。MGD 可以看作 §27 RS2-SAM2 思路的简化版——用 SAM-style 分割先验把 VLM 的粗略判断精化成像素级边界。

**关键结果**：

- **LEVIR-CD**：IoU 与 F1 超过专用 CD 网络（BIT、ChangeFormer、CDMamba）。
- **SYSU-CD / WHU-CD**：同样 SOTA。
- **消融实验**：MGD 对精度的贡献约 4–6 个 IoU 点；RL 对 MIR 的贡献约 2–3 个点——**说明两阶段设计的每一阶段都必要**。

**意义与局限**：

- **意义**：第一次把「VLM 的语义推理」与「分割级 mask 产出」在 RS bi-temporal 场景端到端连通。DeltaVLM + MGD 的组合就是你实验 3 的一个延伸基线。
- **局限**：仍然是两阶段，不是纯 autoregressive 单模型；仍然是 bi-temporal 不支持 >2 时相；输出仍是像素 mask 而非矢量多边形；MIR 的 SFT 数据依赖现有 CD benchmark 的标注，data scaling 仍是瓶颈。**这正是 §28 留给你研究的缺口**。

🔗 [https://arxiv.org/abs/2512.23244](https://arxiv.org/abs/2512.23244)

# 关于 §28 五篇的机制追问

## 一、从单时相 VLM 到多时相 VLM：架构改造的三条路线

### 路线 1 · 时间共享视觉编码器 + LLM 自回归处理时序

代表：TEOChat、ChangeChat。把多张图分别过共享 ViT，特征 token 拼接后一起喂给 LLM，让 LLM 自己在语言空间里处理「这张是 t1、那张是 t2」。**优点**：最小架构改动，只需在 prompt 里加时间标注 token；**缺点**：没有显式 temporal attention，LLM 在多张图间的时序关系是隐式学出来的，长序列（>4 帧）会线性退化。

### 路线 2 · 显式差异感知模块 + 指令引导压缩

代表：DeltaVLM。不是简单拼接两张图的特征，而是在视觉侧先算差异（CSRM 跨语义相似度），再用 Q-former 把与指令相关的差异信号压缩给 LLM。**优点**：让 LLM 看到的是「语义差异」而非「两张原图」，大幅降低 LLM 的处理负担；**缺点**：算差异的模块本身成为新的瓶颈，且这条设计只适用于 bi-temporal，不容易扩展到 >2 时相。

### 路线 3 · 时间适配器 + 结构化序列编码

代表：TAMMs 的 TAM。在视觉 → 语言桥接处插入轻量模块，专门处理时序；用相对时间戳嵌入替代简单拼接。**优点**：模块可插拔，frozen MLLM 不动；**缺点**：TAM 的表达能力有限，对复杂时序模式（周期性、突变 + 渐变混合）仍弱。

### 三条路线的残差

**至今没有人做真正的「时间方向原生 attention」**——类似 Qwen2.5-VL 的 MRoPE 把旋转位置编码从 1D 文本扩展到「文本 + 时间 + 高 + 宽」的 4D。RS 的多时相 + 多尺度 + 多传感器本质上需要一个 5D 甚至 6D 的位置编码，但目前所有架构都避开了这个问题。**这是 §28 第一条残差**，也是你可以动手的架构方向。

## 二、多时相 RS 的数据稀缺：四种标注类型的不完备交集

### 四种标注类型

- **像素级二值变化 mask**：LEVIR-CD（32,507 对）、SYSU-CD、WHU-CD、S2Looking。只标「变化 / 不变化」，无语言。
- **像素级语义变化 mask**：SECOND（4,662 对，30 类）、HRSCD（291 对，6 类）。标「从 A 类变到 B 类」，无自由语言。
- **文本级变化描述**：LEVIR-CC（10,077 对，50,385 句）、DUBAI-CCD、SECOND-CC。有自由语言描述，无像素级 mask。
- **指令追随对话**：ChangeChat-87k / ChangeChat-105k / TEOChatlas（554k）。有多轮对话和多任务指令，mask 层面覆盖不一。

### 四者交集为零

**没有任何一个 RS 数据集同时提供「双时相图 + 自由语言指令 + 像素级变化 mask + 多轮对话」四者**。这意味着：

- 想训 ViLaCD-R1 这类端到端 VLM → mask 模型，必须拼多个数据集。
- 想评测「语言指引下的变化 mask 产出」，必须自构造测试集。
- 想评测「多轮对话下的 mask 一致性」，根本没有 benchmark。

**这是 §28 第二条残差**。构造这样一个「四者合一」的 benchmark 本身就是一篇 ISPRS / TGRS 论文，且会成为 RS-VLM 时序方向的基础工作。

### 合成数据的潜力

RCDGen（Korkmaz 等, WACV 2026, arxiv 2512.11719）和 ChangeAnywhere（arxiv 2404.08892, 100k 合成对）用 latent diffusion 生成 post-change 图像 + change map，绕过人工标注瓶颈。**合成数据 + VLM 指令生成**（用 GPT-4V 描述合成对）是构造「四合一」数据集的可行路径。

## 三、bi-temporal 与 multi-temporal（>2 时相）之间的 gap

### 现状：所有 §28 五篇都是 bi-temporal 或松散时序

- TEOChat：支持任意长度序列，但训练数据仍以 bi-temporal 为主，3+ 时相样本稀少。
- ChangeChat / DeltaVLM：严格 bi-temporal。
- TAMMs：支持 3–5 帧 SITS，但 benchmark 有限。
- ViLaCD-R1：严格 bi-temporal。

### 为什么 multi-temporal 这么难

**（1）标注成本非线性增长**：bi-temporal 只需标「哪里变了」；multi-temporal 需标「何时开始变 / 何时结束 / 中间过程」——这是从「二值 mask」升到「时空轨迹」的维度跃迁。

**（2）架构不对称**：bi-temporal 只需看 delta；multi-temporal 要求模型在时间方向同时能做 detection（某时刻发生变化）、tracking（变化的连续过程）、forecasting（预测未来变化）。这需要不同的归纳偏置，难以用单一架构统一。

**（3）计算成本**：T 个时相，ViT token 数 × T，LLM 的 context length 成为瓶颈。即使 Qwen2.5-VL 的 128k context，也只能处理约 20–30 个 HD 时相。

### 可行的路径

RS 领域有大量 SITS 数据（Sentinel-1/2 每 5 天重访，十年级连续覆盖全球）但无 VLM 适配。**时间维度的 token 压缩**（如 Qwen2.5-VL 的 3D 动态分辨率）+ **稀疏时间采样**（只选变化显著的时相）+ **时序总结 token**（把一段连续无变化的时相压缩为一个 token）——这三件事的组合是通往 multi-temporal RS-VLM 的现实工程路径。**这是 §28 第三条残差**。

---

## 四、从变化 mask 到矢量多边形的时序一致性问题

### §27 残差 1 在时序上的加倍困难

§27 指出单时相 VLM 产出的像素 mask 缺乏拓扑一致性，需要 raster-to-vector 后处理。在时序上这个问题被加倍：

**（1）同一地物在不同时相的矢量多边形应保持拓扑稳定**：一栋建筑在 t1 是矩形 4 顶点，在 t2 扩建后应是 6 顶点，不应变成 50 顶点锯齿形。

**（2）变化多边形应是「t1 与 t2 多边形的集合差」**：变化区域 = (t2 的新增区域 ∪ t1 的拆除区域) 而非两张独立 mask 的像素差。

**（3）变化多边形的时间属性**：需要标注「何时出现 / 何时消失」，而非仅空间位置。

### 研究空白

PolyBuilding / PolyFootNet / HiSup 等矢量多边形方法只处理单时相。TAMMs 做时序 forecasting 但输出是图像而非矢量。**目前没有任何工作把「VLM + 时序 + 矢量多边形」三者组合**。

### 可行的开路方案

借 GeoGround 的 Text-Mask 技术，把矢量多边形编码为顶点序列 token（形如 "P1:(x,y);P2:(x,y);...;Pn:(x,y)"），让 VLM 自回归生成。对时序问题，模型输出「t1 多边形序列 ; t2 多边形序列 ; 变化类型：新增 / 扩建 / 拆除」这样的结构化序列。拓扑约束通过 GGL 式一致性损失夹（首尾闭合、顶点间无自交、t1 与 t2 多边形的拓扑关系合理）。**这是 §28 第四条残差**，直接汇入 IV 生成对齐章。

---

## 五、§28 的独特位置 —— 阶段 III 的时间维度闭环

### 与 §27 的关系

§27 写的是「空间维度上的 RS-VLM 前沿」——从 bbox 到像素 mask 的产出形式演化。§28 写的是「时间维度上的 RS-VLM 前沿」——从单时相到多时相的推理能力演化。两者合起来画出 RS-VLM 的完整二维坐标：

```
              产出形式（空间）
                ↑
                │
矢量多边形 │    ?        ?          ?         ?
                │  PolyBuilding  ?         ?         ?
像素 mask │   GeoPixel    ?         ViLaCD-R1 ?
                │  GeoGround   ?         TAMMs    ?
bbox            │  GeoChat    TEOChat   ChangeChat ?
                │  LHRS-Bot   DeltaVLM
文本            │  RSICC      LEVIR-CC  TEOChat   ?
                │
                └──────────────────────────→ 时间维度
                   单时相    双时相    3–5 时相   多时相 SITS
```

**坐标系的右上角——多时相 SITS × 矢量多边形——是完全空白的**。这是 §28 残差图里最大的空白区域，也是你研究最有机会建立长期护城河的方向。

### §28 对你研究方向的定位

你的研究三角「推理 + 空间定位 + 图斑产出」在 §28 的语境下变形为「**时序推理 + 变化定位 + 时序图斑产出**」。三条新边：

- **时序推理 ↔ 变化定位**：评测在 >2 时相条件下，模型是否能正确识别变化的起止时间和位置。当前 benchmark（LEVIR-CD 等）只有 bi-temporal，无法评测这一条。
- **变化定位 ↔ 时序图斑产出**：像素级变化 mask 到时序矢量多边形的转换。需要你构造自己的 benchmark。
- **时序推理 ↔ 时序图斑产出**：推理链加长（对变化的多步解释）时图斑产出精度是否保持。这是 §26 残差 1 在时序上的具体化。

## 小结：§28 的谱系内逻辑

- **TEOChat**（Stanford, ICLR 2025）：首个时序 EO VLM。TEOChatlas 554k 指令；时间共享 ViT + LLaMA 2；LoRA + 8bit 低成本训练。起点位置。
- **ChangeChat**（ICASSP 2025）：首个 bi-temporal VLM。ChangeChat-87k；Vicuna-based；CoT 变化分析策略。从通用时序收窄到双时相。
- **DeltaVLM**（北外, MDPI RS 2025）：交互式差异感知。ChangeChat-105k；CSRM + 指令引导 Q-former；frozen LLM + 只调视觉。把 VQA 升级为多轮对话 agent。
- **TAMMs**（2025/06）：变化理解 + 未来预测联合。TAM + SFCI + ControlNet；Temporal Consistency Score 度量。把时序 forecasting 第一次纳入 VLM 框架。
- **ViLaCD-R1**（2025/12）：首个端到端 VLM → 像素变化 mask。MIR（SFT + RL） + MGD；LEVIR-CD / SYSU-CD / WHU-CD SOTA。打通最后一步。

**五篇合起来画出三条主线**：

- **指令数据扩展**：TEOChatlas 554k → ChangeChat-87k → ChangeChat-105k → 未来 10M+ SITS 指令数据
- **架构维度加深**：时间共享 ViT → 差异感知 + Q-former → 时间适配器 + ControlNet → VLM + RL + Mask Decoder
- **产出形式升级**：文本描述 → bbox localization → 未来帧生成 → 像素级变化 mask → **矢量多边形 + 时间属性（空白）**

**你研究方向的坐标**：第三条主线的末端 + 第二条主线的进一步深化。具体说就是：在 ViLaCD-R1 两阶段 MIR + MGD 的基础上加 TAMMs 的 TAM，把 bi-temporal 扩到 3–5 时相；或在 GeoPixel 的像素 mask 基础上加 GeoGround 的 Text-Mask 编码，让模型输出时序矢量多边形序列。这两个方向都是 2026 前沿 + 顶刊级。

---

## 本组合力的残差（汇入 V 与 IV）

1. **时间方向的原生 attention 架构尚不存在**。TEOChat 用时间共享 ViT + LLM 自回归；DeltaVLM 只处理 bi-temporal；TAMMs 的 TAM 是最接近的，但表达能力有限。真正需要的是把 Qwen2.5-VL 的 MRoPE 从 3D（文本 + 高 + 宽）扩到 4D（文本 + 时间 + 高 + 宽）或 5D（再加传感器通道），并在大规模 SITS 数据上预训练。这条残差 **汇入 V 反身度量**——需要新的 benchmark 评测长时序推理能力。
2. **时间序列长度 >2（SITS 多日期序列）的研究几近空白**。所有 §28 五篇主力都停留在 bi-temporal 或 3–5 时相。真实 RS 业务是 Sentinel-2 每 5 天重访 × 十年，数据量巨大但无 VLM 处理。通往 multi-temporal 的路径是：时间维度 token 压缩 + 稀疏采样 + 时序总结 token。这条残差 **汇入 V 反身度量**——SITS 评测度量体系需要重新设计。
3. **语言-时间-图斑三位一体 benchmark 缺失**。LEVIR-CD 有 mask 无语言；LEVIR-CC 有语言无 mask；ChangeChat-105k 有指令对话但 mask 覆盖不全；TEOChatlas 有时序但无像素 mask。**需要一个「双时相图 + 自由语言指令 + 像素级变化 mask + 多轮对话」四合一数据集**——构造这个数据集本身就是一篇基础工作。这条残差 **汇入 V 反身度量**。
4. **矢量多边形 + 时间的联合生成为零**。PolyBuilding 等只做单时相矢量化；TAMMs 做时序但输出图像；GeoGround 做统一几何但只单时相。让 VLM 原生输出「t1 矢量多边形序列 ; t2 矢量多边形序列 ; 变化类型」这种结构化时序输出，拓扑约束通过 GGL 式一致性损失夹——这是完全空白的方向。这条残差 **汇入 IV 生成对齐**。
5. **多传感器多时相（SAR 时序 + 光学时序）的 VLM 摄入与对齐无方法**。EarthGPT / EarthDial 在单时相做了多传感器适配，TAMMs 在光学时序做了预测，但「SAR 时序 + 光学时序」的联合 VLM 摄入完全空白。SAR 的复数数据、多极化、相干性 —— 这些都需要新的视觉编码器架构。这条残差 **汇入 IV 生成对齐**——为多传感器多时相设计生成对齐的目标函数。

## 补充：你可以立即着手的三个实验方向

**实验 1 · 构造「四合一」双时相 RS-VLM benchmark**：以 LEVIR-CD（双时相图 + 像素级 mask）为基础，用 GPT-4V 或 Qwen2.5-VL 对每对图 + mask 生成自由语言描述（「新建了 3 栋建筑在东北角」）和多轮对话（3–5 轮围绕变化展开问答）。规模目标：20k 对双时相图 × 5 轮对话 = 100k 对话样本。**潜在发表**：ISPRS / TGRS / NeurIPS D&B。这一 benchmark 一旦建立，后续所有 §28 第 3 条残差相关的工作都要引用它——具有长期影响力。

**实验 2 · TAMMs 的 TAM + ViLaCD-R1 的 MIR + MGD 三件套组合**：以 InternVL3 或 Qwen2.5-VL 为基座，插入 TAM 模块处理 3–5 时相序列，然后用 MIR（SFT + RL）做块级变化判断，最后 MGD 精化到像素 mask。数据用 SECOND + HRSCD（语义变化）+ 你的「四合一」benchmark。**潜在发表**：CVPR / ICCV / ISPRS。这是一个工程量适中但技术新颖的组合工作。

**实验 3 · VLM 原生输出时序矢量多边形**：以 GeoGround 为基座（Text-Mask 技术），把矢量多边形编码为顶点序列 token「P1:(x,y);...;Pn:(x,y)」。模型输入双时相图 + 文本指令，输出「t1 多边形 ; t2 多边形 ; 变化类型」。拓扑约束：首尾闭合损失 + 无自交损失 + 时序一致性损失（t1 与 t2 同一地物的顶点数不能相差 >2 倍，除非拆除 / 新建）。数据用 PolyBuilding 的单时相数据 + 你自构造的双时相矢量变化标注。**潜在发表**：CVPR / ICCV / ISPRS / TGRS。这是最有原创性的方向，直接填补 §28 第 4 条残差的空白。

这三个实验按原创性递增、工程量递增，但都在你的研究方向上，且每一个都直接对应本章标注的一条残差。实验 1 是基础工作（必须先做），实验 2 是组合工作（快速出结果），实验 3 是原创工作（护城河）。理想路径是实验 1 同步启动，实验 2 用实验 1 的初步数据先出一篇，实验 3 作为长线研究。
