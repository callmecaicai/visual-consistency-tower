# §27 · 机制展开：四张账单的多根因分析与两条正交出口

§27 不再把阶段 III 的账单压成 caption bias 单根因。caption bias 很重要，但它只是多根因结构中的第一层。阶段 III 的问题是：视觉进入公共语义契约后，语义主权仍由语言制度掌握。

---

## 一、四张账单

| 账单 | 暴露什么 | 不是简单什么 |
|---|---|---|
| MMVP / CLIP-blindness | caption 没命名、全图对齐压扁的视觉差异 | 不是单个 benchmark bug |
| MMStar / no-image baseline | 很多题目可被语言先验答出 | 不是模型“真的看懂” |
| 62% 感知故障 | 被标成推理错的失败，常常先是看错 | 不是只需更长 CoT |
| Look Light, Think Heavy | 推理越长，视觉证据可能越被语言链稀释 | 不是单纯 attention 可视化趣闻 |

这四张账单共同说明：语言共形能扩大公共语义半径，也能遮蔽视觉闭合不足。

### 账单证据等级

| 账单 | 证据等级 | 当前风险 |
|---|---|---|
| MMVP / CLIP-blindness | 较强：具体 benchmark 与类别盲区明确 | 需要区分 CLIP 盲区与 VLM 继承关系 |
| MMStar / no-image baseline | 强：适合揭露 benchmark shortcuts | 不能推出所有 VLM 都不看图 |
| 62% 感知故障 | 中-强：取决于标注协议与错误分类 | 需要说明来源、样本和分类标准 |
| Look Light, Think Heavy | 中：机制启发强 | attention 衰减不等同于因果证明 |

### 账单验证卡

| 账单 | 根因候选 | 阶段残差 | 缓解方式 | 证伪方式 |
|---|---|---|---|---|
| MMVP | caption 稀疏 + global alignment bottleneck | 未命名视觉被压扁 | 细粒度视觉塔、反例数据、局部监督 | 模型在同类 adversarial pairs 上稳定区分方向、计数、状态 |
| MMStar | benchmark language shortcuts + LLM prior takeover | 能答不等于看见 | no-image control、视觉必要性筛题 | 去图性能接近随机，且扰动图像会改变答案 |
| 感知故障 | visual evidence bandwidth 不足 | 推理链建立在错误感知上 | 强视觉塔、证据回指、grounding audit | 失败样本中感知错误占比显著下降 |
| Look Light | reasoning-time visual decay | 思考越长越不看图 | re-grounding、visual memory、stepwise verification | 多步推理中视觉证据调用持续且与答案因果相关 |

## 二、多根因结构

```text
caption distribution bias
  -> 文本不描述的属性难以保留

global alignment bottleneck
  -> 全图向量压扁空间、计数、方向、状态

connector / token bottleneck
  -> 视觉证据进入 LLM 前被压缩

LLM prior takeover
  -> 语言后端用世界知识和文本模式补全视觉证据

benchmark language shortcuts
  -> 题目本身可被语言先验解出

reasoning-time visual decay
  -> 多步文本推理中视觉证据调用下降
```

单独说“caption 偏见”不够。MMVP 更多是 caption bias + global bottleneck；MMStar 更多是 benchmark shortcuts + LLM prior takeover；62% 感知故障暴露视觉证据带宽；Look Light Think Heavy 暴露推理时视觉衰减。

---

## 三、范式内缓解

| 方向 | 能做什么 | 边界 |
|---|---|---|
| 换视觉塔 | 用 SigLIP / DINO / stronger ViT 改善感知与 dense feature | 不能自动给视觉语义主权 |
| 换 connector | Q-Former、Resampler、visual expert、memory 增加视觉带宽 | 成本、训练复杂度和上下文压力上升 |
| 换 LLM | 改善推理、表达、知识和指令遵循 | 可能更强地接管视觉证据 |
| 换数据 | 反例、细粒度 caption、视觉困难题降低伪视觉 | 仍在语言制度内塑形 |
| 换评测 | 设计 vision-centric benchmark 和 no-image control | 反映问题，但不直接解决表征 |

这些不是无用补丁。它们可以显著提高系统能力。但只要系统仍以语言后端作为语义主权中心，视觉就仍处于共享而非自主状态。

---

## 四、两条正交出口

阶段 III 的出口不是“生成和自监督不兼容”。更稳的结构是：

| 出口 | 处理的残差 | 后继阶段 |
|---|---|---|
| 外显生产 | 视觉可被语言调用，但不能反向生产观测 | 阶段 IV |
| 内部持有 | 视觉进入公共语义，但不自持对象、几何、遮挡、对应和状态 | 阶段 V |

阶段 IV 和阶段 V 是一外一内。生成能力回答“能否把内部状态外化成观测”；状态持有回答“能否不依赖语言而持有视觉世界”。未来强系统很可能同时需要这两种能力。

---

## 五、阶段 III 的自我否定

阶段 III 被自己的成功否定：

1. 命名越成功，未命名视觉越被压扁。
2. 指代越成功，越暴露指到某处不等于持有状态。
3. 调用越成功，越暴露接口灵活不等于视觉主体性。

因此，阶段 III 的失败不是语言不够强，而是语言太强：它能替视觉完成公共语义工作，也能替视觉掩盖自己仍未闭合的部分。
