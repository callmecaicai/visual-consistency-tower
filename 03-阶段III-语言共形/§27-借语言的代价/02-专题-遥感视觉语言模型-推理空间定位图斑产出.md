**主题**：遥感成像自带四大域偏移（垂直视角 / 多尺度共存 / 小目标密布 / 基底时变），当通用 VLM 被当作出发点时，这四条都会打出符。更关键是这条产业链的输出不是对话而是「可进 GIS 的图斑」——这对 VLM 的产出形式又加了三重约束：从对话空间投射到像素空间的回映精度、像素 mask 到矢量多边形的拓扑一致性、多个实例的点-块关系和属性保持。五篇合起来画出了 2024–2025 两年 RS-VLM 的技术链曲线，并把你研究的核心技术张力点（推理 ↔ 空间定位 ↔ 图斑产出）的三条边全部暴露出来。

## 1 · GeoChat —— *arxiv 2311.15826 / CVPR 2024* (Kuckreja 等, MBZUAI)

**共形贡献**：首个面向遥感的 grounded 大视觉语言模型。从通用 VLM 迁到 RS 域的三件关键事：

**数据**：利用现有 RS 数据集（LRBEN / NWPU-RESISC45 / SAMRS / FloodNet 等）构造了约 318k 条 RS instruction-following 数据。包含：图像级 / 区域级对话、视觉接地（visually grounded conversations）、referring object detection、VQA、scene classification。

**架构**：基于 LLaVA-1.5，保持 ViT + MLP + Vicuna 的基架，但引入两项 RS 特化：「**任务 token**」（[grounding] / [vqa] / [caption] / [refer]）指示模型切换输出形式；「**空间位置表示**」把 bbox 用特殊 token 序列编码进语言空间（形如 {<x1><y1><x2><y2>|<angle>}）。这让 VLM 能生成坐标（但仅仅是 bbox 级别的、且是水平框或带角度的 OBB）。

**关键结果**：在 DIOR-RSVG（指称表达式 bbox grounding benchmark）上，GeoChat 的 accuracy@0.5 为 **31.4%**（LHRS-Bot-Nova 在同 benchmark 上后来达 73.5%，Falcon 达 56.9%，VHM 55.9%，EarthDial 46.1%）。**GeoChat 的绝对数据不高，但它建立了评测框架**：后来所有 RS-VLM 都在这套 benchmark 上较其位置。

**局限**：GeoChat 只支持盒级输出。对你的「图斑产出」目标而言，它是起点但不是终点——它把问题从「能不能」推进到「精度不够高」。

🔗 [https://arxiv.org/abs/2311.15826](https://arxiv.org/abs/2311.15826)

## 2 · LHRS-Bot-Nova —— *ISPRS Journal 2025 (Li 等, 南京大学 + TUM)*

**共形贡献**：国产高质量 RS-VLM。其前身 LHRS-Bot（ECCV 2024, arxiv 2402.02544）首次利用 **VGI（Volunteered Geographic Information，OpenStreetMap）** 大规模自动构造 RS 图文对齐数据集 LHRS-Align，一举把 RS 图文训练数据从点 10⁶ 量级推到点 10⁷。Nova 版本（ISPRS 2025）基于 feature-guided image recaptioning 重新生成高质量描述，并专门设计空间识别指令数据来提升「**空间认知**」能力（bbox 定位 / 方位理解 / 成对相对位置）。

**三件事的意义**：

（1）**VGI 是 RS-VLM 的非平衡资源**。OpenStreetMap 有十亿级地理实体 + 属性描述，与全球可得的 Sentinel-2 / 高分共形，理论上几乎是 RS 版的「Common Crawl」。

（2）**空间认知 ≠ 视觉认知**。Nova 的 instruction 数据里专门构造了形如「水域在建筑的什么方位」「哪些车辆坐标在 [x, y] 左侧 100m 内」这种任务，RS-VLM 在这种细粒度空间关系上往往比在粗题级 VQA 上差很多。

（3）**会告诉模型「不知道」（Hnst 指令）**。Nova 同时强调模型不应针对「图中哪里有 X」但图里实无 X 的提问给出肯定回答——这与 VHM（AAAI 2025, arxiv 2403.20213）的 HnstD 数据集理念同源，是 §24 末「幻觉抵抗」在 RS 域的直接响应。

**关键数据**：DIOR-RSVG acc@0.5 = **73.5%**，在 2024–2025 的开源 RS-VLM 中位列前列。

🔗 [https://arxiv.org/abs/2411.09301](https://arxiv.org/abs/2411.09301) 、 [https://arxiv.org/abs/2402.02544](https://arxiv.org/abs/2402.02544)

## 3 · RMSIN + RRSIS-D —— *arxiv 2312.12470 / CVPR 2024* (Liu 等)

**共形贡献**：把「**指称表达式分割**（Referring Image Segmentation, RIS）」这一自然图像的任务正式引入遥感领域，并提供两件基础品：

**RRSIS-D 数据集**：**17,402 个 image-caption-mask 三元组**，是首个大规模 RS referring 分割 benchmark。数据基于 RSVGD，包含遥感常见的 20 类目标，词条表达式复射（从「最左侧那台飞机」到「距主跑道南部最近的建筑」等空间关系表达）。RRSIS-D 在 §27 的作用类比于 RefCOCO 在自然图像 RIS 里的作用。

**RMSIN 架构**：针对 RS 图像的四种艰难定制设计了三件事：

- **多尺度共存**：同一图中车辆（~20 px）与飞机跑道（~2000 px）一起存在 → Intra-scale Interaction Module（IIM，在单尺度内精确处理）+ Cross-scale Interaction Module（CIM，跨尺度信息整合）。
- **多方位**：RS 目标任意朝向，不像自然图像中人 / 车多竖直 → 旋转卷积（Rotated Convolution）换光获取旋转不变性。
- **小目标稀散**：传统 RIS 在 RefSegRS 上表现差 → RMSIN 通过 IIM 在细尺度上充分保留细节。

**关键结果**：RRSIS-D（testA），RMSIN mIoU = 64.20%，oIoU = 74.66%，较当时传统 RIS 方法（LAVT 等）提升约 10 个点。

**后续演进**：RS2-SAM2（arxiv 2503.07266, 2025/03）：用 SAM2 作为 mask decoder，RS 特化的提示生成器挤压文本表达式的理解效率——这一方向直接用 SAM2 的分割先验来卡 referring 分割的高质量 mask。

**给你的概念使用模板**：RMSIN 和 RS2-SAM2 等代表的是 **非 VLM 的专用架构**——输入图像 + 文本，输出 mask，不做对话。这条路线精度更高，但缺乏推理、多轮对话、多任务能力。这与下面的 GeoPixel / GeoGround 路线形成对立。

🔗 [https://arxiv.org/abs/2312.12470](https://arxiv.org/abs/2312.12470)

## 4 · GeoPixel —— *arxiv 2501.13925 / ICML 2025* (Shabbir 等, MBZUAI)

**共形贡献**：**第一个端到端的高分辨率 RS-LMM，支持像素级 grounding**。输入分辨率支持 4K HD 任意长宽比；输出以 **交错式 mask 生成**（interleaved mask generation）的形式嵌入对话流——也就是说可以在一段对话里「这边有 <mask 1>，那边有 <mask 2>，两者的距离关系是……」这样混排输出文字和图斑。

**关键设计**：

（1）**高分辨率的多片覆盖（multi-tile）**：将 4K 图切为 448 × 448 的 tiles，每个 tile 过 ViT，结果拼回；类似 InternVL 的模式，但针对 RS 的极高长宽比特殊处理。

（2）**GeoPixelD 数据集**：通过「set-of-marks prompting + 空间先验」的自动化管道构造了 RS 特化的 grounded conversation 数据。成本低、覆盖面广。

（3）**Mask tokens 输出**：模型词表扩展了 <mask_start> <mask_end> 特殊 token，语言模型生成的 mask segment 或由分割头解码成二值 mask。**这稍有制度施上类似于 GLaMM（2024）在自然图像上的做法**，但针对 RS 场景的小目标 / 多实例 / 多产出做了打磨。

**关键结果**：在 GCG（Grounded Conversation Generation）标准 benchmark 上，单目标和多目标分割都超过 GLaMM / LISA 等通用域像素级 LMM；在 RRSIS-D 上 mIoU 超过 RMSIN 方类。

**给你的概念使用模板**：GeoPixel 是「**对话 + 推理 + 图斑**」三件事真正端到端绑定的第一个 RS-VLM。你的研究方向最对应的参考框架，但它的工程较重，训练成本高，对多目标实例的服务仍有 mask 计数不一致的残差。

🔗 [https://arxiv.org/abs/2501.13925](https://arxiv.org/abs/2501.13925)

## 5 · GeoGround —— *arxiv 2411.11904* (Zhou 等, 上海交大 VisionXLab, 2024/11)

**共形贡献**：统一 HBB（水平框）+ OBB（旋转框）+ mask 三种几何输出的 RS 视觉定位 VLM。**不定制架构**，而是用「Text-Mask」技术让通用 VLM 原生输出像素级分割：把 mask 表达为语言模型可生成的词序列（正形 mask 编码、或边界组点编码），将分割问题化约为文本生成问题。

**关键设计**：

（1）**提示辅助学习（Prompt-Assisted Learning, PAL）**：用 HBB 作为弱提示辅助 mask 预测。即模型先学「给定 bbox 识别一个 mask」，再学「直接从文本推到 mask」。

（2）**几何指导学习（Geometry-Guided Learning, GGL）**：一致性约束——同一目标的 HBB / OBB / mask 在互一致的前提下训练（HBB 应包含 OBB 包含 mask）。这一约束直接夹在损失层面。

**与 GeoPixel 的区别**：

- **GeoPixel**：架构定制（专门的 mask 解码头），输出是真像素 mask。
- **GeoGround**：不改架构，输出通过文本序列编码 mask，精度靠 text 模型划一划，但更而长的是**同一模型输出三种几何形式而不需要多头**。

**这对你的实战意义**：你并不总是要像素级 mask；有的时候只要 bbox + 方向，有时候要矢量图斑，有时候要粗片 mask。GeoGround 的统一显式把「**产出形式的选择**」交给推时解码——从模型设计第一性原则看更 Notion native。

🔗 [https://arxiv.org/abs/2411.11904](https://arxiv.org/abs/2411.11904)

# 关于 §27 五篇的机制追问

## 一、为什么通用 VLM 在遥感上失效 —— 四大域偏移

### 域偏移 1 · 垂直视角

VLM 的预训练图片（LAION / WebImg）中 95%+ 是地面水平视角。人、车、建筑、动物大部分以竖直轴为主的视觉结构进入 ViT。遥感是**正上方垂直视角**：所有目标从背谈出现，无天空地面关系。**视觉编码器尽管在文本上读懂「一张建筑的图」，但组成特征时依然是地面视角的主纹 prior**。GeoChat / LHRS-Bot 等 RS-VLM 必须通过大规模 RS 指令数据微调去「覆盖」这一先验。

### 域偏移 2 · 多尺度共存

一张高分辨率 RS 图里，一个车（~20 px）与一个飞行跑道（~2000 px）同时出现。自然图像中的目标尺度跨度很少这么大。**ViT 的固定 patch size（如 14）对小车而言只挺个 patch、对飞行跑道而言 > 1000 个 patch**——两者限制下尺度不匹配。MM1 发现 A（分辨率 > connector 架构）在 RS 上的意义比自然图像更大。

### 域偏移 3 · 小目标稀散

遥感中**一张图可能有上百个小目标**（一片荣地的车辆、渔群、油罐、风台短候），而自然图像 VLM 预训练每张图往往 1–5 个主体。**默认 ViT 的 attention 分配机制是为少数大目标设计的，针对密散小目标的场景实时如毛**。这是 RMSIN 和 RS2-SAM2 要专门设计 IIM / CIM 的原因。

### 域偏移 4 · 多光谱 / 多传感器 / 多时相

RS 有可见光 RGB、近红外、短波红外、SAR 雷达、LiDAR、红外、多时相序列——**通用 VLM 的 ViT 是 3 通道的**。大部分 RS-VLM（包含 GeoChat / LHRS-Bot / GeoPixel）还停留在 RGB 处理。EarthGPT（TGRS 2024）等尝试扩展到光学 + SAR + 红外，但仍是改改模式适配而非元生式摄入。EarthDial（CVPR 2025）对多传感器交互式对话做了一些尝试。

### 四条域偏移的合成后果

**通用 VLM 直接放到 RS 场景上，accuracy 大约会从自然图像的 70%+ 扫平后扫平到 15–30%**。这不是 RS 任务本身难，而是通用 VLM 的预训练 prior 与 RS 领域的不匹配。整个 RS-VLM 研究就是从这四条域偏移一维维约束回来的过程。

## 二、GeoChat 的后续改进的三条路线

GeoChat（CVPR 2024）的 31.4% DIOR-RSVG 的结果有点是一个「结因结果」。后续所有 RS-VLM 都在三条路线上改进。

### 路线 1 · 数据规模化

- GeoChat：318k 指令数据
- LHRS-Bot：**LHRS-Align 数据集 10.5M 图文对（VGI 驱动）**，指令集 390k
- LHRS-Bot-Nova：再扩展 5M 图文对，feature-guided recaptioning 后质量提升
- SkyEyeGPT：968k 指令数据
- Falcon（2025）：7M+ RS 指令数据

**关键洞见**：RS 图文配对数据的瓶颈问题不是图（Google Earth / Bing / Sentinel / 高分系列免费），而是**对齐的文**——**VGI（OSM）是唯一底料充足的天然标签源**。Nova 的 ISPRS 论文将 VGI 的作用推到达极致。

### 路线 2 · 在 grounding 格式上推进

- GeoChat：HBB 与 OBB（包含角度 token）
- GeoGround：统一 HBB + OBB + mask
- GeoPixel：直接交错式 mask
- RMSIN / RS2-SAM2：专用分割架构 mask

**演化方向**：从 bbox 逐步推向像素级 mask。你的「**图斑产出**」研究方向正好在这条曲线的末端。

### 路线 3 · 在认知 / 语义精度上推进

- GeoChat：通用指令追随
- SkySenseGPT：细粒度理解 + **场景图生成**（image-level scene graph generation）
- LHRS-Bot-Nova：空间认知专项指令
- VHM：「幻觉抵抗」（HnstD 数据集专门训「不知道」的能力）
- EarthDial：多传感器 + 多时相 + 交互式对话

这条线是**从「出片」级别拐进到「可信输出」级别**。

## 三、RRSIS / RRSIS-D 的四条特殊难点与其意义

### 难点 1 · 指称表达的空间性极强

自然图像的 RIS：「穿红衣的人」。RRSIS：「**左下方跑道南端附近的第二架飞机**」。空间关系描述成为指称表达的主构成，模型必须同时懂：坐标系（东/西/南/北）、拓扑关系（跑道的南端）、成个关系（第二架）。这直接评测的是 VLM 的 §25 末残差「空间推理独立性」。

### 难点 2 · 多尺度的 mask 生成

动车（~20 × 50 px）与飞行跑道（~2000 × 50 px）的 mask 需求池质量很不一样。传统 RIS（如 LAVT）用单尺度的 decoder 聊对两者皆不理想。RMSIN 的 IIM + CIM 就是为此设计。

### 难点 3 · 旋转不变性

RS 目标朝向任意，而自然图像中人多竖、车多平。RMSIN 的旋转卷积与 OBB 表达是因应。但也有更深的问题：**VLM 的 RoPE / 位置编码是为水平文本设计的，视觉 patch 的 2D 位置编码缺失旋转等变性**——Qwen2.5-VL 的 MRoPE 没解决这一点，即使是 InternVL3 的 V2PE 也不解决。

### 难点 4 · 指称属性的细粒度

「朱红色屋顶的最南侧建筑」：颜色（朱红）、部位（屋顶）、位置（最南侧）、类别（建筑）四种信息合成的指称。图像规范化色调（不同传感器 / 光照 / 季节下色调差异）会导致「朱红色」被误判。**色调 robustness 在 RS 中重要度远超自然图像域**。

### RRSIS 的研究意义

把一条 referring segmentation 任务分解为三个按顺序的步骤：

（1）指称解析（语义：亮红色 + 屋顶 + 建筑 + 最南侧）

（2）空间心理化（定位到图里相对位置）

（3）像素圈定（mask 生成）

这三个步骤中第 1 步对应 LLM 语言理解、第 2 步对应 §25 末残差里的空间推理独立性、第 3 步对应像素级视觉产出。**RRSIS 的精度差实际上可以反向归因到三步中的哪一步——这为你的研究提供了一个天然的诊断框架**。

## 四、GeoPixel vs GeoGround —— 两条图斑产出路线的对比

### 架构路线

- **GeoPixel**：VLM + 专用 mask decoder（类 GLaMM / LISA 模式）。词表扩展 <mask_start> / <mask_end>；LLM 的 hidden state 投入 mask head 生成二值 mask。精度高，但训练成本重、架构不通用。
- **GeoGround**：VLM 不改架构，mask 编码为文本 token 序列（比如边界多边形的顶点序列）。模型通过文本自回归直接「写」出 mask。精度受 LLM 的 token 生成准确度限制，但架构轻、泛化好、同一模型支持 HBB / OBB / mask 多种输出。

### 精度 vs 灵活性权衡

这个权衡与 §22 / §23 的「融合式理解 vs 挂接口」对立在表现层面同构：

- GeoPixel 路线 = 专用重架构（类似 BLIP / CoCa 的融合式理解），精度高，不灵活
- GeoGround 路线 = VLM 原生输出（类似 LLaVA / MiniGPT-4 的挂接口），灵活，精度受上游约束

### 你研究的实战意义

- 如果你的下游纯是「给我个 mask 去 GIS 矢量化」——GeoPixel 类是对的
- 如果你的下游是「对话式地做图斑产出 + 对话式地对图斑做属性问答 + 对话式地修改图斑」——GeoGround 类的灵活性胜出
- 在两者间还有一类 **新路线**（尚未成熟）：VLM 输出**矢量多边形的顶点序列**（PolyBuilding / PolyFootNet 类），直接绕过 mask 到矢量的转换。这一方向 2025–2026 正在起步，与你研究方向的未来最对齐。

### 图斑产出的拓扑一致性残差

无论 GeoPixel 还是 GeoGround，都输出**像素 mask**——但 GIS 需要的是**矢量多边形**（vertex-edge-face 结构，拓扑闭合、无自交、规则）。像素 mask 到矢量多边形的转换（raster-to-vector）会丢失拓扑信息：锯齿化边界、不闭合、自交、多连通。**VLM 直接产出矢量多边形是 2026 RS-VLM 的真正前沿**，但目前仍无成熟方案。PolyBuilding (arxiv 2211.02506)、PolyWorld、HiSup 等在非 VLM 框架下有进展；PolyFootNet（2025 IGARSS）在 off-nadir 场景提出 Self Offset Attention；但与 VLM 结合仍空白。

## 五、§27 的独特位置 —— 阶段 III 的反身应用

### 阶段 III 的回路闭合

前 §21–§26 讲的是「如何让模型懂图 + 懂文 + 推理」：

- §21 外挂的工程极限：冻结 detector + BERT 式融合暴露前史天花板
- §22 CLIP：对比对齐建立语义桥
- §23 语言锚定稠密任务：开放词汇进入检测/分割/深度
- §24 统一多模态骨干：对比、caption、VQA、grounding 统一承接
- §25 VLM 的诞生：LLM 成为语义后端
- §26 语言条件可提示视觉：语言指令回到像素级动作

§27 是把这些全部拉出来在一个最苛刻的应用领域接受检验：**RS 的四大域偏移 + 工业化图斑产出要求**。前面所有章节的残差在 §27 这里被放大为具体的工程信号。

### 三条"阶段 III 残差"在 §27 的放大

- **§21 残差（对比对齐 ≠ 细粒度）** 在 RS 上放大为：RemoteCLIP 等模型即使在 RS 图文对齐上做了微调，对「朱红色屋顶」等细粒度属性仍无高精度识别。
- **§24 残差（视觉推理 = LLM 文本推理 + 初始条件）** 在 RS 上放大为：RRSIS 的复杂空间指称要求模型真的在空间里推理，但多数 RS-VLM 只是用文本串出「最南侧」→「预测一个 bbox」的模板，不真正具有方位计算能力。
- **§26 残差（推理链加长 → 视觉 grounding 脆弱）** 在 RS 上放大为：让 RS-VLM 做长链推理（比如「这几个建筑的变化判断」）时，后续步骤脱离对原图的 attention，产生固化于文本的错误推断。

### 你研究的三角张力

**推理 ↔ 空间定位 ↔ 图斑产出** 三角中的每一对边都对应一条技术挑战：

- **推理 ↔ 空间定位**：空间推理独立性（§25 残差 3 在 RS 上的具体形式），评测：复杂空间指称下的 grounding 精度
- **空间定位 ↔ 图斑产出**：像素 mask 的拓扑一致性与矢量化，评测：raster-to-vector 的边界锯齿、自交、IoU 稳定性
- **推理 ↔ 图斑产出**：推理链加长时 mask 产出精度是否衰减（§26 残差 1 在 RS 上的具体形式），评测：单轮 vs 多轮对话下的 mask IoU 曲线

**这三条边你都可以立实验检验**——每一条都是发顶刊论文的机会。

## 小结：§27 的谱系内逻辑

- **GeoChat**（MBZUAI, CVPR 2024）：RS-VLM 开山作。任务 token + 空间位置表示；bbox 级输出；31.4% DIOR-RSVG——基线位置。
- **LHRS-Bot-Nova**（南大 + TUM, ISPRS 2025）：数据规模化路线的代表。VGI 驱动；feature-guided recaptioning；空间认知指令；HnstD 幻觉抵抗；73.5% DIOR-RSVG。
- **RMSIN + RRSIS-D**（CVPR 2024）：referring 分割 benchmark 确立 + 专用架构。17,402 三元组；IIM + CIM + 旋转卷积应对 RS 四大难点。
- **GeoPixel**（MBZUAI, ICML 2025）：第一个端到端高分辨率 RS-LMM + 像素级 grounding。4K HD；交错式 mask 生成；对话 + 推理 + 图斑三合一。
- **GeoGround**（上交 VisionXLab, 2024）：统一 HBB + OBB + mask。Text-Mask 技术让 VLM 原生输出分割；PAL + GGL 一致性学习；同一模型多几何形式。

**五篇合起来画出两条主线**：

- 横线（数据 + 架构适配）：GeoChat → LHRS-Bot-Nova →（SkyEyeGPT / VHM / Falcon / EarthDial）
- 纵线（产出精度 + 形式）：GeoChat (bbox) → GeoGround (统一几何) → GeoPixel (像素 mask) → （未来：直接矢量多边形）

**你研究方向的坐标**：在纵线的末端 + 横线的交点处——需要 VLM 原生的图斑产出 + RS 域数据规模化 + 空间推理能力。这一交点是 2026 年 RS-VLM 最值得投入的前沿点之一。

---

## 本组合力的残差（汇入 V 与 IV）

1. **VLM 产出的序列化 mask 缺乏拓扑一致性**。GeoPixel / GeoGround 输出的 mask 是像素级的二值矩阵，但 GIS 需要的是矢量多边形（vertex-edge-face 结构，拓扑闭合、无自交、简化规则）。从像素 mask 到矢量多边形（raster-to-vector）会丢失拓扑信息。**直接从 VLM 产出矢量多边形顶点序列是 2026 RS-VLM 的真前沿**——PolyBuilding / PolyFootNet 等在非 VLM 框架下有工作，但与 VLM 的结合尚空白。这条残差 **汇入 IV 生成对齐**——把「模型产出的几何形式」视作一种生成问题，与图像生成共享生成对齐的研究工具。
2. **多传感器（SAR / 红外 / 多光谱）仍未被 VLM 真正摄入**。当前绝大多数 RS-VLM 停留在 RGB。EarthGPT / EarthDial 做了多传感器适配，但本质上是多个 3 通道 RGB 的拼接。**把 SAR 的复数 L 波段、多光谱的 13 通道、LiDAR 的 3D 点云真正作为 VLM 的第一等输入**——还需要视觉编码器架构的原生改造。这条残差 **汇入 IV 生成对齐** 与 **V 反身度量**——前者需要为多传感器设计生成对齐的目标函数，后者需要为多传感器设计评测度量。
3. **多时相 / 变化推理仍无原生架构**。RS 的核心应用（土地利用变化 / 灾害监测 / 建筑新增）都是多时相问题：t1 和 t2 两个时间的两张图，描述 + 定位 + 产出变化区域的图斑。当前 RS-VLM 把多时相当作两张独立图并简单拼接——没有「时间方向」的原生 attention。**多时相 VLM + 变化图斑产出**是你最有机会建立护城河的子方向。这条残差 **汇入 V 反身度量**——评测多时相推理能力需要新的度量体系，当前 benchmark（LEVIR-CD 等）全是像素级监督，没有语言 + 时间 + 图斑三位一体的评测。

## 补充：你可以立即着手的三个实验方向

**实验 1 · §26 残差 1 在 RS 上的定量验证**：以 GeoPixel 或 GeoGround 为基座，在 RRSIS-D 上测「推理链加长 vs mask IoU」的曲线。假设：随推理 token 数增加，mask IoU 先升后降，拐点约在 token 数 100–200 附近。这一实验复现 Yan 等 2025（arxiv 2509.23322）的核心观察于 RS 场景。**潜在顶刊**：TGRS / ISPRS / CVPR。

**实验 2 · VLM 直接产出矢量多边形**：把 mask token 换成 vertex token，模型输出 [(x1,y1), (x2,y2), ...] 顶点序列。拓扑约束通过 GGL 式损失夹：要求序列首尾相连、无自交、顶点数量约束。对比对象：GeoPixel（像素 mask → raster-to-vector）vs 你的直接顶点输出。**潜在顶刊**：CVPR / ICCV / ISPRS。

**实验 3 · 多时相变化图斑产出**：输入 t1 图 + t2 图 + 文本指令「找出新增的建筑」；输出变化区域的图斑。基线：GeoPixel 对 t1/t2 分别产出 mask 后取差。你的方法：时间方向的 attention（类似 Qwen2.5-VL 的 MRoPE 但应用于时间轴）+ 变化 token 的原生生成。**数据**：LEVIR-CD + 你自构造的文本指令。**潜在顶刊**：ISPRS / TGRS。

这三个实验按难度递增，但都在你当前的研究方向上，且每一个都直接对应本章标注的一条残差。
