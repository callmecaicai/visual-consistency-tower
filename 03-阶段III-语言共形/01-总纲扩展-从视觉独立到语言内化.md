**本章位置**：阶段 II 把度量穿到了最后一个像素、最后一个实例，但第 81 类能被识别，是因为 CLIP 的对比预训练见过这个词——**语义是借来的**。阶段 III 的使命，是让语言从外挂变成**视觉骨干内部共同承载表征的一部分**。这一步不是「加了个文本分支」，是载体本身的扩张：从「像素流」变成「像素 + token 在同一条流里」。

## 一、从阶段 II 继承的三条残差（精确接力）

阶段 II 终章已经把稠密共形的极限写清楚：

1. **语义外包** —— 开放词汇挂载 CLIP / 语言模型的外部表示，视觉侧自己不产语义。
2. **结构化关系缺席** —— 两个实例之间的关系（「在上面」、「握着」、「导致」）仍不在稠密输出里。
3. **反身度量未触及** —— IoU / mAP / PQ / RMSE 外部给定，网络不能质疑或修订自己的度量。

阶段 III 的正面任务是第 1 条：**把语义的生产权从外部挂件收回视觉骨干**。第 2 条在阶段 III 内部被部分吸收（VLM 能"说出"关系，但不能稠密输出），真正的结构化出口留到 IV/V。第 3 条整条留到 V。

## 二、阶段 III 的本体论命题

🔷
**让语言从视觉系统之外的对齐挂件，变成视觉骨干内部共同承载表征的一部分。**

这句话有三重含义：

- **载体内化**：文本 token 与视觉 token 走同一条前向流，不再只在 loss 端碰头。语言不是外接的分类字典，是骨干的一部分。
- **主体性上升到刻度 2（共享式）**：系统对用户指令做持续响应——它不再是被动的检测器/分类器，但也不是自主体。它是对话中的「另一方」。
- **理解侧，不是生成侧**：阶段 III 的工作面是「能不能对 x 说出正确的 y」——能描述、能问答、能推理、能指令对齐。**能不能从零合成 x（GPT-4o 画图 / Chameleon）不属于本阶段，归 IV**。这条分界线是阶段 III 最容易被误读的地方，必须一开始就写明。

## 三、六维度上的具体位移

| 维度 | 阶段 II 的状态 | 阶段 III 的位移 |
| --- | --- | --- |
| 载体 | 单图单模态；语言仅作为外部分类字典 | 图文共形；语言进入骨干的前向流 |
| 主体性 | 无 / 准主体（SAM 式提示响应） | 共享式（刻度 2）；对话响应者 |
| 度量 | Dice / IoU / mAP / PQ / Hungarian | 对比式图文对齐（CLIP）+ 指令响应质量（LLaVA / VQA）+ 视觉问答 / 描述的文本度量 |
| 材料 | CNN / ViT + query | ViT 视觉编码 + LLM 骨干 + 跨模态适配层（Q-Former / Linear / Resampler） |
| 动力 | 稠密损失 + 二分匹配 | 对比预训练 + 指令微调（SFT）+ 偏好对齐（RLHF / DPO） |
| 时间尺度 | 单步前向 | 单步前向仍为主；开始出现「多轮对话」这一准时间轴 |

阶段 III 的核心移动发生在 **载体** 与 **主体性**；度量、材料、动力是跟随位移。真正把主体性推到刻度 3（生产者）的工作留给 IV；推到刻度 4（内转）的工作留给 V。

## 四、阶段 III 的内部接力线（残差自生下一组）

§21–§27 不是按年代或模型家族平铺，是按**残差传递**串起来的。每一节的入口都由上一节的残差形状决定。

1. **§21 外挂的工程极限**（ViLBERT / LXMERT / UNITER / VinVL / OSCAR）——冻结 detector + BERT 式融合把视觉锁死在二手 region feature 上。
    - 残差：视觉塔必须端到端参与训练，监督必须脱离 bbox，数据必须走 web-scale。
2. **§22 对齐即预训练：CLIP 与对比式对齐**（CLIP / ALIGN / LiT / SigLIP / EVA-CLIP / MetaCLIP）——视觉-语言第一次用大规模配对语料把两个表征拉到同一空间。
    - 残差：全图单向量丢空间结构，caption 偏见开始积累，语言友好空间不等于视觉友好空间。
3. **§23 语言锚定稠密任务**（GLIP / Grounding DINO / OWL-ViT / CAT-Seg / Depth Anything）——语言从全图相似度进入检测、分割、深度等稠密任务。
    - 残差：稠密任务能被语言打开，但每个任务仍有独立接口，视觉语义仍依赖外部文本锚。
4. **§24 统一多模态骨干**（CoCa / BLIP / BLIP-2 / Florence-2 / PaLI / OFA / Unified-IO）——对比、caption、VQA、grounding 由一个骨干同时承担。
    - 残差：骨干统一了，但任务能力仍强依赖训练混合比例与语言侧语义后端。
5. **§25 VLM 的诞生**（Frozen / Flamingo / Kosmos / LLaVA / InstructBLIP / Qwen-VL / InternVL / CogVLM）——LLM 成为语义后端，视觉通过适配层进入语言推理空间。
    - 残差：指令对齐做得好，但视觉细粒度容易被 LLM 语言先验淹没。
6. **§26 语言条件可提示视觉**（Grounded-SAM / SEEM / Ferret / Shikra / KOSMOS-2 / PixelLM / GLaMM / LISA）——语言指令开始回到像素级动作。
    - 残差：语言能调用视觉动作，但仍是单向调用，不能反向生产观测，也不能自持视觉状态。
7. **§27 借语言的代价**（MMVP / CLIP-blindness / 视觉细粒度崩塌）——收拢阶段 III 的理解侧到底留下了哪些看不清的东西，把账单明确移交。
    - **出口残差 A**：视觉可被语言锚定，但不能被反向合成 → 送给 IV（生成共形）。
    - **出口残差 B**：语义对齐仍依赖配对语料，系统的「懂」是被数据喂出来的 → 送给 V（表征共形）。

第 7 节的两条残差**直接定义阶段 IV 与阶段 V 的入口**。这是整本书第一次出现「一个阶段向两个后继阶段同时分叉」——阶段 III 是共形演进的岔路口。

## 五、阶段 III 的截止线

**做完了**：

- 载体从「单模态像素」扩张为「像素 + token 在同一条流里」——语言内化基本达成。
- 主体性从无 / 准主体推到刻度 2（共享式）——系统正式成为对话中的响应者。
- 指令对齐、视觉问答、视觉描述、视觉推理——理解侧的主要任务面被打开。

**没做（留给阶段 IV / V 与反身维度）**：

- **能生成像素的主体性（刻度 3）**。GPT-4o 画图、Chameleon、统一生成模型都归 IV，不在本阶段。
- **表征层的自预测**。JEPA / DINO 对比目标这一路——不依赖配对语料、在表征空间自组织语义——留给 V。
- **反身度量**。VLM 的「对不对」仍由人类标注 / 偏好给出；系统不能质疑 MMVP 这张账单本身——留给 V 的反身维度。
