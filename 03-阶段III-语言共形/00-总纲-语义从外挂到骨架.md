## §0 · 阶段 III 的本体命题

阶段 III 的纯存在态不是 image-text pair，也不是 VLM，而是：

```text
Visual Difference -> Public Linguistic Contract
```

更展开地写：

```text
delta_v -> name_l -> refer(x, name_l) -> act_or_answer(x, instruction_l)
```

视觉世界不再只要求“哪里是什么”，而要求这些视觉差异能否进入公共语言。在这张契约中，语言提供公共概念、指代表达、任务指令和评价话语；视觉提供图像证据、空间定位和部分感知支撑。阶段 III 的全部技术史，都是为了让这张契约更深地嵌入模型内部；阶段 III 的全部残差，也都来自这张契约不是视觉自身的主权。

更完整的中层协议见：[阶段 III 核心语法：命名、指代、调用、遮蔽](02-阶段III核心语法-命名指代调用遮蔽.md)。

阶段 II §16 留下的两条残差——**语义是借来的**与**结构化关系不在稠密输出里**——自然流进这里。阶段 III 的命题不是「在视觉模型旁边加个语言编码器」，而是：

> **在开放公共语义与社会可调用接口上，视觉无法仅靠内部空间结构闭合——它必须借语言获得公共语义载体。语言不是附加信号，而是阶段 III 的公共契约制度。**
> 

这一命题下，视觉获得的是共享式主体性，而不是自主语义主权。阶段 III 的真正判断不是「视觉夺回语义」，而是「视觉接受语言主权，以换取公共可调用性」。

边界也必须写清：阶段 III 说的是**公共语义闭合**，不是说所有视觉闭合都必须经由语言。对象、几何、对应、遮挡等图像世界状态，仍可能形成非语言的自持闭合；这正是阶段 V 要接走的问题。

## §1 · 从外挂 → 对齐 → 骨架：三段位移

阶段 III 内部按「语言进入视觉的深度」递进：

1. **外挂**：早期 VLM（ViLBERT / LXMERT / VinVL）把视觉 region 特征和语言编码器拼接，语言和视觉是「拼装」关系。
2. **对齐**：CLIP / ALIGN / SigLIP 用对比损失把全图表征与文本嵌入拉到同一空间，语言成为**预训练目标**。
3. **骨架**：BLIP-2 / Florence-2 / Flamingo / LLaVA 把 LLM 直接用作语义后端，视觉塔只负责「如何把像素喂进 LLM」——语言正式进入骨干。

每段位移的背后是同一条曲线：视觉对语言的依赖从「可选接口」变为「不可分割的表达基底」。

这条三段位移在机制层可细分为 L0-L6：

| 级别 | 名称 | 语言进入哪里 | 代表 |
|---|---|---|---|
| L0 | 标签语言 | 训练标签 / 类别名 | ImageNet、检测类别表 |
| L1 | 后端外挂 | region feature + BERT 融合 | ViLBERT、LXMERT、UNITER、VinVL |
| L2 | 损失内化 | 文本成为预训练目标 | CLIP、ALIGN、LiT、SigLIP |
| L3 | 稠密锚定 | 文本进入检测 / 分割 / region 输出链 | GLIP、Grounding DINO、CAT-Seg |
| L4 | 统一骨干 | 多任务图文理解共用骨干 | CoCa、BLIP、OFA、Florence-2 |
| L5 | LLM 后端 | 视觉 token 被送入语言推理系统 | Flamingo、BLIP-2、LLaVA、Qwen-VL |
| L6 | 像素调用 | 语言指令产生 mask / grounding 输出 | LISA、GLaMM、PixelLM |

因此，阶段 III 的“语言内化”不是单一事件，而是语言角色逐层进入损失、接口、骨干、后端和像素输出链。

## §1.5 · 四条机制线

| 机制线 | 对应章节 | 本体功能 | 自我否定 |
|---|---|---|---|
| 命名 | §21-§22 | 视觉差异进入标签、caption、文本嵌入和公共概念空间 | 名字来自语言分布，不来自视觉自身 |
| 指代 | §23 / §26 | 语言概念落到框、mask、region、属性和关系 | 指对了不等于持有空间状态 |
| 调用 | §24-§26 | prompt / instruction 让视觉成为可调用服务 | 接口扩张不等于状态闭合 |
| 遮蔽 | §27 | 语言先验暴露视觉盲区 | 能说对，不等于看对 |

这四条线先于具体模型。模型只是在不同历史节点上承担这些机制。

## §1.6 · 存在 / 无 / 成为：阶段 III 的主线表

| 环节 | 存在 / 设定 | 无 / 反面 | 成为 / 技术制度 | 成功 | 内在残差 |
|---|---|---|---|---|---|
| §21 外挂 | 视觉区域必须被语言消费 | region feature 是二手视觉 | ViLBERT、LXMERT、UNITER、VinVL | 语言能消费 detector 产物 | detector 上限锁死语义 |
| §22 对齐 | 图像与文本必须进入同一相似度空间 | 全图对齐压扁空间细节 | CLIP、ALIGN、SigLIP、MetaCLIP | web-scale 公共语义坐标成立 | caption 可描述不等于视觉本身 |
| §23 稠密锚定 | 语言必须落到框、mask、region | 语言类别不能自然产生空间 | GLIP、Grounding DINO、CAT-Seg、FC-CLIP | 开放词汇进入稠密输出 | 指代不等于状态持有 |
| §24 统一骨干 | 图文任务必须共用接口 | 任务统一不等于世界统一 | BLIP、CoCa、OFA、Florence-2 | 多任务图文接口统一 | text decoder 语义浅 |
| §25 LLM 后端 | 视觉必须进入语言推理空间 | 语言推理可能接管视觉证据 | Flamingo、BLIP-2、LLaVA、Qwen-VL | 对话、推理、指令能力跃升 | VLM 能说但未必看 |

阶段 III 的内在展开不是「语言越来越强」这一条线，而是：语言作为公共语义制度逐步获得命名权、指代权、调用权和评价权，同时让视觉自身的证据带宽、状态持有和语义主权越来越可疑。

## §2 · §21–§27 的接力线

- **§21** 融合式 VLM 前史（ViLBERT / LXMERT / UNITER / VinVL / OSCAR）——「外挂」的工程极限。
- **§22** 对比对齐规模化（CLIP / ALIGN / LiT / SigLIP / EVA-CLIP / MetaCLIP）——「对齐」本身成为预训练；图文材料制度见 [§22-B 公共语义材料制度](§22-对齐即预训练CLIP/03-公共语义材料制度-从caption到judge-rubric.md)。
- **§23** 语言锚定稠密任务的本体展开（GLIP / Grounding DINO / OWL-ViT / APE / CAT-Seg / FC-CLIP / Depth Anything）——他们在 §16 已作为「出口节点」首次出场，在这里作为本体展开。
- **§24** 统一多模态骨干（CoCa / BLIP / BLIP-2 / Florence-2 / PaLI / OFA / Unified-IO）——对比、caption、VQA、grounding 由一个骨干同时承担。
- **§25** LLM 作为语义后端：VLM 的诞生（Frozen / Flamingo / Kosmos / LLaVA / InstructBLIP / Qwen-VL / InternVL / CogVLM / Cambrian-1 / Qwen2.5-VL）——视觉「借道」已完成语义空间的 LLM；视觉证据能否存活，见 [视觉证据带宽](§25-VLM的诞生/06-视觉证据带宽-语言后端能吞下多少可见性.md)。
- **§26** 语言条件可提示视觉（Grounded-SAM / SEEM / Semantic-SAM / Ferret / Shikra / KOSMOS-2 / PixelLM / GLaMM / LISA）——语言指令触发空间化输出的单向闭合。
- **§27** 出口：两条残差一分为二 + 一笔「借语言的代价」账单。

## §3 · §16 → §23 的接缝说明

阶段 II §16 已把 SAM / Grounding DINO / APE / CAT-Seg / Depth Anything 作为「稠密共形的出口节点」引入。在阶段 III §23，他们以**本体**重新出场：重点从「如何打开类别闭集」转为「语言如何改写稠密输出的结构」。这不是重复，是「引子 vs 正文」的关系——和阶段 I §5 → 阶段 II §10 的交接同构。

## §4 · 出口残差：一分为二 + 一笔账单

**残差 A → 阶段 IV**：视觉可被语言锚定、描述、调用，但**不能反向生产观测**——「画出」「演出」「延续」进入 diffusion + LLM 的生成大动脉，阶段 IV 自然接入 LLM。

**残差 B → 阶段 V**：语义对齐绑定 caption / 配对语料；若要摆脱语言主权，必须走视觉状态自持与纯视觉表征（DINO / DINOv3 / JEPA）。

这两条不是互斥路线，而是阶段 III 出口残差的一外一内：阶段 IV 处理外显生产，阶段 V 处理内部持有。未来强系统很可能同时需要生成能力与状态持有能力。

**未偿账单（借语言的代价）**：

- MMVP / CLIP-blindness：CLIP 把「朝向左的狗」和「朝向右的狗」压成同一向量——被语言没编码的视觉属性被压扁。
- MMStar：多个主流 benchmark 上不给图可达随机线 + 20%——「视觉外壳」下是语言先验。
- 62% 故障源自感知：「推理错误」的一大半是识别错误假装成的。
- Look Light, Think Heavy：VLM 在 CoT 内对视觉 token 的注意力单调衰减——思考越长越不看图。

这些问题由阶段 III **正式命名**，但不由它解决——它们或被阶段 V 的纯视觉 SSL 重新大盘解决，或被阶段外的世界模型 / 具身智能接续。

## §5 · 阶段 III 的一句话

> 视觉从「被加工的客体」进入「与语言共享主体性的合同人」。这不是它的终点，是它与语言签下的一张临时协议——协议到期日是视觉自己能生成、能持有世界的那天。
>
