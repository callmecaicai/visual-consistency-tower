# §21 · 机制展开：外挂范式为什么触到天花板

本页补足 §21 的机制承重：外挂范式不是“错误路线”，而是阶段 II 对象制度被语言消费后的自然极限。它的核心问题不是融合层不够复杂，而是视觉已经在 detector 中被解释过一次。

---

## 一、统一计算图

```text
image
  -> detector / region proposal
  -> region feature + object tag
  -> BERT-style fusion
  -> MLM / ITM / region-word alignment / VQA
```

ViLBERT、LXMERT、VisualBERT、UNITER、OSCAR、VinVL 的差异主要发生在融合方式和预训练目标上。它们共享同一个底层事实：视觉证据先被 detector 切成 region，再被语言模型消费。

## 二、代表节点

| 工作 | 共形贡献 | 残差 |
|---|---|---|
| ViLBERT | 双流 cross-attention 让 region feature 与文本 token 交互 | 两个流仍建立在 detector 输出上 |
| LXMERT | 分离语言、视觉、跨模态编码器，并加入多任务预训练 | 任务多，但视觉粒度仍由 region 决定 |
| VisualBERT | 用单流拼接简化融合结构 | 简化融合不等于重写视觉 |
| UNITER | 统一融合结构，强化 ITM / WRA / MRM / MRFR | 目标更丰富，但证据仍是二手视觉 |
| OSCAR | 把 object tag 作为锚点词，缓解图文对齐噪声 | tag 让语言更容易接管视觉 |
| VinVL | 强 detector + 属性头把外挂路线推到高点 | 暴露 region-feature 外挂路线的天花板 |

## 三、二手视觉的本体问题

region feature 不是原始视觉证据。它已经被检测器类别表、box 制度和 Visual Genome 标注体系解释过一次。语言模型接收到的不是“图像”，而是检测器筛过的对象摘要。

这导致三个结构性后果：

1. detector 没有提出的对象，融合层很难重新发现。
2. detector 没有编码的属性，语言层只能用先验补。
3. region-word 精标越强，数据扩张越慢，web-scale 弱对齐越显得必要。

## 四、为什么 VinVL 是天花板暴露

VinVL 的意义不是证明外挂范式“错了”，而是把可优化项几乎都优化了一遍：更强 backbone、更强 region 标注、更细 object tag、更丰富预训练目标。它说明：继续加固 detector 前端可以提高外挂 VLM，但不会改变“视觉先被 detector 解释，语言再消费二手结果”的制度。

## 五、出口

§21 的三条残差共同推向 §22：

1. 视觉塔需要更深地参与图文预训练。
2. 监督信号需要摆脱 bbox / region-word 精标约束。
3. 数据规模需要从精标数据库转向 web-scale 弱配对。

CLIP 不是凭空出现的突破，而是外挂范式在材料、监督和载体三条线同时触顶后的范式置换。
