## §0 · 本体命题

> VLM 的胜利不是视觉的胜利,是**视觉把自己交给 LLM 的胜利**——主体性在这里第一次出场,但它是借来的。
> 

## §1 · 承接点

- §24 BLIP-2 的 Q-Former 桥 → 正式拥抱 LLM
- 视觉塔冻结,语言后端接管全部语义和推理

## §2 · 三代 VLM 的谱系

### 第一代 · 桥式(2022–2023 早期)

- **Frozen**(2021):最早的冻结 LLM + 可训视觉前缀
- **Flamingo**(2022,DeepMind):Perceiver Resampler + gated cross-attention
- **Kosmos-1**(2023,Microsoft):交错多模态 token 序列
- **BLIP-2**(2023):Q-Former + 冻结 LLM
- **InstructBLIP**(2023):BLIP-2 + instruction tuning

### 第二代 · 简化(2023 中–2024)

- **LLaVA / LLaVA-1.5 / LLaVA-NeXT**(Liu et al.):**2 层 MLP 就够**,证明 Q-Former 可简化
- **MiniGPT-4**:类似路线,更轻量

### 第三代 · 生产级(2024–)

- **Qwen-VL / Qwen2-VL / Qwen2.5-VL**:NaViT 式 dynamic resolution + MRoPE
- **InternVL 系列**:把视觉塔做到 6B(InternViT),对抗「LLM 是瓶颈」的命题
- **CogVLM**:visual expert,视觉 token 走独立 QKV
- **MiniCPM-V**:端侧方向

### 反思类

- **Cambrian-1**(2024,NYU):系统消融,证明**视觉塔的选择比想象中重要,但不如 LLM 关键**
- **Prismatic VLMs**(2024):训练配方的系统研究

## §3 · 三种视觉-LLM 接口

| 接口 | 代表 | 参数量 | 特点 | Perceiver Resampler | Flamingo | 约 100M | 表达力强,训练难 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| Q-Former | BLIP-2 | 约 100M | 解耦优雅,收敛慢 | 线性投影 / MLP | LLaVA | 约 10M | 简单粗暴,**竟然最赢** |

**为什么越简单越赢**:LLM 本身已经足够强,视觉 token 只需要落在 LLM embedding 空间的「合法区域」——复杂的桥反而引入 bottleneck。这是一个反直觉但经得起验证的工程现实。

## §4 · 训练配方(三阶段几乎是共识)

1. **Pretrain(投影对齐)**:冻结视觉塔和 LLM,只训投影层,用 caption 数据
2. **SFT(指令微调)**:解冻 LLM,用指令数据(LLaVA-Instruct / ShareGPT4V 等)
3. **Preference tuning(DPO / RLHF)**:对齐人类偏好,降低幻觉

每一步的数据质量决定上限——**合成数据的质量成为瓶颈**(GPT-4V 给的标注好不好,决定下一代 VLM 的天花板)。

## §5 · 关键穿透:LLM 是瓶颈,不是视觉塔

Cambrian-1 / Prismatic 的系统消融都指向同一结论:

- 换视觉塔(CLIP / SigLIP / DINOv2)对最终分数影响**有限**
- 换 LLM(7B → 13B → 34B / 70B)对最终分数影响**巨大**
- **VLM 的能力分布 ≈ LLM 的能力分布 + 一个视觉折扣系数**

这个结论直接决定产业路径:**不做 LLM 的人做不出顶级 VLM**。

## §6 · 视觉 token 的工程战争

视觉 token 数量是 VLM 的核心瓶颈:

- 单图 CLIP ViT-L = 576 tokens(24×24)
- 高分辨率图 / 多图 / 视频 → token 预算爆炸
- 解决方案:
    - **pooling / perceiver**(Flamingo / BLIP-2)
    - **dynamic resolution**(Qwen2-VL 的 NaViT,InternVL 的动态切图)
    - **AnyRes**(LLaVA-NeXT):把大图切成多 patch + 全局 thumbnail
    - **token 合并 / 剪枝**(FastV, PruMerge):推理时动态减少

## §7 · VLM 的隐性失败模式

- **Look Light, Think Heavy**:CoT 越长,对视觉 token 的注意力越低(详见 §27)
- **幻觉**:LLM 的先验压倒视觉证据,尤其是少见物体 / 长尾属性
- **多图混淆**:多个图的 token 在 LLM 内相互污染,VLM 很难做「这两张图的差异」
- **OCR / 计数 / 空间推理**:三大长期痛点

这些失败在 §27 集中兑现。

## §8 · 出口残差

- **残差 A → 阶段 IV**:VLM 只能理解不能生成 → diffusion + LLM 的融合(GILL / Emu / Chameleon)
- **残差 B → §27**:Look Light Think Heavy + CLIP-blindness + MMStar + 62% 感知故障
- **残差 C → 阶段 V**:视觉塔依附 LLM 的命运必须被反转

## §9 · 一句话收束

> VLM 是 LLM 的眼睛,不是视觉的嘴——阶段 III 的主体性**借自语言**,这一借让视觉跃升一个量级,也让它欠下必须由阶段 V 偿还的债。
>
