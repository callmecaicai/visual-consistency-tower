## §0 · 本体命题

> VLM 的胜利不是视觉的胜利,是**视觉把自己交给 LLM 的胜利**——主体性在这里第一次出场,但它是借来的。
> 

更精确地说：在主流 LLM 后端式 VLM 中，VLM 更像 LLM 长出眼睛，而不是视觉长出嘴。但这不是所有多模态系统的本质定义，而是阶段 III 最典型的语义主权结构。判断一个 VLM 是否真正看见，不能只看回答质量，还要检查视觉证据在语言后端中的存活带宽，详见 [视觉证据带宽](06-视觉证据带宽-语言后端能吞下多少可见性.md)。

## §2 · 三代 VLM 的谱系

### 第一代 · 桥式(2022–2023 早期)

- **Frozen**(2021):最早的冻结 LLM + 可训视觉前缀
- **Flamingo**(2022,DeepMind):Perceiver Resampler + gated cross-attention
- **Kosmos-1**(2023,Microsoft):交错多模态 token 序列
- **BLIP-2**(2023):Q-Former + 冻结 LLM
- **InstructBLIP**(2023):BLIP-2 + instruction tuning

### 第二代 · 简化(2023 中–2024)

- **LLaVA / LLaVA-1.5 / LLaVA-NeXT**(Liu et al.):在强 LLM、强数据和静态图像指令任务中,2 层 MLP projector 往往足够强,证明 Q-Former 可在部分条件下简化
- **MiniGPT-4**:类似路线,更轻量

### 第三代 · 生产级(2024–)

- **Qwen-VL / Qwen2-VL / Qwen2.5-VL**:NaViT 式 dynamic resolution + MRoPE
- **InternVL 系列**:把视觉塔做到 6B(InternViT),提示视觉侧在高分辨率、细粒度和 vision-centric 任务中会重新成为瓶颈
- **CogVLM**:visual expert,视觉 token 走独立 QKV
- **MiniCPM-V**:端侧方向

### 反思类

- **Cambrian-1**(2024,NYU):系统消融,证明**视觉塔的选择比想象中重要,但不如 LLM 关键**
- **Prismatic VLMs**(2024):训练配方的系统研究

## §3 · 三种视觉-LLM 接口

| 接口 | 代表 | 参数量 | 特点 |
| --- | --- | --- | --- |
| Perceiver Resampler | Flamingo | 约 100M | 表达力强,训练难 |
| Q-Former | BLIP-2 | 约 100M | 解耦优雅,收敛慢 |
| 线性投影 / MLP | LLaVA | 约 10M | 简单便宜,在静态图像指令任务中常很强 |

**为什么简单 projector 经常有效**:在强 LLM、强数据和静态图像指令任务中,视觉 token 只需要落在 LLM embedding 空间的「合法区域」,复杂桥接模块可能引入额外训练难度。但在视频、多图、高分辨率、长上下文和结构化视觉状态中,Perceiver / Q-Former / visual expert / memory connector 仍可能重新变重要。

## §4 · 训练配方(三阶段几乎是共识)

1. **Pretrain(投影对齐)**:冻结视觉塔和 LLM,只训投影层,用 caption 数据
2. **SFT(指令微调)**:解冻 LLM,用指令数据(LLaVA-Instruct / ShareGPT4V 等)
3. **Preference tuning(DPO / RLHF)**:对齐人类偏好,降低幻觉

每一步的数据质量决定上限——**合成数据的质量成为瓶颈**(GPT-4V 给的标注好不好,决定下一代 VLM 的天花板)。

## §5 · 关键穿透:LLM 后端与视觉瓶颈的条件分工

Cambrian-1 / Prismatic 的系统消融都指向同一结论:

- 在许多通用指令问答型 VLM 上,换 LLM(7B → 13B → 34B / 70B)对高层语言推理与回答质量影响很大
- 换视觉塔(CLIP / SigLIP / DINOv2)在部分综合榜单上影响较小,但在 vision-centric benchmark、细粒度空间、计数、深度、对应、定位和多图差异任务上会重新成为瓶颈
- 更稳妥的表述是:**通用对话能力高度依赖 LLM 后端,视觉真实性高度依赖视觉塔与连接器**

这个结论直接决定产业路径:通用对话型 VLM 的上限高度依赖语言后端;缺少强语言模型支撑的视觉系统,很难在开放式对话评测上达到顶级水平。但这不是“视觉塔不重要”的定理,而是当前评测与架构制度下的条件性结构。

## §6 · 视觉 token 的工程战争

视觉 token 数量是 VLM 的核心瓶颈:

- 单图 CLIP ViT-L = 576 tokens(24×24)
- 高分辨率图 / 多图 / 视频 → token 预算爆炸
- 解决方案:
    - **pooling / perceiver**(Flamingo / BLIP-2)
    - **dynamic resolution**(Qwen2-VL 的 NaViT,InternVL 的动态切图)
    - **AnyRes**(LLaVA-NeXT):把大图切成多 patch + 全局 thumbnail
    - **token 合并 / 剪枝**(FastV, PruMerge):推理时动态减少

## §7 · VLM 的隐性失败模式

- **Look Light, Think Heavy**:CoT 越长,对视觉 token 的注意力越低(详见 §27)
- **幻觉**:LLM 的先验压倒视觉证据,尤其是少见物体 / 长尾属性
- **多图混淆**:多个图的 token 在 LLM 内相互污染,VLM 很难做「这两张图的差异」
- **OCR / 计数 / 空间推理**:三大长期痛点

| 证据问题 | 技术现象 | 残差 |
|---|---|---|
| 证据压缩 | projector / Q-Former / token pooling | 细节丢失 |
| 证据稀释 | CoT 越长越不看图 | 视觉衰减 |
| 证据污染 | 多图 token 混淆 | 差异比较失败 |
| 证据替代 | LLM prior takeover | no-image baseline |
| 证据回看 | visual verification / self-correction | 推向 V / 反身 |

这些失败在 §27 集中兑现。

## §8 · 出口残差

- **残差 A → 阶段 IV**:VLM 只能理解不能生成 → diffusion + LLM 的融合(GILL / Emu / Chameleon)
- **残差 B → §27**:Look Light Think Heavy + CLIP-blindness + MMStar + 62% 感知故障
- **残差 C → 阶段 V**:视觉塔依附 LLM 的命运必须被反转

## §9 · 一句话收束

> 在主流 LLM 后端式 VLM 中，VLM 更像 LLM 长出眼睛，而不是视觉长出嘴；这一借让视觉跃升一个量级，也让它欠下必须由 §27、阶段 IV 和阶段 V 清算的债。
>
