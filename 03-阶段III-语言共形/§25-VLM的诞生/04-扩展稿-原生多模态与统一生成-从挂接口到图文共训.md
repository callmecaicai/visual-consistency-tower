## 1 · VILA —— *VILA: On Pre-training for Visual Language Models* (Lin 等, NVIDIA + MIT, CVPR 2024)

**共形贡献**：系统回答「要做原生多模态预训练需要什么」的第一篇大规模实证工作。三大核心发现（每一条都是对 §24 隐含假设的肢解）：

发现 1 · **预训练中解冻 LLM 会触发 in-context learning 能力**。§24 预训练阶段默认冻 LLM，只在指令微调阶段解冻。VILA 的消融显示：**预训练阶段就解冻 LLM 是多图 in-context few-shot 能力涌现的必要条件**。冻住的 LLM 仅支持单图问答，解冻后才能在推时看见示例图文对就会模仿新任务。

发现 2 · **交错图文文档（interleaved）比图文对更重要**。VILA 对比两种预训练数据：纯粗大图文对（LAION 风格）vs 有交错结构的文档（MMC4 风格，文档里多张图插入有上下文）。结果：交错文档在下游 benchmark 上全面赢。图文对只教会对齐，交错文档教会上下文建模 + in-context 学习。

发现 3 · **预训练加入纯文本数据能恢复 LLM 的语言能力**。解冻 LLM 预训练带来的副作用是纯文本 benchmark 显著下降（灾难性遗忘）。VILA 发现：在多模态数据里回揺一部分纯文本数据（约 1:1 比例），可以在不损害多模态性能的前提下基本恢复语言能力。「训多模态要把语言数据也带上」是一条启示很大的经验定律。

🔗 [https://arxiv.org/abs/2312.07533](https://arxiv.org/abs/2312.07533)

## 2 · MM1 —— *MM1: Methods, Analysis & Insights from Multimodal LLM Pre-training* (McKinzie 等, Apple, 2024)

**共形贡献**：迪今最详尽的多模态预训练消融研究，基于 3B-30B 参数、数千次小型实验给出认为统计显著的结论。关键发现：

发现 A · **图像分辨率与图像 token 数量的影响 >> VL connector 架构的影响**。无论用 cross-attention resampler、average pooling、Q-Former 还是简单 MLP，最终下游性能差异在 1–2 个百分点；但图像分辨率从 224 升到 336 再到 672 带来 3–5 个百分点的持续提升。**这一条观察不仅内部重排了 §23 的优先级，也在下一节直接解释了为什么 Fuyu 式的「直接投影 patch」能行**。

发现 B · **数据混合的最佳比例在§ interleaved : caption : text-only ≈ 5 : 5 : 1 § 附近**。交错数据对 few-shot 能力关键（与 VILA 一致）；caption 数据对 0-shot 关键；纯文本保护语言能力。三者缺一不可。

发现 C · **视觉编码器的规模比 LLM 的规模对下游影响更敏感**。把 ViT 从 ViT-L 换成 ViT-H 或 ViT-g带来的提升大于把 LLM 从 7B 换成 13B。对 §23 / §24 常见的「ViT 选标配 + LLM 尽量大」的配方习惯是一个纠正。

🔗 [https://arxiv.org/abs/2403.09611](https://arxiv.org/abs/2403.09611)

## 3 · Fuyu-8B —— *Fuyu-8B: A Multimodal Architecture for AI Agents* (Bavishi 等, Adept Blog / Technical Report, 2023/10)

**共形贡献**：极简化原生多模态架构的存在证明。完全舍弃 ViT / Q-Former / 适配器链路，直接把图像切成 patch 、线性投影后和文本 token 拼接，一起喂给 decoder-only transformer。两个重要结果：

- 任意分辨率、任意长宽比支持自然获得——没有了 ViT 的 fixed-grid约束。可输入文档 / UI 截图 / 图表等非标准形状的图像，这对 agent / 调用界面之类的下游很关键。
- 证明了一件重要的事：**视觉编码器本身并非理解图像的必要架构**，LLM 自己就能在透过线性投影的 patch 特征上建立视觉语义。

关键判断：这和 MM1 的发现 A 相互印证——如果「分辨率 >> connector 架构」，那么「没有 ViT」本身就不应该是火吞问题，更大的苦功在于预训练数据和算力。Fuyu 以实物波形给出了这个推论的实战证明。

🔗 [https://www.adept.ai/blog/fuyu-8b](https://www.adept.ai/blog/fuyu-8b)

## 4 · Chameleon —— *Chameleon: Mixed-Modal Early-Fusion Foundation Models* (Chameleon Team, Meta FAIR, 2024)

**共形贡献**：首个大规模 early-fusion 多模态基础模型。图像通过 VQ-VAE 式编码器被离散化成 token，与文本 token 在**同一个词表**里混合，由**同一个 transformer decoder** 用同一个 next-token 交叉熵损失从零预训练。没有 ViT、没有适配器、没有阶段性训练。论文里最值得引用的两个观察：

观察 1 · **混合模态训练在扩扩到 34B 规模时出现严重的训练发散**。原因是不同模态 token 的 softmax 输出分布差异大，导致后期层的 logits 映速爆炸。解决方案：**query-key normalization 在 attention 输入前先归一化 Q 和 K**，同时将 softmax 前的 logits 盖上一层 layer norm。这个技术细节看似个小改动，但展示了 early-fusion 在规模化时遇到的并不是架构问题，而是**训练动力学问题**。

观察 2 · **在相同 FLOPs 下，Chameleon 的纯文本表现与同规模纯文本 LLM（Llama-2）处于相似水平**。这一条经验性结论非常关键：它表明「加入视觉模态 ≠ 损害语言能力」在足够多数据与足够的训练稳定性下是成立的——这是为 §23 / §24 设计中「冻 LLM 以保护语言能力」这个隐含前提提供了否证。

🔗 [https://arxiv.org/abs/2405.09818](https://arxiv.org/abs/2405.09818)

## 5 · Transfusion —— *Transfusion: Predict the Next Token and Diffuse Images with One Multi-Modal Model* (Zhou 等, Meta + Waymo, 2024)

**共形贡献**：统一理解与生成的最优雅路线。在一个 transformer 里同时使用两种损失：

- 文本部分用 next-token prediction（离散损失）
- 图像部分用 diffusion loss（连续差价）——图像 token 是连续 latent patch，不需要 VQ 离散化

关键判断：**离散化图像会损失信息；把扩散目标直接放到一个 transformer 中让它和 NTP 共存，可以同时得到两个世界的好处**。可直接引用的结论：

结论 I · **Compute-matched 下 Transfusion 的图像 FID 优于 Chameleon 的量化路线约 2 个数量级**（在 7B 规模上 Transfusion FID 约 6.78，Chameleon 的类型方法几十）。

结论 II · **文本能力付出可忽略**。加入图像 diffusion 目标与纯 NTP 预训练相比，文本 benchmark 性能基本持平。

结论 III · **不需要多阶段训练、不需要范式切换**，这是一种真正的单一目标统一训练。

🔗 [https://arxiv.org/abs/2408.11039](https://arxiv.org/abs/2408.11039)

# 关于 §25 五篇的机制追问

## 一、「原生多模态」到底指什么：三层逐步暴暴内核

「原生多模态」一词在业界被滥用很严重。精确的定义需要分三层查看：

| 层次 | 特征 | 代表方法 | 与 §23 / §24 的关系 |
| --- | --- | --- | --- |
| 弱意义：多模态预训练 | 解冻 LLM 在预训练阶段就开训，而非只在指令微调中 | VILA / MM1 / Idefics2 / InternVL | 缓解了 §23 的「冻 LLM」假设，但架构层仍是 ViT + 适配器 + LLM |
| 中义：抽掉适配器 | 去掉 ViT / Q-Former，直接 patch 投影 | Fuyu-8B | 抽掉了架构中间层，但 next-token 损失只用在文本上 |
| 强意义：early-fusion 共享词表 / 统一目标 | 图文 token 进同一词表，一个 transformer 一个损失从零共训 | Chameleon / Transfusion / Emu3 | 完全打破「视觉适配语言」的不对称，真正的平权 |

### 为什么强意义才是真正挑战 §24 方程式的

§24 末残差第 1 条为「视觉推理 = LLM 文本推理 + 视觉初始条件」。

- 弱意义的原生多模态（VILA / MM1）并没有挑战这个方程式——它们只是**完善了待后联的接口，让这个方程式跑得更好**。视觉软 token 进了 LLM 的后面的推理仍全部走在语言 token 空间里。
- 中义（Fuyu）部分挑战了它——因为没了 ViT 和适配器，图像 patch 更直接地在 transformer 内部和文本 token 混合。但损失函数还是 next-token，目标最终还是预测文本。
- 强意义（Chameleon / Transfusion）真正挑战了它——图像 token 也是损失目标的一部分，要么与文本共享词表 + NTP损失（Chameleon），要么共存的 diffusion loss（Transfusion）。当模型必须也产出图像 token 时，它不再能把视觉只当「初始条件」对待。

为什么这一区别在 V 反身度量里会重要：如果模型从来不需要生成视觉输出，那么「模型真正理解了图」和「模型把图转换为语言描述」是不可区分的——因为评估途径只有语言。只有强意义的原生多模态才能把这两者在操作上区开。

## 二、VILA 与 MM1 的五条经验定律——「原生预训练配方」的工程综合

把两篇的实证结果拼起来，可以得到五条待引用的经验定律：

### 定律 1 · 预训练中必须解冻 LLM（VILA 发现 1）

&ldquo;We find that unfreezing the LLM during pre-training is crucial for inheriting in-context learning capabilities&rdquo;——这句话是 VILA 对 §23 「冻 LLM」隐含假设的正式翻转。LLaVA / BLIP-2 时代因为数据不够、又担心灾难性遗忘，默认冻 LLM 了。VILA 让人第一次看到：在足够数据条件下，解冻的收益远远超过损失。

### 定律 2 · 预训练数据必须有交错文档（VILA 发现 2 + MM1 发现 B）

仅有图文对（CLIP 时代的数据形态）不够；图文交错文档（MMC4 / OBELICS 风格）才能让模型学到「在图像上下文中展开语言」的模式。这与 NLP 侧「C4 从论坛 / 预训练文档 里学得更多语言能力」是结构类比：**自然序列数据 >> 配对数据**。

### 定律 3 · 必须回揺纯文本数据以防灵语言退化（VILA 发现 3 + MM1 发现 B）

解冻 LLM + 多模态数据 不可避免地造成纯语言 benchmark 下降。回揺约 1:1 比例的纯文本数据乎全量恢复语言能力。「混训能保留继承的能力」——这个结论直接用于 Chameleon / Transfusion 等 early-fusion 方法的数据配方设计。

### 定律 4 · 分辨率与图像 token 数量的收益 >> 适配器架构的收益（MM1 发现 A）

在 MM1 的数千次实验中，替换 VL connector 的下游影响在 1–2 pp；分辨率从 224→336→672 卖到 3–5 pp 的持续提升。这意味着 §23 謼后一年多的「谁胜出：线性 vs MLP vs Resampler vs Q-Former」之争在数据规模出来之后大部分是假问题。**真问题是视觉 token 够不够**。

### 定律 5 · 视觉编码器规模的边际收益 > LLM 规模的边际收益（MM1 发现 C）

将 ViT 从 L 升到 H/g 的提升大于将 LLM 从 7B 升到 13B。这条对工程配方是反直觉的——过去大家抽的多模态模型报告中 ViT 跟 LLM 的标准搭配都是「小 ViT + 大 LLM」。MM1 让这条习惯需要被纠正。

### 这五条在做的事

§24 → §25 弱意义的合理地表现和方法变得明确：**把 §23/§24 里隐含的「挂接口 — 冻 LLM — 尽量只调适配器」这一套范式拆解开，每一项全部用消融实验重新审视**。VILA / MM1 不是架构创新论文——它们是「用实证重写 §23 方法论」的论文。

## 三、early-fusion 的两条路：图像离散化 vs 图像连续表示

### 两条路线的对比

| 路线 | 代表 | 图像如何入词表 | 存失分管 | 关键优势 | 关键代价 |
| --- | --- | --- | --- | --- | --- |
| 离散化 | Chameleon / Emu3 / Show-o | VQ-VAE 编码 → 离散 token | 单一 NTP 损失 | 架构极简 / 完全复用 LLM 基础设施 | VQ 量化损失 / 训练稳定性 |
| 连续 | Transfusion | 保留 continuous latent patch | NTP + diffusion 共存 | 无量化损失 / FID 显著优于离散化路线 | 需要两种损失维保交叉 |

### 可直接引用的实证对比（Transfusion 论文表 2）

Transfusion 论文主结果：

- 在 7B 参数 + 相同 compute budget 条件下，Transfusion 的 FID 约 6.78，Chameleon 风格的量化统一模型 FID 约 26.74。**一个数量级以上的差距**。
- 在同等 compute 下 Transfusion 的 CIDEr（对话能力）与纯 NTP 基线持平。加了图像生成能力 **没有损害文本**。

### 两条路线背后的深层关切

离散化的吸引力：所有 LLM 基础设施（KV cache / 延迟延序 / causal mask / 批量推理）都直接复用。**重用性 >> 信息精度**的技术权衡。

Transfusion 的吸引力：注意到图像的本质是连续转化器，VQ 量化本身就是有信息损失。**信息完整性 >> 重用的便捷性**。

哪条路最终胜出？这取决于：**图像生成质量是否成为主要评估指标**。如果是，Transfusion 路线赢；如果只是理解任务 + 偶尔生成，离散化路线径较短。这个问题在 IV 生成对齐里是中心话题。

## 四、Chameleon 的训练稳定性问题：一个看似小但意义重大的技术观察

### 问题描述

Chameleon 在扩大到 34B 时训练损失出现爆炸。原因追踪到：

- 图像 token 与文本 token 的**输出分布尺度不同**。视觉 token 的 softmax 分布更尖锐，文本 token 更平滑。在高 transformer 层中，两种 token 的 logits 幅值远跳可达 50–100 标准差之差，从而导致 attention 的 softmax 补釀压平一模态。
- 模态同时训练时，这种不平衡在高层通过 LayerNorm 累积放大，导致 logits 映速爆炸。

### 解决方案：QK-Norm + 附加 LayerNorm

在 attention 计算前对 Q 和 K 分别打 LayerNorm（这一技术称为 QK-Norm，而不是来自 Chameleon——它由 Dehghani 等 2023 提出，在 ViT-22B 中被证明能稳定大规模 transformer 的训练）。Chameleon 的贡献是〉：把 QK-Norm 用在混合模态训练上、并发现它能解决混合模态特有的不平衡问题。

### 这一观察为什么重要

上一层的含义：§24 的豱隔策略（冻 LLM + 弱维固适配器）其实在**正是避开这种训练动力学的不平衡问题**。当你冻住 LLM 时，混合模态不平衡不会撕经预训练好的 transformer。Chameleon 的试验告诉我们：强意义的原生多模态不是「只要把数据混着喂就行」，而是**给多模态设计的专用训练动力学**的工程问题。

下一层的含义：这揭示了 §25 还不是一门科学。§23 / §24 的方法成功，部分原因是它们**绕开**了这些动力学问题。§25 的强意义等一来——隐藏的问题涌现了。这一负担催生 §26 对多模态 scaling laws 的系统化追求。

## 五、原生多模态真的解决了「视觉推理 = LLM 文本推理 + 初始条件」这个方程式吗？

这是 §24 末残差第 1 条人交给 §25 的任务。从现有实证看，**答案仍在争议中**。

### 持肯定含种的证据

· **Chameleon 在 「识别图中结构的分布」任务上表现过 LLaVA 等字骨 §23/§24 架构**。特别是条件图像生成 / 图文交错创作任务。因为这些任务需要模型真正“在视觉 token 空间里做回归”——只计算一个语言 token 而不能真正产出视觉输出的架构做不了。

· **Transfusion 展示了理解与生成在单一模型里的互相收益**。加上生成目标后，理解任务的 CIDEr 基线持平而生成 FID 大幅提升。这假证了强意义原生多模态的加值。

· **Emu3 论文的中心论点：「Next-token prediction is all you need」**（论文标题）。单一 NTP 在统一的词表 上在理解和生成两侧均可达 SOTA。这支持一种强版本的挑战陈述：多模态不需要特殊的推理机制，只需要统一的 token 空间和足够的数据。

### 持怀疑含种的证据

· **DeepSeek Janus (2024) 的反例观察**：理解任务需要的视觉表示与生成任务需要的视觉表示**不是同一种**。理解款表示需要语义压缩（快说重点），生成款表示需要保留细节（像素级破密）。Janus 用两套视觉 encoder 分开处理，性能优于共享 encoder 的 Chameleon 风格模型。这是「统一模式是一升高但不是万能」的警示。

· **空间推理 benchmark 上原生与非原生的差异不显著**。VLM 在空间关系理解、几何推理、图形 puzzle 上的系统性弱势当前未因强意义原生多模态而解决（读者可参考 Tong 等 2024 《Eyes Wide Shut?》 arxiv 2401.06209，以及 What'sUp arxiv 2310.19785）。这意味着：即使架构已经让视觉 token 参与损失计算，VLM 在需要真正视觉推理的任务上仍然很脆弱。

### 临时判决

**架构的挑战是必要条件但不是充分条件**。原生多模态做对了解决方程式的**形式条件**（让视觉 token 参与输出），但视觉推理的**实质能力**仍未被充分关疑解决。缺乏的可能是约当的视觉元推理 objective、以及深度够的组合个视觉 token 间交互的所需的适配式捕捉 scaling 。这个争议将在 V 反身度量里重新被启动：我们现在判断「模型会视觉推理」的標准已经是一套语言表达的 benchmark，此中的闭合回路可以被打开吗？

---

## 小结：§25 的谱系内逻辑

- **VILA**（NVIDIA + MIT， 2023/12）：系统消融结论 —— 解冻 LLM + 交错数据 + 纯文本回揺 是原生多模态预训练的三块必备。
- **MM1**（Apple， 2024/03）：更大规模的消融 —— 分辨率 >> 接口架构；ViT 规模比 LLM 规模更敏感；数据混合比例亲测。
- **Fuyu-8B**（Adept，2023/10）：极简原生架构的存在证明 —— 没有 ViT / 没有适配器，直接 patch 线性投影也可行。
- **Chameleon**（Meta FAIR，2024/05）：强意义原生多模态，图文离散化 + NTP 共享词表，代价是需要 QK-Norm 等新的训练稳定性技术。
- **Transfusion**（Meta + Waymo，2024/08）：强意义的最优雅路线，不量化图像、NTP+diffusion 共存于单一 transformer，FID 显著赢过量化路线。

五篇合起来确立了一件事：**§24 的「冻 LLM + 挂接口」是一个工程临时方案**，对指令微调任务表现不错是因为任务后端就是语言；但指令微调走不到「真正的视觉理解」和「理解与生成统一」的東场。§25 是为了走到那里所做的架构转向。但它还未完成——训练稳定性、模态冲突、scaling laws 各项来到目前还处于「工艺而非科学」的阶段。

---

## 本组合力的残差

1. **模态冲突仍未完全解决**。强意义原生多模态里，理解需要语义压缩的表示、生成需要保留细节的表示，两者在单一 backbone 里分还未完美。DeepSeek Janus 的双视觉 encoder 答桌如果胍向，则 scaling law 将不仅为一条，可能为多条并行。这条残差**汇入 §26**（系统协同的系统化追求）。
2. **训练稳定性与数据配方仍是工艺而非科学**。Chameleon 的 QK-Norm、MM1 的 5:5:1 比例、VILA 的 1:1 文本回揺——这些都是经验发现，未很好的理论解释。多模态 scaling laws 的系统化研究目前仅在初始阶段（对比 Hoffmann 等 2022 的 Chinchilla scaling laws 在纯语言侧的成熟度）。这条残差**催生 §26** 对 scaling laws 和协同机制的追求。
3. **「视觉推理独立于语言推理」这个目标仍未被充分关疑实现**。Tong 等 2024《Eyes Wide Shut?》、MMVP benchmark 、空间推理 What'sUp benchmark 都显示：最强的原生多模态模型在需要视觉细节判断的任务上仍然表现出 CLIP-like 视觉编码器的系统性盲区。也就是说，架构换了，但视觉表示本身的脆弱性未能被完全修复。这条残差**汇入 V 反身度量**——当我们说 「模型会视觉推理」的时候，我们是用什么标准在判断？标准本身是否是另一个语言场的投影？

第 1 条与第 2 条汇入 §26；第 3 条汇入 V 反身度量。**§25 是阶段 III 的转弯点**——它把架构偏见清掉了，但也把领域带进了一个「新的建制工程问题比架构挑选更突出」的阶段，也把「视觉推理是否足够独立」的问题更清晰地押到下一章和下一阶段去。
