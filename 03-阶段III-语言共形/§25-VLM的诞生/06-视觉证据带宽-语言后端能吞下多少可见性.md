## 一、视觉证据带宽公式

阶段 III 的关键问题不是“视觉有没有接入 LLM”，而是：

> 接入之后，还有多少可见性活着？

可以写成一个工作公式：

```text
Visual Evidence Surviving in LLM
=
visual encoder fidelity
× token budget
× connector preservation
× attention persistence
× task relevance
- language prior takeover
```

这不是实验指标，而是阶段 III 的机制语法。它要求每个 VLM 说明：视觉证据在哪一层被压缩、在哪一层被稀释、在哪一层被语言先验替代。

## 二、五类证据故障

| 问题 | 技术现象 | 残差 |
|---|---|---|
| 证据压缩 | CLIP global token、Q-Former、projector、token pooling | 细节、局部关系和计数丢失 |
| 证据稀释 | CoT 越长越少回看视觉 token | 推理链越来越语言化 |
| 证据污染 | 多图、多帧、多对象 token 混淆 | 差异比较和变化理解失败 |
| 证据替代 | LLM prior takeover / no-image baseline | 模型能答但未必看 |
| 证据回看 | visual verification、self-correction、tool re-grounding | 推向阶段 V 与反身审计 |

§27 的四张账单不是孤立 benchmark，而是这些证据故障在不同评估制度下的显影。

## 三、连接器不是中性管道

不同 VLM 连接器不是“把视觉送给语言”的等价通道。

| 连接方式 | 代表 | 证据带宽特征 | 典型风险 |
|---|---|---|---|
| 全局投影 | 早期 CLIP-to-LLM / 简单 image token | 便宜、容易训练 | 空间细节被压扁 |
| Q-Former | BLIP-2 / InstructBLIP | 可学习 query 压缩视觉 | query 数量限制证据容量 |
| Perceiver Resampler | Flamingo | 固定 token 汇聚多帧/多图 | 训练复杂、证据选择黑箱 |
| MLP projector | LLaVA 系 | 简单、高效、强数据下足够 | 高分辨率与结构状态弱 |
| dynamic resolution / visual expert | Qwen-VL、InternVL、CogVLM | 增加视觉侧带宽 | 成本、上下文和对齐难度上升 |

连接器决定的不是模型风格，而是视觉证据能以什么形态进入语言推理空间。

## 四、多图、视频与遥感变化是压力测试

单图 VQA 常常允许语言先验补位；多图、视频和遥感变化理解更难伪装，因为它们要求模型保留差异：

```text
image_1 evidence
image_2 evidence
delta evidence
temporal / spatial relation
```

如果视觉证据带宽不足，模型会出现三类失败：

1. 把两张图的对象混在一起。
2. 用文本先验猜变化，而不是比较证据。
3. 在长推理中遗忘第一张图或局部区域。

因此，遥感变化理解不是阶段 III 主线的替代，而是阶段 III 账单的放大镜：它逼问 VLM 是否真的保存了可见差异，而不只是把图像送进语言后端。

## 五、验证协议

任何声称“VLM 看得更清”的改进，至少应报告：

| 检查项 | 最小验证 |
|---|---|
| no-image control | 去掉图像后性能是否显著下降 |
| visual perturbation | 改方向、计数、位置、遮挡后答案是否跟随变化 |
| token ablation | 删除局部视觉 token 是否破坏对应答案 |
| attention persistence | 多步推理中视觉注意力是否持续 |
| multi-image binding | 每张图的证据是否被分别保留 |
| re-grounding | 答案能否回指到区域、mask、bbox 或像素证据 |

这套协议不证明模型拥有世界状态，但能排除“语言替视觉说话”的伪闭合。

## 六、一句话

> VLM 的核心不是语言后端能说多少，而是可见证据穿过语言后端后还剩多少。
