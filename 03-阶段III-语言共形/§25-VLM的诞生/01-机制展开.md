# §25 · 机制展开：LLM 作为语义后端

§25 是阶段 III 的顶点。这里的关键不是“视觉模型会说话”，而是视觉 token 被送入已经完成语言预训练的 LLM 后端，视觉获得了公共语义推理接口，也把主体性继续寄存在语言里。

---

## 一、三代 VLM 谱系

| 代际 | 代表 | 机制 | 残差 |
|---|---|---|---|
| 桥式 | Frozen、Flamingo、KOSMOS-1、BLIP-2、InstructBLIP | 冻结或半冻结 LLM，用视觉前缀、Perceiver 或 Q-Former 接入 | 桥接复杂，视觉带宽受限 |
| 简化 | LLaVA、LLaVA-1.5、MiniGPT-4 | 视觉塔 + MLP projector + 指令微调 | 依赖数据质量和 LLM 后端 |
| 生产级 | Qwen-VL、Qwen2-VL、InternVL、CogVLM、MiniCPM-V | 动态分辨率、强视觉塔、visual expert、OCR/多图/视频增强 | token 成本、细粒度感知和空间状态仍是瓶颈 |
| 反思类 | Cambrian-1、Prismatic | 系统消融视觉塔、连接器、数据和 LLM 的贡献 | 说明 VLM 不能只按语言榜单评价 |

阶段 III 的主轴不是某一代胜出，而是视觉以越来越工程化的方式接入 LLM 语义后端。

---

## 二、接口的带宽问题

| 接口 | 代表 | 优点 | 代价 |
|---|---|---|---|
| 线性投影 / MLP | LLaVA | 简单、便宜、易训 | 信息压缩粗，细粒度空间弱 |
| Q-Former / Perceiver | BLIP-2、Flamingo | 可学习压缩，显式控制视觉 token 数 | 训练复杂，可能成为瓶颈 |
| 动态分辨率 / 切图 | Qwen2-VL、InternVL、LLaVA-NeXT | 保留高分辨率细节 | token 成本爆炸，需要裁剪、合并和调度 |
| Visual expert / memory connector | CogVLM、视频 VLM | 给视觉保留专门路径 | 架构复杂，训练配方敏感 |

“越简单越赢”只在一部分条件下成立：强 LLM、强数据、静态图像、通用指令任务。对于视频、多图、高分辨率、长上下文、结构化视觉状态，connector 会重新变成核心战场。

---

## 三、LLM 后端与视觉瓶颈的条件分工

在通用指令问答型 VLM 上，LLM 后端常常决定高层语言推理与表达质量。因此，更强 LLM 往往显著提升综合对话榜单。

但在 vision-centric benchmark、细粒度空间、计数、深度、对应、定位、多图差异和遥感变化理解中，视觉塔与连接器会重新成为瓶颈。这里不能把“LLM 很重要”误写成“视觉塔不重要”。

更稳的公式是：

```text
VLM 能力 = 语言后端能力
         + 视觉证据带宽
         + connector 保真度
         + 数据/指令配方
         - 语言先验接管风险
```

---

## 四、视觉 token 工程战争

视觉 token 是阶段 III 的硬瓶颈。单图 576 tokens 已经昂贵，高分辨率图、多图和视频会迅速压爆 LLM 上下文。

主要路线：

1. pooling / perceiver：压缩视觉 token 数。
2. dynamic resolution：让不同分辨率输入产生不同 token 数。
3. AnyRes / tiling：局部切块 + 全局 thumbnail。
4. token merging / pruning：推理时动态减少无用 token。
5. visual memory：让视频、多图和长上下文不必把全部视觉证据一次性塞进 LLM。

这些工程都在回答同一个问题：公共语义后端能消化多少视觉证据。

---

## 五、出口

§25 的账单有三条：

1. VLM 能说、能答、能推理，但不能反向生产观测 -> 阶段 IV。
2. 语言后端会用先验补全视觉证据，形成 no-image baseline、幻觉和 CoT 视觉衰减 -> §27。
3. 视觉主体性仍借自语言，必须由阶段 V 的状态持有与纯视觉表征重新接走。
