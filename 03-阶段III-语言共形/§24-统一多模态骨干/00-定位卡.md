## §0 · 本体命题

> 统一骨干的意义不是「一个模型打多个 benchmark」,是**承认语言是所有视觉理解任务的公共基底**——这一步让视觉塔可以进一步退化为 adapter,为 §25 的 LLM 后端铺路。
> 

## §1 · 承接点

- §22 CLIP 提供对齐目标(单目标)
- §23 语言锚定稠密提供空间 / 类别目标(多目标)
- 本节把所有目标塞进同一个骨干

## §2 · 统一骨干的三种哲学

| 类型 | 代表 | 作用 |
| --- | --- | --- |
| 多目标 VLP 骨干 | CoCa、BLIP | 统一 ITC / ITM / LM |
| Seq2Seq 任务统一 | OFA、Unified-IO、Florence-2 | 把任务输出序列化 |
| 冻结 LLM 桥接 | Flamingo、BLIP-2、LLaVA | 视觉接入语言推理后端 |

三类路线不是简单淘汰关系。真正通向 §25 的不是一般 Seq2Seq,而是视觉 token 被送入已完成语言预训练的 LLM 后端。

## §3 · 代表工作

- **CoCa**(2022,Yu et al.):对比 + captioning 双目标,视觉和语言解码器共享
- **BLIP**(2022):三目标(ITC / ITM / LM),CapFilt 数据清洗
- **BLIP-2**(2023):Q-Former 以极清晰形式让冻结视觉编码器与冻结 LLM 通过可学习 query 桥接——这是通往 §25 的桥
- **OFA**(2022):一切皆 seq2seq,任务用自然语言 prompt 描述
- **Unified-IO / Unified-IO 2**:进一步统一到生成任务(图像、音频)
- **PaLI / PaLI-3**(Google):多语言 + 统一训练
- **Florence-2**(2024,Microsoft):**prompt-as-task**——用 task token 替代 task head

## §4 · BLIP-2 为什么是关键转折

BLIP-2 的 **Q-Former**(Querying Transformer):

- 输入:冻结的视觉特征 + 32 个可学习 query token
- 输出:32 个压缩后的视觉 token
- 意义:BLIP-2 不是“首次”把视觉接到冻结语言模型上,但它用 Q-Former 给出极其清晰的桥接模块——视觉塔只负责抽特征,Q-Former 负责转成 LLM 能吃的格式,LLM 冻结

这个结构直接预言了 §25 的大部分桥接范式。后来 LLaVA 用「2 层 MLP」替代 Q-Former,证明在强 LLM、强数据和静态图像指令任务中,桥接模块可以极简化;但 BLIP-2 的解耦思想仍是关键节点。

## §5 · Florence-2 的 prompt-as-task

Florence-2 用一个统一的 seq2seq 模型做:

- caption / detailed caption / OD / OCR / grounding / segmentation
- 所有任务用自然语言 prompt 触发(如 `<OD>`、`<CAPTION>`)
- task token 替代 task head

意义:**任务本身成为 prompt**,这是 prompt-as-task 的极端代表,也是通往阶段 IV「指令即任务」的桥。它证明大量视觉任务可以被 task prompt + structured token output 统一,但这种统一仍是接口统一,不等于世界状态统一。

## §6 · 工程代价(被低估的部分)

- 统一骨干在单任务上**常常打不过专用模型**
- 「统一」的价值不是 SOTA,是**下一步借 LLM** 时的接口一致性
- 训练成本高(多任务 loss 平衡、数据配比、grad 冲突)
- **多数产业应用仍用专用模型**,统一骨干主要活在研究和旗舰 VLM 里

## §6.5 · 统一了什么，没有统一什么

| 统一类型 | 统一了什么 | 没统一什么 |
|---|---|---|
| 目标统一 | ITC / ITM / LM 等 loss | 世界状态 |
| 任务统一 | caption / OD / OCR / grounding 输出格式 | 语义主权 |
| 后端统一 | 视觉 token 接入 LLM | 视觉证据真实性 |

统一骨干不是被淘汰，而是被重新定位：它从“最终语义主体”退到“标准视觉接口与多任务材料组织者”。

## §7 · 包装 vs 工程

- **包装**:「统一多模态理解」「单模型多任务」
- **工程**:真正的价值在于**为 LLM 提供标准视觉接口**——BLIP-2 的 Q-Former 的历史地位远高于它的 benchmark 分数

## §8 · 出口残差

- 统一骨干的语义能力仍被**自身 text decoder 封顶**
- 所以必须把 text decoder 换成 LLM → §25
- 生成能力仍缺席(caption ≠ 生成)→ 阶段 IV

## §9 · 一句话收束

> 统一骨干是 VLM 的过渡骨架——它的历史任务是**铺好 LLM 能够插入视觉的标准接口**；它不是失败，而是从最终语义主体退到公共接口组织者。
>
