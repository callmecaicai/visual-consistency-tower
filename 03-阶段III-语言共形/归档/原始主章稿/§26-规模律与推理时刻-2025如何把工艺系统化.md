**主题**：2025 是多模态从「架构探索」走向「规模律透视」的一年。三股力量同时到位：训练时的原生多模态规模律被系统给出，推理时规模律（o1 / R1 / QvQ）从纯文本迁移进多模态，开源侧（InternVL3 / Qwen2.5-VL）把 §25 的架构洞见落成工程化系统。但随之当 Yan 等 2025揭示：**推理链加长会让视觉 grounding 渐步丢失**——这直接提供了 §24 / §25 残差「视觉推理 = LLM 文本推理 + 初始条件」的经验证据。五篇合起来是 2025 的前沿前沿。

## 1 · Scaling Laws for Native Multimodal Models —— *arxiv 2504.07951 / ICCV 2025* (Shukor 等, Apple + Sorbonne)

**共形贡献**：首个大规模原生多模态 scaling law 研究，训练了 **457 个不同架构与数据混合的模型**。主要结论直接反驳了业界的标准假设：

结论 1 · **晚期融合（ViT + 适配器 + LLM）相比早期融合并无固有优势**。两种架构的规模律曲线乘 flop 在同一条岸上。**在参数量较小时早期融合反而更强、训练更高效、部署更简单**。这一结论直接推翻了 §23 / §24 多年叠加的「晚期融合更样本效率」的过程经验。

结论 2 · **MoE + 模态无关路由（modality-agnostic）败模态感知路由（modality-aware）**。模态感知路由强行把视觉 token 投到「视觉专家」、文本 token 投到「文本专家」；而模态无关路由让路由函数自己判断，结果是后者在规模律上始终赢。**这个反直觉结论揭示：模态之间的路由应该是「按任务 / 按语义」的而不是「按模态身份」的**。这一条是对 DeepSeek Janus 「视觉双 encoder」的一种并行反方程：Janus 用架构强行分开，Shukor 2025 让路由自行学分工。

结论 3 · **训练 FLOPs 分配的最优点与纯语言 Chinchilla 不同**。纯语言的 Chinchilla 给出 「数据 token 数 : 参数数 ≈ 20」；Shukor 2025 发现原生多模态的最优比例更靠向**数据**（数据 token 数 / 参数数 更大），并且这一比值随图像 token 的比例而漂移。**多模态模型比纯语言模型「更相对数据饥渴」**——这与直觉相对成立（因为图像 token 噪声更大、语义密度更低）。

🔗 [https://arxiv.org/abs/2504.07951](https://arxiv.org/abs/2504.07951)

## 2 · InternVL3 —— *arxiv 2504.10479* (OpenGVLab 团队, Shanghai AI Lab, 2025/04)

**共形贡献**：开源原生多模态的系统化样板，把 §25 的几块分散洞见整合成一套可复用的训练配方。三大关键技术：

**V2PE（Variable Visual Position Encoding）**：视觉 token 的位置编码应随分辨率 / 长宽比 / patch 数量动态调整，不再是文本同一套 RoPE 直接沿用。这一技术直接让长上下文多模态（长文档 / 视频等任务）性能显著提升。

**混合偏好优化（MPO）**：结合 SFT + 偏好对齐的统一目标，相比传统 DPO / RLHF 在多模态任务上更稳定。

**测试时规模化 + VisualPRM**：推时用 best-of-N 采样 + VisualPRM（Visual Process Reward Model）筛选推理路径。**提时规模化的思路第一次被系统化地应用到多模态场景**。

结果方面：InternVL3-78B 在 MMMU 上得 72.2（GPT-4o 70.7），在 MathVista 上 79.0（GPT-4o 63.8）。**更重要的是它的文本能力超过基座 Qwen2.5-72B**：原生多模态预训练不仅没损害语言能力，反而提升了它。这是对 Chameleon 的「视觉 ≠ 损害语言」结论的 2025 版本再确认，而且正面方向。

🔗 [https://arxiv.org/abs/2504.10479](https://arxiv.org/abs/2504.10479)

## 3 · Qwen2.5-VL —— *arxiv 2502.13923* (Qwen Team, Alibaba, 2025/02)

**共形贡献**：工程化规模化的杂作。在图像与视频综合多模态上推动一批提升经验技术的成熟：

**视频 + 动态 FPS 采样**：视频不再以固定帧率采样，而是**在空间与时间两个维度上动态采样**，快动作就密采、慢动作就疏采。直接给出长视频理解的一条新直路。

**MRoPE（Multi-Resolutional Rotary Position Encoding）**：把 time-id 与绝对时间维下对齐对齐，事件节奏 / 时刻定位的捕捉从「第几帧」推到「哪一秒」。这是纯视频多模态在时间维的 MRoPE；对比纯语言 RoPE 在位置维度的突破。

**ViT 窗口注意力**：ViT 内部使用窗口注意力而不是全局注意力，即配合高分辨率的训练。这与 MM1 的发现 A（分辨率 >> connector 架构）形成对皮——既然分辨率决定性能，那就突破 ViT 的计算瓶颈来支持高分辨率。

论文中的一个秀观察：**在同样训练 FLOPs 下，动态 FPS + MRoPE 对视频任务的提升大于将 LLM 从 7B 扩到 72B 的提升**。这是 MM1 发现 C（视觉编码器 > LLM）在时间维的推广：对视频任务，**时间编码机制 > 语言模型规模**。

🔗 [https://arxiv.org/abs/2502.13923](https://arxiv.org/abs/2502.13923)

## 4 · QvQ-72B-Preview + LlamaV-o1 —— *Qwen Team (2024/12) + Thawakar 等, ACL 2025*

**共形贡献**：把推理时规模律（test-time compute scaling）从纯文本迁移到多模态。

**QvQ-72B-Preview（Qwen, 2024/12）**：直接在视觉输入上做 long chain-of-thought 推理（类似 OpenAI o1 的风格，数千 token 的内部独白）。基于 Qwen2-VL-72B + RL + 推理数据微调。评测上：在 **Visual Reasoning Evaluation of Grok, DeepSeek Janus, Gemini, Qwen, Mistral, and ChatGPT**（arxiv 2502.16428）里，QvQ-72B 的拒绝准确率 85.5%（满分指标，表示被问到无答案的问题时不胡猜）高于 ChatGPT-o1 的 70.0%。

**LlamaV-o1（MBZUAI, ACL 2025）**：提出专门的多步视觉推理基准 VCR-Bench，同时给出一个开源的视觉推理模型，在 MMStar / MMBench / MathVista / Hallusion 多个标准 benchmark 上超过 Gemini-1.5-Flash / GPT-4o-mini / Llava-CoT。与 QvQ 的区别：LlamaV-o1 的推理步骤是**分段的**（每一步一个小目标）而不是连续独白，这让视觉 grounding 在链上更稳。

**两者的共形意义**：「视觉指令微调」进化到「视觉推理微调」。§24 学的是「听指令」，§26 这一侧学的是「听问题 + 做长推理」。关键识别：这个扩展直接致敗于推理持续时视觉 grounding 的问题——这是第 5 篇的主题。

🔗 [https://huggingface.co/Qwen/QVQ-72B-Preview](https://huggingface.co/Qwen/QVQ-72B-Preview)、 [https://arxiv.org/abs/2501.06186](https://arxiv.org/abs/2501.06186)

## 5 · Decoupling Reasoning and Perception —— *arxiv 2509.23322* (2025/09)

**共形贡献**：系统性地给出了第 4 篇两个模型背后的核心问题的实证观察——**推理链加长会让 MLLM 渐步丢失视觉 grounding**。论文的关键观察以 QvQ-72B 为案例：

**观察**：在视觉推理任务开始，QvQ 能准确描述图像的核心元素；但随着推理链展开，模型逐渐**丢失与原始视觉输入的联系**，开始在文本空间里自我消耗，甚至幻觉出图里不存在的细节（论文里的例子：一个缺失图案推理任务中，模型在第 3–4 步后幻觉出一个「阴影」来描述被填充的象限，后续推理全部基于这个不存在的细节）。

**原因框架**：推理链越长，模型就越依赖自己已生成的文本 token 进行下一步预测，原始视觉 token 在 attention 中的**相对权重指数下降**。这与 §24 末尾的核心命题「视觉推理仍是 LLM 文本推理，视觉只是初始条件」提供了**直接的可观测经验证据**：推理链越长，则这个“初始条件”的影响越衰减。

**论文提出的解**：将推理与感知解耦 —— 一个 LLM 负责推理链，另一个 LMM 在需要时被调用重新感知原图——这在理论上承认了 §24 末认定的方程式，且把解决的提交到「规范流程而非架构」层面。

🔗 [https://arxiv.org/abs/2509.23322](https://arxiv.org/abs/2509.23322)

# 关于 §26 五篇的机制追问

## 一、原生多模态的 scaling laws 到底给出了什么

### Shukor 2025 的三组结论的综合影响

第一组（晚期融合 ≈ 早期融合）：**业界默认方案没有到目前为止所假设的不可替代性**。过去一年半，frontier labs 的标准配方都是 ViT + 连接器 + LLM（晚期融合）。Shukor 2025 对比同 FLOPs 下两条路线——早期融合模型（无 ViT / 无连接器 / patch 直接投影）和晚期融合模型在大每个参数规模点上曲线行为上相似。推翻的意义：**框架不该是架构选择的优先依据**，工程上更太嗨泛视的选择（早期融合部署更简单、样本效率相当）推成新默认。

第二组（MoE 模态无关路由 > 模态感知路由）：**模态冲突不需要通过架构解决**。§25 末残差第 1 条说模态冲突未解——DeepSeek Janus 用双 encoder 的方式解决（架构让图一个“理解视觉”、另一个“生成视觉”分工）。Shukor 2025 提出替代方案：用 MoE 加模态无关路由，让路由自己学会「这个 token 适合哪个专家」，关键是**路由的判断基于语义而不是基于模态标签**。实验上模态无关路由从头到尾赢。这一条对 §25 末残差第 1 条的意义是：**模态冲突的解是路由策略而不是架构层面的分工**。

第三组（多模态的数据-参数比率 > 纯语言的 Chinchilla 比率）：**多模态模型需要比纯语言模型更多的数据**。这与直觉不完全一致——一些人先验地认为多模态应该「毫无图交写之力」处于前模型为知识富含。但图像 token 噪声更大、语义密度更低，需要更多样本来撈取同等的语义信息。这一观察直接影响下一批前沿模型的数据工程优先级。

### Shukor 2025 与 MM1 / VILA 的层级关系

- MM1 / VILA（2024）：深入研究**在传统晚期融合架构内**执行最佳配方。结论本身受架构假设约束。
- Shukor 2025：跨**架构**比较，解放了架构假设。结论情况应用范围更广。

没有矛盾。类似于 Chinchilla scaling laws 与具体模型的 best-practice 之间的层级关系——后者在前者所定的框架内调优。

## 二、InternVL3 的三件事合在一起做的那件事

把三件技术分开看：

**V2PE（绿字用 Chen 等 2024）**：觉程位置编码随视觉 token 数量动态调整。解决的问题：**长上下文多模态中，视觉 token 和文本 token 数量差异巨大**（一张高分辨率图可能是数千 token，一句话仅数十 token）。若使用同一套 RoPE，会导致位置标 id 迅速用尽，模型无法理解跞以增量上下文。V2PE 让视觉 token 的位置编码增量更粗粒度，模型在长上下文多模态上取得显著提升。

**MPO（Mixed Preference Optimization）**：SFT + DPO 统一目标。传统多模态对齐是 SFT → DPO 两阶段，会有阶段间的漂移和失调。MPO 把两者糕合成单目标，在多模态上性能稳定。类比于 NLP 侧：正如 InstructGPT 演化到 Direct Preference Optimization，MPO 是这条路线在多模态上的同构物。

**测试时规模化 + VisualPRM**：推理时从模型采样 N 个解，VisualPRM（独立训练的过程奖励模型）为每个解的每一个推理步骤打分，选出得分最高的解。闭合跟随 o1/R1：**推时计算不仅是更长的个体推理链，也包括并行采样 + 过程奖励模型筛选**。这一策略首次被系统性地应用到多模态场景。

### 三件事合起来实现的一件东西

**把 §25 的一些工艺数字变成可复现的工程模板**。VILA 、MM1 的消融结论是「一个平台、一种配方」的解读； InternVL3 把它们合一处理成「一个开源模型 + 一套工具链 + 一条能在简约硬件上复现的推时 流水线」。这类似于 §24 的 LLaVA-1.5 对 LLaVA 的角色——从概念验证到可复现工程模板的工程化跃迁。

### 为什么 InternVL3 值得专门一段

InternVL3 主要是一篇系统整合而非原创结论的工作，但它的重要性不在理论创新而在「**在这一年里把多模态大模型的工程模板结构就结洞了**」。与 InternVL3.5（2025/08）的 Cascade RL 进一步下抚推理能力。

## 三、Qwen2.5-VL 的时间维架构突破

### 从空间到时间

§23 / §24 / §25 的 ViT-based 模型都有一个集高点：**时间维的处理用的是「给每一帧过 ViT」 + 「拼接成序列送进 LLM」**。这条路的问题：

- 固定 FPS 采样在快动作场景（体育 / 调低锯四调入口）丢帧；在静态场景（讲座视频）浪费计算。
- 一秒钟 30 帧 vs 60 帧的时间定位描述跟送入模型的 token 下标没有对应关系，模型要裂开「第几帧」和「第几秒」靠纯文本上下文。

### Qwen2.5-VL 的两个协同修改

**动态 FPS 采样**：模型依据视频内容动态決定每段的采样密度。原理：快动作部分 motion energy 高，采样密。慢动作部分 motion energy 低，采样稀。输入到模型的 token 总数保持可控而视频的关键信息不丢失。

**MRoPE（multi-resolutional RoPE）**：位置编码维度扩展到**时间维上是给每一帧分配一个「绝对时间 id」，而不是「序列位置 id」**。这样，在 1 秒钟还是 2 秒钟的事件位置是模型可直接看到的，不需要用文本 token 作中介转接。

### 为什么时间维的处理对 2025 这里变得必顶好

2024 前的多模态主要绩阵是**静态图像**。但互联网的视频数据量远超静态图像，而且 agent / embodied AI / 真实使用中的需求也以视频为主。Qwen2.5-VL 在 2025 初给出的时间维架构突破直接决定了后续的多模态方向——**从图文班底到视频班底的转向**。

重要的浙骡观察（论文 Table 7）：**在同样的训练 FLOPs 下，动态 FPS + MRoPE 对视频任务的提升大于将 LLM 从 7B 扩到 72B 的提升**。这是 MM1 发现 C 在时间维上的类似结论：**多模态下的特殊机制（视觉编码 / 时间编码）的收益 > LLM 规模的收益**。

## 四、推理时规模律迁移进多模态：不是简单复制 o1 的亨步

### o1 / R1 引爆的范式转移

OpenAI o1（2024/09）和 DeepSeek-R1（2025/01，Nature 2025）确立了一个新范式：**从扩大训练时计算转移到扩大推理时计算**。不再是「更大的模型 + 更多数据」，而是「让模型在推时多想一会」。这一范式转移到多模态有两条技术路线：

**QvQ 的连续独白风格**：模型在输出里直接打出数千 token 的推理，总建等似一个内部独白，最后给出答案。训练机制：推理过程数据 + RL。

**LlamaV-o1 的分段式推理**：模型推理被分成明确段落，每一段是一个子目标，两段间有明确的上下文表达。

### 这两条路线的真正差异

看似是风格上的区别，实际上跟视觉 grounding 保持有关：

- 连续独白风格（QvQ）：模型在推理中没有重新回看图像的点。一旦初始视觉 token 在深层 attention 中被冲淡，后续推理全部基于已生成的文本 token。
- 分段推理（LlamaV-o1）：每一段为一个子目标，开始新段时模型重新 attention 到当前的上下文 + 视觉 token，随部分恢复视觉 grounding。

### 一个死範数据：拒绝准确率

Visual Reasoning Evaluation 标准 benchmark（arxiv 2502.16428）：

- ChatGPT-o1：总准确率 82.5%、拒绝准确率 70.0%
- Gemini 2.0 Flash： 70.8%
- QvQ-72B-Preview：拒绝准确率 **85.5%**（各模型中最高）

拒绝准确率的含义：当题目没答案（如图像无法回答的细节）时，模型说「不知道」而不打乱讲的比例。这一指标在 2025 已经是多模态模型评估的第一等指标——体现的是模型是否能知道自己不知道。QvQ 在这一项上赢 o1，是开源视觉推理模型的一个里程碑。

但这并不意味着已解决“视觉推理独立性”的问题——第 5 篇的内容。

## 五、核心反例：推理链加长会让视觉 grounding 逐渐脆弱

### Yan 等 2025 的关键观察

*Decoupling Reasoning and Perception*（arxiv 2509.23322, 2025/09）提供了 §24 末残差第 1 条「视觉推理 = LLM 文本推理 + 初始条件」的**直接经验证据**。论文使用 QvQ-72B 训练实例进行定量分析：

观察 1 · **视觉推理链的前 3–5 步：QvQ 的视觉描述精准率 > 90%**。模型能准确识别图中的核心元素、布局、颜色等特征。

观察 2 · **推理链的第 10–15 步：精准率降至 ~60%**，开始出现「幻觉细节」——模型声称图中存在实际不存在的元素。

观察 3 · **推理链的后半部分（>20 步）：精准率降至 ~30%**，模型已经完全依赖其自身生成的文本 token 继续推理，**原始视觉 token 在 attention weights 中的份额降至起始值的 10% 以下**。

观察 4 · **幻觉的一致性而非随机性**：一旦模型在第 5 步幻觉出了某个细节，后面的推理会一直基于这个幻觉的细节。这是一个「**错误信息在文本链里被复制放大**」的过程，而不是随机漂移。

### 理论同向的解释

§24 / §25 的结论。推理链越长，**模型的下一步预测越依赖文本 token**（因为它们数量多、每一步更新、徽标 attention 权值高），**原始视觉 token 尽管仍在上下文里，实际被 attention 计算释徕放下了**。这是「视觉被当作初始条件」的结构性后果。

### 这一观察的深层含义

**对 §26 的**：推理时规模律并没有跳过 §24 / §25 的残差。相反，它**放大**了残差：推理链越长，视觉 grounding 越脆弱。

**对 V 反身度量的**：当我们说「模型会视觉推理」时，实际上我们在说「模型的文本推理链在前中期仍被原始视觉输入指导」——这种指导有直接可观测的衰减曲线。V 反身度量会重新对待一个问题：我们在评估多模态模型时使用的视觉推理 benchmark，大部分测试的是**短链视觉推理**（前 3–5 步就结束的推理）。在短链里视觉 grounding 仍强。长链表现有没有被充分评估？

**对 IV 生成对齐的**：如果基础模型在 20+ 步后已丢弃了视觉 grounding，那么下游任何需要长链视觉推理的任务（专家审论 / 视频分析 / 多步空间推理）都会继承这个问题。这不仅是推理模型的问题，也是**连接 §26 与 IV 的桥**。

## 小结：§26 的谱系内逻辑

- **Shukor 2025**（Apple + Sorbonne，ICCV 2025）：原生多模态的第一块规模律里程碑。主要结论：早期融合 ≈ 晚期融合、MoE 模态无关路由赢模态感知路由、多模态比纯语言更数据饥渴。
- **InternVL3**（OpenGVLab，2025/04）：开源原生多模态的工程化整合——V2PE + MPO + VisualPRM 测试时规模化；多模态预训练反而提升基座文本模型的语言能力。
- **Qwen2.5-VL**（Alibaba，2025/02）：时间维架构突破：动态 FPS + MRoPE + 窗口 ViT。时间编码机制 > LLM 规模的发现是 MM1 发现 C 在视频侧的延续。
- **QvQ-72B-Preview + LlamaV-o1**（2024/12 + 2025 ACL）：推理时规模律从 o1 / R1 迁移到多模态，两条技术路线（连续独白 vs 分段推理）。QvQ 在拒绝准确率上超 o1。
- **Decoupling Reasoning and Perception**（Yan 等 2025）：揭示推理链加长 → 视觉 grounding 渐步丢失——这为 §24 / §25 末的核心命题「视觉是被一个初始条件的角色出演的」提供了直接可观测的计量性证据。

**五篇合起来定义了 2025 多模态的三股同时向前的力量**：训练时规模律的系统化、开源的工程化整合、推理时规模律的多模态迁移。但同时，推理时规模律最新的观察问中，最核心的残差（视觉 grounding）反而暴露得更充分。§26 不是「解决了 §25 的问题」的章，而是「把问题从工艺问题提升为可测量问题」的章。

## 本组合力的残差

1. **推理链越长，视觉 grounding 越脆弱**——Yan 等 2025 的观察 不仅是一个技术问题，而是 §24 末残差第 1 条「视觉推理 = LLM 文本推理 + 初始条件」的直接可观测证据。当我们让模型「多想一会」时，模型思考的就越不是原始图像、而是自己对原始图像的文本描述。这条残差**直接汇入 V 反身度量**——我们需要重新审视评估多模态推理的标准，大部分现有 benchmark 测的是短链推理，长链推理的系统性脆弱被掩盖了。
2. **多任务 / 多模态的统一 scaling law 仍未完成**。Shukor 2025 给出了理解侧（text + image → text）的规模律；但统一理解与生成的规模律、视频/音频/3D 的规模律、推理时计算与训练时计算的综合最优点 —— 这些仍未系统化。比如 InternVL3 的推理时规模化 + MPO 两件事并行的最佳配方、推理时计算与训练时计算的权衡仍是经验侧打磨。这条残差**汇入 §27 账单**。
3. **模态冲突在 MoE 路由下缓解但未消除**。Shukor 2025 的 modality-agnostic routing 证明了路由策略可以承载大部分模态分工，但极端模态冲突（如理解 vs 像素级生成）仍有 DeepSeek Janus 的结果显示专用架构胜。绘给出了“单一 backbone + 路由学习分工”是否像纯语言那样是单曲线，或是多曲线？没有系统性回答。这条残差也**汇入 §27 账单**。

第 1 条汇入 V 反身度量；第 2 、 3 条汇入 §27 账单。**§26 是阶段 III 的收束章**，它把阶段 III 中緈累的所有技术账单估算清楚，把未解决的核心谜团（视觉推理独立性）计量化，并把他们传递到 §27 与 V。
