# §23 · 机制展开：语言如何改写稠密输出的结构

§16 已把 Grounding DINO / APE / CAT-Seg 等作为“稠密共形的出口节点”引入。本章的视角转换是：§16 问“类别闭集如何被打开”，§23 问“语言如何改写稠密输出的内部结构”。

---

## 一、三种注入位置

| 注入方式 | 语言进入哪里 | 代表 | 闭合对象 | 残差 |
|---|---|---|---|---|
| text-as-classifier | 输出端分类头 | LSeg、OpenSeg、CAT-Seg | 每个位置与文本类别原型对齐 | 语言仍像外挂分类器 |
| text-as-query | decoder / DETR query | MDETR、OWL-ViT、Grounding DINO | 文本主动从 dense feature 中索取对象 | query 与真实对象绑定仍需 matching 制度 |
| text-as-region-prompt | 匹配端区域短语 | GLIP、GLIPv2 | phrase 与 box / region 共同训练 | 指代歧义和文本偏见进入检测 |

语言进入输出链越早，灵活性越高；但训练越难，歧义越大，越需要更强的数据和 assignment 制度。

---

## 二、代表路线

| 工作 | 机制贡献 | 本体位置 |
|---|---|---|
| GLIP | 把 detection 与 phrase grounding 写成同一个 grounding 任务 | text-as-region-prompt |
| OWL-ViT / OWLv2 | 用文本 embedding 作为开放词汇检测分类器 | text-as-classifier / query 之间 |
| Grounding DINO | 把 DINO 的 query / matching 制度与 GLIP 的语言 grounding 合并 | text-as-query |
| CAT-Seg | 用 CLIP cost volume 做 dense prediction | text-as-classifier |
| FC-CLIP | 把 CLIP 全卷积化，推进 panoptic 统一 | text-as-classifier |
| APE | 试图用 all-purpose embedding 统一 detection / grounding / referring | embedding-unification |

这些路线不是“谁替代谁”的单线史，而是在回答同一个问题：语言到底作为输出分类器、解码查询，还是区域匹配协议进入稠密视觉。

---

## 三、CLIP dense 化的真实难点

CLIP 训练的是全图-整句匹配。它没有 pixel-level 监督，也没有要求 patch token 与文本空间局部对齐。直接把 CLIP 用于 dense prediction，常见问题是“语义塌陷”：所有位置都倾向图像主体，局部属性和小对象被全图语义吞掉。

CAT-Seg、FC-CLIP、SAN 等路线本质上都是在补 CLIP 的空间盲：要么构造 cost volume，要么全卷积化 CLIP，要么用 side adapter 保留原始 CLIP 能力同时补 dense head。

---

## 四、开集检测、开集分割、几何任务的差异

开集检测相对容易，因为 box 是几何原语，开放语义主要进入分类头或 query 匹配。开集分割更难，因为 mask 本身携带语义边界，语言不只要说“是什么”，还要塑造“哪里属于它”。

深度、法线、对应等连续场任务更特殊。它们不天然需要类别名，也不天然落入语言 query。把这些任务强行写成语言条件任务，往往会遮蔽它们更接近阶段 V 的事实：几何闭合可能主要依赖视觉表征自组织。

---

## 五、Depth Anything 是边界节点

Depth Anything 不应被简单归入“语言条件几何”。它更像阶段 III 内部的反证节点：深度、几何和连续场任务，可能更依赖强视觉 SSL、伪标签和视觉数据规模，而不是语言对齐。

这说明阶段 III 的语言契约不是所有视觉任务的终局。对于几何、对应、状态和世界持有，阶段 V 的纯视觉表征可能更接近根本解。

---

## 六、APE 的预测地位

APE / embedding-unification 路线具有长期扩展潜力，因为它试图把 detection、grounding、referring 都纳入同一 embedding 匹配制度。但这仍是预测，而不是事实结论。它需要接受跨域、长尾类别、多任务迁移和强 prompt pipeline 的反证。

可以登记为预测：

```text
P-III-1:
embedding-unification 型开放词汇检测 / grounding，
长期会优于任务专用 prompt pipeline。

可证伪方式:
跨任务、跨域、长尾类别迁移评估中，
embedding-unification 是否稳定优于专用 prompt pipeline。
```
