## §0 · 本体命题

> 稠密任务的类别是无限长尾,标注带宽是有限的——必须把语言作为类别空间本身,才能让稠密共形的出口打开。
> 

§23 的本体不是 open-vocabulary detection 本身，而是公共语义从全图向量返回投影表面的过程。它把 §22 的 caption-stabilized visual subspace 拉回框、mask 和 region，但仍没有让视觉拥有状态主权。

## §1 · 承接点

- 阶段 II §16 把这几个工作作为「稠密共形的出口节点」引入(引子)
- 本章作为正文:重点从「如何打开类别闭集」转为「**语言如何改写稠密输出的结构**」
- §22 的 CLIP 提供语言编码器;本章把它落到像素 / region / depth

## §2 · 语言注入稠密任务的三种方式

| 注入方式 | 语言扮演什么 | 代表工作 | 本体位置 |
| --- | --- | --- | --- |
| text-as-classifier | 分类头的权重向量 | LSeg / OpenSeg / CAT-Seg | 语言在输出端 |
| text-as-query | DETR / decoder query | OWL-ViT / Grounding DINO / MDETR | 语言在解码端 |
| text-as-region-prompt | 检测阶段的词级区域匹配 | GLIP / GLIPv2 | 语言在匹配端 |

三种方式的深层差异:**语言进入视觉输出链的位置越靠前,灵活性越高,但训练越难**。

## §3 · 开集检测线

- **GLIP**(2022,Li et al.):phrase grounding + detection 统一为同一任务
- **Grounding DINO**(2023):DINO + GLIP 思路,真实 zero-shot 检测的代表
- **OWL-ViT / OWLv2**(Google):ViT + detection head,一步到位
- **YOLO-World**(2024):把 GLIP 思想推到实时
- **APE**(2024):All-Purpose Embedding,统一检测 / grounding / referring

## §4 · 开集分割线

- **LSeg**(2022):最早的语言条件分割
- **CAT-Seg**(2023):把 CLIP cost volume 用来做 dense prediction
- **FC-CLIP**(2023):CLIP 全卷积化,一个模型做 panoptic
- **ODISE**(2023):借 Stable Diffusion 的隐空间做分割
- **SAN**(2023):side adapter,不改 CLIP 本体
- **SEEM**(2023):支持 text / click / box / mask 多种 prompt

## §5 · 语言条件几何

- **Depth Anything v1/v2**(2024):不应被简单归入“语言条件几何”。它更像阶段 III 内部的边界节点:即使在语言共形时代,深度 / 几何这类 dense task 仍可能主要依靠强视觉 SSL、伪标签和视觉数据规模,而不是语言对齐。
- 这条路提示阶段 V:几何类 dense task 可能说明纯视觉 SSL / dense feature 比语言对齐更干净。

| 任务类型 | 是否天然需要语言 | 更接近阶段 |
|---|---|---|
| 开放词汇检测 | 需要 | III |
| referring / grounding | 需要 | III |
| 开放词汇分割 | 需要，但受空间盲限制 | III / II |
| 深度 / 法线 / 对应 | 不天然需要 | V |
| 变化状态 / 多时相 | 不只需要语言，更需要状态 | V / 应用域 |

## §6 · 真假难点

- **表面难点**:「zero-shot 分割」「开集检测」
- **真实难点**:**如何让 CLIP 特征在空间维度上不崩**
    - CLIP 本身是全图对齐,没学过空间定位
    - 直接用 CLIP 做 dense 会出现「语义塌陷」——所有位置激活都指向图像主体
    - CAT-Seg / FC-CLIP / SAN 的各种 trick 本质都是**补 CLIP 的空间盲**

## §7 · 三种路线的本体差异

- **Grounding DINO**:语言作为 query,bbox 作为输出——「语言→位置」
- **OWL-ViT**:每个 patch 独立分类,语言作为分类器——「位置→语言」
- **APE**:embedding space 统一,所有任务都是 embedding 匹配——「语言⇌位置」

**预测性判断**:APE / embedding-unification 路线具有长期扩展潜力,因为 embedding 统一更容易跨检测、grounding、referring 和开放词汇迁移。但它是否会在长期评估中胜出应登记为预测,不能在正文中当成既成事实。可证伪方式:跨任务、跨域和长尾类别迁移评估中,embedding-unification 是否稳定优于任务专用 prompt pipeline。

## §8 · 出口残差

- 语言可以**定位**像素,但不能**生成**像素 → §24 的统一骨干试图用一个模型同时做理解 + 生成(未完全成功,最终让位给阶段 IV)
- 语言条件稠密仍受 CLIP-blindness 限制 → §27
- Dense prediction 的 SSL 路线(DINOv2 / JEPA)反而更干净 → 阶段 V

## §9 · 一句话收束

> 语言锚定稠密任务不是「稠密任务长出了嘴」,是**视觉的空间维度被迫借用语言的分类维度**——它解决了类别闭集,没解决空间本身。
>
