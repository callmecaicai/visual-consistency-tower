## §1 · 为什么 §35 必须出现

§34 结尾留下的问题是:CFG 给了“以一个 `c` 为等效接口”的通用语法,但这个 `c` 到底该长什么样?

- 类标签(ImageNet 1000 类)?太书——只能表达 1000 种意图
- one-hot 向量?有限、不组合
- 图像本身(image-to-image)?能做风格迁移,但不能从零创造

人类自己表达意图的最常用方式是什么?**自然语言**。语言具备几条无与伦比的质:

- **无穷组合性**:“穿红色大衣 + 宇航员 + 火星 + 风沙 + 下午光”中每个概念可自由搭配
- **抽象层次**:从物体(猫)到属性(蒙时光)到风格(点彩派)到情绪(孤独)
- **人类天然会用**:不用培训
- **可翻译**:中文 / 英文 / 西班牙文 → 同一意图

§35 的必然论断:**用自然语言充填 `c`,生成主体就与整个人类文化的表达系统对接。**我们不是在训一个更好的分类器,而是给生成主体装了一个**听觉器官**。

但要让这个听觉官真正起作用,需要四件事:

1. **把语言变成稠密向量**(CLIP / T5)
2. **把向量注入扩散网络**(cross-attention)
3. **放大这个向量的影响**(CFG 的文本版)
4. **让人类会写 prompt**(prompt engineering,后来交给 LLM 扩写)

§1 到 §8 依次拆完这条管道。

## §2 · 文本编码器——CLIP vs T5

文本编码器的职责:把一句人话(一串 token)变成一个或一组稠密向量,让扩散网络可以消化。两大派系在 2021/2022 并行诞生。

### 2.1 CLIP(Radford et al., OpenAI 2021)——图文对齐派

**核心思想**:收集 4 亿对 (图, 文本) 配对(网络爬取的 alt-text),训两个网络:

- 图像编码器 `f(x)`(ViT 或 ResNet)
- 文本编码器 `g(t)`(Transformer)

损失函数是 **InfoNCE 对比学习**:

$$
L = -\frac{1}{N}\sum_{i=1}^{N}\log\frac{\exp(\langle f(x_i), g(t_i)\rangle / \tau)}{\sum_{j=1}^{N}\exp(\langle f(x_i), g(t_j)\rangle / \tau)}
$$

**人话**:一个 batch 里 `N` 对 (图, 文) 配对。让模型把“配对的图与文”的内积拉高,“乱匹配的图与文”的内积拉低。`τ` 是温度参数(控制而对比的锐度)。

**训成之后**:图和文的嵌入落在**同一个空间**中,且语义对齐。“猫的照片”的图向量 ≈ “a photo of a cat” 的文向量。

**CLIP 对扩散的意义**:它提供了一个图文对齐的多模态语义空间。把文本推进这个空间,得到的向量天然“带着图像语义”。DALL·E 2 / SD1.5 / SDXL 用的都是 CLIP 文本编码器。

### 2.2 T5(Raffel et al., Google 2020)——纯语言派

T5 是一个纯文本 encoder-decoder 模型(在《C4 语料》预训),与图像没任何关系。较大的版本 T5-XXL 是 11B 参数。

**关键不同**:T5 只用过文本,没在图像上对齐过。它的语言理解“纯”而深。

Imagen(Google 2022)发现了一个冲击性的结论:**用 T5-XXL 的文本编码器比用 CLIP 文本编码器生成的图更好——且扩大图像网络计算不如扩大文本网络来得有效**。换言之:文生图的瓶颈在文,不在图。

### 2.3 两派对照

| 尺度 | CLIP(图文对齐) | T5(纯语言) |
| --- | --- | --- |
| 训练任务 | 对比学习 | span masked language modeling |
| 语言深度 | 中等(短 caption 训) | 深(长文本预训) |
| 语义与图距离 | 已对齐 | 完全无关——由扩散网络自行搭桥 |
| 代表作品 | DALL·E 2, SD1.5, SDXL | Imagen, Parti, SD3 |

**SDXL 的妥协**:同时用 CLIP-ViT-L + OpenCLIP-ViT-bigG 两个编码器,拼接嵌入向量。**SD3 的最终解**:三个编码器拼一起——CLIP-L + OpenCLIP-bigG + T5-XXL,同时获得多模态对齐与深语义。

## §3 · cross-attention——文本如何进入扩散网络

有了文本向量,怎么“注入”到扩散网络?标准答案是 **cross-attention**。

### 3.1 cross-attention 的本质公式

self-attention 是文本内部 token 互看。cross-attention 是**图像特征向文本特征看**。

设:

- 图像侧特征 `h \in \mathbb{R}^{(H×W) \times d}`(每个空间位置一个向量)
- 文本侧特征 `c \in \mathbb{R}^{L \times d}`(`L` 个 token,每个一个向量)

三个投影:

$$
Q = h W_Q, \quad K = c W_K, \quad V = c W_V
$$

注意:**Query 来自图像**,**Key / Value 来自文本**。

$$
\text{CrossAttn}(h, c) = \text{softmax}\left(\frac{Q K^\top}{\sqrt{d}}\right) V
$$

**人话**:

1. 每个图像空间位置问每个文本 token 一道问题——“我这个位置应该与你多相关?”(`Q K^\top`)
2. softmax 得到权重分布——这个位置与每个 token 的注意力
3. 拿这个权重从文本侧 `V` 加权重汇汇闭,返回给这个图像位置

直观:“红色大衣”这个词会在角色身体对应的空间位置激活,“火星表面”在背景位置激活。Cross-attention 将文本语义**空间化地裁剪**到图像结构中。

### 3.2 扩散 U-Net 里的文本注入点

Stable Diffusion 的 U-Net 在每个层级都插入了 cross-attention 层:

$$
h_\text{new} = h + \text{CrossAttn}(\text{SelfAttn}(h), c)
$$

翻译:一个 block 会连续做“自己看自己”(self-attention,整合空间关系)和“自己看文本”(cross-attention,整合文本条件)。结果跟输入相加再进下一层。

**关键细节**:注意力矩阵 `\text{softmax}(QK^\top/\sqrt{d})` 的尺寸是 `(H×W) \times L`——即“每个图像位置 × 每个文本 token”。这是**prompt-to-prompt 编辑**的数学基础——想把“猫”改成“狗”,只要在替换文本 token 后,把旧注意力矩阵的空间心 copy 过来,就能保持空间结构只改属性。

### 3.3 为什么是 cross-attention 而不是 concat / AdaIN

历史上有三种注入方案试过:

- **Concat**:把文本向量平铺与图像拼接——简单但损失空间结构
- **AdaIN / FiLM**:用文本向量去调整图像特征的均值 / 方差——能调整全局特征但不能对应空间位置
- **Cross-attention**:每个空间位置独立跟文本互动,最富表达

Stable Diffusion 的测试表明 cross-attention 质量显著赢,今日不争议。DiT 与 MMDiT 将 cross-attention 升级为“图文 token 统一 attention”,但仍是 attention 家族,本质未改。

## §4 · 文生图第一代——DALL·E 1(2021)

在扩散成熟之前,文生图已经在走路。DALL·E 1(Ramesh et al., OpenAI 2021 年初)是第一个在公众看来“最么是行”的演示。架构:

- dVAE:把图像压为离散 token(类似§33 VQ-VAE)
- GPT 风的自回归 Transformer:`[文本 tokens; 图像 tokens]` 一条序列,从左到右生成

效果:能产生趣味性(“牧翊上的牛沿牌钢琴”)但质量一般,大部分时候解剖模糊。

**历史意义**:它证明了**文本调度图像生成是可行的**,在业内引起尾环(Google Parti 仍是这条线的后裔)。但文生图真正的爆发要等扩散的加入。

## §5 · 文生图第二代——DALL·E 2 / Imagen / Parti(2022)

2022 年是文生图的爆发年。三大路线同时出现,代表三种哲学。

### 5.1 DALL·E 2(Ramesh et al., OpenAI 2022 年4 月)——unCLIP 架构

官方名称是 **unCLIP**,因为它在概念上是 CLIP 的反操作。两阶段:

**阶段一—Prior**:训一个扩散模型 `p(z_i \mid t)`,从文本嵌入 `z_t = \text{CLIP}_\text{text}(\text{prompt})` 生成 CLIP 图像嵌入 `z_i`。

**阶段二—Decoder**:训一个扩散模型 `p(x \mid z_i)`,从 CLIP 图像嵌入生成真图素。

采样:文本 → CLIP text embed → (prior 扩散) → CLIP image embed → (decoder 扩散) → 图。

**为什么这样那么绕**?因为 CLIP 图像嵌入空间是语义严格的中层。直接从文本到像素跳过这个中层,模型要同时学习“将词解释为图像语义” + “将语义化为像素”两件事,很难。unCLIP 把它分为两步——主要负担由 prior 光明地学习。

**特点**:图像属性由 CLIP 嵌入决定,图质良好;但结构细节(文本拼写、数量关系)既 CLIP 本身在这些位置上也弱,故 DALL·E 2 突出特色不在编辑等可控性。

### 5.2 Imagen(Saharia et al., Google 2022 年5 月)——T5-XXL + 级联扩散

**价值直接冲击的工程洞见**:**文本编码器的效能比图像网络的容量重要**。用 T5-XXL(11B)做文本编码器,FID 响亮提升,特别是对细腻描述(方位、数量、属性绑定)有撵很好的处理。

**级联扩散**:三个扩散模型串联:

$$
\text{Text}\xrightarrow{\text{基本扩散}} 64\times 64 \xrightarrow{\text{超分 1}} 256\times 256 \xrightarrow{\text{超分 2}} 1024\times 1024
$$

每个模型自行行高响亮的工作。LDM 以“潜空间扩散”解决相同问题,但 Imagen 用像素空间级联的方式证明“原生像素扩散 + 强文本编码器”也能走得通。

### 5.3 Parti(Yu et al., Google 2022 年6 月)——自回归路线的最后工域

Parti 是§33 VQ-GAN 路线的工域版:

- ViT-VQGAN 把图压为离散 token(`32×32 = 1024` 个 token,码本尺寸 `K = 8192`)
- T5-XXL 提供文本条件
- 20B 参数的 encoder-decoder Transformer 写图像 token

**结论**:Parti 在文本跟随性上特别强(能画出“一只精美造型的煞脸火鸡 + 拿着洗号 + 上面写着触角乩鱼星星”这种长长复杂描述),但总体质感输给 Imagen / DALL·E 2。

**历史位置**:Parti 是自回归路线在文生图上的**最后一次强号的尝试**。在它之后,业内全面倒向扩散,自回归路线直到 2024 年 Chameleon / Emu3 等图语统一模型里才重新进入主流视线(那已经在阶段边缘)。

## §6 · 文生图第三代——云化设施(2022-2023)

### 6.1 Stable Diffusion(2022 年8 月)

LDM 的开源化版本。用 OpenCLIP ViT-L/14 文本编码器,不到 8GB VRAM 即可跑通。一夜之间 Hugging Face / Replicate / Automatic1111 / ComfyUI 生态爆炸。

**此版本赢在**:开源 + 足够好 + 能微调。它从此成为业内所有微调 / LoRA / ControlNet / Adapter 级态卷着的得做的实验台。

### 6.2 SDXL(Stability AI 2023 年7 月)

**双文本编码器**:CLIP-ViT-L + OpenCLIP-ViT-bigG 嵌入拼接到 `d = 2048`。文本理解响亮增强。

**其它改进**:

- 更大 U-Net(2.6B vs SD1.5 的 860M)
- 多尺寸训练(支持非正方形比)
- Refiner 模型(二阶段打磨)
- 条件扩充:尺寸、裁剪、开始时间步等条件全部打进 CFG

SDXL 是 2023-2024 开源生态的真正标准线。

### 6.3 DALL·E 3(OpenAI 2023 年10 月)——Prompt 工程化的真正义临界点

DALL·E 3 边聊不是模型架构的革命,而是**两个数据/工艺革命**:

**(1) 极高质 caption 训练**:OpenAI 用 GPT-4V 重新为图像数据集自动生成“长、结构化、描述性强”的 caption。模型学到的“文本 vs 图”对应关系直接准确。

**(2) GPT 自动 prompt 重写**:用户输入简短 prompt(“一只猫”),ChatGPT 改写成详细 prompt(“一只虚木豹猫,结构高高松张,毛发柔亮,有沈思眼神,下午花园背景…”),再交事扩散模型。

**关键洞见**:DALL·E 3 的实际效果飞跃,很大部分来自第二项——**用户写的 prompt 总是太短太模糊,让 LLM 替用户扩写,扩散模型就能充分发挥。**这开转了 2024 年后所有主流图像产品的设计范式:**LLM 为 Prompt 工程师,扩散为画师**。

## §7 · Prompt Engineering——让条件真正可用

### 7.1 Prompt 的典型结构

在开源社区经验变成准模板:

```
[主体描述], [风格], [构图], [光影], [材质细节], [质量增强词]
样例:“A portrait of an elderly fisherman, oil painting style,
centered composition, dramatic side light, weathered skin texture,
masterpiece, ultra detailed, 8K”
```

这个结构跟随模型对语言的实际语义收解。模型训练数据里最常见的 caption 格式就是这种紧凑标签序列。

### 7.2 CFG 与 Prompt 的交互

§34 的 CFG 公式 `\tilde\epsilon = \epsilon_\text{uncond} + w(\epsilon_\text{cond} - \epsilon_\text{uncond})` 在文生图里的翻译:

- `w = 1`:安静地生成一张图,但可能与 prompt 关系不大
- `w = 7.5`:标准值——图与 prompt 严实关联
- `w = 15`:过饱和、色彩崩坏、构图变形

**理解**:prompt 写得精密时 `w` 可调低(模型不用被迫强化);prompt 模糊时需要 `w` 高(忽视条件风险大)。

### 7.3 Negative Prompt——CFG 的对称麻放

替换 `\epsilon_\text{uncond}`(无条件)为 `\epsilon_\text{neg}`(负面 prompt):

$$
\tilde\epsilon = \epsilon_\theta(x_t, t, c_\text{neg}) + w \cdot \bigl[\epsilon_\theta(x_t, t, c) - \epsilon_\theta(x_t, t, c_\text{neg})\bigr]
$$

**翻译**:不再是“朝 `c` 方向放大”,而是“从 `c_\text{neg}` 朝 `c` 的方向放大”。模型会主动**远离负面 prompt 描述的状态**。

经典负面 prompt:"blurry, low quality, deformed hands, extra fingers, bad anatomy"——模型学习而认得往相反方向走。Stable Diffusion 社区的“负面口诀”就是这条机制的工程点照。

### 7.4 Prompt 扩写——DALL·E 3 和新一代工业实践

人类写 prompt 的平均长度约 5-15 词,无法激发模型的全部能力。解决方案:**LLM 扩写**。

一般流程:

1. 用户输入“A cat on a tree”
2. LLM(通常是 GPT-4 或 Claude)扩写成 80-150 字的结构化描述(主体、构图、光影、风格)
3. 把扩写 prompt 交给扩散模型

**这不是辅助功能,这是新一代文生图产品的核心工艺**。DALL·E 3 / Ideogram / Flux.1-dev 的实际追踪能力很大一部分来自 prompt 扩写,而不是模型本身。

## §8 · Adapter 革命——条件接口的插件化

文本只是条件的一种。人类有时想要更具体的控制:“我想要这个姿势”、“按这张设计草图出成品”、“用这个人物的脸”。Adapter 家族将 CFG 条件接口推广到**任意模态**。

### 8.1 ControlNet(Zhang et al., Stanford 2023 年2 月)

**核心想法**:把预训扩散 U-Net 的 encoder **复制一份**,作为条件支线。支线的输入是条件图(边缘 / 深度 / 姿态),输出在每个 decoder 位置相加到主线:

$$
y = \mathcal{F}(x, \theta) + \mathcal{F}_c(x, \theta_c + \mathcal{Z}_1(c; \theta_{z1}); \theta_c)
$$

**关键设计**:条件支线和主线之间的连接是**零卷积**(权重初始化为 0)。微调初期,条件支线的贡献是 0,主线保持预训能力。以后随训练逐渐学会利用条件信号。这是 “**不破坏主网络 + 增量添加条件**” 的干净范例。

**支持的条件模态**:

- Canny 边缘
- HED 軟边缘
- 深度图(MiDaS / Depth Anything)
- 法线图
- 语义分割图
- OpenPose 人体姿态
- Scribble 简笔

**意义**:ControlNet 以后,图像生成成为工业设计的可用工具——设计师画草图 → ControlNet 完成。**Stable Diffusion 从玩具评价为生产工具的转点在此**。

### 8.2 T2I-Adapter(Mou et al., Tencent 2023 年2 月)

轻量条件注入:训一个小的 CNN 给条件图提特征,特征被加到主网络的 encoder 输出。比 ControlNet 小 10 倍,效果略差但够用于大部分场景。

### 8.3 IP-Adapter(Ye et al. 2023 年8 月)——图像作为 prompt

**关键思想**:不仅文本可做 prompt,图像也可以。用 CLIP 图像编码器把一张参考图变成嵌入向量,插入一个额外的 cross-attention 分支,与文本 cross-attention 并行。

人话:`“一个参考图” + “一个文本 prompt” = “像参考图的图 + 文本要求”`。`“这张人物照” + “油画风格”` = “这个人的油画版”。

**IP-Adapter 的产业意义**:人物身份、品牌风格、产品参考——所有“有视觉参考的生成”都因此可行。

### 8.4 LoRA(Hu et al., Microsoft 2021)——个性化微调日常化

LoRA(Low-Rank Adaptation)本身不是为扩散做的,但语法恶恶 fit:在预训权重 `W_0 \in \mathbb{R}^{d \times k}` 旁路加一个低秩矩阵:

$$
W = W_0 + B A, \quad A \in \mathbb{R}^{r \times k},\; B \in \mathbb{R}^{d \times r},\; r \ll \min(d, k)
$$

**人话**:主网络 §W_0§ 冻结,只训 `A, B` 两个小矩阵。参数量从“一亿”降到“几百万”——更换小而快。

**在文生图的度计**:训一个角色 LoRA 仅需 5-20 张图、半小时。训一种风格 LoRA 类似。Civitai 上的几十万个 LoRA 更换是此接口的社会化产物——**个性化生成从学术论文变为日常实践**。

### 8.5 Adapter 家族的总体哲学

| Adapter | 条件什么 | 参数规模 | 典型用途 |
| --- | --- | --- | --- |
| ControlNet | 姿态 / 边缘 / 深度等结构信号 | 大(复制 encoder) | 设计 / 准确构图 |
| T2I-Adapter | 同上但轻量 | 小(10%) | 快迭代 / 移动端 |
| IP-Adapter | 参考图 | 中 | 角色身份 / 风格参考 |
| LoRA | 一小批图定义的语义子集 | 极小(1%) | 个性化角色 / 风格 |
| Textual Inversion | 一批图定义的新 token | 极小(仅 embedding) | 专有概念 / 特定对象 |

> **总体哲学**:条件接口从“文本单一通道”进化为“可组合的插件生态”。一张图的生成可以同时受 “文本描述 + 姿态约束 + 风格参考图 + 角色 LoRA” 四个接口控制。**这是阶段 IV 在工业层面的成熟标志——生成主体开始有“接口单”**。
> 

## §9 · 非 diffusion 对照线——为什么产业选了扩散

公平对待:与扩散并行的文生图路线还有两条。

### 9.1 自回归 + VQ Token(Parti / Chameleon / Emu3)

- 优势:与语言模型架构统一、跟随长 prompt 特别好
- 劣势:采样顺序性,`1024 × 1024` 图§ 16 × 16 = 256 token§ 仍需边生产、质感输扩散
- 今日定位:多模态统一模型的基础训练,不是单点文生图的主力

### 9.2 MaskGIT / Muse(并行 token 解码)

- 优势:8-12 步出图,比自回归快
- 劣势:质感还是略输 SDXL / Flux,且训练复杂度大
- 今日定位:学术路线持续活跃,产业上被扩散 + Consistency Models 覆盖

### 9.3 为什么产业倒向扩散

三个关键原因:

1. **质感**:扩散挂载 `p(x)` 全区域,不模式崩溃;自回归在低概率组合上容易出错
2. **可控**:CFG / ControlNet / LoRA / IP-Adapter 都是在扩散架构上最先实现的,生态积累丰富
3. **工程**:LDM 把计算量打下来了,消费级可跑

自回归在 2024 后以 Chameleon / Emu3 / GPT-4o image 的**图语统一模型**身份重回主流——但那不是“生成质量升”而是“语言与图像在同一 Transformer 里生成”,是阶段边界面的助换。在阶段 IV 本体里,扩散是无争议的主轴。

## §10 · 一步步走向 2025 前沿——从 DALL·E 3 到 Nano Banana

前面的章节按逻辑走完了"文本接口 + Adapter 生态"。这一节按**时间线**把 2023-2025 的主要模型串一遍,让演进脉络清楚——每个模型**多了什么**、**处于哪一代**、**把范式推向何处**。

### 10.1 五代路径图

| 代 | 时间 | 代表作 | 核心范式 | 主要突破 |
| --- | --- | --- | --- | --- |
| I | 2021 | DALL·E 1, CogView | dVAE + GPT 自回归 | 文生图可行性证明 |
| II | 2022 | DALL·E 2, Imagen, SD1.5 | 扩散 + 独立文本 encoder | 文生图质量爆发,开源民主化 |
| III | 2023 | SDXL, DALL·E 3, Midjourney V6 | 扩散 + 多 encoder + caption 工艺 | Prompt 工程化、产业日常化 |
| IV | 2024 | SD3, Flux.1, Imagen 3 | MMDiT + Rectified Flow | 架构与训练目标双重升级 |
| V | 2025 | GPT-4o image, Gemini 2.5 Flash Image(Nano Banana) | **原生多模态**(LLM 内部直接生成) | 多轮编辑 / 角色一致性 / 图像成为 LLM 内在能力 |

纵看这张表,2021–2024 都在“扩散为主、语言作为外挂 encoder”的架构里演进。**2025 是一次范式跃迁**——图像生成从“LLM 外挂的独立模块”变为“LLM 体内的原生能力”。这个跃迁正好站在阶段 IV 与阶段 V 的交界面上。

### 10.2 三代标杆(2023 下半年)——三个哲学共存

**SDXL**(Stability AI, 2023/7):开源路线展开到标准产品。双 encoder + 细粧度条件输入 + Refiner 二阶段打磨。为 LoRA / ControlNet / IP-Adapter 生态提供标准底座。

**DALL·E 3**(OpenAI, 2023/10):闭源路线的最高点。**Prompt 工艺化**是核心——GPT-4V 重新做 caption + ChatGPT 自动扩写用户 prompt。章也站稳了“LLM 为 prompt 工程师,扩散为画师”的产业范式。

**Midjourney V6**(2023/12):闭源美学路线的顶峰。技术细节不公开,但在**艺术感**与**光影综合**上独步。代表“不追求最大似然但追求美感均值”的设计哲学。

> 这三份在 2023 年末站稳了“产业级文生图”的下限。到这里,技术架构仍是 **扩散 + 独立文本 encoder**,从§34 一路演进下来的同一主干。
> 

### 10.3 四代架构跃迁(2024 年)——MMDiT + Rectified Flow

2024 年最重要的不是又一个更大的模型,而是底层架构换了。

**Stable Diffusion 3**(Stability AI, 2024/6)和 **Flux.1**(Black Forest Labs, 2024/8)两家同时完成了两项在§34 和 §35 里都曾预告的转型:

**转型一:U-Net → DiT → MMDiT**

SD1.5 / SDXL 用的 U-Net 是 CNN + 少量 attention 的杂种架构。DiT(Diffusion Transformer, Peebles & Xie 2022)证明纯 Transformer 在扩散里一样行,且更好缩放。MMDiT(Multi-Modal DiT)更进一步——把文本 token 和图像 token **拼成一条序列**,走同一套 self-attention。Cross-attention 变成了 self-attention 的一种特例。

**转型二:DDPM → Rectified Flow**

§34 §9 计划的“Flow Matching / Rectified Flow”在 SD3 上正式上线。不再需要 `\beta_t` 调度表,不再需要 DDPM 的马尔可夫推导,训练目标简化为在 `x_0`（噪声）和 `x_1`（图）之间学一个速度场。采样步数经过 reflow 提约到 20 步以内。

**Flux.1** 具体版本:

- Flux.1 [pro]:闭源旗舰版
- Flux.1 [dev]:12B 参数开源(非商业)——**当前开源 SOTA**,文本跟随与手画质感超越 SDXL
- Flux.1 [schnell]:蒸馏少步版,4 步即出图
- 文本 encoder 用 **T5-XXL + CLIP-L 双轨**——继承 Imagen 的深语义 + CLIP 的图文对齐

> **§34 预告的 Rectified Flow 兑现。§35 预告的 “语言与图像 token 同源 attention” 兑现。**这是 2024 年最大的技术跃迁,目前所有 SOTA 开源模型都演化到 MMDiT + Rectified Flow。
> 

### 10.4 Imagen 3 与闭源路线(2024 年中后半)

**Imagen 2**(Google, 2023/12):Imagen 1 的跟进版,修复文本拼写和数量准确性。

**Imagen 3**(Google, 2024/5):全面重写,**T5-XXL 深语义** + **潜空间扩散**(向 LDM 靠拢),集成进 Gemini。对长复杂 prompt 的追踪性上与 DALL·E 3 / Flux 差不多,**结构清晰度**更优。

**Ideogram 2.0**(2024/8):专注**字体渲染**与**版式设计**,在海报 / logo / typographic poster 上主可以申。技术核心是为字体 token 做了专门优化。

**Midjourney V7**(2025 初):进一步推美学与风格的可控性,增加对话式编辑的雏形。

> 到 2024 年底,扩散路线在**质感 + 文本跟随 + 字体**三个维度上已经打磨极致。产业界的惊喜不再来自“生成质量”,而是来自**交互范式**的核变——这正是 2025 的头条。
> 

### 10.5 GPT-4o Image Generation(OpenAI, 2025/3)——原生多模态元年

2025 年 3 月 25 日,OpenAI 将图像生成能力 **直接内置进 GPT-4o**。这不是一个更好的 DALL·E,而是一个根本不同的架构。

**与 DALL·E 3 的关键区别**:

| 维度 | DALL·E 3(2023) | GPT-4o image(2025) |
| --- | --- | --- |
| 架构 | GPT-4 扩写 prompt → 独立扩散模型出图 | GPT-4o 本体直接输出图像 token |
| 图像和语言的关系 | 两个模型的串联 | 同一个模型的两种输出 |
| 修改已有图 | 重新生成(不能保证一致性) | 多轮对话式编辑(保持角色与场景一致) |
| 文字渲染 | 一般 | 显著提升(能稳定写长字符串) |
| 上下文 | 单轮为主 | 能利用整个对话历史 |

**技术上的推断(公开线索)**:GPT-4o 将图像表示为一串**离散 token**(VQ 类型),在 Transformer 内部自回归生成,然后通过一个 decoder(可能是扩散 decoder)变为像素。这反而 **继承了§33 VQ-VAE / VQ-GAN 的语法**,但放在一个 2025 级别的语言模型里——§9.1 里预告的“Parti 自回归路线在图语统一模型里重回主流视线”、经容正式兑现。

**主要影响**:

- 图像生成从“独立产品”变成“对话式交互中的内嵌能力”
- 图文混合输出——可以文本解释 + 图 + 再文本,在一条 response 里交织
- 一贯性编辑(问“把背景换成夜晚”,前一张的人物角度/表情保留)
- 审美不及干净的 Midjourney,但**指令遵循**与**在对话背景的深关联**换回了巨大用户价值

### 10.6 Gemini 2.5 Flash Image / Nano Banana(Google, 2025/8)——多轮编辑的标杆

Google 的回击,2025 年 8 月在 Gemini 2.5 Flash 里上线。官方名称是 Gemini 2.5 Flash Image,开发期代号 **Nano Banana**(在 Google AI Studio 和 LMSys Arena 里灵光一现时就是这个名字)。

**特性**:

- 同样原生多模态——Gemini 2.5 本体输出图像 token
- **多轮对话式编辑**:连续对话中能精细改“衣服颜色”、“头发长度”、“背景场景”,同时保持角色一致性——这一点上对标 GPT-4o image 更稳
- **Flash 速度**:1-2 秒出图,快于当时所有闭源 T2I
- **图像输入支持**:上传一张图让模型 "继续画"、"扩展边缘"、"改变风格"——类似 Inpaint + Outpaint + Style Transfer 用对话语言做
- 与 Gemini 生态(Veo 视频、NotebookLM)深集成

**产业意义**:Nano Banana 的出现将“多轮图像编辑”行为标准化。之前要一针到位改图要链 ControlNet + inpaint mask + LoRA 三样工具。Nano Banana 让这些步骤被一句话替代:"把她的衣服改成蓝色, 背景换成东京夜景"。

两件轫动出后,Adobe 接入 Nano Banana 到 Firefly/Photoshop, Figma 、 Canva 接入 GPT-4o image,权增加汽车,这成为 2025 下半年的产业风景。

### 10.7 五代演进的本体论解读

把这五代站距离看,一个清晰的轨迹浮现:

| 轴度 | 2021 | 2022-2023 | 2024 | 2025 |
| --- | --- | --- | --- | --- |
| 图像与语言的关系 | 拼接序列 | 两个模型串联 | 架构内部对称 (MMDiT) | 同一个 LLM 的两种输出 |
| 技术主轴 | 自回归 token | 扩散 + cross-attention | Rectified Flow + MMDiT | LLM 内的自回归 token + decoder |
| 交互范式 | 实验室演示 | 单轮 prompt | 稍复杂 prompt | 多轮对话式编辑 |
| 主体关系 | - | 文 → 图(外部翻译) | 文 ↔ 图(架构对称) | 文 = 图(内在合一) |
| 阶段定位 | 阶段 IV 的前夜 | 阶段 IV 的灰黄期 | 阶段 IV 的工艺高峰 | **阶段 IV ↔ 阶段 V 的交界面** |

**五代的本体运动**:**图像生成正在从“外挂的独立能力”回归为“主体的内在能力”。**

何言?阶段 I-III 的文生图架构里,图像生成主体是一个独立的技术设施——它有自己的网络(U-Net)、自己的训练范式(扩散)、通过 CFG 从外部接收文本 encoder 的输出。语言与图像是两个仵事的主体,数据跨过一个折叠器(文本 encoder) 方能对话。

一直到 MMDiT(2024),架构层面给出了第一个“对称”的答案:文本和图像在同一个 Transformer 里共享 self-attention。但图像仍然以扩散的方式迭代生成,语言仍然以 LLM 的方式用 token 生成。

2025 的原生多模态给出的答案更彻底:**图像不再是独立主体,而是 LLM 内部 token 流的一种输出。**一个模型,一套参数,交错地输出文本与图像。

> **这是为什么 §34 · §35 不断强调“LLM 是条件供给者而非阶段命名者”这个判断**。如果那个论断在 2025 这一年被突破,我们就踏出了阶段 IV 的边界。Nano Banana 和 GPT-4o image 这样的存在没有破坏那个论断,但它们【把阶段 IV 和 V 的边界放在了桃子里】。
> 

### 10.8 2025 后的两条续线

伴随原生多模态的出现,产业会同时走两条线:

**线 A专用生成主体**(SDXL / SD3 / Flux / Midjourney / Imagen 3):极致追求生成质量、艺术品味、细节控制。给设计师 / 艺术师 / 专业工作流。ControlNet / LoRA / IP-Adapter 生态在这里持续活跃。

**线 B原生多模态 LLM**(GPT-4o / Nano Banana / Gemini / 未来的 Claude-vision 演进):不追求极致质感,追求**在对话中嵌入图像**。给普通用户 / 业务场景 / 广普同定。

两条线不是替代关系而是分工关系。线 A 保留阶段 IV 的纯粹性,线 B 把阶段 IV 的技术成果玩进阶段 V 的架构里。

> **阶段 IV 中心论断仍成立**:在线 A 上,扩散作为独立生成主体持续挑战高级书面。在线 B 上,图像成为 LLM 的内在能力——但这已经不是阶段 IV 的主题,而是阶段 V 开展的**表征共形**的前曲。
> 

## §11 · LLM 在阶段 IV 的地位——条件供给者而非阶段命名者

这是本章的一个不可徽哲学判断。

不少人会将 “DALL·E 3 是 GPT-4 + 扩散” 视为 “语言模型侵入图像用阶段”,认为阶段 IV 的主轴已经从生成转移到语言。这个判断混淆了主体和接口。

**正确的读法**:

- 阶段 IV 的主体是 **生成主体**(VAE / GAN / diffusion 探寻出来的图像生成体系)
- 阶段 IV 的任务是让这个主体拥有**动力学、内部语、意图接口**
- LLM / CLIP / T5 在这里是**供给条件的官吗**,不是主体本身

作个比喻:导演是生成主体,编剧是 LLM / 编码器——电影在阶段 IV 依然是导演主导的艺术,编剧是必要的服务提供商。

> **语言主体本身的本体地位当什么时候站稳,以及语言与图像主体如何真正合一**,是**阶段 V** 的主题,而不属于阶段 IV。在阶段 IV,语言的身份始终是“服务生成的语义供给者”。
> 

## §12 · 收束与向 §36 的传递

§35 把满了 CFG 里留下的接口空位 `c`。五件事已完成:

1. CLIP / T5 把语言变成稠密向量
2. cross-attention 把向量送进扩散网络的每一层
3. CFG / Negative Prompt 放大文本影响
4. Prompt Engineering 与 LLM 扩写让人类能真正使用
5. Adapter 家族把条件接口扩到任意模态

生成主体从此拥有:内部语(§33) + 内部时间(§34) + 意图接口(§35)。三者合起来,生成主体成为一个完整具行动力、内部状态、外部接收能力的技术有机体。

> **§35 的结论**:文本成为生成主体的通用意图接口。Adapter 家族将这个接口扩到任意模态。LLM 作为 prompt 供给者进入工艺流程——但仍是供给者,不是命名者。
> 

> **§36 的课题**:接口和动力学都成熟了,但扩散的**架构演化**(U-Net → DiT → MMDiT)、**采样器数学**(DPM++ / UniPC / Heun 2)、**蒸馏与加速**(LCM / SDXL-Turbo / ADD / DMD)等工艺线所有系统的最终全图还没有生成。§36 把这条最后一线走正完,并展示 2024-2025 的最前沿:SDXL / SD3 / Flux / Pixart-Σ 的架构与工程收尾。
