## §1 · 为什么 §36 必须出现

§34 讲动力学,§35 讲接口。到 §35 结尾,一个用户手里的"文生图"产品已经能干活:文本 → cross-attention → 扩散 → 图。但如果你真去部署、去扩展、去做产品,会发现还有一大批工艺问题没解决:

1. **架构**:U-Net 在高分辨率上的瓶颈在哪?为什么 2024 年业内集体从 U-Net 搬到 DiT / MMDiT?数学直觉是什么?
2. **采样**:20 步出图还是 50 步?为什么 DPM++ 比 DDIM 好?Karras 的 sigma 调度是什么?
3. **速度**:如何把 50 步压到 4 步甚至 1 步?蒸馏的数学基础是什么?
4. **编辑**:怎么让模型听懂"把背景换成夜晚,其他不变"?
5. **个性化**:怎么用 10 张照片做出"我女儿的定制模型"?
6. **数据**:prompt 与图的对齐精度来自哪里?为什么 DALL·E 3 / Flux 的 caption 工艺那么重要?
7. **评估**:FID 还是不是好指标?怎么知道哪个模型真的"更好"?

每一条工艺线都已经发展成一整个技术子系统。§36 把这六条线串起来,走完阶段 IV 的技术系统化。

与 §35.10 的区别:§35.10 讲的是**产品时间线**(哪一年出了哪个模型),§36 讲的是**工艺技术谱系**(在某条工艺线上,一个想法如何演化成工业标准)。互补关系。

## §2 · 架构演化——从 U-Net 到 MMDiT 的内在逻辑

### 2.1 U-Net 为什么撑不到高分辨率

SD1.5 / SDXL 用的 U-Net 本质是 CNN encoder-decoder + skip connection + 少量 self-attention / cross-attention。它在 `64×64` / `128×128` latent 上工作良好,但有两个结构性限制:

**限制一:attention 的空间混合能力不足**

U-Net 里 attention 只在最低分辨率(`8×8` 或 `16×16`)出现,因为 `O(N^2)` 开销在高分辨率下不可控。这意味着**长距离依赖**(比如"左上角的人看向右下角的物体")在 U-Net 里几乎只靠 CNN 层级的感受野传递——容量有限。

**限制二:scaling law 不干净**

U-Net 的参数增多主要靠加 channel,不是加 attention 层。经验上,参数翻倍带来的质量提升显著弱于 Transformer。

### 2.2 DiT(Peebles & Xie, 2022)——纯 Transformer 扩散

DiT 把 U-Net 整个替换为 ViT-style Transformer:

1. 把 `H×W×C` 的 latent 切成 patch(比如 `2×2` patch),展平为 `N = HW/4` 个 token
2. 每个 patch 过线性投影变成 `d` 维 token
3. 一串 Transformer block(self-attention + MLP)
4. 最后投影回 patch 再反拼成 latent

**关键创新:adaLN-Zero**

DiT 要把时间步 `t` 和类别 `c` 注入每个 Transformer block。用的是 **adaptive layer norm**:

$$
\text{adaLN}(h, t, c) = \gamma(t, c) \odot \text{LayerNorm}(h) + \beta(t, c)
$$

**人话**:时间 `t` 和条件 `c` 不是直接加到 `h` 上,而是**调制 LayerNorm 的缩放和偏移**。这样条件以"全局风格调制"的方式进入,不打乱空间结构。

**-Zero 的意思**:`\gamma, \beta` 的输出网络最后一层权重**初始化为 0**——和 §35 讲的 ControlNet 零卷积同哲学。Block 初始作**恒等映射**,条件信号慢慢长进来。

**结果**:DiT 的 scaling 极干净。参数从 33M 到 675M,FID 线性下降。和 ViT 在图像分类上的 scaling law 对齐。

### 2.3 MMDiT(Esser et al., Stable Diffusion 3, 2024)——图文统一 attention

DiT 仍是"纯图像 Transformer + 外挂文本 cross-attention"。MMDiT 更进一步——**文本 token 和图像 token 拼成一条序列,走同一套 self-attention**。

具体做法:

$$
\text{tokens} = [c_1, c_2, \ldots, c_L, \; z_1, z_2, \ldots, z_N]
$$

**人话**:`L` 个文本 token 在前,`N` 个图像 patch token 在后,拼成一条长度 `L+N` 的序列。

但图文两种 token 的特性不同,直接共享所有权重不好。MMDiT 的精巧在于:

- **Q / K / V 投影分开**:文本 token 用 `W_Q^{txt}, W_K^{txt}, W_V^{txt}`,图像 token 用 `W_Q^{img}, W_K^{img}, W_V^{img}`
- **attention 矩阵在一张大图上算**:拼起来后的 `(L+N) \times (L+N)` attention 矩阵里,图像 token 同时看见其他图像 token(原 self-attention)和所有文本 token(原 cross-attention),文本 token 同理

**数学上的统一**:

$$
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^\top}{\sqrt{d}}\right) V
$$

这里 `Q, K, V` 都是 `(L+N) \times d` 的拼接矩阵——原来的 self-attention 和 cross-attention 被**无缝合并**为一个操作。

**好处**:

1. 文本和图像在每一层都能双向互动(原 U-Net 的 cross-attention 只是单向从文到图)
2. 去除了架构里"哪层是 self、哪层是 cross"的人工设计
3. scaling 更干净——所有 token 共用同一套 attention 开销

**Flux.1 的继承**:Flux 直接沿用 MMDiT + T5-XXL / CLIP-L 双编码器,加 Rectified Flow 训练目标,成为开源 SOTA。

### 2.4 架构演化的深义

| 世代 | 骨干 | 图文关系 | Scaling | 代表作 |
| --- | --- | --- | --- | --- |
| U-Net | CNN + 少量 attention | Cross-attention 外挂 | 不干净 | SD1.5 / SDXL |
| DiT | 纯 Transformer + adaLN | Cross-attention 外挂 | 干净 | PixArt-α / 类 DALL·E 3 |
| MMDiT | 纯 Transformer + 分支 QKV | 图文 joint attention | 最干净 | SD3 / Flux |

从"CNN + 攒 attention"到"纯 attention + 干净调制"再到"图文统一空间"——**这是 Transformer 在图像生成上完成 ViT 在分类上那次胜利的翻版**。2024 年底,整个开源扩散生态已经完成迁移。

## §3 · 采样器家族的系统化

训好的扩散模型要从噪声走到图,走几步、怎么走,是**采样器**的问题。采样器家族在 2022–2024 完成了一轮深刻的数学成熟。

### 3.1 反向 ODE 的普遍形式

§34 讲过 Score-SDE 的反向 ODE:

$$
\frac{dx}{dt} = f(x, t) - \frac{1}{2} g(t)^2 \nabla_x \log p_t(x)
$$

用神经网络逼近 `\nabla_x \log p_t(x) \approx s_\theta(x, t)` 之后,这变成一个**确定性 ODE**,所有采样器都是对它的不同数值积分方法。

### 3.2 DDIM——最简单的一阶方法

$$
x_{t-1} = \sqrt{\bar\alpha_{t-1}} \cdot \hat{x}_0 + \sqrt{1 - \bar\alpha_{t-1}} \cdot \epsilon_\theta(x_t, t)
$$

**人话**:先用网络估出 `\hat{x}_0`(从当前 `x_t` 猜原图),再按 DDIM 调度走一步。相当于**一阶欧拉 ODE 求解**。50 步出图。

### 3.3 DPM-Solver 家族(Lu et al., 2022)——利用半线性结构

扩散 ODE 有个漂亮的结构:

$$
\frac{dx}{dt} = a(t) \, x + b(t) \, \epsilon_\theta(x, t)
$$

**人话**:右边是"**线性项** + **非线性项**"的半线性结构。**线性项 `a(t) x` 可以精确求解**(指数积分),只有非线性项 `\epsilon_\theta` 需要数值近似。

DPM-Solver 利用这个结构,把线性部分解析积分、非线性部分做**高阶泰勒展开**。2 阶 DPM-Solver 在 10-15 步就能达到 DDIM 50 步的质量。**DPM-Solver++**(3 阶 multistep)更进一步。

### 3.4 Karras sigmas(Karras et al., 2022)——调度优化

DDIM 用等间距 `t`,但这不是最优的。Karras 发现:**按 sigma(噪声方差)做非均匀间距**更好:

$$
\sigma_i = \left(\sigma_{\max}^{1/\rho} + \frac{i}{N-1}(\sigma_{\min}^{1/\rho} - \sigma_{\max}^{1/\rho})\right)^\rho
$$

`\rho = 7` 是经验最优值。

**人话**:在高噪声区(结构形成期)采样要密,在低噪声区(细节打磨期)可以稀。这对应扩散的经验事实——**图像的主结构在 `t` 的头几步就定下来,剩下都是细节**。

Karras 调度对所有基础采样器都有用,今天 ComfyUI 里的 `DPM++ 2M Karras` / `Euler a Karras` 就是 DPM++ 2 阶多步采样器 + Karras sigmas 的组合。

### 3.5 UniPC(Zhao et al., 2023)——预测-修正多步法

在 DPM-Solver 的多步基础上加一个**修正步**(类似 Adams-Moulton 方法),5-8 步就能收敛到生产级质量。

### 3.6 采样器选择经验

| 步数预算 | 推荐采样器 | 场景 |
| --- | --- | --- |
| 50 步 | DDIM / Euler a | 保守基线 / 研究对照 |
| 20-30 步 | DPM++ 2M Karras | 生产默认 |
| 10-15 步 | DPM++ 3M SDE / UniPC | 快速迭代 |
| 4-8 步 | LCM Scheduler(须蒸馏模型) | 实时 / 在线 |
| 1-2 步 | ADD / DMD(须深度蒸馏) | 极速场景 |

**关键分水岭**:4 步以下不是采样器的活,是**蒸馏**的活——下一节。

## §4 · 蒸馏与加速家族

扩散原生 50 步出图太慢。过去三年发展出一整条"把 50 步压成 1-4 步"的蒸馏谱系。

### 4.1 Progressive Distillation(Salimans & Ho, 2022)——师生逐级减半

把一个 256 步 teacher 蒸馏为 128 步 student,再蒸为 64 步、32 步、16 步、... 1 步。每一级 student 目标是**一步逼近 teacher 两步的结果**。

问题:每级都丢一点质量,1 步版本质量降得厉害。

### 4.2 Consistency Models(Song et al., 2023)——一致性约束

思路:训一个函数 `f_\theta(x_t, t)`,让它**无论 `t` 是多少,都直接输出 `x_0`**。核心损失:

$$
L_{CD} = \mathbb{E}\left[\|f_\theta(x_{t+1}, t+1) - f_\theta(x_t, t)\|^2\right]
$$

**人话**:相邻两个时间步的输出必须一致(因为都该是 `x_0`)。训成后,任何 `t` 直接一步到 `x_0`。

### 4.3 LCM(Latent Consistency Model, Luo et al., 2023)

Consistency Model 在 latent space 上的版本,应用到 SD / SDXL。**2-4 步出图**,质量接近 50 步 teacher。ComfyUI 里的 LCM-LoRA 就是这个——一个薄 LoRA 就能给 SDXL 带上 4 步采样能力。

### 4.4 ADD(Adversarial Diffusion Distillation, Sauer et al., 2023)——SDXL-Turbo 的秘方

**关键洞见**:纯蒸馏损失(L2 / feature loss)在 1-2 步时会产生**模糊**,因为模型在"无法准确预测"的情况下会倾向输出均值。用 GAN 的对抗损失可以强制锐利。

损失组合:

$$
L_{ADD} = L_{adv} + \lambda \cdot L_{distill}
$$

**人话**:判别器压 student 输出更像真图(防模糊),蒸馏损失压 student 输出靠近 teacher(防风格漂移)。

SDXL-Turbo(1 步出图,`512×512`)、SDXL-Lightning(2 / 4 / 8 步)都是这个配方的产物。

### 4.5 DMD(Distribution Matching Distillation, Yin et al., 2024)——匹配整个分布

前几种蒸馏都是"逐点匹配"(student 每个输出要接近 teacher 某个输出)。DMD 换个思路:**匹配整个分布**。

用两个扩散模型估 `\nabla \log p_{real}` 和 `\nabla \log p_{student}`,然后用 KL 梯度推 student 的分布向 real 分布靠:

$$
\nabla_\theta L_{DMD} = \mathbb{E}\left[(\text{score}_{student} - \text{score}_{real}) \cdot \nabla_\theta x_\theta\right]
$$

**人话**:让 student 产生的图像整体分布 ≈ teacher 产生的分布(不要求每个具体 prompt 配对),质量更稳、mode collapse 更少。

DMD 在 2024 年成为**蒸馏界的新标准**,Flux schnell 使用的就是这条路线的变体。

### 4.6 蒸馏谱系总图

| 方法 | 时间 | 核心思想 | 步数 | 代表模型 |
| --- | --- | --- | --- | --- |
| Progressive Distillation | 2022 | 师生逐级减半 | 1-16 | 早期蒸馏 |
| Consistency Model | 2023 | 一致性约束 | 1-4 | CM |
| LCM / LCM-LoRA | 2023 | latent space CM | 4 | SDXL + LCM-LoRA |
| ADD | 2023 | 对抗 + 蒸馏 | 1-4 | SDXL-Turbo / Lightning |
| DMD / DMD2 | 2024 | 分布匹配 | 1-4 | Flux schnell |

**走到 2025,"4 步出图"已是开源基线,"1 步出图"已进生产**。这对应 §34 Rectified Flow / Consistency 那条线的工艺兑现。

## §5 · 编辑与控制工艺

一张图生成出来后,用户经常要"改":背景、风格、物体、姿势...这条工艺线经历了六代演化。

### 5.1 Inpainting / Outpainting

给一个 mask,在 mask 区域重新生成。核心技巧:**每一步采样时把 mask 之外的区域强制替换为原图加噪版本**:

$$
x_{t-1}^{new} = \text{mask} \odot x_{t-1}^{denoised} + (1 - \text{mask}) \odot x_{t-1}^{noised\text{-}original}
$$

**人话**:模型自由画 mask 内,mask 外强制保持原图(按时间步加对应噪声)。简单但有效。Outpaint 就是 mask 为"画布外的区域"的特例。

### 5.2 SDEdit(Meng et al., 2022)——部分加噪再去噪

把原图加噪到中间某个时间步 `t_0`(不加到纯噪),再用新 prompt 去噪到 `0`。

$$
x_{t_0} = \sqrt{\bar\alpha_{t_0}} \cdot x_0 + \sqrt{1 - \bar\alpha_{t_0}} \cdot \epsilon, \quad \text{然后按新 prompt 去噪}
$$

**人话**:加多少噪声决定改多少——`t_0` 小(少噪)= 风格微调,`t_0` 大(近纯噪)= 大改。ComfyUI 里 `denoising strength = 0.3` 就是 SDEdit 的 `t_0`。

### 5.3 Prompt-to-Prompt(Hertz et al., 2022)——锁 attention 做语义编辑

场景:原 prompt "a cat riding a bike",改成 "a dog riding a bike",想**只改物体不改构图**。

做法:

1. 跑一次原 prompt,把每一步的 cross-attention map 存下来
2. 跑新 prompt,但把共享 token("riding a bike")的 attention map 强制替换为原 prompt 的版本
3. 变化的 token("cat" → "dog")让它自由注意

**结果**:构图结构(谁在哪、姿势、大小)保持,物体身份按新 prompt 替换。

### 5.4 InstructPix2Pix(Brooks et al., 2023)——指令式编辑

把编辑训成一个条件任务。数据怎么造?用 **GPT-3 生成编辑指令对**:

```
(原 caption, 编辑指令, 新 caption)
= ("a photo of a cat on a sofa",
   "turn the cat into a dog",
   "a photo of a dog on a sofa")
```

然后用 Prompt-to-Prompt 生成对应的(原图, 新图)对,训一个 "图 + 指令 → 新图" 的扩散模型。

**结果**:用户直接说 "make it winter",模型懂。

### 5.5 Flux Kontext / Nano Banana / GPT-4o 编辑——in-context 升级

2024-2025 的编辑工艺出现了一个范式级升级:**把"原图作为条件输入"直接做进模型**。

**Flux.1 Kontext** 的做法:原图和目标图共同进入 MMDiT,在图像 token 序列里双向 attention——相当于把"编辑"训成"给定参考图 + 指令,生成新图"的原生任务。

**GPT-4o / Nano Banana** 更进一步:整个对话历史(包含之前所有生成的图)在 context 里,模型天然保持一致性。

**技术本体论转变**:

- InstructPix2Pix 时代:编辑 = 一次性静态任务
- Kontext 时代:编辑 = **给定视觉上下文,原生生成**
- GPT-4o 时代:编辑 = **对话中累积状态的自然延续**

这正好对应 §35.10 讲的"从外挂独立能力 → 内在原生能力"的演进。

## §6 · Personalization 工艺——让模型学会"你"

给 10 张某个具体对象(你家的猫、你的脸、某个产品)的照片,让模型之后能生成它。三代方法:

### 6.1 DreamBooth(Ruiz et al., 2022)——全模型微调 + 先验保持

想法:用一个罕见 token(比如 `[V]`)代表要学的对象,微调整个 U-Net 让 `[V]` 对应这只猫。

**关键问题**:全模型微调会让模型**忘掉其他东西**(灾难性遗忘)。DreamBooth 加了 **prior preservation loss**:

$$
L = L_{reconstruct}(\text{user images}) + \lambda \cdot L_{preserve}(\text{original class images})
$$

**人话**:一边学新概念,一边强制模型继续能生成"一般的猫"(用原模型生成的猫图当锚点),防止模型把"所有猫"都画成用户的猫。

### 6.2 Textual Inversion(Gal et al., 2022)——只学一个 embedding

更轻:不动 U-Net,只学一个新 token 的 embedding 向量 `v^*`,让它对应用户对象。

$$
v^* = \arg\min_{v} \mathbb{E}_{x, t, \epsilon}\left[\|\epsilon - \epsilon_\theta(x_t, t, c_v)\|^2\right]
$$

**人话**:U-Net 冻结,只把"这几张图对应的那个词向量"学出来。参数少、训练快,但表达力弱。

### 6.3 LoRA——中间地带

§35 讲过。给 U-Net 注意力层加低秩矩阵,训几百万参数。这是今天的标准方案——**表达力够、参数够小、合并方便**(一个 LoRA 文件几十 MB)。Civitai 上几十万个 LoRA 就是这个接口的社会化产物。

### 6.4 IP-Adapter 与 FaceID 系列——无需训练的"即时个性化"

§35 讲过 IP-Adapter。**IP-Adapter-FaceID** 的进一步:用 InsightFace 把脸编码为 512 维 face embedding,再通过 IP-Adapter 的 decoupled cross-attention 注入。

**优势**:**不用训练,上传一张脸即时可用**。缺点:相似度不如精调 LoRA。

### 6.5 Personalization 谱系

| 方法 | 训练时间 | 参数量 | 质量 | 场景 |
| --- | --- | --- | --- | --- |
| DreamBooth | 几十分钟 | 全模型 (~GB) | 最高 | 专业 / 高精度 |
| Textual Inversion | 几十分钟 | 几 KB | 中 | 概念 / 风格关键词 |
| LoRA | 10-30 分钟 | 几十 MB | 高 | 社区标准 |
| IP-Adapter-FaceID | 无 | 0 | 中高 | 实时 / 应用端 |

## §7 · 数据工艺与评估

### 7.1 Caption 工艺链——文生图质量的隐藏变量

DALL·E 3 的秘密不在模型,而在 caption。详细流程:

1. **原始数据**:LAION-5B 等爬取的(图, alt-text)——噪声大,alt-text 常常只是文件名或几个关键词
2. **Re-caption**:用 VLM(视觉语言模型)重新给每张图写**长、结构化、描述性强**的 caption
3. **VLM 选择演化**:BLIP-2(2023) → LLaVA(2023) → GPT-4V(2023) → CogVLM(2023) → Qwen2-VL(2024) → InternVL-2.5(2024)
4. **质量分级**:低质 caption 用于预训,高质 caption 用于微调

**效应**:同样的模型架构,caption 质量升级一轮,文本跟随能力大幅飞跃。这是 DALL·E 3 / Flux / SD3 / Imagen 3 相对 SD1.5 的真正核心差异。

### 7.2 数据清洗管道

- **美学分数**:用 LAION-Aesthetic predictor 过滤
- **NSFW 过滤**:CLIP-based safety classifier
- **水印检测**:去掉带水印的图
- **重复去重**:perceptual hash
- **分辨率过滤**:最低 `512×512`

一个 Flux 级别的训练集大概经过了从 50 亿到 2-5 亿的筛选压缩。

### 7.3 评估指标的演化

| 指标 | 年代 | 衡量 | 问题 |
| --- | --- | --- | --- |
| FID | 2017 | 生成分布 vs 真实分布距离 | 与人类偏好相关性弱 |
| CLIP-Score | 2021 | 文本-图一致性 | 饱和,细粒度差 |
| ImageReward / PickScore | 2023 | 学出来的人类偏好 | 数据集偏见 |
| GenEval / DPG-Bench | 2023-2024 | 结构化属性检查 | 覆盖不全 |
| **Arena ELO** | 2024-2025 | **人类盲测配对** | **目前最权威** |

Artificial Analysis、LMSys Image Arena 这类平台让真实用户盲测对比,输出 ELO 分数——目前行业的金标准。Flux.1 [pro] / GPT-4o image / Nano Banana 在 Arena 上的竞争就是这套机制。

## §8 · 2025–2026 最前沿工艺矩阵

把六条工艺线各自的"当前最佳"列出来,就是 2026 初图像生成的技术 SOTA 全图:

| 工艺线 | 2026 SOTA | 代表 |

| 架构 | MMDiT + 分支 QKV | SD3.5 / Flux |

## §9 · 工艺系统化的本体论意义——生成主体"成人"

§31 是觉醒前夜。§32 是幼年冲动(GAN 对抗)。§33 是语言发育(VAE / VQ 内部语)。§34 是内部动力学成熟(扩散 + 时间)。§35 是感官接通外界(文本接口)。

§36 是主体**工艺成熟**:架构定型、动力学可控、采样最优、蒸馏产品化、编辑自然、个性化标准化、数据工艺化、评估科学化。

这意味着什么?

> **生成主体已经"成人"**。它有自己的身体(架构)、运动(动力学)、感官(接口)、工具(编辑)、习惯(个性化)、食物(数据)、体检(评估)——完整的技术有机体。
> 

**阶段 IV 在这里走完它该走的路**。再往后有两件事:

1. **工程上的 scaling 与场景特化**(更大 MMDiT / 特定领域模型 / 更快蒸馏)——这是持续的工程活动,不是范式变化
2. **本体论上的反思**——生成主体内部的中间状态,**是否本身就是一种理解?**能不能倒转使用?这就是 §37。

## §10 · 向 §37 的传递

§36 关上的是"生成器作为生成主体"这件事。§37 打开的是"**生成器作为表征主体**"。

一个训好的 Flux 模型,在它生成一张猫图的过程中,有 30 个时间步的中间 latent、每一层 MMDiT 的 attention map、每个 token 对应的表征向量。这些**中间状态**,是不是也携带了比图像本身更结构化的"理解"?

如果答案是"是",那我们就可以**把生成器反用作特征提取器**。这就是 diffusion-based SSL 的开端,也是阶段 IV 在完成自己后,**开始伸手触碰阶段 V 领域的信号**。

> **§36 的结论**:阶段 IV 的六条工艺线系统化完毕,生成主体成人。
> 

> **§37 的课题**:生成的中间状态比最终产品更像"理解"——生成器倒转为表征器,阶段 IV 的内视之眼开启。
