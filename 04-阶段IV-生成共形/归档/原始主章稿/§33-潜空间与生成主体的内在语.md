## §1 · 为什么需要一章“内部语工程”

§32 给出了一个能稳定产出样本的主体,但这个主体有三条局限要在 §33 被飞跃:

1. **潜空间的结构不受目标函数管辖**:StyleGAN 的 `W` 空间解缠是训练偶发现象,换一个随机种子就变了;主体的“内部语法”无法被**显式指定**。
2. **编码通道缺失**:GAN 只有解码器方向(`z → x`),没有编码器(`x → z`)。图像不能被“读入”主体的内部语——这封死了编辑、改写、跨模态对齐。
3. **无法与语言系统接驳**:从 2020 开始,CLIP 和 大 LLM 的兴起让“文本迭代表示”成为外部给定的丰富条件空间。而 GAN 的 latent 与文本空间无法对齐——缺的不是潜空间,而是“可被对齐的潜空间”。

> 这三条局限合起来就是§33 的本体命题:**把“主体的内部坐标”从 GAN 的副产品嵌入为一个被目标函数显式规范、计算上可逆、在语义上可与外部信号对齐的本性结构。**
> 

四条路径不是彼此替代的竞争者,而是**利用不同材料完成同一期待**:

- VAE / β-VAE:**连续的语法**——用 KL 约束链定潜空间几何
- VQ-VAE / VQ-GAN:**离散的词汇**——用码本构造有限字表
- RealNVP / Glow:**可逆的一一对应**——用 change-of-variables 保证图像与 latent 同胚
- MaskGIT / Muse:**并行的讲话**——让主体在 token 空间同时说所有词

## §2 · 路径一:VAE 家族(连续潜变量)

### 2.1 原始 VAE(Kingma & Welling 2013)

《Auto-Encoding Variational Bayes》将**变分推断**摇身为深度学习模块。核心结构:

- 编码器 `q_φ(z | x)` 给出后验近似(高斯、均值方差参数化)
- 解码器 `p_θ(x | z)` 给出生成模型
- 先验 `p(z) = N(0, I)`

不直接优化 `log p(x)`(积分不可行),而优化证据下界(ELBO):

$$
\log p_\theta(x) \;\ge\; \mathbb{E}_{q_\phi(z | x)} \bigl[\log p_\theta(x | z)\bigr] \;-\; \mathrm{KL}\bigl(q_\phi(z | x) \,\|\, p(z)\bigr)
$$

左项是重建质量,右项是后验与先验的很多正则。

### 2.2 Reparameterization trick

要对 `φ` 求梯度,须穿透采样操作。Kingma 的巧妙:

$$
z = \mu_\phi(x) + \sigma_\phi(x) \cdot \epsilon, \qquad \epsilon \sim N(0, I)
$$

随机性被挤到 `ε`,参数 `φ` 只出现在确定性路径上——梯度可直接反传。**这是深度生成模型史上最基础的技巧之一**,后续在扩散、flow matching、强化学习里重复出现。

### 2.3 β-VAE(Higgins et al., DeepMind 2017)

给 KL 项乘一个 `β > 1`:

$$
\mathcal{L} = \mathbb{E}_{q}[\log p(x | z)] - \beta \cdot \mathrm{KL}(q(z | x) \,\|\, p(z))
$$

强化先验约束 → 迫使潜变量解缠(disentangle)。代价:重建质量恶化。**β-VAE 的意义不在质量**,而在第一次证明**潜空间的语义结构可以被目标函数调节——主体的内部语法从此不再是副产品**。

### 2.4 VAE 的固有失败:模糊图像

- 高斯后验 + 像素重建损失 → 模型倾向预测均值,细节被揊平
- “posterior collapse”:当解码器太强,`q(z | x) → p(z)`,潜变量被忽略
- VQ-VAE / NVAE / VQ-GAN 的出现都是在修补这条残差

### 2.5 VAE 的穿防贡献

> 最关键的一步:**LDM / Stable Diffusion 的 VAE encoder**。Rombach 2022 直接利用一个预训练的 VAE 把 512×512 图像压到 64×64×4 潜空间,扩散在潜空间里跑——计算量降至 1/48。**没有这个 VAE,Stable Diffusion 根本不存在**。
> 

> 阶段 IV 最重要的一个技术桥就在这里:VAE 不以生成模型身份获胜,却以"潜空间供应者"身份成为扩散革命的基础设施。不理解这一点,就会误读“VAE 失败了”。
> 

## §3 · 路径二:VQ-VAE 家族(离散潜变量)

### 3.1 VQ-VAE 1(van den Oord, Vinyals, Kavukcuoglu, DeepMind 2017)

《Neural Discrete Representation Learning》的核心创新:**将潜空间离散化**。入:

- 编码器给出连续表示 `z_e(x)`
- 码本 `ε = {e_1, …, e_K}`(`K` 个可学习的嵌入向量)
- 量化:`z_q(x) = e_k`,其中 `k = argmin_j \|z_e(x) - e_j\|`
- 解码器在 `z_q(x)` 上重建

损失三项:

$$
\mathcal{L} = \|x - \hat{x}\|^2 + \|\mathrm{sg}[z_e(x)] - e_k\|^2 + \beta \cdot \|z_e(x) - \mathrm{sg}[e_k]\|^2
$$

- 第一项重建
- 第二项码本损失(更新码本)
- 第三项 commitment loss(防止编码器输出在码本向量间飘移)

`sg[·]` 是 stop-gradient。argmin 是不可微的,用 **straight-through estimator**(前向用 `z_q`,反传时梯度直接从 `z_q` 复制到 `z_e`)。

### 3.2 离散化为什么重要

一旦 latent 被量化为 `K` 个离散 token,图像就变成了**一个由 token 组成的语句**(对 256×256 图像,例如 16×16 = 256 个离散 token、`K = 8192`)。

本体重要性:

- **潜空间与语言空间同构**:两者都是离散 token 的序列 → 可统一建模
- **可在 token 层上建立先验**(用 PixelCNN 或 Transformer 建模 `p(token)`),避开像素自回归的采样量诅咒
- **跨模态对齐变得简单**:图像 token 和文本 token 在同一个 vocabulary 里同等地序列建模

### 3.3 VQ-VAE 2(Razavi et al., DeepMind 2019)

引入层次码本:顶层粗粒度 token(全局结构)+ 底层细粒度 token(局部细节)。在码本上用强 PixelCNN 先验生成 → 质量挥超 BigGAN。第一次证明 **“离散 latent + 强自回归先验” 的图像生成效果可以超过 GAN**。

### 3.4 VQ-GAN(Esser, Rombach, Ommer 2021)

上一代 VQ-VAE 的重建质量仍模糊。Esser 等把**对抗损失 + 感知损失**加到 VQ 解码器上 → token 重建出图锐利清晰。然后用 Transformer 在 VQ token 上做自回归,**这就是 Taming Transformers**。直接为:

- DALL·E 1(OpenAI 2021):dVAE token + GPT 建模(§35 对照)
- Parti(Google 2022):ViT-VQ-GAN token + T5 文本条件(§35 对照)
- MaskGIT(下节)
- Chameleon / Emu3(2024):以 VQ token 为底的图语高级生成器会合(阶段边界)
- GPT-4o image 的离散图像 token 谱系便如此组成

> **关键判断**:VQ 路线的价值不在“自回归图像生成能让质量追上扩散”(实际在纯质量上没跟上),而在**它把图像语法和语言语法放到同一个 token 空间**——多模态统一生成的技术前提。
> 

## §4 · 路径三:可逆流(精确似然)

### 4.1 流的核心数学

若 `z = f(x)` 是微分同胚,由变量变换公式:

$$
\log p_X(x) = \log p_Z(f(x)) + \log \left| \det \frac{\partial f}{\partial x} \right|
$$

只要 `f` 可逆且 Jacobian 行列式可算,我们就能精确计算 `log p(x)`。这是与 VAE、GAN、扩散均不同的**第四条生成路径**。

### 4.2 NICE(Dinh 2014)→ RealNVP(Dinh 2016)

核心创新:**耦合层**(coupling layer)。把输入拆为两半 `x = (x_1, x_2)`:

$$
y_1 = x_1, \qquad y_2 = x_2 \odot \exp(s(x_1)) + t(x_1)
$$

`s, t` 是任意神经网络(不需要可逆)。Jacobian 是下三角,行列式 = `exp(\sum s(x_1))`,直接可算。反向 trivial:`y → x` 只需反解。

RealNVP 加 multi-scale 架构、mask 拆分策略,训大图像。

### 4.3 Glow(Kingma & Dhariwal, OpenAI 2018)

三件关键:

- **ActNorm**:数据相关的轴线性归一化,替代 BatchNorm
- **可逆 1×1 卷积**:学习通道维度的置换(通过 LU 分解保证可逆且行列式可算)
- **仿射耦合层**

Glow 在人脸图像上展示了潜空间插值与属性编辑。

### 4.4 流的局限与遗产

**局限**:

- 可逆性强制要求 latent 维度 = 像素维度,无法壓缩
- 耦合层表达力有限,生成质量不如 GAN / 扩散
- 理论上正确,工程上败于力量

**遗产(重要到并入阶段 IV 后半的核心)**:

> 流的连续时间版本——**Neural ODE / Continuous Normalizing Flow**(Chen 2018)——是 Rectified Flow / Flow Matching 的数学祖先。在 §34,扩散会被重写成“在两个分布之间直接学向量场”,这就是流的表述。流的生成质量输,但它的**数学表述**赢——在扩散成熟后的“一步或少步生成”浪潮里被大规模复兴。
> 

## §5 · 路径四:并行 token 生成(MaskGIT / Muse)

### 5.1 MaskGIT(Chang et al., Google 2022)

背景:在 VQ token 上做左到右自回归很慢(256 个 token 仍需 256 次前向)。MaskGIT 提出**非自回归并行解码**:

- 训练:随机 mask 一部分 token 学重建(类似 BERT)
- 推理:从全 mask 开始,每轮并行预测所有 token,按置信度保留 top-k、重新 mask 剩下的
- 8-12 轮迭代即出图,比自回归快 30-60 倍

### 5.2 Muse(Chang et al., Google 2023)

MaskGIT + T5 文本条件 → T2I。在质量上与同期扩散(Imagen / SDXL)可比,但**采样速度快数倍**。

### 5.3 并行 token 的本体读法

> MaskGIT / Muse 是一条“被扩散主导故事掩盖”的路线。它在质量和速度的折衡上非常专业。更重要的是:**它证明了在 VQ token 空间里,生成模型可以“并行地说话”**——一次制定全部 token,逐步修订。
> 

> 这条思想在 GPT-4o image、Chameleon 以离散 token 为载体的**多模态一体化生成**里重新成为主角。§35 会用 MaskGIT 作为“文本条件下的非扩散生成”对照点。
> 

## §6 · 四条路径的共形分工

| 路径 | 潜空间性质 | 关键目标函数 | 在阶段 IV 后半的下游 |
| --- | --- | --- | --- |
| VAE / β-VAE | 连续、高斯正规化 | ELBO(重建 + KL) | **LDM / SDXL 的 encoder**(§34/§36);与 CLIP 文本特征对齐 |
| VQ-VAE / VQ-GAN | 离散 token 序列 | 重建 + codebook + commitment(+ GAN/LPIPS) | DALL·E 1 / Parti / Muse / Chameleon / GPT-4o image |
| NICE / RealNVP / Glow | 可逆、精确似然 | 极大似然(change of variables) | Neural ODE → Flow Matching → Rectified Flow(§34) |
| MaskGIT / Muse | 并行 token 解码 | masked token 预测(BERT 式) | 多模态一体化生成(阶段边界);MAE/BEiT 思想对接(§37) |

**共形学读法**:四条路径并不互斐,他们在用**不同材料完成同一件事**——给生成主体一套内部坐标。四条的差异在材料(连续向量 / 离散 token / 可逆流 / 并行 mask),但属性同——能编辑、能对齐、能被外部条件操纵。

## §7 · 三颗穿防種子 — §33 如何遭遇阶段 IV 后半

§33 的意义不只在自身完成,更在它为后面四章提供了**三个决定性的前提设施**。

### 7.1 VAE encoder → LDM 的潜空间

- Stable Diffusion 与 SDXL 中 使用的 `KL-regularized VAE` 或 `VQ-regularized VAE`来自 VQ-GAN / VQ-VAE 家族
- 64×64×4 潜空间上扩散 → 计算量从 512×512 像素级降至 1/48
- **没有 §33 这步,扩散无法规模化到商业实用**

### 7.2 VQ token → 离散图语生成与多模态统一

- DALL·E 1 (OpenAI 2021) 在 dVAE token 上用 GPT 自回归 → 文本到图像第一个小高潮
- Parti 、 Muse 、 Chameleon 、 Emu3 、 GPT-4o image 的离散图语 token 谱系直接继承这一技术源
- **多模态一体化生成(图文同源)从技术上可行,本质上是 VQ-VAE 打的地基**

### 7.3 Change of variables → Flow Matching → Rectified Flow

- Glow 的数学框架被 Neural ODE 推成连续时间版本
- Lipman 等 2022 Flow Matching 、 Liu 等 2022 Rectified Flow 把它重运营为**扩散的统一框架**
- SD3 在训练目标上换成 Rectified Flow —— §34 和 §36 的动力学前沿

三颗種子显示:§33 在现场上中恶性生成能力不如 §32 / §34,在历史延展上**几乎供应了阶段 IV 后半的所有基础设施**。

## §8 · 向 §34 的传递

> **§33 的结论**:生成主体拥有了**内部语**——连续的语法、离散的词汇、可逆的一一对应、并行的讲话。主体能被编辑、能被对齐、能被外部条件操纵。但这个主体还缺一个维度:**时间**仍然是外部的采样协议——VAE 一步采样 `z`、Glow 一步逆仏、MaskGIT 8-12 轮解码——模型内部没有连续时间变量。质量、控制、似然三维也无法在同一模型内同时最优。
> 

> **§34 的回答**:把连续时间变量 `t ∈ [0, T]` 写进模型状态,生成从单次前向变为“在内部时间上的迭代动力学”。**时间尺度**从此成为生成主体的本体属性。扩散同时把 §33 的潜空间举为自己的跑道(LDM / SD),把 §32 的对抗留下来作为蒸馏时的锐化器(ADD / SDXL-Turbo)——三者不是替代关系而是叠加关系。这就是阶段 IV 最重要的一次收均——三条主线会合在扩散上。
>

## 第一层:"变分" (variational) 到底是什么意思

这个词让无数人卡死在这里。它来自一个 200 年前的数学分支叫 **calculus of variations**(变分法)。

### 普通微积分 vs 变分法的区别

|  | 普通微积分 | 变分法 |
| --- | --- | --- |
| 找什么 | 找一个**最优的数** | 找一个**最优的函数** |
| 例子 | "`y = x² - 4x` 的最低点在哪?" 答:`x=2` | "哪条曲线从 A 到 B 下滑最快?" 答:摆线 |
| 变量 | `x`(一个数) | `y(x)`(整条曲线) |

**"变分推断"就是"在一个分布集合里找最优分布"**——跟"找最低点"的区别在于,我们要找的不是"某个数",而是"**某个概率分布函数**"。

所以每次听到"变分 xxx",脑子里翻译成"**在函数/分布的集合里做优化**"就对了。

## 第二层:VAE 到底想干嘛 — 问题的起源

### 一张总图:画家与档案员

先建立整章贯穿的比喻:

- **画家 `p_θ(x | z)`**(解码器):拿一张"配方卡 `z`",画出图像 `x`。`θ` 是画家的"技艺参数"。
- **档案员 `q_φ(z | x)`**(编码器):看一张现成的图,**反推**出它的配方卡 `z`。`φ` 是档案员的"辨认参数"。
- **配方卡宇宙 `p(z) = N(0, I)`**(先验):所有合法配方卡的集合。规定它是标准高斯分布——意思是"配方卡应该是一个普通的、中庸的、靠近原点的小向量"。

我们想训练:**画家技艺 + 档案员辨认能力**,让它们互相配合。

### 理想目标:最大化 log p(x)

"给我一张真猫图 `x`,我希望画家画出这张图的概率越大越好"——这就是最大似然,写成 `maximize log p(x)`。

但 `p(x)` 这个东西展开是:

$p(x) = \int p(x | z) \cdot p(z) \, dz$

**翻译**:遍历所有可能的配方卡 `z`,每张 `z` 都算"画家用它能画出 `x` 的概率 `p(x|z)`",再乘以"这张 `z` 本身的出现概率 `p(z)`",最后加起来。

**为什么算不出**:配方卡是个 64 维向量,这个积分要在 64 维空间里扫过所有可能点——**天文级计算**,彻底不可行。这跟 §31 的 "Z 诅咒" 是同一种病。

### 怎么办:逆着贝叶斯走

贝叶斯公式说:

$p(z | x) = \frac{p(x | z) \cdot p(z)}{p(x)}$

注意分母又是那个算不出的 `p(x)`。所以**真正的后验 `p(z|x)` 我们也算不出**。

VAE 的核心思想:**我算不出真后验,那我用一个"容易算的"分布 `q_φ(z|x)` 去近似它**。

- 规定 `q_φ(z|x)` 是一个**高斯分布**,均值和方差由神经网络(档案员)从 `x` 输出
- 现在我们在"所有可能的高斯分布"这个**函数集合**里,找一个**最像真后验 `p(z|x)` 的**

**——这就是"变分推断":在一个函数集合里找最优函数**。所以叫"**变分**"(在分布空间里优化),叫"**推断**"(推断 `z`,也就是给定 `x` 找背后的配方卡)。

---

## 第三层:KL 散度是什么

在继续之前必须讲 KL,因为后面到处是它。

**KL 散度 `KL(q ‖ p)`** = 衡量两个分布 `q` 和 `p` 差多少的一个数:

$\mathrm{KL}(q \,\|\, p) = \int q(z) \log \frac{q(z)}{p(z)} \, dz = \mathbb{E}_q\left[\log \frac{q}{p}\right]$

你不需要会算,只需要记住三件事:

- **`q = p` 时,KL = 0**(两分布完全相同)
- **差异越大,KL 越大**(永远 ≥ 0)
- **不对称**:`KL(q‖p) ≠ KL(p‖q)`

VAE 里会出现两种 KL:

1. `KL(q(z|x) ‖ p(z|x))` — 我们猜的后验 vs 真后验(理想量,但 `p(z|x)` 算不出)
2. `KL(q(z|x) ‖ p(z))` — 我们猜的后验 vs 先验(可算,长得像公式里的那个)

## 第四层:ELBO 公式是怎么推出来的

现在开始逐行拆章节那个恐怖公式:

$\log p_\theta(x) \;\ge\; \mathbb{E}*{q*\phi(z|x)}\bigl[\log p_\theta(x|z)\bigr] \;-\; \mathrm{KL}\bigl(q_\phi(z|x) \,\|\, p(z)\bigr)$

这个公式叫 **ELBO**(Evidence Lower BOund, 证据下界)。"证据"就是 `log p(x)`,"下界"就是 `≥ 右边`。

### 推导(每步都不跳)

**起点**:我们想优化 `log p(x)`。用积分展开:

$\log p(x) = \log \int p(x, z) \, dz$

**第 1 步:塞一个 `q(z|x)` 进去**(凑配分母)

$= \log \int q(z|x) \cdot \frac{p(x, z)}{q(z|x)} \, dz$

把 `q(z|x)` 同时乘除一下,值没变,形式变了。现在积分可以重新读成"按 `q(z|x)` 分布求期望":

$= \log \; \mathbb{E}_{z \sim q(z|x)}\!\left[\frac{p(x, z)}{q(z|x)}\right]$

**第 2 步:用 Jensen 不等式**

`log` 是凹函数,Jensen 不等式说:**`log E[X] ≥ E[log X]`**。直觉:log 把大数往下压得比小数狠,所以"先平均再 log" ≥ "先 log 再平均"。

$\log p(x) \;\ge\; \mathbb{E}_{z \sim q(z|x)}\!\left[\log \frac{p(x, z)}{q(z|x)}\right]$

这已经是 ELBO 了,只是还没化简。

**第 3 步:拆 `p(x, z) = p(x|z) · p(z)`**

$\mathrm{ELBO} = \mathbb{E}_q\!\left[\log \frac{p(x|z) \cdot p(z)}{q(z|x)}\right] = \mathbb{E}_q[\log p(x|z)] + \mathbb{E}_q\!\left[\log \frac{p(z)}{q(z|x)}\right]$

**第 4 步:第二项就是负的 KL**

$\mathbb{E}_q\!\left[\log \frac{p(z)}{q(z|x)}\right] = -\mathbb{E}_q\!\left[\log \frac{q(z|x)}{p(z)}\right] = -\mathrm{KL}(q(z|x) \,\|\, p(z))$

**合起来**:

$\log p(x) \;\ge\; \underbrace{\mathbb{E}*q[\log p(x|z)]}*{\text{重建项}} - \underbrace{\mathrm{KL}(q(z|x) \,\|\, p(z))}_{\text{正则项}}$

**这就是章节里那个公式**。

### 两项的人话翻译

- **重建项 `E_q[log p(x|z)]`**:用档案员反推出来的配方 `z` 给画家,画家再画一张图,**画得跟原图 `x` 越像越好**。本质就是 encoder→decoder 的**自编码重建误差**。
- **KL 正则项 `KL(q(z|x) ‖ p(z))`**:档案员给出的配方分布,不能离"中庸配方宇宙"(标准高斯)太远。否则画家只会用奇怪的配方,整个潜空间一片混乱。

**两项在打架**:

- 光要重建好 → 档案员会把每张图编到独一无二的位置,latent 杂乱无章
- 光要 KL 小 → 档案员把所有图都编到原点,画家啥也重建不出

**VAE 的训练就是在这两项之间找平衡**——这也是为什么 `β-VAE` 这么重要,下面就讲。

## 第五层:Reparameterization Trick — 为什么需要它

训练神经网络要用梯度下降,要求每一步都**可微分**。问题来了:

目标里有 `z ∼ q_φ(z|x)` 这个**随机采样**操作。档案员输出均值 μ 和方差 σ²,然后我们从 `N(μ, σ²)` 里随便抽一个 `z`。

**采样是个"掷骰子"操作,没有梯度可言**——你问"掷出 5 对 μ 的偏导是多少?",这个问题没有意义。

### 巧妙的小技巧

Kingma 的改写:

$z = \mu_\phi(x) + \sigma_\phi(x) \cdot \epsilon, \quad \epsilon \sim N(0, I)$

**翻译**:

- 先从**固定的** `N(0, I)` 里抽一个 `ε`(跟参数无关)
- 然后做**确定性的**放缩平移:`z = μ + σ·ε`

现在随机性只出现在 `ε` 里,跟参数 `φ` 毫无关系。而 `μ_φ` 和 `σ_φ` 都是**神经网络的确定性输出**,梯度可以正常反传。

**一图版**:

```
原版(梯度过不去):  参数φ → [采样 z] → 重建损失 → 不能反传 ❌
改版(梯度通):      参数φ → μ, σ → [z = μ + σ·ε] → 重建损失 → 可反传 ✓
              ε (独立噪声,旁路注入)
```

**为什么这个 trick 很基础**:扩散、flow matching、强化学习、甚至 LLM 里的 Gumbel-softmax,全都是"把随机性旁路、让梯度走确定性路径"的变体。**一次学懂它,整条阶段 IV 都通了**。

## 第六层:β-VAE 就是把 KL 调粗

一眼就能看懂:

$\mathcal{L}_{\beta\text{-VAE}} = \mathbb{E}_q[\log p(x|z)] - \beta \cdot \mathrm{KL}(q(z|x) \,\|\, p(z))$

唯一的改动:**给 KL 项乘个系数 β**。

- `β = 1` 就是原版 VAE
- `β > 1` 意味着**更重视"潜空间要规整"**,不在乎重建质量恶化
- `β < 1` 反之

**为什么 β > 1 会"解缠"**?强化 `KL(q ‖ N(0,I))` 会逼 `q` 的各维度**独立**(因为 `N(0,I)` 各维度独立)。独立 → 每一维编码一个独立属性(如果 β=1 的独立,一维编码"旋转",一维编码"颜色",互不干扰)。

**本体意义**:β-VAE 第一次证明了"**改改目标函数里的一个系数,就能直接塑造潜空间的语义结构**"——这就是章节里说的"主体的内部语法不再是副产品"。

## 第七层:VAE 为什么会模糊 — 两个根本毛病

### 毛病 1:高斯解码器逼你预测均值

VAE 假设 `p(x|z)` 是高斯分布,这等价于用 L2 损失(像素平方误差)。

问题:给定同一个 `z`,可能对应好几张合理的图(比如同一个配方可以画成"左眼先画"或"右眼先画")。L2 损失下,模型会选这**几个可能答案的平均**——也就是一张"模糊但安全"的图。

**比喻**:问"这张脸的下巴应该在哪?"——在像素 (100, 120) 或 (101, 119) 都合理,但 L2 要求你"猜一个最安全的",模型就预测 (100.5, 119.5)——每个像素都是平均值,整张脸就糊了。

### 毛病 2:Posterior Collapse

如果画家(解码器)非常强,它不看配方卡也能画出好图。这时档案员发现:"反正画家不需要我的配方,我干脆给出标准高斯算了"——也就是 `q(z|x) → p(z)`,KL=0,**潜变量被完全忽略**。

**VAE 变成一个普通的 autoencoder(甚至是 decoder-only)**,潜空间失去意义。

**这两个毛病决定了 VAE 的生成质量永远追不上 GAN**。但——

### VAE 的真正胜利战场:给扩散当"压缩器"

Stable Diffusion 做的事:

1. 训一个 VAE,把 `512×512×3` 压到 `64×64×4`——**压缩比 48 倍**
2. 让扩散模型在这个 `64×64×4` 的小空间里跑,**计算量暴降**
3. 生成完了,VAE 的 decoder 把小 latent 还原成高清图

**VAE 不是以生成模型胜利,是以"潜空间供应商"胜利**。没有这一步,Stable Diffusion 在消费级显卡上根本跑不动。

## 第八层:VQ-VAE 的三项损失怎么来

VQ-VAE 把连续 latent 换成**离散 token**。核心操作:

- 编码器输出连续向量 `z_e(x)`
- 有一本"码本" `{e_1, e_2, …, e_K}`,共 K 个嵌入向量
- 找到离 `z_e(x)` 最近的那个码本向量 `e_k`,用它代替 —— 这一步叫**量化 (quantize)**
- 解码器用 `e_k` 重建图

三项损失:

$\mathcal{L} = \underbrace{\|x - \hat{x}\|^2}*{\text{(1) 重建}} + \underbrace{\|\mathrm{sg}[z_e(x)] - e_k\|^2}*{\text{(2) 码本}} + \beta \cdot \underbrace{\|z_e(x) - \mathrm{sg}[e_k]\|^2}_{\text{(3) commitment}}$

### 逐项人话翻译

**`sg[·]` = stop-gradient**,意思是"在这个位置**截断梯度**,反传时当作常数"。

- **(1) 重建项**:熟悉,原图和重建图的像素差
- **(2) 码本项**:`sg[z_e(x)]` 把编码器输出当作"目标",训练码本向量 `e_k` 去向它靠拢。**这项只更新码本,不更新编码器**。
- **(3) commitment 项**:反过来,`sg[e_k]` 把码本向量当作"目标",训练编码器输出 `z_e(x)` 向它靠拢。**这项只更新编码器,不更新码本**。`β` 控制这种"靠拢"的强度。

**为什么要拆成两个方向?**

- 如果不拆,两边同时动,训练会震荡
- 拆开之后:码本知道"我该跟上编码器";编码器知道"我该提交(commit)到最近的码本向量附近"——因此叫 commitment loss

### 那个 argmin 的不可微问题

量化操作 `k = argmin_j \\|z_e(x) - e_j\\|` 是不可微的(硬选 index,没法求导)。

**Straight-Through Estimator** 绕过:

- **前向**(算损失时):用量化后的 `z_q = e_k` 送进解码器
- **反向**(传梯度时):假装 `∂z_q / ∂z_e = 1`,梯度直接从 `z_q` 抄一份传给 `z_e`

**比喻**:进门扫了一下二维码(硬动作),但出门报销时就当作"我正常走过去的",不计二维码扫描的成本。工程暴力但有效。

## 第九层:可逆流的 change of variables 公式

$\log p_X(x) = \log p_Z(f(x)) + \log \left| \det \frac{\partial f}{\partial x} \right|$

这是"精确似然"路径的核心。让我慢讲。

### 一维直觉先建立

假如 `z = f(x)` 是一个可逆的一维函数(比如 `z = 2x`)。

- 你把概率密度从 `x` 搬到 `z`,**总概率守恒**(都必须是 1)
- 但"密度"会随着"拉伸/压缩"变化:
    - 如果 `f` 把一个小区间拉宽 2 倍 → 密度除以 2
    - 如果 `f` 把区间压窄到一半 → 密度乘以 2

公式就是:`p_X(x) = p_Z(f(x)) · |f'(x)|`。取 log:

$\log p_X(x) = \log p_Z(f(x)) + \log |f'(x)|$

**跟上面公式一模一样**,只是一维版的 "`|f'(x)|`" 在高维变成 **Jacobian 行列式的绝对值**。Jacobian 记录了"在这一点附近,`f` 对每个方向的局部拉伸比",行列式是"总体积缩放因子"。

### 为什么这很吸引人

**直接给你精确的 `log p(x)`**——VAE 只能给下界,GAN 给不了任何似然,这是唯一能精确计算概率密度的路径。

### 但为什么工程上输了

**要做到"可逆 + 行列式可算",函数 `f` 被大幅限制**:

- RealNVP 的 **coupling layer**:把向量拆两半,一半保持不变,另一半被另一半"条件仿射变换"
- Jacobian 变成下三角,行列式 = 对角线乘积,好算

**代价**:神经网络的表达力被切成一半再切一半——远不如 GAN / 扩散的自由架构。质量赶不上。

**但它的数学遗产 → Rectified Flow / Flow Matching**(§34):

- 把离散步的可逆函数 `f` 换成**连续时间的常微分方程** (ODE)
- 生成 = 在时间上积分一个"速度场"
- **这正是今天 Stable Diffusion 3 的训练目标**

"**流的质量输,但数学表述赢,在扩散时代以另一种身份复活**"——这句话现在应该能看懂了。

## 第十层:MaskGIT 的"并行讲话"

背景:VQ token 化后,图变成 256 个离散 token。

**自回归方式**:一个一个猜(`p(token_1)`, `p(token_2 | token_1)`, ...)——需要 256 次前向。

**MaskGIT 的类 BERT 思路**:

- 训练:把 token 序列随机 mask 一部分(比如 50%),让模型预测被 mask 的那些
- 生成:从"全部 mask"开始,每轮**并行预测所有位置**,按置信度保留 top-k,剩下的重新 mask
- 迭代 8-12 轮就出图,比自回归快 30-60 倍

**为什么这能工作?** 因为**token 之间的依赖是局部 + 全局混合**,不像像素那样是严格的因果序列。"同时猜 + 多轮修订"比"左到右一个个猜"效率高得多。

**比喻**:自回归像是"默写古诗,一个字一个字写,错了要推倒重来";MaskGIT 像是"先在所有空格上写铅笔草稿,看着满不满意,把不对的擦掉重写"。
