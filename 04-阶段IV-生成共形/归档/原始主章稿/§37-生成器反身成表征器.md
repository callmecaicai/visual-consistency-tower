## §1 · 为什么 §37 必须出现

§36 把阶段 IV 走完:生成主体成人。但走到这一步,一个奇怪的事实开始浮现——

当我们拿一个训好的 Stable Diffusion / Flux 去做分类、分割、对应点匹配、深度估计这些**理解性任务**,效果常常比专门训练的 CNN / ViT 分类器**更好**(或相当)。换句话说:一个**为了生成**训练的模型,它学到的东西可能比**专门为了理解**训练的模型还要完整。

这是 §37 要处理的本体论问题:**生成和理解之间到底是什么关系?**

两条证据线浮现出来,把这个问题逼到必须面对的位置:

- **线 A**:重建式 SSL(MAE / BEiT / iBOT-MIM 等)——只用像素重建作目标,就能训出顶级表征
- **线 B**:扩散特征外溢(DIFT / Diffusion Hyperfeatures / REPA)——为生成训练的模型,其中间激活直接就是顶级表征,REPA 甚至能用表征候指导生成训练

两条线汇合出同一个结论——**生成器内部的中间表征比最终图像本身更像"理解"**。

## §2 · 线 A:重建式 SSL(Masked Image Modeling)

### 2.1 BERT 在图像上的翻版

BERT 在 NLP 的成功方案非常简单:mask 掉10-15% 的词,让模型预测它们。学出来的中间表征在下游任务上极强。2021-2022 年,整个 CV 圈在问:**能不能在图像上做同样的事?**

挑战:图像比文本难——没有天然的离散 token,像素空间连续,mask 一小块的预测难度也不一样。

### 2.2 MAE(Masked Autoencoder, He et al., 2022)

最成功的方案。核心设计:

1. **随机 mask 75% 的 patches**(注意:这个比例惊人地高,BERT 才 15%)
2. **Asymmetric encoder-decoder**:encoder 只见 visible patches(25%),decoder 把 mask token 补齐再重建
3. **重建目标**:L2 回归到原像素

为什么 75% 这么高?——因为图像信息冗余度高,掉 15% 模型靠近邻 copy 就能重建。必须 mask 到 75% 这个"不能靠抄邻居解决"的难度,encoder 才被迫学习**语义级别**的结构理解。

损失:

$$
L_{MAE} = \|\text{Decoder}(\text{Encoder}(x_{visible}) \oplus \text{mask tokens}) - x_{masked}\|^2
$$

**人话**:把 3/4 的图挡掉,用剩下 1/4 猜被挡的部分长什么样。能猜准,说明 encoder 真的学懂了图的语义结构,不是在背紋理细节。

结果:encoder 单独拿出来(扔掉 decoder),在 ImageNet linear probe / fine-tune / COCO detection 上都刷出 SOTA。证据:**为重建训练的模型,学到的表征极强**。

### 2.3 BEiT / BEiT-v2(Bao et al., 2021-2022)

变种:不预测原像素,预测一个**离散 visual tokenizer**(DALL·E 1 的 dVAE 或 VQ-KD codebook)产生的 token。

$$
L_{BEiT} = -\sum_{i \in \text{masked}} \log p(z_i \mid x_{visible})
$$

这里 `z_i` 是 mask 位置的离散 token(而不是像素)。

**优势**:预测离散 token 比预测连续像素更接近 BERT 的形式,分类性能略优于 MAE。

### 2.4 SimMIM(Xie et al., 2022)——最简化版

MAE 的极简变种:用 ViT 随机 mask(比例略低,50%),直接对 mask 位置回归像素。结构简单,性能接近 MAE。

### 2.5 iBOT(Zhou et al., 2022)——MIM + 自蒸馏

iBOT 同时做两件事:

1. **MIM 目标**:mask patch,预测被 mask 部分(属于 IV)
2. **对比 / 自蒸馏目标**:学生 student 和动量 teacher 的不同视角表征一致(属于 V,与 DINO 同宗)

**iBOT 是典型的"双面作品"**——它的两个目标分属 IV 和 V。按目标拆归属,而不是按论文。

### 2.6 为什么 MIM 归阶段 IV(重要)

尽管 MAE / BEiT 最终用作**分类器**,它们的训练目标是**像素重建 / 像素级 token 预测**——这和扩散的"噪声→图像"、GAN 的"噪声→图像"、VAE 的"latent→图像"是同一个家族:**目标都在像素空间(或像素级离散化空间)产生某种输出**。

所以 MIM 在本体论上属于 IV,而不是 V。V 的标志是**目标在抽象状态空间**(JEPA 直接预测表征向量,不碰像素)。

## §3 · 线 B:扩散模型作为表征器

这条线更有趣——**本来是为生成训练的模型,发现它的中间激活本身就是顶级表征**。

### 3.1 DIFT(Diffusion Features, Tang et al., 2023)

实验:拿训好的 Stable Diffusion,取 U-Net 某一层某个时间步的激活作为特征,去做 semantic correspondence(两张不同图里相同语义部位的对应,比如两只猫的左眼)。

结果:**超过了所有专门为匹配训练的 CNN 特征和 DINO 特征**。

关键发现:

- 不是所有层都好。中间层(通常是 U-Net 的 mid-block)最好
- 不是所有时间步都好。`t ≈ 261 / 1000`(中噪声水平)最好
- 直觉:太早(`t` 大、噪声多)表征太混乱;太晚(`t` 小、近原图)表征太倾向低级细节;中间时间步刚刚好对应语义结构形成的时机

### 3.2 Diffusion Hyperfeatures(Luo et al., 2023)

升级:不只取一层一步,而是**跨层跨时间步聚合**,学一个小网络把多尺度特征融合。进一步超过 DIFT。

### 3.3 DiT features 的特性

2024 年更多工作把这套方法拓展到 DiT / MMDiT:

- DiT 的中间 attention map 编码了**强语义空间布局**
- 某些 head 几乎就是 object parser / part segmenter
- 整个模型在**无监督**训练下获得了大量下游任务上的 SOTA 表征能力

结论:**扩散生成训练过程中产生了意外的、极高质量的表征机构**。

### 3.4 经验事实对比

| 下游任务 | 用 Flux / SD 特征 | 用 DINOv2 | 用 CLIP |
| --- | --- | --- | --- |
| Semantic correspondence | **最佳** | 强 | 弱 |
| Part segmentation | 强 | **最佳** | 弱 |
| Depth estimation | 强 | 强 | 弱 |
| Classification linear probe | 中 | **最佳** | **最佳** |

Flux / SD 的**空间结构类任务**极强,**全局语义分类**稍弱——正好对应它们训练目标的差异(生成需要具象空间结构,对比需要抽象语义概念)。

## §4 · REPA 的突破——表征反指导生成训练

### 4.1 REPA(Representation Alignment, Yu et al., 2024)

思路革命性:既然扩散模型自然会学到好表征,能不能**强制扩散模型的中间激活与预训好的 SSL 表征(比如 DINOv2)对齐**,来加速训练?

做法:在 DiT 训练过程中,除了扩散损失,加一个对齐损失:

$$
L_{REPA} = L_{diffusion} + \lambda \cdot L_{align}
$$

$$
L_{align} = -\text{sim}\big(\text{Proj}(h^l_\theta(x_t, t)), \; f_{DINOv2}(x_0)\big)
$$

**人话**:扩散模型某一层(通常 8-12 层)的激活,过一个小投影头后,要和 DINOv2 在原图上的特征**对齐**(余弦相似度)。

### 4.2 惊人效果

- 训练速度**提升 17.5×**:达到同样的 FID,需要的训练步数降到 1/17.5
- 最终质量**反而更好**
- 生成质量和文本跟随都提升

### 4.3 这意味着什么?

**扩散模型之前要自己从零学表征,同时还要学生成。现在我们直接告诉它"目标表征长这样"(由 DINOv2 提供),它只需要把这个表征映射到图。**

这等于在说:**表征是比生成更基础的东西**。生成是建立在表征之上的上层任务。

### 4.4 REPA 之后的工作

- **REPA-E**:enhanced variants,在更多架构上验证
- **用 MAE features / I-JEPA features** 代替 DINOv2
- **Diffusion-as-SSL**:直接把扩散模型作为 SSL 预训范式

2024-2025 年整个 diffusion 社区都在往这个方向收敛——**表征对齐是下一代扩散训练的标配**。

## §5 · 反身的哲学

### 5.1 "生成即理解"公设开始动摇

阶段 IV 的核心公设:**能正确生成 = 能正确理解**。一个能画出猫的模型必然理解猫。

但 §37 的两条线在说:

- **可能不需要生成,也能学到同等好的理解**(MAE 只重建像素,学到的表征和 DiT 生成器相当)
- **生成的过程中产生的中间状态,比生成物本身更像理解**(DIFT 用中间激活做对应匹配 ＞ 最终图本身)

那么**生成本身到底是理解的必要条件,还是副产品?**

### 5.2 中间状态 vs 最终样本

这是阶段 IV 自己向自己提出的问题:

- 我们看最终生成的图,评估模型"会不会画"
- 但真正的"理解"可能在中间 latent / 中间 attention 中
- 最终图是中间理解**被迫压成像素**的结果——在"还原到像素"这个过程中,大量结构化理解被压缩丢失

换个说法:**最终图是理解的投影(低维像素),中间状态是理解本身(高维表征)**。

### 5.3 阶段 IV 的自我超越

这就是反身:阶段 IV 用自己最成熟的工具(大扩散模型)研究自己最深的问题(什么是理解),发现**答案不在自己手里**——而在一个它尚未成为的阶段(阶段 V:抽象状态作为预测目标)。

## §6 · IV → V 的分水岭

### 6.1 两类目标

|  | 阶段 IV(生成共形) | 阶段 V(表征预测) |
| --- | --- | --- |
| 目标空间 | 像素 / 连续信号 | 抽象表征 / 状态向量 |
| 损失 | 像素重建 / 噪声预测 | 表征距离(MSE / 对比) |
| 代表 | DDPM, MAE, VAE, SD, Flux, BEiT | I-JEPA, V-JEPA, DINO, 世界模型 |

### 6.2 "双面作品"归属

- **iBOT MIM 目标** → IV(像素)
- **iBOT Contrastive 目标** → V(表征)
- **BEiT** → IV(尽管预测 token,但 token 是 dVAE 编码的像素级单元)
- **MAE** → IV(纯像素)
- **DINOv2** → V(纯表征)
- **I-JEPA / V-JEPA / LeCun 的世界模型框架** → V

### 6.3 为什么 JEPA 不属于 IV

JEPA 的目标:mask 一块,在**表征空间**而不是像素空间预测被 mask 的内容。这是"理解"在 LeCun 体系里的定义——**拥有对世界状态的预测模型,但不必恢复到具体感官信号**。

这是阶段 V 的起点(会在后续章节详细展开)。

### 6.4 反公设

- 阶段 IV 的公设:"能生成 = 能理解"
- 阶段 V 的反公设:"**生成是形似理解的冗余工作,真正理解 = 对抽象状态的持有与预测**"

§37 就是让这两条公设第一次在同一页上相遇的章节。

## §7 · 向阶段 V 的传递

阶段 IV 在 §37 走到终点。走到终点的标志有三个:

1. **重建式 SSL 证明**:不需要完整生成,只需要像素预测,就能产出高质量表征
2. **扩散特征证明**:为生成训的模型,其中间表征是它**副产品**级别的理解资产
3. **REPA 证明**:表征可以作为生成训练的**先验指导**,表明表征的地位在生成之上

所以阶段 V 的开篇是一个干净的反转:

- IV 的主语:生成器
- V 的主语:**表征器 / 预测器 / 世界模型**

§37 是阶段 IV 给阶段 V 交接的钥匙——它说:"我走完了,但我在走完的最后发现,**真正的理解比我这个阶段里的任何工具都更重要**。这把钥匙交给你。"

> **§37 结论**:IV 的最后一章承认了 IV 不是尽头。生成主体反身发现自己是表征主体的外壳,阶段 V 从这里出发。

## A · OpenAI GPT-4o Image / gpt-image-1(2025-03)

### 架构推测

- **官方明确**🟢:"model is autoregressive, and uses the same architecture as the GPT-4o LLM. The model essentially generates images in the same way as the LLM generates text"[[1]](https://www.promptingguide.ai/guides/4o-image-generation)
- **社区强推测**🟡(Interconnects 分析):**Autoregressive 先生成 patch tokens(从左上到右下 tile-wise),然后把"粗图"交给一个 diffusion decoder 去精细化**,即 `tokens → [transformer] → [diffusion] → pixels`。这个 pipeline 也与 OpenAI 官方白板插画("compose autoregressive prior with a powerful decoder")一致[[2]](https://openai.com/index/introducing-4o-image-generation/)[[3]](https://www.interconnects.ai/p/gpt-4os-images-and-lessons-from-native)
- **来源血脉**🟡:这条路线可以追溯到 **DeepSeek Janus**(LLM + 自回归图像头,arXiv 2410.13848)、以及"LLM 直接预测 rectified flow velocity"的工作(arXiv 2411.07975),Reddit ML 社区认为这是同一条技术谱系[[4]](https://www.reddit.com/r/MachineLearning/comments/1jkt42w/d_gpt4o_image_generation_and_editing_how/)

### 训练方式推测

- **联合分布**🟢:"We trained our models on the joint distribution of online images and text, learning not just how images relate to language, but how they relate to each other"[[2]](https://openai.com/index/introducing-4o-image-generation/)——即 image-text 和 image-image 的联合,这和纯文→图模型的训练目标明显不同
- **统一 post-training stack**🟢:在推理时共享 GPT-4o 的 RLHF / safety / personality 训练栈,这是它能"继承 GPT-4o 的世界知识和对话上下文"的根因
- **改进点**🟢:"model compressed representations, compose autoregressive prior with a powerful decoder"——推测 VQ-VAE 或类似的离散视觉 tokenizer 被替换为**更压缩的表征**,降低 AR 部分的 token 预算

### 技术路径

DALL·E 1(AR + dVAE, 2021) → DALL·E 2(prior + decoder diffusion, 2022) → DALL·E 3(LDM + caption recap, 2023) → **GPT-4o Image(AR + diffusion decoder, 2025)**

OpenAI 在 GPT-4o 这一步**放弃了纯扩散路线**,回归到 DALL·E 1 的自回归谱系——但视觉 tokenizer、backbone 容量、与 LLM 融合程度都是质变级别。这是 §35 讲的"2025 原生多模态元年"的关键转折。

### 留下的痛点

- "varying bit-rate across modalities"(不同模态的信息密度不同,统一 tokenization 损失效率)
- "compute not adaptive"(无法像 diffusion 那样按时间步分配算力)[[2]](https://openai.com/index/introducing-4o-image-generation/)

## B · Google Gemini Flash Image 家族("Nano Banana" 谱系)

三代演进:

- **Nano Banana**(Gemini 2.5 Flash Image, 2025-08)——LMArena 第一次匿名登场就登顶,~13M 首次访问用户 / 4 天内 5B 生成[[5]](https://www.woshipm.com/ai/6345887.html)
- **Nano Banana Pro**(Gemini 3 Pro Image, 2025-11)——引入"System 2"思考、高保真文本渲染[[6]](https://ai.google.dev/gemini-api/docs/image-generation?hl=zh-cn)
- **Nano Banana 2**(Gemini 3.1 Flash Image, 2026-02)——Arena ELO 1279 登顶,$0.067/图,4-6 秒生成[[5]](https://www.woshipm.com/ai/6345887.html)

### 架构推测(知乎 / CSDN 深度分析汇总)🟡

1. **原生多模态 Transformer**:输入序列直接是 `{文本 token, 图像 token, 音频 token, 视频 token}` 的混合序列,attention 跨模态长程依赖[[7]](https://zhuanlan.zhihu.com/p/1977045999781422337)
2. **稀疏 MoE 架构**:FFN 层被替换为一组 experts,每个 token 动态路由到 top-k 专家。理论上专家会**自发分化出模态专精**——纹理专家、文本逻辑专家、空间几何专家——这解释了"带文字的图像"这类混合任务上的极致效果[[7]](https://zhuanlan.zhihu.com/p/1977045999781422337)
3. **高保真视觉 tokenizer**:Gemini 成功的关键在"语言是离散、图像是连续"的桥——必须有高效、高保真的视觉词元化系统[[7]](https://zhuanlan.zhihu.com/p/1977045999781422337)
4. **混合推理架构(Nano Banana 2 独特)**:"彻底摒弃了传统文生图领域单一的扩散管线,转而构建了一套由大型语言模型(LLM)驱动的混合推理架构"[[5]](https://www.woshipm.com/ai/6345887.html)——这和 GPT-4o 的"AR + diffusion decoder"路线在本质上同宗

### 训练方式推测

- **在线知识蒸馏**:Flash 模型不只学 Teacher(Pro / Ultra)的 hard label,还学 soft distribution——"继承了 Teacher 模型在创作时的犹豫和倾向"[[7]](https://zhuanlan.zhihu.com/p/1977045999781422337)
- **AQT(Accurate Quantized Training)**:在训练中就模拟 4-bit / 8-bit 量化,而不是训后量化,从而让 Flash 版本能直接在边缘设备 / 高并发服务器跑
- **思考预算(thinking budget)可调**🟢:Gemini 2.5 系列支持开/关"思考"并设置推理预算,这是 Pro 版本走"System 2 逻辑创作"路线的核心工具[[8]](https://www.woshipm.com/ai/6300978.html)

### 技术路径

Imagen 1/2/3(纯 cascaded diffusion, T5-XXL)→ Gemini 2.0 native image(实验) → **Gemini 2.5 Flash Image**(原生多模态 MoE + 视觉 tokenizer)→ Gemini 3 Pro Image(+ System 2)→ **Gemini 3.1 Flash Image**(Pro 能力下放 + 量化 + Grounding)

Google 的路线和 OpenAI 汇合于同一点:**从独立的 text-to-image 扩散模型转向 LLM 原生多模态生成**。但 Google 通过 MoE 架构更激进地降低了推理成本——这是 Nano Banana 2 能以 $0.067/图 对抗 GPT-4o 的关键。

### Imagen 并未死,而是分工

Google 官方 guidance 显示[[9]](https://firebase.google.com/docs/ai-logic/generate-images-gemini?hl=zh-cn):

- **Gemini (Nano Banana)** 主推:利用世界知识、上下文对话编辑、文本图像交织
- **Imagen** 主推:画质第一、写实、艺术风格、明确宽高比、品牌素材

即:**Imagen 4 依然是标准 cascaded diffusion(T5-XXL frozen encoder + 64→256→1024 super-res + Efficient U-Net + dynamic thresholding)**[[10]](https://imagen.research.google/)[[11]](https://deepmind.google/models/imagen/)——这条经典路线在 Google 内部被保留作为"纯美学 / 纯画质"工艺线,Nano Banana 走"智能 / 多模态 / 编辑"工艺线。**两条线分工并存,不是替代**。

## C · Midjourney V7 / V8

### 官方极度隐晦,全靠版本对比逆向

- **V7 发布** 2025-04-03, 默认 2025-06-17;V8 2025 alpha[[12]](https://docs.midjourney.com/hc/en-us/articles/32199405667853-Version)[[13]](https://www.mindstudio.ai/blog/midjourney-v8-vs-v7-comparison/)
- **V7 新特性**🟢:Draft Mode(10× 更快)、Personalization 默认开启(解锁 5 分钟)、Omni Reference、更好的文本+图像 prompt 联合处理、更精确的手脚身体
- **V8 重架构方向**🟡:"rearchitected around photographic realism, physical coherence, and precise prompt interpretation"[[13]](https://www.mindstudio.ai/blog/midjourney-v8-vs-v7-comparison/)——光线 / 材质 / 反射 / 皮肤次表面散射变"physically grounded"而不只是"atmospheric"

### 架构推测🔴

Midjourney 官方**从不公开架构**,社区共识:

- **底层很可能是 MMDiT + Rectified Flow**(与 Flux、SD3 同宗,Stability 技术已经成为行业标配)
- **强烈差异化的美学后训练**:Midjourney 的独特卖点是"预设审美",这来自巨量人类偏好数据的 RLHF / DPO 后训练——这是 V6 / V6.1 / V7 每次升级明显的"换风格"现象的来源
- **Personalization 默认开启的技术基础**🟡:为每个用户学一个**极轻的个性化 adapter**(很可能是 LoRA-style),5 分钟的解锁流程就是在收集足够的偏好数据拟合这个 adapter。V7 是"第一个 personalization default on"的版本,说明这套 per-user adapter infra 到 V7 才工业化稳定
- **Omni Reference**🟡:V7 引入的 multi-reference 融合,类似 Flux Kontext / IP-Adapter 路线,即在 attention 层注入参考图 CLIP / VAE encoding 做内容约束

### 训练路径

Midjourney 从 V1-V4 的早期 CLIP-guided diffusion,V5 跃升到大规模 latent diffusion(2023),V6 / V6.1 引入细节增强 + 文本 prompt 对齐,V7 换到**新的底层(很可能是 MMDiT + rectified flow + per-user personalization layer)**。

**独特商业路径**:MJ 从未发论文、从未开源,完全靠"我们的画最好看"这条 moat。2025 年在 LLM-native 潮流下仍守住美学第一——证明**美学审美调优**是一条独立的 moat,不是架构可以复制的。

## D · Black Forest Labs(BFL)Flux 家族——半开放旗舰

Flux 是唯一**架构论文公开、Schnell 权重开源、Dev / Pro / Ultra / Kontext 权重闭源**的顶级模型。这让我们能从 schnell 的开源实现直接推断 Pro 的架构。

### 架构(已公开)🟢

- **MMDiT(Multimodal Diffusion Transformer)**:图像 token 和文本 token 在单一 transformer 里 joint attention,没有 cross-attention——这是 Flux / SD3 的共同架构(见 §36 §2)
- **Rectified Flow**:直线 ODE 路径代替 curved diffusion 路径,采样效率更高(见 §34 §9)
- **Guidance Distillation**:Flux.1 Dev 公开的关键技巧——把 Classifier-Free Guidance 的二次前向蒸馏到单次前向,推理速度 ×2
- **ADD(Adversarial Diffusion Distillation)**:Flux Schnell 的 4 步采样基础,SDXL-Turbo 同门工艺
- **Flux.1 Kontext(2025)**:in-context 图像编辑,架构上把参考图作为额外 token 拼接到输入序列

### Pro / Ultra / Kontext 的差异推测🟡

- **Pro**:更大的 MMDiT(社区估计 12B+ vs Dev 的 12B),额外的审美后训练
- **Ultra**:进一步放大 + 更长 / 更精细的后训练;支持更高分辨率 + 细节保真
- **Kontext Pro**:加上编辑专用的 in-context 样本 post-training

### 路径

SD1.5 / SDXL(Stability AI) → **Stability 核心团队出走创立 BFL** → Flux.1(2024-08, schnell/dev/pro)→ Flux.1 Kontext(2025)→ Flux.1 Pro Ultra

BFL 是**技术最透明的顶级商业玩家**——他们用"开源 schnell + 闭源 pro"的模式,既享受社区 ecosystem(ComfyUI 主流 backbone)又保持商业付费护城河。

## 综合结论:商业闭源的三条技术主脉

1. **LLM-native 自回归混合**(OpenAI GPT-4o, Google Nano Banana 家族)——走"统一多模态 + 世界知识 + 上下文对话编辑",核心是 **LLM 驱动 + 视觉 tokenizer + 轻量 decoder(diffusion 或 MaskGIT-style)**。这条线**是未来,但当前画质第一梯队里还没完全取代纯扩散**
2. **MMDiT + Rectified Flow 工业旗舰**(Flux Pro/Ultra, SD3.5, Imagen 4, 推测 MJ V7/V8)——走"最大化画质 + 美学 / 构图控制",核心是**大 MMDiT backbone + 强文本编码器(T5-XXL 或 CLIP+T5) + 大规模美学 RLHF**。这是**当下 2026 年工业画质的实际前沿**
3. **垂直特化 + 价格屠夫**(Ideogram 文本、Seedream 性价比、Recraft 平面设计)——在 MMDiT 基础上做**垂直数据 + 垂直 loss**,以"某一维度最好 + 更便宜"占据细分市场
