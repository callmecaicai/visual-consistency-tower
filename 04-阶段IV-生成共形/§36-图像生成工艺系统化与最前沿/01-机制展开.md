# 阶段 IV · §36 机制展开：当扩散进入工艺系统化

类型: 理论/主章  
阶段: IV · 生成共形  
共形维度: 动力, 时间尺度, 材料  
关键贡献: 把阶段 IV 后半程彻底写成一套可学习的图像生成生产制度，而不是零散论文列表。主线分成两条：一条是“架构与速度”的内核主线，一条是“控制与产品化”的生态主线。
残差 / 它催生了什么: 当图像生成工艺已经高度成熟，真正的残差就不再是“还能不能更会画”，而是“生成过程中形成的内部状态是不是比最终图像更像理解”。这条残差直接送向 `§37`。  
定位标签: LDM, SDXL, DiT, MMDiT, SD3, Flux, DPM++, Karras, UniPC, LCM, ADD, DMD, ControlNet, InstructPix2Pix, DreamBooth, LoRA

## 学习抓手

- 本章不要按“模型年表”读，要按两条主线读：先读架构与速度，再读控制与产品化。
- 如果你只想建立全局理解，先抓六件事：骨干网络、采样器、蒸馏、编辑、个性化、数据评估。
- 本章不是在证明谁绝对最好，而是在说明阶段 IV 怎样从范式革命走到工业成熟。

## 一、本章为什么必须单独存在

`§34` 解决的是动力学，`§35` 解决的是意图接口。但真正能把阶段 IV 推到成熟态的，不是再发明一个新公设，而是把已有公设做成系统工艺。

如果没有本章，阶段 IV 会停留在“扩散很强、文生图很火”的印象层，而不会变成一个真正可组织、可部署、可扩展的知识体系。

因此 `§36` 的任务是收尾，也是生产制度闭合：

> 它要证明外显生产不只是会出图，而是已经拥有完整技术栈。

一个成熟图像生成系统至少需要以下工艺层同时闭合：

| 工艺层 | 代表 | 解决什么 |
|---|---|---|
| 压缩层 | VAE / tokenizer / LDM | 让高分辨率生成可扩展 |
| 主干层 | U-Net / DiT / MMDiT | 决定模型容量与 scaling |
| 条件层 | CLIP / T5 / cross-attention | 把意图接入生成 |
| 轨迹层 | DDPM / DDIM / EDM / Flow | 决定生成路径 |
| 采样层 | DPM-Solver / DPM++ / UniPC | 决定推理效率 |
| 蒸馏层 | Progressive Distillation / LCM / DMD / ADD | 压缩步数 |
| 控制层 | ControlNet / IP-Adapter / inpainting | 保持结构约束 |
| 个性化层 | DreamBooth / LoRA / Textual Inversion | 引入特定身份或风格 |
| 数据层 | caption、过滤、合成、偏好数据 | 决定生成分布和审美 |
| 评估层 | FID、人类偏好、prompt alignment、consistency | 决定什么算“好图” |

## 二、主线 A：架构与速度的内核主线

### 2.1 LDM：把高成本问题降到可扩展尺度

LDM 已经在 `§34` 出现过，但放在工艺角度看，它是整条后半程的地基：

- 把像素问题压到 latent 空间。
- 让高分辨率生成成本可控。
- 让后续所有骨干演化都拥有现实工程前提。

没有 LDM，就没有后面一整串“看似前沿”的工业模型。

### 2.2 从 U-Net 到 DiT

U-Net 长期是扩散骨干标准，但它有两个明显限制：

1. 长距离依赖组织得不够优雅。
2. 随规模扩张时，scaling law 不够干净。

DiT 的意义就在于把扩散主干彻底 transformer 化。它的优势不是一个局部 trick，而是：

- token 化后更适合大规模扩张。
- 自注意力对全局关系更自然。
- 与文本和多模态 token 的统一也更顺手。

因此从 U-Net 转向 DiT，不是简单的“换 backbone”，而是扩散开始在大规模 latent diffusion 中吸收 transformer 的规模化优势。

这不意味着 U-Net 结束。U-Net 在高效、局部结构、产品部署和许多控制场景中仍然长期有价值；DiT 的意义是提供了更自然的 scaling 载体，而不是宣布旧骨干失效。

### 2.3 从 DiT 到 MMDiT

DiT 仍然基本是“图像主干 + 文本外挂接口”。MMDiT 则更进一步，把图像 token 和文本 token 放进更统一的注意力结构中。

这一步的历史意义很明确：

- 图像与文本不再只是外接关系。
- 它们开始在统一 token 空间里共同参与生成过程。

因此 MMDiT 不是一个局部架构名称，它代表的是：

> 阶段 IV 后半程的图文统一工艺成熟。

### 2.4 速度主线：采样器系统化

当骨干稳定以后，速度问题就成为生产级关键问题。

这一支最核心的是：

- DDIM：从原始 DDPM 的慢采样迈向可用。
- DPM-Solver / DPM++：更高效的数值积分。
- Karras sigmas：更合理的噪声调度。
- UniPC：更稳的预测-校正式少步采样。

这里的逻辑非常清楚：

- 骨干决定“会不会画”。
- 采样器决定“多快画出来还不太崩”。

### 2.5 蒸馏与加速：从 50 步到 4 步甚至 1 步

真正把扩散推向近实时应用的，不是单纯换采样器，而是蒸馏。

这一线可以压成：

- Progressive Distillation：逐级压步数。
- Consistency / LCM：把少步生成系统化。
- ADD / DMD：在极少步情况下用更强蒸馏与判别性约束保住锐度。

这说明阶段 IV 后半程已经不满足于“高质量但慢”，而是开始追求一个更成熟的工业指标：

> 质量、速度、稳定性三者要同时成立。

## 三、主线 B：控制与产品化的生态主线

### 3.1 编辑：从“会画”到“按约束改”

真正困难的产品问题往往不是从零生成，而是：

- 改哪里。
- 保留哪里。
- 改完以后全局别散。

因此编辑工艺是本章第二条主线的核心起点。这里典型包括：

- Inpainting / Outpainting
- SDEdit
- Prompt-to-Prompt
- InstructPix2Pix

它们共同回答的是：

> 什么该变化，什么该保持不变。

这比单次生成更高级，因为它要求模型同时守住不变量和变化量。

### 3.2 ControlNet 与结构控制

ControlNet 之所以关键，不是因为它只是一个插件，而是因为它把“控制”从文本描述推进到结构约束：

- 边缘
- 深度
- 姿态
- 分割

这些条件一旦进入扩散工艺，外显生产就不再只是“按概念画”，而能“按布局和几何画”。这是从语义接口走向结构接口的一大步。

### 3.3 个性化：让模型学会“你”

DreamBooth、Textual Inversion、LoRA、IP-Adapter 等路线，解决的是另一个产品核心问题：

> 如何让通用生成系统快速吸收一个具体身份、具体风格、具体参考？

这类方法的重要意义不是只服务个别应用，而是说明外显生产系统已经具备：

- 低成本增量更新能力。
- 在通用性与特化性之间来回切换的能力。

### 3.4 数据工艺：生成质量并不只来自模型

到阶段 IV 后半，大家逐渐看清一件事：好模型不够，数据描述方式本身会决定上限。

因此数据工艺开始成为核心部分：

- 更细的 caption。
- 更好的 recaption。
- 更精的图文清洗。
- 更明确的审美与风格筛选。

这意味着阶段 IV 的成熟，不只是网络和损失的成熟，也是“如何组织训练语料与条件描述”的成熟。

### 3.5 评估工艺：从 FID 到更接近使用场景

FID 在很长时间里是生成质量标准，但后来大家越来越清楚它不够：

- 它不等于文本跟随。
- 它不等于编辑能力。
- 它不等于人类偏好。

因此后面的评估会逐步引入：

- CLIP-based 相似性。
- 对齐度。
- 偏好对比。
- 更接近产品使用的竞技场式评测。

这说明图像生成已经不再只是“看着像不像”，而是在迈向多目标综合评价。

## 四、为什么说阶段 IV 在这里形成生产制度

走到本章结束，阶段 IV 已经具备以下完整链条：

1. 有内部语：潜空间、token 化、内部坐标。
2. 有内部时间：扩散与其连续动力学。
3. 有意图接口：文本、guidance、adapter。
4. 有工业骨干：LDM、DiT、MMDiT。
5. 有速度工艺：采样器、蒸馏、少步生成。
6. 有控制工艺：编辑、结构控制、个性化。
7. 有数据与评估工艺：更稳定地把训练和产品连起来。

这意味着阶段 IV 到这里已经不只是一个理论阶段，而是一整套成熟技术文明。

## 五、本章真正要记住什么

- 阶段 IV 后半程必须按两条线理解：一条是骨干与速度，一条是控制与产品化。
- U-Net → DiT → MMDiT 是骨干统一化的主线。
- DPM++ / Karras / UniPC 到 Consistency / LCM / ADD / DMD 是速度成熟化的主线。
- 编辑、ControlNet、DreamBooth、LoRA、IP-Adapter 说明外显生产开始真正可控、可个性化、可部署。
- 当这些都成熟以后，阶段 IV 的核心问题就不再是“还能不能更会画”。

## 六、向 §37 的传递

正因为阶段 IV 在这里走到成熟态，`§37` 才会变得不可回避。

因为一旦外显生产系统已经极其会画，最值得追问的问题就会转向：

> 真正重要的到底是最终样本，还是生成过程中形成的内部状态？

换句话说，阶段 IV 的最深残差不再是产品指标，而是认识论问题。`§37` 会在这个位置把生成器倒转成表征器。
