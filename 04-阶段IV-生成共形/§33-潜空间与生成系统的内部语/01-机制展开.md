# 阶段 IV · §33 机制展开：潜空间为什么是生成系统的内部语

类型: 理论/主章  
阶段: IV · 生成共形  
共形维度: 材料, 主体性, 动力  
关键贡献: 把 VAE / β-VAE / VQ-VAE / 可逆流 / MaskGIT 等路线统一成一件事：为生成系统构造内部语。它们不是在补一个小技巧，而是在回答“系统内部到底拿什么来组织样本变化”。
残差 / 它催生了什么: 潜空间让生成系统有了内部坐标，但时间还没有被真正内化；质量、控制、似然和速度也尚未被统一。下一步只能是 `§34`，把迭代本身写成生成动力学时间。
定位标签: VAE, β-VAE, VQ-VAE, Glow, RealNVP, MaskGIT, 潜空间

## 学习抓手

- 第一次读本章，只要抓住一句话：`§33` 的核心不是“几种模型谁更强”，而是“生成系统终于开始有自己的内部语言”。
- 最重要的公式只有三块：VAE 的 ELBO、VQ-VAE 的三项损失、Flow 的 change of variables。
- 如果公式暂时看不懂，先记住四条路线各自解决了什么，再回来看公式就会顺很多。

## 一、为什么生成系统必须拥有“内部语”

`§32` 已经让生成样本获得外显可信性，但这条路径主要还是靠外部看见的样本和对抗关系在生存。问题是：

- 如果系统没有内部坐标，它就很难被稳定操纵。
- 如果系统没有内部语法，它就很难把“结构”和“细节”分开组织。
- 如果系统没有明确的内部表征，它就只能生成结果，不能组织自身过程。

因此 `§33` 的真正问题不是“怎么再做一个生成模型”，而是：

> 生成系统内部到底拿什么来表达世界？

这一章的所有路线，本质上都在回答这个问题。

## 二、内部语的五个判据

不是任何 latent 都能被叫作“内部语”。一个内部变量要进入阶段 IV 的内部语闭合，至少要满足五个判据中的若干项：

| 判据 | 含义 | 反例 |
|---|---|---|
| 坐标性 | 内部变量能组织样本变化 | 纯噪声入口不算强内部语 |
| 可采样性 | 内部变量能从先验中生成 | 只用于编码、不用于生成的 feature 不够 |
| 可操纵性 | 内部变量变化能引起可解释输出变化 | 完全纠缠且不可解释的 latent 很弱 |
| 可复用性 | 同一内部单位能跨样本或任务重复使用 | 单图优化 latent 不够 |
| 可组合性 | 内部单位能形成组合语法 | 只有连续滑动，没有结构组合，仍不完整 |

按这个标准看，`§33` 的各路线不是“谁替代谁”，而是在不同材料上构造内部语：

| 模型 | 内部语类型 | 强项 | 残差 |
|---|---|---|---|
| VAE | 连续概率潜变量 | 可采样、可压缩 | 样本发糊，语义可操控性有限 |
| β-VAE | 解耦潜变量倾向 | 试图获得可解释因子 | 解耦不保证语义正确 |
| VQ-VAE | 离散码本 | token 化、可序列化 | 码本不一定有语义 |
| Flow | 可逆坐标变换 | 严格 likelihood | 结构表达成本高，语义可控弱 |
| MaskGIT / Muse | 离散并行 token 生成 | 比像素自回归更并行 | token 语法仍需外部条件和解码器 |

## 三、路径一：VAE 家族把生成写成潜变量问题

### 3.1 VAE 的基本形式

VAE 的直觉很清楚：

- 外部图像 `x` 很高维、纠缠、难直接建模。
- 我们希望先把它压到一个较小的潜变量 `z`。
- 再从 `z` 解码回图像。

它引入两个网络：

- 编码器 `q_\phi(z \mid x)`
- 解码器 `p_\theta(x \mid z)`

训练时直接优化的不是 `\log p(x)`，而是它的下界 ELBO：

$$
\mathcal{L}_{\text{ELBO}}
=
\mathbb{E}_{q_\phi(z \mid x)}[\log p_\theta(x \mid z)]
- D_{\text{KL}}(q_\phi(z \mid x)\,\|\,p(z))
$$

这条式子非常重要，但可以只按两部分理解：

1. 第一项要求 `z` 能把图像重建回来。
2. 第二项要求编码出来的 `z` 不要乱长，而要贴近一个简单先验，通常是高斯分布。

于是 VAE 实际上做的是：

> 一边构造内部语，一边约束这种内部语必须是可采样、可组织、可复用的。

这条公式的最低门槛读法是：

| 问项 | 回答 |
|---|---|
| 公式在问什么 | `z` 既要能把图像重建回来，又要贴近一个可采样先验。 |
| 哪个参数最敏感 | KL 权重、latent 维度、重建损失形式。 |
| 变大 / 变小会怎样 | 正则太强会发糊或塌缩，太弱会潜空间散乱。 |
| 服务哪种闭合 | 连续内部语闭合。 |
| 读不懂时记住 | VAE = 让图像经过一个可采样、可规整的内部坐标。 |

### 3.2 Reparameterization 为什么关键

VAE 训练会遇到一个经典问题：`z` 是随机采样的，梯度怎么穿过去？

解法是：

$$
z = \mu(x) + \sigma(x) \odot \epsilon,\quad \epsilon \sim \mathcal{N}(0, I)
$$

这叫 reparameterization trick。它的本质不是一个小技巧，而是在说：

> 把“随机性”外包给一个独立噪声源，让编码器自己只负责产出可微的参数。

因此 VAE 是第一种真正把“随机潜空间”工程化的主流方法。

### 3.3 VAE 为什么常常发糊

VAE 的重建往往比较平滑，原因主要有两点：

1. 高斯重建目标会鼓励均值化，复杂细节容易被抹平。
2. KL 正则会压缩潜变量带宽，过强时会损伤表达能力。

这说明 VAE 很擅长做“内部语”的雏形，但不天然擅长做最锋利的样本质量。

### 3.4 β-VAE 的意义

β-VAE 把 KL 项权重调大：

$$
\mathcal{L}_{\beta\text{-VAE}}
=
\mathbb{E}_{q_\phi(z \mid x)}[\log p_\theta(x \mid z)]
- \beta D_{\text{KL}}(q_\phi(z \mid x)\,\|\,p(z))
$$

它的逻辑是：宁可牺牲一点重建质量，也要换来更规整、更可解耦、更可解释的潜空间。

这里的 `\beta` 是最直接的旋钮：

| `\beta` 状态 | 可能结果 |
|---|---|
| 变大 | 潜空间更规整、更有解耦压力，但图像细节和重建质量可能下降 |
| 变小 | 重建更好，但内部语更容易纠缠，采样秩序变弱 |

这一步清楚表明：

- `§33` 不只是追求“能生成”。
- `§33` 还在追求“系统内部表征是否可组织”。

## 四、路径二：VQ-VAE 把内部语离散化

### 4.1 为什么要离散

连续潜变量很灵活，但有时过于柔软。语言、token、离散组合等很多后续能力，都更依赖一种可枚举、可拼装、可序列化的内部单位。

VQ-VAE 的方案是引入一个码本 `\{e_k\}`，把连续编码结果量化到最近的码本向量上。

### 4.2 VQ-VAE 的三项损失

典型写法包括三部分：

1. 重建损失。
2. 码本更新损失。
3. commitment 损失，防止编码器频繁漂移。

它们合起来的意义是：

> 生成系统不仅要有内部语，还要把内部语切成可重复调用的离散词汇。

这条路线的后果非常深：

- DALL·E 1 的图像 token 化来自这里。
- Muse、Parti、MaskGIT 这类离散生成路线来自这里。
- 多模态模型里“图像也能像文本一样 token 化”的思想，也来自这里。

### 4.3 VQ-GAN 为什么重要

VQ-GAN 在 VQ-VAE 基础上把对抗感知损失接进来，说明离散潜空间不是只能做重建，它也能支撑高质量图像生成。

它的意义在于把两条线接起来：

- `§32` 的对抗锐化能力。
- `§33` 的离散内部语。

## 五、路径三：可逆流把内部语做成严格变换

### 5.1 Flow 模型在干什么

Flow 模型的目标非常“正统”：

- 不想放弃似然。
- 不想只要隐式采样。
- 希望有一个可逆映射，把简单分布变成复杂图像分布。

change of variables 写成：

$$
\log p_X(x)
=
\log p_Z(f(x))
+
\log \left| \det \frac{\partial f(x)}{\partial x} \right|
$$

这条式子的直觉是：

> 如果我知道一个可逆变换怎样把图像 `x` 变成简单潜变量 `z`，那我就能精确算出图像的概率密度。

### 5.2 Flow 的价值与限制

Flow 的价值很大：

- 精确似然。
- 可逆结构。
- 潜变量和样本之间的双向映射很干净。

但它也有明显限制：

- 架构约束强。
- 训练和表达成本高。
- 在图像生成质量和易用性上，后来没有跑赢扩散。

不过本章要记住的是：Flow 把“内部语”问题推进到了数学最严的形态。后来的 Flow Matching / Rectified Flow 会在 `§34` 和 `§36` 重新回来。

## 六、路径四：并行 token 生成

### 6.1 MaskGIT / Muse 在解决什么

如果说 VQ-VAE 提供了离散词汇，那么 MaskGIT / Muse 这条线就在问：

> 图像 token 能不能像语言一样，通过 mask-and-fill 的方式并行生成？

这和 PixelCNN 那种严格逐像素顺序不同。它尝试让生成变成“先生成一批粗 token，再逐轮修补不确定位置”的过程。

### 6.2 这条线的意义

它说明 `§33` 不是只在研究“潜变量本身”，还在研究：

- 内部语怎样被并行调度。
- 生成系统能否用 token 级别的结构组织图像。

这条线后来会和多模态统一 token 空间发生非常强的联系。

## 七、四条路线的共形分工

把本章压成表格理解最清楚：

| 路线 | 它解决的核心问题 | 主要代价 |
| --- | --- | --- |
| VAE / β-VAE | 连续潜空间、压缩、可采样内部语 | 细节发糊、质量有限 |
| VQ-VAE / VQ-GAN | 离散词汇、token 化、可组合内部语 | 码本与重建设计复杂 |
| Flow | 精确似然、可逆映射 | 架构受限、工程代价高 |
| MaskGIT / Muse | 并行 token 生成 | 需要离散视觉词表作为前提 |

因此 `§33` 的真正结论不是“谁赢了”，而是：

> 生成系统必须拥有内部语，这件事已经无法回避；不同路线只是给出不同的语言形态。

## 八、本章最大的未解残差

虽然系统有了内部语，但一个新的结构问题开始暴露：

- 这些内部语大多还是静态坐标。
- 采样过程仍经常表现为外部算法调度。
- 质量、控制、稳定性和理论统一没有被同时解决。

换句话说：

> 系统知道“自己拿什么表达”了，但它还没有把“自己如何在时间中生成”写成内在本体。

这条残差会直接指向扩散。

## 九、本章真正要记住什么

- `§33` 的使命不是做花样模型，而是为生成系统建立内部语。
- VAE 解决“连续潜语”，VQ-VAE 解决“离散词汇”，Flow 解决“严格可逆”，MaskGIT 解决“并行 token 调度”。
- StyleGAN 的 latent 只是内部语的前兆；`§33` 才开始把内部语做成显式目标。
- 本章的出口残差非常明确：系统有了内部坐标，但时间仍然在外面。

## 十、向 §34 的传递

于是 `§34` 的任务就完全清楚了：

> 如果生成系统已经有了内部语，那么下一步必须让它在“生成动力学时间”中展开，而不再只靠外部采样流程调度自身。

扩散革命正是在这个位置发生的。
