# 阶段 IV · §31 机制展开：觉醒前夜为什么必须写

类型: 理论/主章  
阶段: IV · 生成共形  
共形维度: 动力, 材料, 时间尺度  
关键贡献: 把 RBM / DBN / EBM / PixelRNN / PixelCNN 写成阶段 IV 的胚胎期。它们已经把“让模型刻画哪些样本可能成立”设成问题，但还没有找到可规模化、可控制、可稳定外显的生产路径。
残差 / 它催生了什么: 前夜证明“生成”不是偶然兴趣，而是视觉理论的必然问题；但不同路线留下不同残差：EBM / RBM 受 `Z` 与采样拖累，PixelRNN / PixelCNN 受串行顺序拖累，DBN / 生成式预训练尚未形成可操纵内部语。这些残差分别在 GAN、潜空间和扩散中被重新组织。
定位标签: RBM, DBN, EBM, score, PixelRNN, PixelCNN, 生成前夜

## 学习抓手

- 第一次读本章，只要抓住一件事：`§31` 不是讲“老方法大全”，而是讲“为什么生成问题早就被提出，但一直点不着”。
- 公式里最重要的只有两条：能量模型 `p(x) = exp(-E(x)) / Z`，以及 score `s(x) = \nabla_x \log p(x)`。
- 如果你暂时不懂玻尔兹曼机细节，也不影响理解本章主线。真正需要记住的是它们各自留下了什么残差。

## 一、本章到底解决什么问题

如果阶段 IV 从 GAN 直接开始，整条生成线会被误读成一场工程突变。但真实历史不是这样。GAN 之所以像一次爆炸，是因为更早的模型已经反复把同一个问题抛出来：

> 视觉系统不仅要判断一个样本像什么，还要能够刻画“哪些样本本身可能成立”。

`§31` 的任务，就是把这种“生成冲动”从历史边角里捞出来，并明确它为何长期没有成为主流。

更具体地说，前夜阶段已经提出了三件事：

1. 模型应当拥有某种对数据分布的刻画能力，而不仅仅是判别边界。
2. 样本可以被逐步生成，而不只是被最后一层分类。
3. 生成如果要成立，系统必须在内部拥有某种稳定的动力学，而不是完全依赖外部采样协议。

它们离阶段 IV 很近，但还没有真正成为阶段 IV。

## 二、胚胎线一：能量谱系

### 2.1 能量模型到底在说什么

能量模型最朴素的想法是：

> 不直接给出“这张图属于哪一类”，而是给每个样本一个“成立程度”的分数。

经典写法是：

$$
p(x) = \frac{\exp(-E(x))}{Z}
$$

这里 `E(x)` 是能量，越低表示样本越像真实数据；`Z` 是归一化常数，它把整片空间上的能量都积起来，保证这是一个概率分布。

这条路线的重要性在于，它第一次把“图像世界”想成一个地形：

- 真实样本处在低谷。
- 不像样的东西处在高坡。
- 生成的任务，就是想办法落进低谷。

这比分类更接近阶段 IV 的本体，因为它已经不是“给答案”，而是“构造一个可成立性的地形”。

这条公式的最低门槛读法是：

| 问项 | 回答 |
|---|---|
| 公式在问什么 | 一张图像在模型地形中有多可能。 |
| 哪个量最麻烦 | `Z`，因为它要对整个高维空间归一化。 |
| 变大 / 变小会怎样 | 能量差越尖锐，分布越集中；但如果采样不到足够空间，地形会学偏。 |
| 服务哪种闭合 | 可能性闭合。 |
| 读不懂时记住 | EBM 给图像可能性画地形。 |

### 2.2 从 Boltzmann 到 RBM / DBN

Boltzmann Machine、RBM、DBN 的价值不在于今天还被大量直接使用，而在于它们把两个关键直觉做了出来：

1. 生成可以通过隐变量层级来实现。
2. 深层网络不一定只能靠判别监督训练，生成式预训练也能成为深网的入口。

尤其是 DBN 的历史意义很大。它不只是一个旧模型，而是后面两条路线的先声：

- 一条通往 `§33` 的“内部语”路线：隐藏层不是副产品，而是主体内部坐标。
- 一条通往 `§37` 的“重建式预训练”路线：先让系统从输入里重构结构，再去做下游任务。

### 2.3 EBM 与 score 的种子

后来的 EBM 复兴把焦点放在一个更实用的问题上：既然 `Z` 很难算，能不能不算高度，而只算“往哪里走会更像真数据”？

这就得到 score：

$$
s(x) = \nabla_x \log p(x)
$$

如果代入能量形式，就有：

$$
s(x) = - \nabla_x E(x)
$$

这条公式在本章里非常关键，因为它是 `§34` 的数学前种子。它把“生成”从估计整片分布的绝对高度，转成估计局部方向场。

用直白的话说：

- 能量模型想知道“这里到底多像真图”。
- score 模型只问“从这里往哪里走，会更像真图”。

这一步直接预告了扩散的思想。

score 的公式卡更短：

| 问项 | 回答 |
|---|---|
| 公式在问什么 | 不问绝对概率，只问往哪里走更像数据。 |
| 关键旋钮 | 噪声尺度、扰动范围、score 网络容量。 |
| 服务哪种闭合 | 从可能性地形通向显形轨迹。 |
| 读不懂时记住 | 高度难算，方向更关键。 |

## 三、胚胎线二：像素自回归谱系

### 3.1 自回归的基本直觉

另一条前夜路线不是给地形打分，而是把图像拆成一个顺序过程：

$$
p(x) = \prod_i p(x_i \mid x_{<i})
$$

意思是，图像不是一次性出现，而是一个像素接一个像素、一块 token 接一块 token 地长出来。

这条公式在问：

| 问项 | 回答 |
|---|---|
| 公式在问什么 | 图像概率能否被拆成一连串条件概率。 |
| 关键旋钮 | 生成顺序、上下文窗口、token 粒度。 |
| 变大 / 变小会怎样 | 上下文更长能看更多结构，但采样更慢；token 更粗更快但细节更弱。 |
| 服务哪种闭合 | 显式似然闭合。 |
| 读不懂时记住 | 自回归把生成写成严格但缓慢的接龙。 |

PixelRNN / PixelCNN / PixelCNN++ 代表了这条路线。它们的重要性在于：

- 它们第一次把“生成样本”做成了一个清晰、严格、可训练的过程。
- 它们没有 `Z` 的困难，因为分解后的条件概率都可以直接优化。
- 它们证明图像生成不是玄学，而是可以被明确写成似然问题。

### 3.2 自回归为什么没有成为阶段 IV 主轴

像素自回归的问题不是理论不对，而是它把生成写成了过于狭窄的时间顺序：

- 每一步都依赖前一步，采样速度太慢。
- 结构是按像素顺序展开，而不是按“对象先出现、细节后打磨”的语义顺序展开。
- 条件控制难以自然接入，尤其在高分辨率图像上代价巨大。

所以它虽然是一种生成，但还不是“主体性的生成”。它更像一个勤奋的抄写员，而不是一个能自我组织的生产者。

## 四、前夜阶段的三类残差

前夜模型不是共享同一种病理，而是从不同方向暴露了生成闭合尚未成立。

| 路线 | 它解决了什么 | 它的残差 |
|---|---|---|
| EBM / Boltzmann / RBM | 把图像世界写成能量地形 | `Z` 难算，采样难，训练难 |
| PixelRNN / PixelCNN | 把图像生成写成严格似然分解 | 串行慢，像素顺序不等于语义顺序 |
| DBN / 生成式预训练 | 证明隐变量层级可学习 | 内部表征尚未成为可操控生成坐标 |
| 早期自回归 token 线 | 证明图像可被序列化生成 | 离散化与全局结构控制不足 |

### 4.1 `Z` 与采样的困难

能量模型的根本困难是：

$$
Z = \int \exp(-E(x)) dx
$$

这个积分在高维空间里几乎不可直接计算。结果就是：

- 训练难。
- 采样难。
- 规模化更难。

前夜已经意识到“给样本一个成立程度”是对的，但它无法把这件事做成工业级生成动力学。

### 4.2 像素顺序不等于语义顺序

像素自回归的困难不是理论不清楚，而是它把生成过度绑定到像素顺序上。采样虽然严格，但语义上很别扭：

- 图像对象不是按 raster scan 成立。
- 局部像素依赖不是对象结构依赖。
- 高分辨率下，串行成本迅速压倒可用性。

这意味着它们缺乏阶段 IV 后来非常重要的一种能力：

> 让生成在有限步数里形成“先结构、后细节”的内部节律。

### 4.3 隐变量还没有成为内部语

前夜模型可以生成，但几乎没有真正可操纵的内部表示空间。更准确地说：

- RBM / DBN 的隐层是有意义的，但还没有被工程化为可通用操纵的语义坐标。
- PixelCNN 的世界几乎完全贴在像素顺序上，没有一个干净的主体内部语。

这条残差会直接送去 `§33`。因为生成如果没有内部语，就只能生成“结果”，不能组织“自身”。

### 4.4 DBN 的跨阶段边界

DBN 在阶段 I 中被读作判别训练的预训练地基；在阶段 IV 中被读作生成式隐变量世界的前夜。

同一技术对象可以跨阶段出现，但主归属由它主要解决的残差决定：

```text
阶段 I 读法：深层判别训练如何起步？
阶段 IV 读法：隐变量层级能否成为生成世界的内部坐标？
```

## 五、前夜的哲学位置

`§31` 的核心判断不是“这些方法过时了”，而是：

> 它们第一次把“对象不是只用来被认出，也应该能被产出”这件事摆上桌面。

但它们仍然停留在“生成的意愿”，没有形成稳定的外显生产闭合。

这可以压成一句非常短的话：

- 前夜已经提出了生成问题。
- 前夜还没有找到稳定、可控、可规模化的外显生产动力。

因此它们是阶段 IV 的必要前提，但不是阶段 IV 自身。

## 六、三颗种子如何被后续章节回收

`§31` 不是死历史，它留下了三颗后面会反复发芽的种子。

### 6.1 score 的种子送往 `§34`

从 `\nabla_x \log p(x)` 到扩散的 score matching，这是一条直接的数学继承线。扩散不是凭空出现，而是把 EBM 里最有生命力的局部方向场思想做成了主轴。

### 6.2 生成式预训练送往 `§37`

DBN 的一层层预训练、重建式学习的直觉，最后会在 MAE / BEiT / iBOT 这些重建式 SSL 里复活。那里“为了重建而学习表征”的思想会被重新发现。

### 6.3 条件像素化送往 `§35`

自回归像素路线虽然太慢，但“条件逐步生成”这个接口直觉会在后面的文本条件化、CFG 和条件采样里被重新组织。

## 七、本章真正要记住什么

- `§31` 的意义不在于它今天是否还是最好方法，而在于它证明了“生成问题早已存在”。
- 前夜最重要的三条技术资源是：能量地形、score 方向场、逐步生成。
- 前夜之所以没有完成闭合，不是因为它们不想生成，而是因为它们还没有找到稳定、可扩展、可组织的生成动力学。

## 八、向 §32 的传递

到这里，阶段 IV 的下一步几乎是必然的：

> 如果生成不能靠外部采样协议稳定外显，那就需要一个内部自我驱动的关系结构。

`§32` 的 GAN 正是在这个位置出现的。它的贡献不是单纯“更会画”，而是第一次让生成样本通过系统内部的对抗关系获得外显可信性。
