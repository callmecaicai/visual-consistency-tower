# Update Policy

## Primary Principle

Update the repository by the smallest structural move that preserves consistency.

## Directory Roles

- `00-公理层`: foundations and global ontology
- `01-阶段I` to `05-阶段V`: main theoretical floors
- `横切`: protocols, audits, reflexive documents
- `应用域`: domain evidence and case collections
- `工具`: utilities and viewers
- `skills`: agent operating procedures
- `归档`: old drafts and exports

## Numbering Policy

### Stage Root

- `00-*`: stage entry
- `01-*`: stage-level extension
- `§xx-*`: main chapters
- `结语*`: closure
- `90-*`: navigation and structural helpers

### Chapter Root

- `00-定位卡.md`
- `01-机制展开.md`
- `02-扩展稿-*.md`
- `03-专题-*.md`
- `90-*` only for non-canonical support files

## Editing Policy

- Preserve user-written structure unless the task requires reorganization
- Prefer adding a new file over rewriting an unrelated existing file
- Update navigation only if the visible information architecture changed
- Keep archive material separated from canonical material
- Keep application evidence separated from chapter canon

## Escalation Threshold

Escalate from file update to structural change only when:

1. no existing chapter can host the material
2. the new material opens a persistent residual chain
3. the change affects future reading order, not only current evidence density
